    시작하기 전에

    Not translated
    This page is currently being translated. We will complete the translation as soon as possible.
    Tip

    BioStar X를 시작하기 전에 알아야 할 내용을 안내합니다. BioStar X는 32비트 운영 체제와 64비트 운영 체제를 지원합니다. BioStar X를 설치하려는 PC의 시스템 사양을 확인한 후 설치하세요.

    설치 전 참고 사항​

    BioStar X를 설치하기 전에 다음 사항을 확인하고 설치를 진행하세요.

    • BioStar 2.9.2 버전부터는 32비트 운영체제를 위한 설치 파일을 제공하지 않습니다.

    • BioStar 1이 설치된 PC에 BioStar X를 설치하지 마세요. 프로그램 성능에 문제가 발생할 수 있습니다.

    • BioStar 2.2.1이나 2.2.2 버전을 설치한 상태에서 BioStar 2.3.0을 설치하면 SQLite에 저장된 모든 정보를 새로운 MariaDB로 마이그레이션합니다.

    • BioStar X 최신 버전으로의 업그레이드는 2.6.0 버전부터 지원됩니다. BioStar 2.6.0 버전 이하에서 최신 버전으로 업그레이드하려면 다음 차례로 설치하세요.

      현재 버전업그레이드 순서
      22.2.1 > 2.3 > 2.4 > 2.4.1 > 2.5.0 > 2.6.4 > Latest version
      2.22.3 > 2.4 > 2.4.1 > 2.5.0 > 2.6.4 > Latest version
      2.2.1
      2.2.2
      2.32.4 > 2.4.1 > 2.5.0 > 2.6.4 > Latest version
      2.42.4.1 > 2.5.0 > 2.6.4 > Latest version
      2.52.6.4 > Latest version
      2.6
      2.6.4 버전 이상Latest version

    • Windows 8.1을 사용하는 경우 다음 링크를 참고해 KB2919355 업데이트를 설치하세요. Link 1

    • MS SQL 2012 Express를 사용할 경우 다음 링크를 참고해 서비스 팩 3를 설치하세요. Link 2

    • MS SQL 2014 Express를 사용할 경우 다음 링크를 참고해 서비스 팩 2를 설치하세요. Link 3

    • MS SQL Server와 BioStar X가 서로 다른 PC에 설치되어 있는 경우 BioStar X를 설치한 PC에 Native Client를 설치해야 합니다. Link 4

    • MS SQL Server를 사용할 경우 서버 및 각 테이블의 Collation 옵션을 CI(Case-insensitive)로 설정하세요.

    • 기존 버전의 BioStar X에서 데이터베이스를 백업할 때 모든 서비스를 사용 중지한 후 진행하세요. 또한, AC 데이터베이스와 TA 데이터베이스를 함께 백업하고, 복원하지 않으면 TA 데이터베이스를 사용할 수 없습니다.

    • BioStar X 데이터베이스를 백업하려면 \Program Files\BioStar 2(x64)\util 폴더의 enckey 파일과 \Program Files\BioStar 2(x64) 폴더의 system.conf, setting.conf 파일도 반드시 함께 백업하세요. 그렇지 않으면, 데이터베이스를 사용할 수 없게 됩니다.

    • BioStar X가 사용하는 포트의 기본 값은 아래와 같습니다. 다른 프로그램에서 동일한 포트를 점유한다면, BioStar X가 정상 구동되지 않을 수 있습니다.

      Info

      • HTTP 포트: 80

      • 웹소켓 포트: 9002

      • 데이터베이스 포트: 3312

      • 근태 HTTPS 포트: 3002

      • FastCGI 포트: 9000

      • 통합 게이트웨이 HTTPS 포트: 5002

      • HTTPS 포트: 443

      • API 포트: 9010

      • 근태 HTTP 포트: 3000

      • 근태 클라우드 포트: 52001

      • 통합 게이트웨이 HTTP 포트: 5000

      • 리포트 HTTP 포트: 4213

    데이터베이스 확인​

    BioStar X를 설치하기 전에 사용자가 직접 구성한 데이터베이스를 사용한다면 아래 사항을 확인하세요.

    MariaDB​

    1. my.cnf 파일에서 [mysql] 항목 하단의 옵션을 아래와 같이 변경하세요.

      my.cnf
      character-set-server=utf8
      collation-server=utf8_unicode_ci
      max_connections = 600

    2. my.cnf 파일에서 [mysql] 항목 하단에 아래 옵션을 추가하세요.

      my.cnf
      log_bin_trust_function_creators = 1
      group_concat_max_len = 102400

    3. 루트(root) 권한으로 MariaDB에 접속해 아래 명령어를 실행하세요.

      SQL
      GRANT SUPER ON . TO user_id@'localhost' IDENTIFIED BY "password";
      GRANT SUPER ON . TO user_id@'%' IDENTIFIED BY "password";

    MS SQL Server​

    Info

    • 모든 데이터베이스 및 테이블의 Collation 옵션을 대소문자를 구분하지 않는 CI(Case-insensitive)로 설정하세요.

    • 데이터베이스 이름에는 숫자, 영문(대소문자 구분), 특수 기호(-, _)만 사용할 수 있습니다.

    포트 설정​

    • SQL Server 구성 관리자를 실행한 후 SQL EXPRESS에 대한 프로토콜의 TCP/IP 프로토콜을 사용하려는 포트 번호로 설정하세요.

    • 설정을 적용하려면 SQL Server 서비스를 재시작하세요.

    사용자 및 데이터베이스 생성​

    1. SQL Server Management Studio에 SQL Server 인증을 이용하여 sa 계정으로 로그인하세요.

    2. 보안에서 마우스 오른쪽 버튼을 클릭한 뒤 새 로그인을 클릭하세요.

    3. 로그인 이름 필드에 원하는 이름을 입력한 뒤 SQL Server 인증을 선택하세요.

    4. 암호와 암호 확인 필드에 사용하려는 암호를 입력한 뒤 암호 정책 강제 적용을 해제하세요.

    5. 확인을 클릭하세요.

    6. 데이터베이스에서 마우스 오른쪽 버튼을 클릭한 뒤 새 데이터베이스를 클릭하세요.

    7. 데이터베이스 이름 필드에 원하는 이름을 입력하세요.

    8. 소유자 필드에 3 단계에서 설정한 로그인 이름을 입력하세요.

    Tip

    데이터베이스 파일 섹션에서 아래와 같이 설정할 것을 권장합니다.

    • 처음 크기(MB): 3000

    • 자동 증가/최대 크기: 10MB 단위, 제한 없음

    Caution

    트랜잭션이 많은 환경에서는 로그를 주기적으로 백업하여 로그 파일의 크기가 증가하지 않도록 주의하세요.

    Windows Authentication 데이터베이스 설정​

    사전 설정​

    Microsoft Windows Active Directory를 사용한다면 Windows Authentication 데이터베이스를 설정하기 전에 아래와 같이 사전 설정을 완료하세요.

    1. SQL Server Management Studio에 관리자 계정으로 로그인하세요.

    2. 보안에서 마우스 오른쪽 버튼을 클릭한 뒤 새 로그인을 클릭하세요.

    3. Windows 인증을 선택한 뒤 검색을 클릭하세요.

    4. 사용자 또는 그룹 선택 창에서 위치를 클릭한 뒤 Active Directory 경로를 선택하고 확인을 클릭하세요.

    5. 선택할 개체 이름 입력 필드에 사용자 이름을 입력한 뒤 이름 확인 > 확인을 클릭하세요.

    6. 페이지 선택 항목에서 서버 역할을 클릭하세요.

    7. sysadmin을 선택한 뒤 확인을 클릭하세요.

    8. 페이지 선택 항목에서 사용자 매핑을 클릭하세요.

    9. ac, master, ta, ve를 선택한 뒤 기본 스키마를 dbo로 설정하세요.

    10. 확인을 클릭하여 저장하세요.

    Active Directory 계정으로 MS SQL Windows Authentication을 사용하기 위한 설정​

    1. services.msc를 실행하세요.

    2. SQL Server 데이터베이스의 속성 > 로그온 창에서 계정 지정을 선택한 뒤 도메인 사용자로 로그인하세요.

    3. Windows 방화벽에 1433 포트를 예외로 추가하세요.

    4. SQL Server에 비어있는 ac, ta, ve 데이터베이스를 sysadmin 인증으로 생성하세요.

    5. SQL Server에 도메인 사용자가 Windows Authentication을 사용하도록 설정하고 ac, ta, ve 데이터베이스에 sysadmin를 제외한 나머지 모든 권한을 할당하세요.

    6. 로컬 컴퓨터를 제외한 모든 서비스를 services.msc에서 연결하세요.

    7. 도메인 사용자를 로컬 서비스 컴퓨터에서 관리자로 설정하세요.

    8. MS SQL ODBC를 연결하세요.

    데이터베이스 설정​

    1. SQL Server 구성 관리자를 실행한 뒤 SQL Native Client 하위의 클라이언트 프로토콜을 클릭하세요.

    2. TCP/IP를 선택한 뒤 기본 포트를 확인하세요.

    3. SQL Server 네트워크 구성 하위의 SQL EXPRESS에 대한 프로토콜을 클릭하세요.

    4. TCP/IP에서 ODBC 포트가 기본 포트와 동일하게 설정되었는지 확인하세요.

    5. SQL Server Management Studio에 관리자 계정으로 접속하세요.

    6. 보안 > 로그인을 클릭한 뒤 NT AUTHORITY\SYSTEM을 더블 클릭하세요.

    7. 페이지 선택 항목에서 서버 역할을 클릭하세요.

    8. public, sysadmin을 선택해 체크한 뒤 확인을 클릭하세요.

    9. 페이지 선택 항목에서 사용자 매핑을 클릭하세요.

    10. ac, ta, ve 데이터베이스를 선택한 뒤 확인을 클릭하여 저장하세요.
