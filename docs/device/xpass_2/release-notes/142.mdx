# 펌웨어 버전 1.4.2 (빌드 번호 1.4.2_241025)

릴리즈<Separator/>2024-10-28

:::caution

신규 BLE 칩이 적용된 장치의 경우 펌웨어를 v1.3.1 이상으로 업그레이드 한 뒤 하위 버전으로 다운그레이드 할 수 없습니다. 기존 BLE 칩이 적용된 장치는 하위 버전으로 다운그레이드 할 수 있습니다.

장치의 신규 BLE 칩 적용 여부 및 펌웨어 다운그레이드 가능 여부는 장치의 시리얼 번호를 확인한 뒤 [슈프리마 홈페이지](https://supremainc.com)에 문의하십시오.

:::

## 신규 기능 및 개선

---

1. Device Manager 애플리케이션을 통한 장치 펌웨어 업그레이드 지원

1. 슬레이브 장치의 전송 속도를 설정할 수 있는 기능 추가

1. SDK에서 GetLogWithFilter 함수를 이용해 사용자 정보 업데이트 성공 이벤트 로그를 가져올 수 있도록 개선

1. RS-485 통신 품질 개선

## 버그 수정

---

<BugLists>

1. 장치에 로그가 가득 찬 상태에서 새로운 이벤트가 발생하는 경우 새로 생성된 이벤트와 함께 의도치 않은 로그를 가져오는 문제 <Badge>1.3.1</Badge>

1. 특정 iOS에서 Device Manager 애플리케이션 v1.1.8 실행 시 장치가 연결되지 않는 문제 <Badge>1.0.0</Badge>

1. **키패드로 카드 ID 입력** 옵션이 활성화된 상태에서 키패드를 사용해 등록되지 않은 카드 ID를 입력했을 때 이벤트 로그에 카드 ID 값이 다르게 기록되는 문제 <Badge>1.0.0</Badge>

1. 장치가 서버와 연결되지 않은 상태에서 32자리 사용자 ID를 가진 사용자가 장치에 인증했을 경우, 장치가 서버와 다시 연결된 후에도 BioStar 2에 인증 이벤트 로그가 업데이트되지 않는 문제 <Badge>1.0.0</Badge>

1. 32자로 저장된 사용자 ID가 특정 이벤트 로그에서 31자까지만 표기되는 문제 <Badge>1.0.0</Badge>

1. 펌웨어 업그레이드 후 기존에 사용 중이던 카드가 인증에 실패하는 문제 <Badge>1.4.1</Badge>

1. **서버와 시간 동기화** 옵션을 해제한 후 장치의 시간을 변경할 경우 장치가 비정상적으로 재시작되는 문제 <Badge>1.4.1</Badge>

1. **RS-485** 모드가 **슬레이브**로 설정된 상태에서 장치의 **커스텀 스마트 카드 레이아웃**과 **바이트 순서**가 설정한대로 저장되지 않는 문제 <Badge>1.4.1</Badge>

1. **슈프리마 스마트 카드 레이아웃**에 **기본 사이트 키**만 설정되어 있고, **커스텀 스마트 카드 레이아웃**에 **보조 사이트 키**만 설정된 상태에서 장치를 재시작할 경우, 장치가 커스텀 스마트 카드를 CSN 카드로 인식하는 문제 <Badge>1.4.1</Badge>

1. 신규 SE 칩 펌웨어를 사용하는 환경에서 일부 스마트 카드 인식에 실패하는 문제 <Badge>1.4.1</Badge>

1. **커스텀 스마트 카드 레이아웃**이 **보조 사이트 키**로 설정된 DESFire 스마트 카드가 CSN 카드로 인식되는 문제 <Badge>1.4.1</Badge>

1. **DESFire 고급 설정**을 사용하여 설정된 커스텀 스마트 카드가 인식에 실패하는 문제 <Badge>1.4.1</Badge>

</BugLists>
