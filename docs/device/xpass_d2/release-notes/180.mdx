# 펌웨어 버전 1.8.0 (빌드 번호 1.8.0_241015)

릴리즈<Separator/>2024-10-15

:::caution

신규 BLE 칩이 적용된 장치의 경우 펌웨어를 v1.7.0 이상으로 업그레이드한 뒤 하위 버전으로 다운그레이드할 수 없습니다. 기존 BLE 칩이 적용된 장치는 하위 버전으로 다운그레이드할 수 있습니다.

장치의 신규 BLE 칩 적용 여부 및 펌웨어 다운그레이드 가능 여부는 장치의 시리얼 번호를 확인한 뒤 [슈프리마 홈페이지](https://supremainc.com)에 문의하세요.

:::

## 신규 기능 및 개선

---

1. Device Manager 애플리케이션을 통한 장치 BLE 칩의 펌웨어 업그레이드 지원

1. RS-485 통신 품질 개선

## 버그 수정

---

<BugLists>

1. SDK에서 BS2_GetDeviceCapabilities 함수를 이용해 장치에서 지원하는 정보를 불러올 때 커스텀 스마트 카드를 지원하는 장치임에도 CustomClassicPlus와 CustomDesFireEV1 항목이 False로 표시되는 문제 <Badge>1.7.1</Badge>

1. HID Prox 카드를 연속으로 인증할 경우 간헐적으로 잘못된 값이 출력되는 문제 <Badge>1.0.0</Badge>

1. **키패드로 카드 ID 입력** 옵션이 활성화된 상태에서 키패드를 사용해 등록되지 않은 카드 ID를 입력했을 때 이벤트 로그에 카드 ID 값이 다르게 기록되는 문제 <Badge>1.0.0</Badge>

1. 펌웨어 업그레이드 후 기존에 사용 중이던 카드가 인증에 실패하는 문제 <Badge>1.7.1</Badge>

1. 특정 iOS에서 Device Manager 애플리케이션 v1.1.8 실행 시 장치가 연결되지 않는 문제 <Badge>1.0.0</Badge>

1. 장치를 타사 컨트롤 패널과 OSDP로 연결했을 때 타사 컨트롤 패널의 응답을 받지 못하는 문제 <Badge>1.0.0</Badge>

1. BioStar 2에서 장치의 Micom 펌웨어를 업그레이드한 후, 신규 BLE 펌웨어를 업그레이드 한 경우 Device Manager 애플리케이션에 재접속할 수 없는 문제 <Badge>1.7.0</Badge>

1. SDK를 사용하여 슬레이브로 연결된 장치의 전송 속도를 변경해도 정상적으로 반영되지 않는 문제 <Badge>1.7.1</Badge>

</BugLists>
