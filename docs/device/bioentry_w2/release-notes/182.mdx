# 펌웨어 버전 1.8.2 (빌드 번호 1.8.2_241025)

릴리즈<Separator/>2024-11-01

:::caution

신규 SE 프로세서가 적용된 장치의 경우 펌웨어를 v1.7.1 이상으로 업그레이드한 뒤 하위 버전으로 다운그레이드할 수 없습니다. 기존 SE 프로세서가 적용된 장치는 하위 버전으로 다운그레이드할 수 있습니다.

장치의 신규 SE 프로세서 적용 여부 및 펌웨어 다운그레이드 가능 여부는 장치의 시리얼 번호를 확인한 뒤 [슈프리마 홈페이지](https://supremainc.com)에 문의하세요.

:::

## 신규 기능 및 개선

---

1. 장치 부팅 후 1분 동안 SEOS 설정 카드를 CSN으로 인식하지 않도록 개선

## 버그 수정

---

<BugLists>

1. **슈프리마 스마트 카드 레이아웃**에 **기본 사이트 키**만 설정되어 있고, **커스텀 스마트 카드 레이아웃**에 **보조 사이트 키**만 설정된 상태에서 장치를 재시작할 경우, 장치가 커스텀 스마트 카드를 CSN 카드로 인식하는 문제 <Badge>1.8.0</Badge>

1. 일부 장치가 펌웨어 업그레이드 후 부팅이 되지않는 문제 <Badge>1.5.0</Badge>

1. 일부 카드가 **슈프리마 스마트 카드** 옵션에서 설정한 것과 다르게 인식되는 문제 <Badge>1.4.1</Badge>

1. **일괄 편집**으로 설정한 **커스텀 스마트 카드 레이아웃**이 장치 상세 페이지에서 적용되지 않는 문제 <Badge>1.8.0</Badge>

1. 장치에 로그가 가득 찬 상태에서 새로운 이벤트가 발생하는 경우 새로 생성된 이벤트와 함께 의도치 않은 로그를 가져오는 문제 <Badge>1.7.0</Badge>

1. **서버와 시간 동기화** 옵션을 해제한 후 장치의 시간을 변경할 경우 장치가 비정상적으로 재시작되는 문제 <Badge>1.0.0</Badge>

1. 신규 SE 칩 펌웨어를 사용하는 환경에서 일부 스마트 카드 인식에 실패하는 문제 <Badge>1.7.1</Badge>

1. **커스텀 스마트 카드 레이아웃**이 **보조 사이트 키**로 설정된 DESFire 스마트 카드가 CSN 카드로 인식되는 문제 <Badge>1.8.0</Badge>

1. **RS-485** 모드가 **슬레이브**로 설정된 상태에서 장치 상세 페이지에서 **커스텀 스마트 카드** 옵션이 보이지 않는 문제 <Badge>1.8.0</Badge>

1. **DESFire 고급 설정**을 사용하여 설정된 커스텀 스마트 카드가 인식에 실패하는 문제 <Badge>1.8.0</Badge>

</BugLists>
