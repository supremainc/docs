# 펌웨어 버전 1.6.2 (빌드 번호 1.6.2_241113)

릴리즈<Separator/>2024-11-14

:::caution

신규 BLE 칩 및 SE 프로세서가 적용된 장치의 경우 펌웨어를 v1.5.0 이상으로 업그레이드한 뒤 하위 버전으로 다운그레이드할 수 없습니다. 기존 BLE 칩 및 SE 프로세서가 적용된 장치는 하위 버전으로 다운그레이드할 수 있습니다.

장치의 신규 BLE 칩 및 SE 프로세서 적용 여부 및 펌웨어 다운그레이드 가능 여부는 장치의 시리얼 번호를 확인한 뒤 [슈프리마 홈페이지](https://supremainc.com)에 문의하세요.

:::

## 신규 기능 및 개선

---

1. **커스텀 스마트 카드 레이아웃** 지원

1. **인텔리전트 슬레이브**를 사용할 경우 타사 컨트롤러의 인증 성공/실패 결과를 장치 화면에 표시하는 **컨트롤러 결과 표시** 기능 지원

1. 장치의 인증 결과 화면에서 사용자 ID, 이름이 표시되는 방법을 선택할 수 있는 **인증 결과 표시 옵션** 기능 지원

    - **사용자 ID 표시**

    - **사용자 이름 표시**

1. 슬레이브 장치의 전송 속도를 설정할 수 있는 기능 지원

1. 사용자 유효기간을 최대 '**2037-12-31**'까지 설정할 수 있도록 개선

## 버그 수정

---

<BugLists>

1. 장치 메뉴에서 **릴레이**를 설정한 경우 모바일 카드로 출입문이 열리지 않는 문제 <Badge>1.0.1</Badge>

1. RS-485로 연결된 슬레이브 장치에서 BS2_GetDeviceCapabilities 함수 호출 시 잘못된 값이 출력되는 문제 <Badge>1.0.0</Badge>

1. 신규 SE 칩 펌웨어를 사용하는 환경에서 일부 스마트 카드 인식에 실패하는 문제 <Badge>1.5.0</Badge>

1. 슬레이브 장치에 지문이 입력되어 매칭 중일 때 마스터 장치에 지문 입력이 되지 않는 문제 <Badge>1.0.0</Badge>

1. **시간 제한 안티패스백**의 **초기화 시간**을 0으로 설정하면 안티패스백이 동작하지 않는 문제 <Badge>1.0.0</Badge>

1. **이중 인증**이 설정된 출입문에 위조 지문을 사용하여 인증에 실패한 경우 이중 인증 과정이 처음부터 다시 시작되지 않는 문제 <Badge>1.5.0</Badge>

1. **시간 제한 안티패스백** 위반이 발생했을 때 화면에 출력되는 문구 수정 <Badge>1.0.0</Badge>

</BugLists>
