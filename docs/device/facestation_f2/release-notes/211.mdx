# 펌웨어 버전 2.1.1 (빌드 번호 2.1.1_221130)

릴리즈<Separator/>2022-11-30

:::caution

신규 SE 프로세서가 적용된 장치의 경우 펌웨어를 v2.1.0 이상으로 업그레이드한 뒤 하위 버전으로 다운그레이드할 수 없으며, 하위 버전에서 커스터마이징한 펌웨어도 다운그레이드할 수 없습니다.

기존 SE 프로세서가 적용된 장치는 하위 버전으로 다운그레이드할 수 있습니다.

장치의 신규 SE 프로세서 적용 여부 및 펌웨어 다운그레이드 가능 여부는 장치의 시리얼 번호를 확인한 뒤 [슈프리마 홈페이지](https://supremainc.com)에 문의하세요.

:::

## 신규 기능 및 개선

---

1. 사용자 정보 업데이트 시 원하는 정보만 선택하여 업데이트 할 수 있도록 개선 (BioStar 2.9.0 버전 이상에서 호환)

1. 얼굴 인증 속도 개선

1. **얼굴 검출 설정** 기능 개선

    - **얼굴 인식 거리**, **얼굴 인식 영역 넓히기** 옵션 지원

1. 업그레이드 패키지 암호화 강화

## 버그 수정

---

<BugLists>

1. SDK에서 BS2_GetLogSmallBlobEx 함수를 이용해 장치의 마스크 검출 및 온도 측정 동작에 대한 이벤트 로그를 불러올 때 사용자 ID 정보가 포함되지 않던 문제 <Badge>1.1.2</Badge>

1. **마스크 검출** 기능이 설정되어있고 마스크 확인 방식이 **인증 후 확인**으로 설정된 장치에 인증했을 때 로그가 출입 성공 (확인 전용)으로 기록되는 문제 <Badge>2.0.1</Badge>

1. BioStar 2.9.x 버전에서 비주얼 페이스에 흐릿한 사진이 업로드된 사용자를 장치로 전송했을 때 사용자가 정상적으로 등록되는 문제 <Badge>2.0.3</Badge>

1. 장치에 얼굴이 중복되는 사용자가 있는 경우에도 얼굴이 정상적으로 등록되는 문제 <Badge>2.0.3</Badge>

1. 장치에서 얼굴을 등록하는 과정에서 스캔한 얼굴 이미지가 기울어져 보이는 문제 <Badge>2.0.3</Badge>

1. 카드 인증 시 데이터베이스 및 캐시 메모리가 깨져 인증에 실패하는 것을 방지하기 위한 구조 개선 <Badge>1.0.0</Badge>

1. 입력 신호에 의한 경비 시작/해제를 설정한 후 입력 신호가 발생하면 경비/해제 로그와 도어 잠김 로그가 생성되는 문제 <Badge>1.0.0</Badge>

1. TCM-10을 사용하여 설치된 장치에 **동적 ROI**를 활성화하고 온도를 측정했을 경우 얼굴 외 배경 부분까지 감지되어 실제보다 온도가 높게 측정되는 문제 <Badge>2.0.1</Badge>

1. 특정 기간(약 50일 이상) 얼굴 인증을 사용해온 장치에서 갑자기 얼굴 인증이 동작하지 않는 문제 <Badge>2.0.3</Badge>

</BugLists>
