# 펌웨어 버전 2.1.0 (빌드 번호 2.1.0_221005)

릴리즈<Separator/>2022-10-05

:::caution

신규 SE 프로세서가 적용된 장치의 경우 펌웨어를 v2.1.0 이상으로 업그레이드한 뒤 하위 버전으로 다운그레이드할 수 없으며, 하위 버전에서 커스터마이징한 펌웨어도 다운그레이드할 수 없습니다.

기존 SE 프로세서가 적용된 장치는 하위 버전으로 다운그레이드할 수 있습니다.

장치의 신규 SE 프로세서 적용 여부 및 펌웨어 다운그레이드 가능 여부는 장치의 시리얼 번호를 확인한 뒤 [슈프리마 홈페이지](https://supremainc.com)에 문의하세요.

:::

## 신규 기능 및 개선

---

1. 듀얼 SE 프로세서 지원

1. 출입문 개방 사유 관련 로그 분리

    - 퇴실 버튼에 의한 출입문 개방 요청

    - 관리자에 의한 출입문 개방 요청

1. 퇴실 버튼 입력에 대한 릴레이 비활성 옵션 추가

    - 퇴실 버튼 입력 시 출입문 열림 요청 로그는 발생하지만 릴레이는 동작하지 않도록 설정할 수 있는 옵션 지원

1. 스마트 카드 바이트 순서 설정 지원

    - Wiegand 또는 OSDP 로 출력되는 데이터 값에 대해 바이트 순서를 설정할 수 있도록 지원

1. 슬레이브 장치로 연결된 FaceStation F2에서 보안 탬퍼가 발생한 경우 OSDP ID도 삭제되도록 개선

## 버그 수정

---

<BugLists>

1. 펌웨어를 v1.1.0 이하에서 v1.1.1 이상으로 업그레이드할 경우 마이그레이션 관련 오동작이 발생하는 문제 <Badge>2.1.0</Badge>

    - 마이그레이션이 완료되지 않고 장치가 메인 화면으로 부팅되며, 일부 키가 동작하지 않거나 비정상적인 화면이 표시되는 문제

1. 모바일 카드 인증이 갑자기 동작하지 않는 문제 <Badge>1.0.0</Badge>

1. 장치에 연결된 LAN 케이블을 분리 후 다시 연결했을 때 DHCP로 IP가 할당되는 시간이 오래걸리는 문제 <Badge>1.0.0</Badge>

1. 사용자 얼굴을 연속으로 등록하거나 메인 메뉴를 반복적으로 터치하면 장치가 재시작되는 문제 <Badge>1.0.0</Badge>

1. 카드를 사용하여 장치에 인증할 때 정상적으로 인증되던 카드가 간헐적으로 인증 실패하는 문제 <Badge>1.0.0</Badge>

1. 얼굴 인증 시 장치에서 보여지는 미리보기 화면이 실제 장치 앞의 상황과 다르게 보여지는 문제 <Badge>1.0.0</Badge>

1. 얼굴인증 모드가 꺼져 있을 때 얼굴을 등록한 사용자가 얼굴인증 모드가 켜져 있을 때 얼굴로 인증하면 실패하는 문제 <Badge>1.0.0</Badge>

1. 장치에서 얼굴인증 모드를 1:N으로 설정하고 BioStar 2에서 다수의 비주얼 페이스를 전송했을 때 장치에 저장 가능한 최대 비주얼 페이스 수가 초과되어도 저장되는 문제 <Badge>1.0.0</Badge>

1. 카드 인증 시 데이터베이스 및 캐시 메모리가 깨져 인증에 실패하는 것을 방지하기 위한 구조 개선 <Badge>1.0.0</Badge>

</BugLists>
