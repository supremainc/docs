카메라는 타사 NVR(Network Video Recorder) 및 VMS(Video Management Software)와의 표준 연동을 위해 RTSP(Real-Time Streaming Protocol) 스트리밍과 ONVIF(Open Network Video Interface Forum) 프로토콜을 지원합니다.

아래와 같은 상황에서 이 문서를 참고하세요.

* 타사 NVR/VMS에서 RTSP로 카메라 영상을 가져오고 싶을 때

* ONVIF를 지원하는 NVR/VMS와 카메라를 연동하고 싶을 때

* RTSP 접속 주소나 ONVIF 연동 시 필요한 계정 정보를 확인하고 싶을 때

## RTSP 스트리밍

### 포트 및 전송 방식

RTSP 포트와 전송 방식은 <Cmd sid='network' product='cam' /> 설정에서 변경할 수 있습니다. 포트 설정, 영상 전송 및 암호화 설정에 대한 자세한 내용은 다음 문서를 참고하세요.

* [포트 설정](configure-network#port-settings#port-settings)

* [영상 전송 및 암호화 설정](configure-network#rtsp-settings)

:::info

RTSP 포트를 기본값(554)에서 변경했다면, 아래 **Connection URL**의 `{port}` 부분도 변경된 포트 번호로 입력해야 합니다.

:::

### RTSP Connection URL

타사 NVR/VMS에서 카메라 영상을 RTSP로 가져오려면 채널과 해상도에 맞는 Connection URL을 등록해야 합니다.

| 채널 | 스트림 | Connection URL |
| --- | --- | --- |
| Channel 1 - Fisheye(피쉬아이 영상) | 메인 스트림(고해상도) | `rtsp://{id}:{passwd}@{IP_address}:{port}/fisheye_primary` |
| Channel 1 - Fisheye(피쉬아이 영상) | 서브 스트림(저해상도) | `rtsp://{id}:{passwd}@{IP_address}:{port}/fisheye_secondary` |
| Channel 2 - Dewarp(평면 왜곡 보정 영상) | 메인 스트림(고해상도) | `rtsp://{id}:{passwd}@{IP_address}:{port}/dewarp_primary` |

* `{id}`, `{passwd}`: 카메라에 접속할 계정의 아이디와 비밀번호를 입력하세요.

* `{IP_address}`: 카메라의 IP 주소를 입력하세요.

* `{port}`: RTSP 포트 번호를 입력하세요. 기본값은 554입니다.

## ONVIF 연동

### 지원 사양

카메라는 ONVIF **Profile S** 및 **Profile T**를 지원합니다.

### ONVIF 전용 계정 준비

보안 표준 사양에 따라 기본 관리자(admin) 계정으로는 ONVIF 연결을 지원하지 않습니다. ONVIF로 연동하기 전에 ONVIF 전용 운영자 계정을 추가하고, 해당 계정 정보로 NVR/VMS에서 접속하세요. 운영자 계정을 추가하는 방법은 <DocLink docId='device/vionyx_webserver/manage-operators' />를 참고하세요.

### 연동 시 유의 사항

* **설정 동기화 보호**

   웹 설정 엔진과의 데이터 정합성을 유지하고 오동작을 방지하기 위해, ONVIF를 통해 해상도, 비트레이트, 이미지 센서 속성 등 카메라 내부 설정값을 변경하는 쓰기(Write) 동작은 지원하지 않습니다.

* **VMS 자동 최적화 기능 비활성화**

   일부 VMS는 연결 시 카메라 설정을 자동으로 변경하는 기능(Video Optimization, Auto Configuration 등)을 제공합니다. 이 기능이 켜져 있으면 연동 오류가 발생할 수 있으므로, VMS 측에서 카메라 설정 자동 수정/최적화 옵션을 비활성화하세요.

:::caution

RTSP 전송 방식을 변경하면 기존 RTSP 세션을 다시 연결해야 할 수 있습니다. NVR/VMS에서도 선택한 전송 방식과 호환되도록 설정해야 합니다.

:::
