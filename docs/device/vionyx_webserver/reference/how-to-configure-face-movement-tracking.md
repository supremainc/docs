# Face Movement Tracking으로 얼굴 동선 추적하기

## 개요

Face Movement Tracking을 사용하면 카메라가 감지한 사람의 얼굴을 인식 영역에서 추적하고, 정의된 진입 구역으로의 이동을 감지합니다. 이 기능은 특정 공간에서 사람들의 이동 경로를 파악하고자 할 때 유용합니다.

## 언제 사용하나요?

- 매장 내 고객의 이동 경로 분석
- 사무실 출입 현황 모니터링
- 특정 구역으로의 진입/진출 추적

---

## 1단계: Face Movement Tracking 생성하기

### 1.1 AI Analysis 메뉴로 이동

1. **Settings(설정)** > **AI & Rule(AI & 규칙)** > **AI Analysis**를 클릭합니다.
2. AI Analysis 유형 선택 화면에서 **Face Movement Tracking** 항목을 선택합니다.

![사람 추적 이벤트 생성](주소-예시)

> **참고**: Face Movement Tracking은 Intrusion(침입 감지)이나 Line Crossing(라인 크로싱)과 달리 **얼굴 인식**에 중점을 둔 기능입니다.

### 1.2 이벤트 이름 설정

설정 화면의 **Name(이름)** 필드에 이벤트 이름을 입력합니다.

**이름 설정 규칙:**
- 길이: 1~64자
- 허용 문자: 한글, 영문, 숫자, 공백(중간), 특수문자(`_`, `.`, `-`, `(`, `)`)
- 예시: "로비 출입자 추적", "Meeting Room Entry #1"

기본값으로 자동 생성된 이름(예: "Face Movement Tracking #1")을 사용해도 됩니다.

---

## 2단계: 추적 영역 설정하기

Face Movement Tracking은 두 가지 영역을 설정합니다:

### 2.1 Face Area (얼굴 인식 영역) - 필수

Face Area는 **얼굴을 인식할 주요 영역**입니다. 카메라가 이 영역에서 얼굴을 감지하면 추적을 시작합니다.

**설정 방법:**

1. 설정 화면의 **Face Area** 섹션에서 **+ Add(추가)** 버튼을 클릭합니다.
2. 카메라 영상 위에 마우스로 다각형을 그립니다:
   - 시작점을 클릭 → 다음 점들을 순서대로 클릭 → 완성
   - 우측 상단의 파란색 영역처럼 사각형이나 다각형 모양으로 만들 수 있습니다.
3. 영역을 완성하면 카메라는 자동으로 이 영역 내에서 얼굴을 인식하기 시작합니다.

**Face Area 개수:**
- 최소: 1개 (필수)
- 최대: 10개
- 여러 개를 설정하면 모든 Face Area가 **합집합**으로 동작합니다(구분 없음).

**설정 팁:**
- Face Area는 **얼굴이 가장 명확하게 보이는 영역**으로 설정하세요.
- 예: 출입문 정면, 로비 중앙 등
- 새로 추가하는 Face Area는 기존 영역보다 약간 작은 크기로 자동 생성됩니다.

### 2.2 Entry Area (진입 구역) - 선택사항

Entry Area는 **추적 대상이 진입하는 구역**입니다. Face Area에서 인식된 얼굴이 Entry Area로 이동하면 이벤트가 발생합니다.

**설정 방법:**

1. **Entry Area** 섹션에서 **+ Add** 버튼을 클릭합니다.
2. 카메라 영상에 진입 구역을 그립니다.
   - 예: 로비 입구, 미팅룸 입구 등

**Entry Area 개수:**
- 최소: 0개 (선택사항)
- 최대: 10개
- 설정하지 않으면 Face Area에서의 얼굴 감지만 추적합니다.

**Entry Area와 Face Area의 관계:**

| 상황 | 동작 |
|------|------|
| Entry Area 없음 | Face Area에서 얼굴이 감지되면 즉시 이벤트 발생 |
| Entry Area 있음 | Face Area의 얼굴이 Entry Area로 들어올 때만 이벤트 발생 |
| Entry Area가 Face Area 범주 내 | 미묘한 움직임도 감지 가능 |
| Entry Area가 Face Area 범주 외 | 두 영역 사이 이동이 명확할 때만 감지 |

## 3단계: 얼굴 인식 설정하기

### 3.1 Face Matching 활성화

**누가 지나갔는지** 확인하려면 Face Matching을 활성화해야 합니다.

1. 설정 화면의 **Option(옵션)** 섹션에서 **Face Matching** 토글을 **ON**으로 전환합니다.
2. 다음 팝업 메시지를 읽고 **동의(확인)** 버튼을 클릭합니다:
   > "얼굴 인식 사용은 해당 지역의 관련 규제 법규 준수가 필요합니다. 슈프리마는 이에 대한 책임을 지지 않습니다."
3. 동의 내역은 시스템 로그에 기록되며, 이후에는 팝업이 표시되지 않습니다.

**Face Matching의 의미:**
- **ON**: 감지된 얼굴을 시스템의 사용자 데이터베이스와 비교하여 신원 확인
  - 일치: 사용자 이름 표시 (예: "Tim berland")
  - 불일치: "Unknown" 표시
- **OFF**: 단순히 얼굴 감지 이벤트만 발생 (신원 정보 없음)

### 3.2 Min Face Size Guide (최소 얼굴 크기)

얼굴이 너무 작으면 인식되지 않을 수 있습니다. 최소 크기 가이드를 확인하세요.

1. **Min Face Size** 섹션에서 **Visualize Guide(가이드 시각화)** 버튼을 클릭합니다.
2. 카메라 영상에 최소 크기를 나타내는 박스가 표시됩니다.
3. 이보다 **작은 얼굴은 인식되지 않을 수 있**으니 참고하세요.

## 4단계: 이벤트 조건 설정하기

### 4.1 Event Condition (진입/진출 감지)

Entry Area가 설정된 경우, **진입(Entry), 진출(Exit), 또는 양쪽 모두**를 감지할지 선택합니다.

**옵션 설명:**

| 옵션 | 동작 | 사용 사례 |
|------|------|---------|
| **Entry** | Entry Area로 들어올 때만 | 영역 진입만 추적 |
| **Exit** | Entry Area에서 나갈 때만 | 영역 진출만 추적 |
| **Entry, Exit 모두** | 진입과 진출 모두 감지 | 완전한 출입 관리 |

**기본값**: Entry, Exit 모두

> **권장**: 출입 통제가 필요하면 **Entry, Exit 모두** 선택

### 4.2 Event Delay Duration (이벤트 지연 시간)

사람이 감지된 후 이벤트가 발생하기까지의 **대기 시간**입니다.

**범위**: 0~60초  
**기본값**: 0초

**의미:**

| 설정값 | 동작 | 장단점 |
|--------|------|--------|
| 0초 | 감지 직후 즉시 이벤트 발생 | 빠른 반응, 하지만 오류 가능성 높음 |
| 1~5초 | 일정 시간 경과 후 이벤트 발생 | 오류 최소화, 안정적인 감지 |
| 10초 이상 | 오래 기다린 후 이벤트 발생 | 응답 속도 느림 |

**설정 팁:**
- 처음에는 **0초**로 시작하여 오류가 많으면 **1~3초**로 조정
- 사람 이동 속도가 느린 공간이면 더 길게 설정 가능
- 빠른 반응이 중요하면 짧게 설정

---

## 5단계: 설정 저장 및 활성화

### 5.1 Apply (적용) 버튼 클릭

모든 설정을 완료한 후 우상단의 **Apply(적용)** 버튼을 클릭합니다.

**저장 후 확인:**
- 성공 메시지: "Event created successfully" 토스트
- 설정된 Face Movement Tracking이 AI Analysis 목록에 추가됨
- 자동 ID 부여 (ID: 1, 2, 3...)

### 5.2 Live View에서 확인

1. **Live(라이브)** 메뉴로 이동합니다.
2. 우상단의 **AI 버튼**을 클릭하여 AI 기능을 활성화합니다.
3. 카메라 영상에 설정한 Face Area 영역이 표시됩니다.
4. 사람이 Face Area에 들어오면 얼굴 주변에 **바운딩 박스**가 표시됩니다.

## 문제 해결

### 얼굴이 인식되지 않습니다

**원인 1: 최소 얼굴 크기 미충족**
- Min Face Size Guide를 확인하세요.
- Face Area를 더 크게 설정하거나 카메라 위치 확인

**원인 2: 조명 부족**
- 촬영 공간의 조명을 개선하세요.
- 역광이 있으면 얼굴이 어둡게 보여 인식 어려움

**원인 3: 얼굴 각도**
- 얼굴이 카메라를 향해야 인식됩니다.
- 옆모습이나 뒷모습은 인식되지 않음

### Entry Area에서 이벤트가 발생하지 않습니다

**확인 사항:**
- Face Area와 Entry Area의 위치 관계 확인
- Entry Area가 Face Area 범주 내에 있는지 확인
- Event Condition이 올바르게 설정되었는지 확인

### 너무 많은 오류 이벤트가 발생합니다

**해결방법:**
- Event Delay Duration을 0초에서 2~3초로 증가
- Face Area 크기 조정 (너무 크지 않게)
- Min Face Size 기준 상향 조정
