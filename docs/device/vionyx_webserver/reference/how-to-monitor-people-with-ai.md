# AI 기능으로 출입자 동선 파악 및 모니터링하기

## 개요

ViOnyx 카메라의 AI 분석 기능을 활용하면 촬영 영상에서 사람들의 동선을 자동으로 추적하고, 얼굴 인식을 통해 누가 지나갔는지 확인할 수 있습니다. 이 가이드는 AI 기능을 설정한 후 실시간으로 사람을 모니터링하는 전체 과정을 단계별로 설명합니다.

## 작업 흐름

AI 기능으로 출입자를 모니터링하려면 다음 단계를 거쳐야 합니다:

1. **Settings에서 AI Analysis 설정** - 추적하려는 구역과 방식 정의
2. **Live View에서 AI 모니터링** - 실시간으로 사람 추적 및 얼굴 인식 확인

---

## 1단계: AI Analysis 설정

### 1.1 AI Analysis 추가하기

1. **Settings** > **AI & Rule** > **AI Analysis**로 이동합니다.
2. 표시된 AI 분석 유형 중에서 필요한 기능을 선택합니다:
   - **Face Movement Tracking** (얼굴 동선 추적) - 얼굴을 인식하여 이동 경로를 추적
   - **Intrusion** (침입 감지) - 특정 구역에 들어오는 사람 감지
   - **Line Crossing** (라인 크로싱) - 가상 라인을 통과하는 사람 감지

> **팁**: 출입자 관리 시에는 **Intrusion**이나 **Line Crossing**과 Face Matching을 함께 사용하면 누가 진입했는지 명확하게 파악할 수 있습니다.

### 1.2 추적 영역 설정하기

AI 분석 유형을 선택한 후, 촬영 영상 위에 추적할 영역들을 그려야 합니다.

**Face Movement Tracking의 경우:**
- **Face Area** (얼굴 인식 영역): 얼굴을 인식할 주요 영역을 최대 10개까지 그립니다.
- **Entry Area** (진입 구역): 추적 대상이 진입하는 영역을 최대 10개까지 그립니다. (선택사항)

**Intrusion의 경우:**
- **Intrusion Area** (침입 감지 구역): 진입을 감지할 구역을 최대 10개까지 그립니다. (필수)

**Line Crossing의 경우:**
- **가상 라인**: 통과를 감지할 라인을 최대 10개까지 그립니다. (필수)

영역을 그릴 때는 마우스로 시작점을 클릭하여 다각형 모양을 만듭니다. 원하는 모양이 만들어지면 마우스를 놓으면 됩니다.

### 1.3 얼굴 인식 활성화하기 (Face Matching)

누가 지나갔는지 확인하려면 **Face Matching**을 활성화해야 합니다.

1. AI Analysis 설정 창에서 **Face Matching** 옵션을 **ON**으로 전환합니다.
2. 얼굴 인식 관련 법규를 확인하는 팝업이 표시됩니다. 내용을 읽고 **동의**를 클릭합니다.
   - ⚠️ 얼굴 인식 기능 사용 시 해당 지역의 관련 법규를 준수해야 합니다.

### 1.4 이벤트 발생 조건 설정하기

각 AI 기능의 동작 방식을 세밀하게 조정할 수 있습니다:

| 옵션 | 설명 | 권장값 |
|------|------|--------|
| **Event Condition** | 진입(Entry) 또는 진출(Exit) 감지 | Entry, Exit 모두 |
| **Event Delay Duration** | 사람 감지 후 이벤트 발생까지의 지연 시간 | 0~5초 |
| **Min Face Size** | 인식할 최소 얼굴 크기 | 기본값 사용 |

**Event Delay Duration**의 의미:
- 0초: 감지되는 즉시 이벤트 발생
- 1~5초: 일정 시간 경과 후 이벤트 발생 (오류 감지 최소화)

### 1.5 설정 저장하기

1. 모든 설정을 완료한 후 **Apply** 버튼을 클릭합니다.
2. 토스트 메시지로 "Event created successfully"가 표시되면 설정이 완료됩니다.
3. 설정된 AI Analysis가 이제 활성화되어 Live View에서 사람 추적을 시작합니다.

## 2단계: Live View에서 AI 모니터링하기

### 2.1 AI 기능 활성화하기

1. **Live** 메뉴로 이동합니다.
2. 우상단의 **AI 버튼**을 클릭하여 AI 메타데이터 표시를 활성화합니다.
   - AI 버튼이 녹색으로 표시되면 AI 기능이 켜진 상태입니다.

### 2.2 실시간 이벤트 알림 확인하기

Live View 화면의 여러 영역에서 AI 분석 결과를 확인할 수 있습니다:

**좌상단 - 이벤트 토스트 알림**
- 사람이 설정된 구역에 진입/진출하면 토스트 메시지로 알림이 표시됩니다.
- 메시지 형식: [이벤트명] [구역명] [IN/OUT]
- 알림은 설정된 시간(기본 1초) 후 자동으로 사라집니다.

**우상단 - 바운딩 박스 (객체 감지 박스)**
- 카메라가 감지한 사람 주변에 박스가 표시됩니다.
- 박스 색상과 라벨은 상태에 따라 변합니다:
  - 일반 박스: ID 번호 표시 (카메라 부여)
  - 초록색 박스: 얼굴 검출 중
  - 파란색 박스: 얼굴 인식 성공 (사용자 이름 표시)
  - 빨간색 박스: 얼굴 인식 실패 (Unknown 표시)

### 2.3 Face Shot 패널에서 인식된 얼굴 확인하기

**Face Shot 패널** (우측 사이드)에는 인식된 모든 얼굴 목록이 시간순으로 표시됩니다.

**패널 구성:**
- 상단: 재생/일시정지, 필터, 전체화면 버튼
- 중앙: 캡처된 얼굴 사진과 인식 정보
- 필터: All Events (기본값), Auth Success, Auth Fail (Unknown)

**표시 정보:**
- **인식 성공**: 일치도(%) + 사용자 이름 + 사용자 ID + 시간
- **인식 실패**: "Unknown Face" + ? 아이콘 + 시간

> **팁**: 필터를 "Auth Fail"로 설정하면 신원을 확인하지 못한 사람들만 볼 수 있습니다. 보안이 필요한 구역에 미인가자가 진입했는지 빠르게 파악할 수 있습니다.

### 2.4 설정된 구역 표시 확인하기

Live View 영상에 설정된 구역들이 오버레이로 표시됩니다:
- Settings에서 그린 Entry Area, Intrusion Area, Line Crossing 라인이 영상 위에 표시됩니다.
- 마우스 호버 시 구역명이 툴팁으로 표시됩니다.

## 3단계: 출입자 추적 활용하기

### 일반적인 사용 사례

**케이스 1: 특정 출입문 모니터링**
- Line Crossing으로 출입문을 통과하는 모든 사람 추적
- Face Matching ON으로 누가 진입/퇴출했는지 기록
- Face Shot 패널에서 진입한 사람들 목록 확인

**케이스 2: 제한 구역 침입 감지**
- Intrusion으로 접근 제한 구역에 대한 진입 감지
- 미인가자(Unknown) 진입 시 즉시 알림 받기
- 필필 필터를 "Auth Fail"로 설정하여 미인가자 사진 기록

**케이스 3: 고객 동선 분석**
- Face Movement Tracking으로 고객의 이동 경로 파악
- 어떤 구역을 거쳐 어디로 이동하는지 시각적으로 확인
- 매장 개선이나 보안 강화에 활용

## 자주 묻는 질문

**Q: Face Matching을 사용했을 때 얼굴을 인식하지 못합니다.**
- 얼굴이 카메라를 향하고 있는지 확인하세요.
- Event Delay Duration을 1~3초 정도로 설정하여 인식 시간을 확보하세요.
- Face Area를 촬영 범위에 맞게 조정하세요.

**Q: 이벤트 토스트가 너무 빨리 사라집니다.**
- Settings > AI & Rule > AI Analysis에서 표시 시간을 1초에서 3~5초로 늘려보세요. (개발팀 확인 예정)

**Q: Live View에서 AI 메타데이터가 보이지 않습니다.**
- 우상단의 AI 버튼이 활성화되어 있는지 확인하세요.
- Settings에서 최소 하나 이상의 AI Analysis가 활성화되어 있어야 합니다.

**Q: Unknown (미인가자)는 무엇입니까?**
- 등록되지 않은 사람으로, 시스템의 얼굴 데이터베이스에 없는 사람입니다.
- 얼굴 인식 기능이 인식 대상을 찾지 못했음을 나타냅니다.

## 관련 용어

| 용어 | 설명 |
|------|------|
| **Face Movement Tracking** | 얼굴 인식 영역에서 사람의 얼굴을 감지하고 이동하는 경로를 추적 |
| **Intrusion** | 설정된 구역에 진입/진출하는 사람 감지 |
| **Line Crossing** | 가상 라인을 통과하는 사람을 지정된 방향으로 감지 |
| **Face Matching** | 감지된 얼굴을 미리 등록된 사용자 데이터와 비교하여 신원 확인 |
| **Entry Area** | Face Movement Tracking에서 추적 동선이 진입하는 구역 |
| **Event Delay Duration** | 사람 감지 후 이벤트 발생까지의 대기 시간 (0~60초) |
| **Face Shot 패널** | Live View의 우측에서 인식된 얼굴들의 목록과 정보를 표시하는 패널 |
| **Unknown** | 얼굴 인식 결과가 가지를 못한 미인가자 |
