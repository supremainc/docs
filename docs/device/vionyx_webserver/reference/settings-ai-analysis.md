h3. 3.6 Face Matching 동작 원리 (공통)
{quote}*⚠️ 개발 참고:* Face Matching ON 시 동작 방식
{quote}
||단계||동작||
|1|이벤트 조건 발생 (진입/진출/크로싱 등)|
|2|Event Delay Duration 대기|
|3|지연 시간 경과 후 *얼굴 매칭 수행*|
|4|매칭 결과와 함께 이벤트 발생|

 

{{[이벤트 조건 감지] ──▶ [Event Delay 대기] ──▶ [얼굴 매칭 수행] ──▶ [이벤트 발생 + 매칭 결과]
 (0~60초) (지연 후 실행)}}
||항목||Face Matching OFF||Face Matching ON||
|이벤트 발생|즉시 (지연 시간 후)|지연 시간 후|
|얼굴 매칭|수행 안 함|*지연 시간 경과 후 수행*|
|결과 데이터|이벤트만|이벤트 + 얼굴 정보 (또는 Unknown)|

h2. 4. Face Movement Tracking (얼굴 동선 추적)
----
h3. 4.1 기능 설명
||항목||내용||
|목적|얼굴 인식 + 동선 추적|
|동작|얼굴 인식 영역에서 얼굴 감지 → 진입 구역 이동 추적 → 이벤트 발생|

|신규 생성|기존 영역보다 약간 작게 생성|

--> 10개 확정
{quote}

{quote}*� [개발팀 확인 필요]* Counting Area 최대 개수 확정 (30개?)

h3. 7.3 영역별 설정
||항목||설명||옵션/범위||
|Color|영역 표시 색상|흰색(기본), 초록색, 빨강색, 파랑색, 회색, 검정색|

h3. 7.4 이벤트 조건 (Condition)
||조건||설명||기본값||

||Count Limit||범위||
|기본값|10명|
|최대|100명|
{quote}*� [개발팀 확인 필요]* Count Limit 최대값 (예: 100명? 999명?)
{quote}
----
h3. 7.5 Duration (지연 시간)
||항목||설명||
|동작|개별 인원이 해당 시간 이상 머물러야 카운트|

|범위|0 ~ 60초|
|기본값|0초|
