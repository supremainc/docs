# AI 규칙(Rule) 설정하기

AI 이벤트가 발생했을 때 어떤 동작을 실행할지 설정합니다. 이 문서에서는 규칙 추가, 조건 설정, 동작 설정(오디오/웹훅), 저장 방법을 안내합니다.

## 상황/목적

다음과 같은 상황에서 AI 규칙을 설정하세요.

- 특정 AI 이벤트가 발생할 때 외부 서버로 웹훅을 전송하고 싶을 때
- 특정 AI 이벤트가 발생할 때 장치에서 오디오를 재생하고 싶을 때
- 이벤트 유형별로 다른 후속 동작을 자동화하고 싶을 때

## 사전 준비

규칙 설정을 시작하기 전에 다음을 확인하세요.

- AI 이벤트가 먼저 생성되어 있어야 합니다.
- 설정 메뉴에서 AI 규칙 화면에 접근할 수 있어야 합니다.
- 웹훅을 사용할 때 수신 URL을 미리 준비하세요.
- 오디오를 사용할 때 재생할 오디오 클립을 미리 준비하세요.

## 실행 방법

### 1. 규칙 설정 화면 열기

1. 상단 메뉴에서 설정을 클릭하세요.
2. 왼쪽 메뉴에서 AI & 규칙 > 규칙 설정을 클릭하세요.
3. 규칙 목록 화면 오른쪽 상단의 + 추가 버튼을 클릭하세요.

### 2. 규칙 이름 입력하기

1. 규칙 추가 화면에서 이름을 입력하세요.
2. 이름은 필수 항목입니다.

### 3. 조건(Condition) 설정하기

#### 3-1. 채널 확인

- 채널은 Ch 1로 고정됩니다.
- Fisheye 뷰만 지원됩니다.

#### 3-2. 유형 선택

유형에서 다음 중 하나를 선택하세요.

- 사람 추적
- 침입 감지
- 라인 통과 감지
- 인원 카운팅

#### 3-3. 이벤트 선택

1. 이벤트 항목에서 사용할 AI 이벤트를 선택하세요.
2. 이벤트 목록에는 선택한 유형에 해당하는 AI 이벤트가 표시됩니다.
3. 이벤트 표시는 (ID:N) AI event - 이벤트명 형식으로 표시될 수 있습니다.

#### 3-4. 유형별 추가 조건 설정

유형에 따라 추가 항목을 설정하세요.

- 사람 추적: 진입 영역을 선택할 수 있습니다.
- 침입 감지: 침입 영역을 선택하세요.
- 라인 통과 감지: 이벤트에 연결된 라인을 선택하세요.
- 인원 카운팅: 이벤트에 연결된 카운팅 영역을 선택하세요.

#### 3-5. 이벤트 유형 선택

사람 추적을 선택한 경우 이벤트 유형에서 얼굴 인식, 진입 영역을 선택할 수 있습니다.

- 진입 영역을 선택하지 않았거나 진입 영역이 없을 때: 얼굴 인식이 자동 선택되며 해제할 수 없습니다.
- 진입 영역을 선택했을 때: 얼굴 인식 또는 진입 영역 중 최소 1개를 선택하세요.

### 4. 동작(Action) 설정하기

하나의 규칙에 여러 동작을 추가할 수 있습니다.

1. 동작 섹션에서 유형을 선택하세요.
2. 필요하면 동작 스위치를 켜서 활성화하세요.
3. - 추가 버튼으로 동작을 추가하세요.

지원 동작 유형:

- 오디오
- 웹훅

### 5. 오디오 동작 설정하기

유형에서 오디오를 선택한 뒤 다음 항목을 설정하세요.

- 오디오 클립: 재생할 클립 선택(필수)
- 반복: 반복 횟수 설정
- 일정: 기본값 항상

### 6. 웹훅 동작 설정하기

유형에서 웹훅을 선택한 뒤 다음 항목을 설정하세요.

- 메소드: POST 고정
- URL: http:// 또는 https:// 주소 입력(필수)
- 헤더: 기본값 Content-Type / application/json
- 바디: 전송할 필드 선택 또는 사용자 항목 추가
- 일정: 기본값 항상

#### 6-1. 헤더 설정

- 기본 헤더 1개가 제공됩니다.
- - 추가 버튼으로 헤더를 추가할 수 있습니다.
- 헤더는 최대 8개까지 설정할 수 있습니다.

#### 6-2. 바디 설정

바디에서 기본 제공 항목을 체크해 전송할 수 있습니다.

예시 항목:

- 외부 사용자 ID(user_id)
- 카메라 시리얼(device_serial)
- 이벤트 타임스탬프(event_at)
- 이벤트 타입(event_type)
- 영역 ID(area_id)
- 진입/퇴장 방향(direction)

추가 규칙:

- 바디는 최소 1개 이상 선택 또는 입력해야 합니다.
- - 추가 버튼으로 사용자 정의 항목을 추가할 수 있습니다.
- 사용자 정의 항목은 비워 둘 수 없습니다.
- 사용자 정의 항목은 최대 10개까지 추가할 수 있습니다.

### 7. 저장하기

1. 설정을 완료한 뒤 적용을 클릭하세요.
2. 변경을 취소하려면 취소를 클릭하세요.
3. 저장에 성공하면 규칙 목록에 새 규칙이 추가됩니다.

## 상세 정보

### 규칙 목록에서 확인 가능한 항목

- 켜짐/꺼짐: 규칙 활성화 상태
- 이름: 규칙 이름
- 유형: 연결된 AI 이벤트 유형
- 이벤트: 연결된 AI 이벤트 이름
- 일정: 현재 일정(기본 항상)
- 수정: 규칙 수정 또는 삭제

### 규칙 ON/OFF 동작

- OFF에서 ON으로 변경하면 동작이 다시 실행됩니다.
- ON에서 OFF로 변경하면 동작이 실행되지 않습니다.

### 규칙 삭제

- 삭제 전에 확인 메시지가 표시됩니다.
- 삭제한 규칙은 복구할 수 없습니다.

## 참고

- 채널은 현재 Ch 1로 고정되어 있습니다.
- 일정은 현재 항상만 지원합니다.
- 웹훅 발신 기록과 응답 코드는 장애 분석을 위해 시스템 로그에서 확인할 수 있어야 합니다.
- 인원 카운팅은 별도 조회 API(GET)로 현재 카운트 값을 조회할 수 있습니다.
