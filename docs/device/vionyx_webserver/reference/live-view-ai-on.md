h1. AI Live View 개발 사양서
h2. 1. 개요

AI 카메라의 실시간 영상에 AI 분석 결과(객체 인식, 얼굴 인증, 이벤트 알림 등)를 오버레이하여 표시하는 Live View 화면 개발
----
h2. 2. 화면 구성
h3. 2.1 메인 레이아웃
||영역||위치||설명||
|비디오 뷰어|중앙|Fisheye 카메라 영상 표시|
|상단 네비게이션|상단|Live, Settings, People, System 탭 + 카메라 정보 (모델명, IP)|
|AI 토글 버튼|우상단|AI 메타데이터 표시 ON/OFF|
|박스 토글 버튼|우상단|이벤트/사용자 박스 표시 여부 및 표시 시간 설정|
|Face Shot 패널|우측 사이드|얼굴 인증 결과 리스트 (접기/펼치기 가능)|
|이벤트 토스트|좌상단|구역 진입 등 이벤트 발생 시 팝업 (스택 형태로 누적)|
|사용자 정보 박스|우하단|인증된 사용자 정보 표시|
----

h3. 3.2 구역 표시 (Entry Area)

*기능:*
 * 설정된 진입 구역을 영상 위에 오버레이
 * 마우스 호버 시 구역명 툴팁 표시
 * 얼굴 인증 영역은 별도 표시 없음
 * 복수 구역 겹침 시 별도 우선순위 처리 없이 그대로 표시

*이벤트 발생 시 동작:*
 # 구역 마크 색상 변동 + 깜빡임 애니메이션
 # 좌상단 토스트 메시지 표시
 # 우하단 사용자 정보 박스 표시 (FACE MATCH ON인 경우만)

*토스트 메시지 내용:*
 * 이벤트명
 * 구역명
 * IN/OUT 구분

구역 표시 색상, 깜빡임 효과 스타일
{panel}

h3. 3.3 Face Shot 사이드 패널

*표시 조건:*
 * AI Analysis 설정됨
 * Intrusion / Line Crossing / People Counting 중 하나라도 FACE MATCH ON

*패널 구성:*
{code:java}
┌─────────────────────┐
│ [▶❚❚] [필터▼] [⛶] [◀] │ ← 컨트롤 바
├─────────────────────┤
│ ┌───┐ Unknown Face │
│ │ ? │ 13:36:21 │
│ └───┘ 2025/09/09 │
├─────────────────────┤
│ ┌───┐ 85% Matching │
│ │사진│ Tim berland │
│ └───┘ 2201310007 │
│ 13:40:03 │
└─────────────────────┘
{code}
*리스트 아이템 정보:*
||필드||인증 성공||인증 실패||
|썸네일|캡처 이미지|캡처 이미지|
|상태|"XX% Matching"|"Unknown Face|}|
|이름|사용자 이름|-|
|ID|사용자 ID|-|
|시간|HH:mm:ss|HH:mm:ss|
|날짜|YYYY/MM/DD|YYYY/MM/DD|
|아이콘|프로필 이미지 (우하단)|? 아이콘|

*필터 옵션:*
 * All Events (기본값)
 * Auth Success
 * Auth Fail (Unknown)

*컨트롤:*
 * 재생/일시정지: 실시간 업데이트 토글
 * 전체화면 보기
 * 접기/펼치기

*정렬:* 시간 역순 (최신순)

h3. 3.4 이벤트 토스트 (좌상단)

*동작 방식:*
 * 이벤트 발생 시 새 토스트가 상단에 추가되며 기존 토스트는 아래로 밀림 (스택 구조)
 * 설정된 표시 시간 경과 후 개별 토스트 자동 사라짐

*표시 시간 설정:*
 * 기본값: 1초
 * 설정 범위: 1초 ~ 5초

*최대 표시 개수:* 제한 필요 (개수 TBD)
{panel:title=� 디자인 확인|borderColor=#cccccc|titleBGColor=#f5f5f5|bgColor=#f5f5f5}
토스트 크기, 간격, 사라짐 애니메이션
{panel}
----
h3. 3.5 사용자 정보 박스 (우하단)

*표시 조건:* FACE MATCH ON + 구역 진입 이벤트 발생

*표시 정보:*
 * 프로필 사진
 * 사용자 이름
 * 사용자 ID
 * 발생 시간 (HH:mm:ss)
 * IN/OUT 표시

*비인가자 진입:*
 * "Unknown" 표시
 * ? 아이콘

*다중 이벤트:* 복수 발생 시 여러 박스 표시

*Entry Area 설정에 따른 표시:*
 * IN/OUT 모두 설정: 둘 다 표시
 * IN만 설정: IN만 표시

*표시 시간 설정:*
 * 기본값: 1초
 * 설정 범위: 1초 ~ 5초

{panel:title=� 디자인 확인|borderColor=#cccccc|titleBGColor=#f5f5f5|bgColor=#f5f5f5}
복수 박스 배치 방식, 박스 크기
{panel}

*이벤트 발생 조건:*

----
h3. 3.7 Intrusion / Line Crossing
 * 구역 진입 이벤트와 동일한 UI 패턴 적용
 * 이벤트 발생 시 토스트 + 사용자 정보 박스 표시

----
h3. 3.8 AI 메타 보기 토글

*위치:* 우상단 AI 버튼

*기능:*
 * ON: 바운딩 박스, 구역, 카운트 등 모든 AI 메타데이터 표시
 * OFF: 순수 영상만 표시

*기본값:* ON (Enabler 디폴트 켜기)
----
h3. 3.9 오류 상태 처리

*표시 조건:* 카메라 연결 끊김, AI 분석 실패 등 예외 상황 발생 시

*표시 방식:*
 * 비디오 영역 중앙에 로딩 인디케이터(서큘레이터) 표시
 * 에러 메시지 또는 팝업 표시

h2. 4. 화면 상태별 표시 조건
||조건||Face Shot 패널||사용자 정보 박스||토스트||
|AI OFF|숨김|숨김|숨김|
|AI ON + FACE MATCH OFF|숨김|숨김|표시|
|AI ON + FACE MATCH ON|표시|표시|표시|
----
h2. 5. 데이터 처리 흐름
{code:java}
카메라 AI 분석 → 객체 인식 → RE:ID 부여 → 얼굴 검출 → 얼굴 인증(성공/실패) 
→ 구역 진입 감지 → 이벤트 발생 (웹훅 송신) → UI 업데이트
{code}
