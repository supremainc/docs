## Pre-requisites

* Software House C•CURE 9000 (versions 2.90, 3.0, 3.10)

* Suprema BioStar 2 (version 2.9.0 and later)

:::info

You must install BioStar 2 server and C•CURE 9000 Server in the SAME machine.

:::

## Considerations

* For specific instructions on how to install and configure C•CURE 9000 or BioStar 2, please refer to the installation manual for each of these platforms.

* If you are using MS SQL 2014 Express (a newer version is better), install the Service Pack 2 by referring to the following web page.

   > https://www.microsoft.com/en-us/download/details.aspx?id=53168

* Please bear in mind that due to security reasons this integration does not support HTTP, only HTTPS (over TLS 1.2) then please make sure your BioStar 2 is set to handle HTTPS only.

* The default value for the ports used are as follows. If another program uses the same port the pre-requisites or this integration may not work properly.

   * Suprema - C•CURE Integration

      <div className='overflow-x cols3--20-20-60'>
      | Type | Port | Description |
      | :--: | :--: | -- |
      | Port | 7733 | HTTPS over TLS 1.2 |
      | Port | 51211 | Suprema Devices |
      </div>

   * Suprema BioStar 2 backend service

      <div className='overflow-x cols3--20-20-60'>
      | Type | Port | Description |
      | :--: | :--: | -- |
      | Port | 443 | HTTPS over TLS 1.2 |
      | Port | 3312 | When MariaDB schema is used. |
      | Port | 9002 | Web Socket |
      </div>

   * C•CURE 9000 v2.9 or v3.0

      <div className='overflow-x cols3--20-20-60'>
      | Type | Port | Description |
      | :--: | :--: | -- |
      | Port | 161 | For collecting and organizing information about devices. |
      | Port | 2901/2 | iSTAR Ultra G2 GCM listening port for IP-ACM2 SSL encrypted connection with default certificate/key. |
      | Port | 28014 | TLS 1.3 Signed Certificate delivery port. iSTAR Ultra G2 opens the port. Host delivers the signed certificates. |
      </div>

   * Microsoft SQL server
      <div className='overflow-x cols3--20-20-60'>
      | Type | Port | Description |
      | :--: | :--: | -- |
      | Port | 1433 | Or as configured |
      </div>

* The data is transferred under TLS 1.2 with private certificates, sensitive data (user, passwords, configurations) besides the encryption in transport layer, are stored encrypted.

* Before Installation starts, please ENSURE that MS SQL is being configured to:

   * Allow IP connections in the desired port (1433 default), from the desired IP address(es) as shown in the following images.

      <Image src='/img/integration/ccure/tcp-ip-properties1.png' alone />

      <Image src='/img/integration/ccure/tcp-ip-properties2.png' alone />

   :::info

   A PC restart/reboot might be required for the new settings to take effect in MS SQL.

   :::

* Allow SQL Authentication:

   * In the SQL Server Management Studio: Object Explorer, right click on the **Server** → **Properties** → **Security** → Select **SQL Server and Windows Authentication mode** → **Apply**.

      <div className='grid-container'>
      <Image src='/img/integration/ccure/allow-sql-authentication1.png' alone />
      <Image src='/img/integration/ccure/allow-sql-authentication2.png' alone />
      </div>

   * Windows Authentication and SQL Authentication (Local and Domain based) are supported for this integration.

   * For Windows Authentication please be aware that the current Windows user account (where the integration has been installed) must have clearance to C•CURE databases, either for local or domain users. Otherwise, the Integration CT will fail to connect.

   * Please mind specific Database configurations when integrating with Active Directory, if you get any issue with the Database connection, please open a ticket to request technical support.

This guide is for the ADVANCED version.

## Login

The Suprema - C•CURE Integration Configuration Tool is a web-based tool which you can access from anywhere in your Network as long as you have access to the machine where it was installed.

Through this Configuration Tool you can adjust many settings from this integration.

## First run

1. Run your web browser.

   * We recommend Brave Browser, Microsoft Edge, or Google Chrome browser.

1. The Suprema - C•CURE Integration Configuration Tool can be accessed by:

   > https:\//BioStar 2_server_IP_or_Domain_Name:PORT/modules/SupremaCCURE/index.html

   Where the PORT should be the same as in the BioStar 2 Server configuration. You can find the appropriate port by checking the "BioStar 2 Settings" app from the Start Menu on the PC where you’ve installed Both C•CURE 9000 Server and BioStar 2 Server.

   > https://127.0.0.1:443/modules/SupremaCCURE/index.html

   :::info

   Troubles finding the IP address or domain name? Please refer to the Troubleshooting section.

   :::

1. You shall login with the Suprema Administrator credentials, the administrator account ID is *admin* and the password is the one you configured when installing BioStar 2 server.

1. When you log in for the first time, you may see the "*Not secure*" warning in screen, this is because private (auto signed) certificates are used, please click **OK** and you’ll see the following login screen.

   <Image src='/img/integration/ccure/first-run1.png' width={400} alone />

   If right after the previous screen shows, the following notice shows:

   <Image src='/img/integration/ccure/first-run2.png' width={400} alone />

   Please go to the troubleshooting section: what to do when the **Service Not running** notice shows in the Integration CT.

1. In the first run or when you do not have a valid license in the system, you will see the following notice on your screen:

   <Image src='/img/integration/ccure/first-run3.png' width={400} alone />

1. The Integration Configuration Tool shall show a warning message saying you need to go to **System Settings**.

1. Once in the **System Settings** you will be able to activate your license(s).

<Image src='/img/integration/ccure/adding-devices1.png' width={400} alone />

Right next to the device icon you’ll see the device status, if green means the devices is online and well, if red the device is not online.

You will also visualize the Device ID, specific model, and the name with which was registered.

If you want to add more devices, you can refer to the column on the right side of your screen:

<Image src='/img/integration/ccure/adding-devices2.png' width={400} alone />

After clicking the **Search** button.

<Image src='/img/integration/ccure/adding-devices3.png' width={400} alone />

The results may appear as shown in the following image.

<Image src='/img/integration/ccure/adding-devices4.png' width={500} alone />

Now, the Normal search will perform a UDP search for Suprema devices on your network, while the Advanced Search:

<Image src='/img/integration/ccure/adding-devices5.png' width={400} alone />

These operations are available for your convenience as a way to cold-start with the Integration if you have an system with a number of existing users and credentials under CCURE.

### Pre-Enrollment

The *Pre-Enrollment* Menu is available in the Integration Configuration Tool:

<Image src='/img/integration/ccure/main-ico-pre-enrollment.png' width={140} alone />

The Data shown in this dialog will allow you to select individually or massively send an E-mail to users to enable Facial Biometrics Pre-Enrollment from their mobile devices.

<Image src='/img/integration/ccure/user-available-pre-enrollment1.png' width={800} alone />

Please be aware that although you select a user if the selected user(s) doesn’t have an email address already set in their profiles, they will not be included in the process.

This feature also validates that all needed pre-requisite settings have been already configured, if not it will show you a feedback window for you to know what you still need to configure prior sending the Pre-enrollment links, for example:

<Image src='/img/integration/ccure/user-available-pre-enrollment2.png' width={400} alone />

With this feedback you can direct yourself to the specific setting and revise your configuration, after you do that, you can select the <Image src='/img/menus/ico-send-pre-enrollment-link.png' ico alone /> button once again and the process will flow correctly.

After the process have finished you will be able to receive an email as the one show below:

<Image src='/img/integration/ccure/user-available-pre-enrollment3.png' alone />

You should follow the instructions given in the URL shown in the email message, and once it is done your pre-enrollment facial credential will be ready the first time you step in front of any supported Suprema device.
