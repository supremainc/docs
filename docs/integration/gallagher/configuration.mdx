### Gallagher Command Centre

:::info

Process to generate Gallagher API Key.

:::

1. Open Gallagher Command Centre, set the Logon name as a system, and press the **OK** button.

   <Image src='/img/integration/gallagher/command-centre1.png' alone />

1. **Set** Logon Name as a **system** and set the password for Gallagher Command Centre and press **OK**.

   <Image src='/img/integration/gallagher/command-centre2.png' alone />

1. Now, go to Administration from the top left corner and **select** Configuration Client, the window will be open.

   <Image src='/img/integration/gallagher/command-centre3.png' width={800} alone />

1. The below tab will be open.

   <Image src='/img/integration/gallagher/command-centre4.png' width={600} alone />

1. To configure server properties, **Select File** → **Server Properties**.

   <Image src='/img/integration/gallagher/command-centre5.png' width={600} alone />

1. The new window will open, now go to **Web Services**, in the **REST API** section, enable local network connection, **set** the server port as **8904**, **uncheck** the Enable REST client with no client certificate, and **press** the **OK** button.

   <Image src='/img/integration/gallagher/command-centre6.png' width={600} alone />

1. **Click** over to **Gallagher Client**.

   **Click** on **Manage**. Within the Dropdown, please **select Operator Groups**.

   <Image src='/img/integration/gallagher/command-centre7.png' width={600} alone />

1. The Operator Group window will open, **press right-click** and select **New** → **Operator Group**.

   <Image src='/img/integration/gallagher/command-centre8.png' alone />

1. In the general setting, the name should be **set** as **REST Client Operator Group**. Then Click **Apply**.

   <Image src='/img/integration/gallagher/command-centre9.png' alone />

1. Now **select** the **Operator Privileges** setting on Operator group and go to the Gallagher configuration client window Manage → Operator Privileges. Now **drag and drop**.

   You need to add 3 privileges to the Operator Group (Add Card or Credential, Edit Cardholders, View Cardholders) from operator privileges to operator group and **press** the **OK** button, and **close** the operator privileges.

   <Image src='/img/integration/gallagher/command-centre10.png' width={600} alone />

   <Image src='/img/integration/gallagher/command-centre11.png' width={800} alone />

1. Go to Configuration Client and **select Cardholders**.

   <Image src='/img/integration/gallagher/command-centre12.png' width={600} alone />

1. **Press right-click** New → Cardholder.

   <Image src='/img/integration/gallagher/command-centre13.png' alone />

1. In the general setting, the First Name as **REST Client**, and **press Apply**.

   <Image src='/img/integration/gallagher/command-centre14.png' alone />

1. **Click** on the **Operator Configuration** Tab on the left side within the REST Client Properties. **Drag and Drop** the REST Client operator group that we have created from Operator Group to Operator Group Membership.

1. **Drag** the REST Client Operator Group over to Operator Group Membership: like the image below:

   <Image src='/img/integration/gallagher/command-centre15.png' width={800} alone />

1. **Set** the Logon name to **REST** and **set** the password the same as we set for Gallagher Command Centre. **Click OK**, and also **Close** the Operator Group window.

   <Image src='/img/integration/gallagher/command-centre16.png' width={600} alone />

1. On **Configuration** Client go to Configure → Services & Workstations.

   <Image src='/img/integration/gallagher/command-centre17.png' width={600} alone />

1. Now **right-click** New → REST Client.

   <Image src='/img/integration/gallagher/command-centre18.png' alone />

   <Image src='/img/integration/gallagher/command-centre19.png' width={800} alone />

1. The Gallagher API key is there, in the API key section.

   :::info
   
   Save the Gallagher API Key in a safe place.

   :::

   <Image src='/img/integration/gallagher/command-centre20.png' width={500} alone />

1. To get the thumbprint, go to the `C:\Program Files\Suprema Integration with Gallagher\Prerequisites`, open the thumbprint file, and copy the value of the thumbprint inside it.

   :::info
   
   If the thumbprint file is missing. If thumbprint is not generated while running the EXE file.

   <Image src='/img/integration/gallagher/command-centre21.png' width={600} alone />

## Hardware configuration

:::info

* Skip to [BioStar 2 Setup](#biostar2-setup) if the hardware has already been configured.

* We are using controller 6000 as a reference, if you have another controller please do the steps accordingly.

:::

1. First, we have to create controller 6000 (or any other controller you have) to **connect** to the Gallagher controller.

   Open the Gallagher Configuration Client window and go to Configure> Hardware.

1. In the hardware window press **right-click** then **New** → **Controllers** → **Controller 6000** (if you have a different controller it will display here) and **press apply**. It will create a controller in hardware.

   <Image src='/img/integration/gallagher/configuration-hw1.png' alone />

1. Then **go back** to Gallagher Configuration Client and go to configure → Alarm Zones and press **right-click** to create a new alarm zone.

   <Image src='/img/integration/gallagher/configuration-hw2.png' alone />

1. To configure the Alarm zone in General settings change the name to **System Alarm Zone** and **press** the **OK** button.

   <Image src='/img/integration/gallagher/configuration-hw3.png' alone />

1. Let’s configure Controller 6000, In the General setting **set** the name as **Controller 6000** (set name same as your controller).

   <Image src='/img/integration/gallagher/configuration-hw4.png' width={600} alone />

1. Now **go to** Event Response and select Alarm Zone for all events and select **System Alarm Zone**. And **press** the **Apply** button.

   <Image src='/img/integration/gallagher/configuration-hw5.png' width={600} alone />

   Now go to System Alarm Zone Properties and **select** Connections and in the controller dropdown **choose Controller 6000**. **Press** the **OK** button and **close** the Alarm Zone window.

   <Image src='/img/integration/gallagher/configuration-hw6.png' alone />

1. Now, we have to **create** an Access Zone in configure.

   Go to the Access Zone and the window will open up. Now **Press right-click** in the Access Zone Window and **Select** New → Then **Click** on Access Zone, and it will create an access zone.

   <Image src='/img/integration/gallagher/configuration-hw7.png' alone />

1. **Open** the Access Zone Properties that we have created in the General Setting. Then **set** the name as **Suprema Access Zone**.

   <Image src='/img/integration/gallagher/configuration-hw8.png' alone />

1. **Go to** Access Mode Tab Setting. **Select** Schedule as Default Access Zone Secure and **Press** the **OK** button.

   <Image src='/img/integration/gallagher/configuration-hw9.png' alone />

1. Now **go to** the Hardware Window then **right-click** on Controller 6000. Select New → HBUS Devices → HBUS I/O. This step will create HBUS under Controller.

   <Image src='/img/integration/gallagher/configuration-hw10.png' width={600} alone />

1. Let's Configure the HBUS that we have created by **double-clicking** on it to edit the properties. In the general setting, **set** the name as **HBUS I/O Wiegand Module (Suprema)**.

   **Go to** the Connections Tab and **choose** Controller as Controller 6000, set Hardware Variant as 4 in 2 out, and then press the **OK** button.

   <Image src='/img/integration/gallagher/configuration-hw11.png' width={600} alone />

1. To connect your Gallagher Controller to the system, **open** Controller 6000 properties and **Click** on the Server Address Tab. Tick use different IP for the server and type your PC/ServerIP address. Then **Press** the **Apply** button.

   <Image src='/img/integration/gallagher/configuration-hw12.png' width={600} alone />

1. In the Controller Address setting set your controller MAC address (Please look at your C6000 controller), Also **set** the IP address assigned for the Controller, Subnet, and Gateway. **Press** the **Apply** button.

   <Image src='/img/integration/gallagher/configuration-hw13.png' width={600} alone />

1. In the Setup Tab, set Plug in Module as **4H Module** (choose according to your controller) and in Hardware RS485 Bus 1 as **OSDP** and RS485 Bus 2 as **HBUS** and Press Apply.

   <Image src='/img/integration/gallagher/configuration-hw14.png' width={600} alone />

   :::info

   Your Setting could vary depending on your Gallagher devices & components.

   :::

1. In the Status and Overrides options **tick** both checkboxes Disable Front Tamper and DisableRear Tamper.

   <Image src='/img/integration/gallagher/configuration-hw15.png' width={600} alone />

1. To connect your controller to the system, **turn off** the controller, and wait until the red light goes off. Then **turn on** the 2nd button from the left.

   :::info

   Before performing this step, please ensure that your PC firewall is turned off.

   :::

   <Image src='/img/integration/gallagher/configuration-hw16.png' width={400} alone />

1. When the status on the controller turns normal then **turn off** 2nd button (down/dip the 2nd button).

   :::info
   
   Once the controller is online and normal, you can turn all dip switches off including 2.

   :::

   <Image src='/img/integration/gallagher/configuration-hw17.png' width={400} alone />

   **Additional note first time power-up and new controller connection:**

   You only need \#2 on when you first power up a new controller and connect to a server.

   Also, Jumper \#2 can be used when doing some resetting of the controller. You want it to be fixed if you are doing upgrades to the controller.

   <Image src='/img/integration/gallagher/configuration-hw18.png' width={600} alone />

   :::info
   
   Connect Devices to access HBUS in the controller HBUS setting. (To connect devices please refer to BioStar 2 setup)

   :::

1. In the HBUS Devices option of Controller 6000 **drag and drop** HBUS I/O 4 in 2 Out module - Wiegand from **Plugin Modules** to **Undetected devices** HBUS I/O.

   <Image src='/img/integration/gallagher/configuration-hw19.png' alone />

1. To Create Door go to Configure, Door then **press right-click** and create a new door.

   <Image src='/img/integration/gallagher/configuration-hw20.png' alone />

1. **Set** the Door name as **Suprema Door** and go to Event Response and **choose** alarm as System Alarm Zone.

   <Image src='/img/integration/gallagher/configuration-hw21.png' alone />

1. In **Connections select** Controller as Controller 6000 and Press **Apply** and **OK**.

   <Image src='/img/integration/gallagher/configuration-hw22.png' alone />

1. To create an **OSDP connection; Right-Click on the Hardware window. Select New** → **OSDP Device** → **Then click on OSDP Reader**, to create OSDP Reader on Hardware.

   A new window opens up. **Set** its name as Suprema OSDP Reader, and in the Event Response settings, Tab **set** Alarm Zone for all events: as System Alarm Zone. Then **Click** Apply & **Press** OK to save the changes.

   <Image src='/img/integration/gallagher/configuration-hw23.png' alone />

   **Go back** to the Hardware Window. **Double Click** on Controller 6000 for the Properties. **Click** on OSDP Devices Tab.

1. **Drag** the OSDP reader from the **Hardware Window to** the **Controller properties** in the OSDP Devices Tab. **Set** Configured Address as 0, Suprema OSDP Device Address as 0, and Baudrate as **115200**.

   <Image src='/img/integration/gallagher/configuration-hw24.png' width={600} alone />

   :::info

   If you have a 2x OSDP Reader please change the OSDP address within BioStar 2.

   :::

1. To create the Wiegand Reader. **Right-click** on the Hardware Window. Click on New → Third-Party Reader. A New Window appears; **Set** its name as Suprema Wiegand Reader, then in the Event Response Tab. **Choose** the Alarm Zone as System Alarm Zone and **press OK**.

   <Image src='/img/integration/gallagher/configuration-hw25.png' alone />

1. **Go to** the Door that we have created in properties **Entry Zone** set Access Zone as **Suprema Access Zone** and reader as **Suprema Wiegand Reader** and **Suprema OSDP Reader**, and **press** the **OK** button.

   <Image src='/img/integration/gallagher/configuration-hw26.png' alone />

1. In **HBUS I/O** properties go to **Reader** settings and set Reader 1 as Suprema Wiegand Reader, **press** the **OK** button. **Close** the door and Controller window.

   <Image src='/img/integration/gallagher/configuration-hw27.png' width={600} alone />

1. To create a card type Configure → Card Type **right-click** and create a new **Card Type**. **Set** its name as **Suprema Card Type**.

   :::info

   * A QR Device license is needed to read the Gallagher QR Code. Please contact Suprema for it.

   * Please use the same terms as mentioned for the fields.

   :::

   <Image src='/img/integration/gallagher/configuration-hw28.png' alone />

1. In Card type properties **go to** setup option Region code as **A** and Facility code as **10000** card number range as **1 to 99999999999999999999 (20 times)** and enable card expiry notification. **Press OK**.

   <Image src='/img/integration/gallagher/configuration-hw29.png' width={600} alone />

1. Create another card type Configure → Card Type **right-click** and create a new **Card Type**. set its name as **HID Seos Corporate**. And **go to the** properties(setup) to set up card type. Use the below configurations to configure the card and **press** Apply and **OK**.

   <Image src='/img/integration/gallagher/configuration-hw30.png' width={600} alone />

1. Create another card type Configure → Card Type **right-click** and create a new **Card Type**. Set its name as **Suprema QR Code**.

   <Image src='/img/integration/gallagher/configuration-hw31.png' alone />

1. In Card type properties **go to** setup option Region code as **A** and Facility code as **10000** card number range as **4194304 to 8388607** and enable card expiry notification. **Press OK**.

   <Image src='/img/integration/gallagher/configuration-hw32.png' width={600} alone />

1. To create a universal card type Configure → Universal Card Format **right-click** and create a new universal card format. We have to make 4 card formats.

   1. **Create** a new format name as **Suprema 26 bits** Wiegand and set the format as the below image:

      <Image src='/img/integration/gallagher/configuration-hw33.png' alone />

   1. **Create** a second card format as **Suprema CSN 32 bits**, and in the format **set** as shown below.

      <Image src='/img/integration/gallagher/configuration-hw34.png' alone />

   1. **Create** a third card format as **Suprema CSN 56 bits**, and in the format **set** as shown below.

      <Image src='/img/integration/gallagher/configuration-hw35.png' alone />

   1. **Create** a fourth card format as **Suprema Virtual Card (25bit)**, and in the format **set** as shown below.

      <Image src='/img/integration/gallagher/configuration-hw36.png' alone />

   1. **Create** a fifth card format as **HID 48bit**, and in the format **set** as shown below.

      <Image src='/img/integration/gallagher/configuration-hw37.png' alone />

   1. **Create** a sixth card format as **Suprema QR 23 bits**, and in the format **set** as shown below.

      <Image src='/img/integration/gallagher/configuration-hw38.png' alone />

1. **Drag** all six card types and drop them on Controller properties, Card Format Setting, and **Apply** the settings. **Press** the **OK** and **Close** all the windows of Gallagher Configuration Client.

   <Image src='/img/integration/gallagher/configuration-hw39.png' width={600} alone />

1. **Go to** Config Client Configure → Hardware.

   <Image src='/img/integration/gallagher/configuration-hw40.png' width={600} alone />

   Then **Right Click** on the Window and **create** output and **set** the name as **Wiegand Output** in the general tab.

   <Image src='/img/integration/gallagher/configuration-hw41.png' alone />

1. Now **Click** on Advance on the left side **set** 1 second in the Output times option and **press Apply** and **OK**.

   <Image src='/img/integration/gallagher/configuration-hw42.png' alone />

1. Now **go to** hardware window → **double click** on HBUS I/O Wiegand Module (Suprema) on the left side **go to** Outputs and **select** Wiegand Output **press Apply** and **OK**.

   <Image src='/img/integration/gallagher/configuration-hw43.png' width={600} alone />

## BioStar 2 Setup {#biostar2-setup}

1. To open BioStar 2 **type** `https://[Your system IP Address]/#/login` in your browser it will open the login form.

   <Image src='/img/integration/gallagher/biostar2-setup1.png' width={800} alone />

1. **Fill** in your BioStar 2 credentials to log in, **go to** the setting page, and **select** Card Format.

   <Image src='/img/integration/gallagher/biostar2-setup2.png' width={800} alone />

1. In the card, setting **click** on the **edit** button to create a new card format.

   <Image src='/img/integration/gallagher/biostar2-setup3.png' width={800} alone />

1. Here also we have to create three card types **56-bit**, **32-bit**, **and 26-bit**.

   1. **Set** name as **56-bit Gallagher** total bits as 56 and in ID field **press** add **set** start bit as 0 and end bit as 55.

      <Image src='/img/integration/gallagher/biostar2-setup4.png' width={800} alone />

   1. **Set** name as **32-bit Gallagher** total bits as 32 and **ID field** and in the ID field **press** add & **set** start bit as 0 and end bit as 31.

      <Image src='/img/integration/gallagher/biostar2-setup5.png' width={800} alone />

   1. **Set** the name as **26-bit Gallagher** total bits as 26 and **ID field** and in the ID field **press** add & **set** the start bit as 0 and end bit as 25.

      <Image src='/img/integration/gallagher/biostar2-setup6.png' width={800} alone />

1. Now inside the card format **go to** the smart card section by **pressing** the smart card button and **click** on the ADD SMART CARD button.

   <Image src='/img/integration/gallagher/biostar2-setup7.png' width={800} alone />
   
1. Now enter the **name** as AOC Inside MIFARE type **enter 1** for Start Block Index, **tick** the checkbox for Use Face template, and **press the** Apply button.

   :::info

   If you don’t want to use a visual face and only want to use a fingerprint on the smart card you can untick the checkbox.

   :::

   <Image src='/img/integration/gallagher/biostar2-setup8.png' width={800} alone />

1. To connect devices to the system **go to** the device panel and **press** the SEARCH DEVICE button.

   <Image src='/img/integration/gallagher/biostar2-setup9.png' width={800} alone />

1. Now **press** the search button and **select** the device and **press** Add button.

   <Image src='/img/integration/gallagher/biostar2-setup10.png' width={800} alone />

1. The connected devices will be listed on the device panel on the left side.

   <Image src='/img/integration/gallagher/biostar2-setup11.png' width={800} alone />
