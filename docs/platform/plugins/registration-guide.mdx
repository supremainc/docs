개발한 플러그인을 **BioStar X Service Manager**에 등록하고 관리하는 방법을 안내합니다.

## 등록 전 준비사항

1. **플러그인 서버 실행**

    플러그인을 등록하기 전에 반드시 플러그인 서버가 실행 중이어야 합니다. BioStar X는 등록 과정에서 플러그인 서버에 접근성을 확인합니다.

1. **네트워크 접근 확인**

    BioStar X 서버에서 플러그인 서버로 접근할 수 있는지 확인하세요.

    ```bash
    # BioStar X 서버에서 테스트
    curl -H "X-BioStar-Ping: test123" http://localhost:8000/bsx
    ```

## 플러그인 등록

1. Windows에서 <Start/> → **BioStar X** → **BioStar X Service Manager**를 클릭하세요.

1. 화면 왼쪽 사이드 메뉴에서 <Cmd>PLUGINS</Cmd>를 클릭하세요.

1. 화면 오른쪽 상단의 <Cmd>Add Plugin</Cmd> 버튼을 클릭하세요.

    <Image src='/img/common/biostarx-sm-plugins-add.png' />

1. <Cmd>Add New Plugin</Cmd> 화면에서 플러그인 정보를 입력하세요.

    <Image src='/img/common/biostarx-sm-plugins-add-new.png' />

1. 플러그인 정보를 입력하고 등록을 완료하려면 <Cmd>Apply</Cmd> 버튼을 클릭하세요.


### 플러그인 정보 입력

* <Cmd>Plugin Name</Cmd>: 플러그인의 이름을 입력하세요. 최대 48자까지 입력할 수 있습니다. 기존 등록한 플러그인과 같은 이름을 사용할 수 없습니다.

* <Cmd>Description</Cmd>: 플러그인의 설명을 입력하세요. 최대 500자까지 입력할 수 있습니다.

* <Cmd>Plugin Address</Cmd>: 플러그인의 주소를 입력하세요. `http://` 또는 `https://`로 시작하는 유효한 URL이어야 하며, 기존 등록한 플러그인과 같은 주소를 사용할 수 없습니다.

* <Cmd>Plugin Icon</Cmd>: 플러그인의 아이콘을 업로드하세요. 

  * 지원되는 형식은 JPEG, PNG, SVG, GIF입니다. 
  
  * 파일의 최대 크기는 1MB이며, 0KB 빈 파일은 업로드할 수 없습니다. 
  
  * 업로드된 이미지는 104×104 픽셀로 자동 리사이징됩니다. 

* <Cmd>Session Bridge</Cmd>: 세션 브리지를 사용하려면 활성화하세요. 이 기능을 사용하면 BioStar X 사용자의 권한을 자동 상속하여 플러그인 접근 시 별도 로그인이 필요 없습니다.

:::info

* 아이콘을 업로드하지 않으면 플러그인 이름의 첫 글자로 아이콘이 자동 생성됩니다. 업로드한 아이콘은 <IcTrash2 /> 버튼을 클릭해 삭제할 수 있으며, 삭제 시 자동 생성 아이콘으로 변경됩니다.

* <Cmd>Session Bridge</Cmd>를 활성화하면 비활성화할 수 없으며, 보안상의 이유로 플러그인 주소를 변경할 수 없습니다.

:::


## 등록 과정

### 1. 정보 입력 완료
모든 필수 정보와 원하는 선택 정보를 입력합니다.

### 2. Apply 버튼 클릭
**Apply** 버튼을 클릭하여 등록을 시작합니다.

### 3. 서버 접근성 확인
BioStar X가 입력된 URL로 ping 요청을 전송합니다:

```http
GET /your-plugin-path
Headers:
  X-BioStar-Ping: <random_token>
```

### 4. 등록 완료 또는 실패
- **성공**: 플러그인이 목록에 추가되고 Launcher에서 사용 가능
- **실패**: 오류 메시지 표시 후 등록 취소

**일반적인 등록 실패 원인:**
- 플러그인 서버 미실행
- 네트워크 접근 불가
- 잘못된 URL 형식
- ping 엔드포인트 미구현

## Session Bridge 인증서 관리

Session Bridge를 활성화한 플러그인의 경우 추가적인 보안 설정이 필요합니다.

### 인증서 다운로드

1. **플러그인 상세 페이지 접속**: 등록된 플러그인 목록에서 해당 플러그인 클릭
2. **인증서 다운로드**: "Download Certificate" 버튼 클릭
3. **파일 저장**: `public_key.pem` 파일을 플러그인 서버에 저장

### 인증서 갱신

- **새 인증서 발급**: 다운로드 버튼 클릭 시마다 새 인증서 생성
- **기존 인증서 무효화**: 새 인증서 발급 시 기존 인증서 사용 불가
- **주의사항**: 운영 중인 플러그인의 인증서 갱신 시 서비스 중단 가능성

## 플러그인 관리

### 플러그인 목록 확인

Service Manager의 Plugins 페이지에서 등록된 모든 플러그인을 확인할 수 있습니다.

**표시 정보:**
- 플러그인 이름 및 아이콘
- URL 정보
- Session Bridge 사용 여부
- 등록 순서 (슈프리마 플러그인이 상단에 우선 표시)

### 플러그인 수정

**수정 가능 항목:**
- Plugin 이름
- Plugin 설명
- Plugin 아이콘

**수정 불가 항목 (Session Bridge 활성화 시):**
- Plugin URL
- Session Bridge 설정

### 플러그인 삭제

#### 외부 플러그인 삭제
1. **플러그인 선택**: 삭제할 플러그인을 목록에서 선택
2. **삭제 버튼 클릭**: "Delete" 버튼 클릭
3. **확인 팝업**: 삭제 확인 메시지에서 "확인" 클릭
4. **삭제 완료**: 성공 메시지 표시 후 목록에서 제거

#### 슈프리마 플러그인 삭제
- **Service Manager에서 삭제 불가**: "Delete" 버튼 클릭 시 오류 메시지 표시
- **삭제 방법**: Windows 프로그램 추가/제거를 통해서만 삭제 가능

```
오류 메시지 예시:
"슈프리마에서 제공하는 플러그인은 Service Manager에서 
삭제할 수 없습니다. 프로그램 추가/제거를 통해 제거하세요."
```

## 제한사항

### 등록 개수 제한
- **전체 플러그인**: 최대 12개 (Launcher UI 제한)
- **외부 플러그인**: 최대 10개
- **슈프리마 플러그인**: 최대 2개 (자동 계산됨)

### 표시 순서
- **슈프리마 플러그인**: 항상 목록 상단에 표시
- **외부 플러그인**: 등록 순서대로 표시
- **Launcher 표시**: 한 줄에 6개씩 두 줄까지 표시

## 문제 해결

### 등록 실패 시 확인사항

1. **서버 실행 상태**: 플러그인 서버가 실행 중인지 확인
2. **네트워크 연결**: BioStar X에서 플러그인 서버로 접근 가능한지 확인
3. **방화벽 설정**: 필요한 포트가 열려있는지 확인
4. **엔드포인트 구현**: GET 요청에 대한 올바른 응답 구현 확인

### 세션 브리지 오류 시 확인사항

1. **인증서 파일**: `public_key.pem` 파일이 올바른 위치에 있는지 확인
2. **암호화 구현**: RSA/AES 암호화 함수 구현 확인
3. **API 호출**: 세션 브리지 API 호출 형식 확인
4. **네트워크 설정**: HTTPS 인증서 및 방화벽 설정 확인
