CLUe 웹 포털을 처음 사용한다면, 관리자 권한 구조를 이해하는 것부터 시작하세요. 이 문서에서는 브랜치 하위에 대리점, 공간 그룹, 공간을 추가하는 방법과 출입 방식 설정 방법을 안내합니다.

## CLUe 웹 포털이란?

CLUe는 여러 지점에서 고객 서비스를 운영하는 유통 채널 환경을 위해 설계된 3단계 계층 구조의 멀티 테넌트 시스템입니다. CLUe 웹 포털을 처음 시작한다면 관리자 레벨과 그에 따른 권한을 이해하는 것이 중요합니다.

<Image src='/img/clue/clue-admin-levels.png' alt='CLUe Admin Levels' />

- **브랜치 관리자**(Branch Admin): 브랜치 관리자는 **대리점**/**공간 그룹**/**공간**을 하위 항목으로 생성할 수 있습니다.

- **대리점 관리자**(Disti Admin): 대리점 관리자는 **공간 그룹**/**공간**을 하위 항목으로 생성할 수 있습니다.

- **공간 그룹 관리자**(Place Group Admin): 공간 그룹 관리자는 **공간**을 하위 항목으로 생성할 수 있습니다.

하나의 브랜치는 여러 대리점을 가질 수 있고, 하나의 대리점 역시 여러 공간 그룹을 가질 수 있습니다. 대리점 관리자 레벨이라면, 자신의 대리점과 하위 항목만 볼 수 있습니다.

이메일을 통해 고객을 초대하고, 초대받은 고객은 이메일에 포함된 링크를 통해 가입한 후 초대받은 공간 그룹에 액세스할 수 있습니다.

:::note

**고객이나 최종 사용자는 어느 항목에 해당하나요?**

위 계층 구조에서 **공간 그룹**(Place Group)이 고객 또는 서비스 제공자에게 할당되는 항목입니다. 공간 그룹은 여러 공간을 운영하는 고객 단위를 나타내므로, 공간 생성 시 해당 고객의 관리자 이메일을 입력해야 합니다.

:::


## 기본 설정

### 하위 항목 설정

1. [CLUe 웹 포털](https://clue.moon.supremainc.com)에 로그인하세요.

1. 로그인하면 초대받은 브랜치 목록을 확인할 수 있습니다. 원하는 브랜치를 선택하세요.

1. 선택한 브랜치의 하위 항목인 대리점/공간 그룹/공간 목록이 표시되면 화면 오른쪽 상단의 <IcAirMore title='more' /> 버튼을 클릭하세요.

1. 팝업 메뉴에서 <Cmd sid='add_child' product='clue' />을 클릭하세요.

   <Image src='/img/clue/clue-basic-setup-add-child.png' alt='Add Child' />

   :::note

   마찬가지로 대리점, 공간 그룹, 공간 목록에서 <IcAirMore title='more' /> →  <Cmd sid='add_child' product='clue' />을 클릭하면 하위 항목을 추가할 수 있습니다.

   :::

1. <Cmd sid='distributor' product='clue' /> <Cmd sid='add' product='clue' /> 팝업이 나타나면 각 필드를 입력하세요.

   <Image src='/img/clue/clue-basic-setup-add-distributor.png' alt='Add Distributor' />

1. 입력을 모두 완료한 후 하단의 <Cmd sid='add' product='clue' /> 버튼을 클릭하세요.

1. 항목이 추가되면 자동으로 <Cmd sid='basic' product='clue' /> 페이지로 이동되며, 이름이나 사진을 변경할 수 있습니다.

   <Image src='/img/clue/clue-basic-setup-add-distributor-profile.png' alt='Distributor Profile' />

:::note

**관리자 이메일을 입력해야 하는 이유**

기본적으로 최고 관리자는 장치 또는 출입문 설정과 같이 고객 개인 정보와 관련 없는 메뉴에만 접근할 수 있습니다. 고객 개인 정보에 접근하려면 고객의 동의가 필요하므로 먼저 고객 관리자를 초대해야 합니다.

이후 개인 정보 접근이 필요하다면, 고객은 최고 관리자를 추가로 초대할 수 있습니다. 초대된 최고 관리자는 사용자, 관리자, 모니터링 등 개인 정보가 포함된 메뉴에 접근할 수 있습니다.

:::

:::info

<Cmd sid='admin' product='clue' /> 메뉴는 기본적으로 표시되지 않습니다. 처음에는 모든 개인 정보에 접근할 수 없기 때문입니다. 실제 대리점 또는 공간 그룹 관리자로부터 초대를 받았을 때만 접근할 수 있습니다.

<Image src='/img/clue/clue-basic-setup-admin-information.png' alt='Admin information' />

:::

### 다른 관리자 초대

<Cmd sid='admin' product='clue' /> 메뉴에 접근할 수 있다면, 이메일 주소를 입력해 다른 관리자를 쉽게 초대할 수 있습니다. 초대받은 사용자가 가입을 완료하면 동일한 수준의 관리자 권한을 갖게 됩니다.

<Image src='/img/clue/clue-basic-setup-invite-admin.png' alt='Invite Admin' />

- <Cmd sid='main_admin' product='clue' /> / <Cmd sid='sub_admin' product='clue' />: 기능적인 차이는 없습니다. 이는 최소 한 명의 메인 관리자가 유지되도록 보장하기 위한 것입니다. 다만, 서브 관리자는 메인 관리자를 삭제할 수 없습니다.


## 서비스 설정

공간 그룹 설정에서 출입 서비스 설정을 구성하려면 화면 왼쪽 사이드바에서 <Cmd sid='service' product='clue' />를 클릭하세요.

<Image src='/img/clue/clue-basic-setup-services.png' alt='Services' />

:::note

**서비스란?**

CLUe는 두 가지 모드를 지원합니다. 하나는 출입문을 열기 전 API를 통해 클라이언트에 출입 승인을 요청하는 방식이고, 다른 하나는 외부 요청 없이 CLUe가 독립적으로 동작하는 방식입니다. 전자의 경우 <Cmd sid='vendor' product='clue' />을 설정할 수 있으며, 하위 장소들은 상위 장소 그룹의 설정을 따르도록 할 수 있습니다. 후자의 경우 <Cmd sid='membership' product='clue' />를 설정할 수 있으며, 외부 요청 없이 내부 출입 레벨을 기반으로 출입 제어를 결정할 수 있습니다.

각 항목을 선택할 때, 사용할 크리덴셜을 선택할 수 있으며 이는 하위 장소의 기본 크리덴셜로 설정됩니다.

:::

### <Cmd sid='vendor' product='clue' className='normal' />

기본적으로 <Cmd sid='vendor' product='clue' /> 설정은 아래 그림과 같이 API를 통해 클라이언트에 출입 결정을 요청하도록 그룹을 설정하는 것입니다.

<Image src='/img/clue/clue-basic-setup-qr-code.png' alone caption='QR Case' />

<Image src='/img/clue/clue-basic-setup-face.png' alone caption='Face Case' />

따라서 <Cmd sid='vendor' product='clue' /> 코드와 함께 CLUe 서버가 고객/파트너 API를 어디서, 어떻게 호출하는지 입력해야 합니다.

<Image src='/img/clue/clue-basic-setup-access-service-verify.png' alt='Access Service Verify' />

#### <Cmd sid='vendor' product='clue' className='normal' /> 코드

<Cmd sid='vendor' product='clue' className='normal' /> 코드는 슈프리마 CLUe 팀이 어떤 클라이언트의 API를 호출할지 결정하기 위해 사전에 테스트하고 개발한 코드 형식입니다. 이 코드를 입력하면 하위 장소에서 추가 설정 없이도 각 인증 방식에 맞는 지정된 API를 자동으로 호출하고 지정된 응답 값을 기반으로 출입 제어를 수행할 수 있습니다.

:::note

**<Cmd sid='vendor' product='clue' className='bold' /> 코드를 얻는 방법**

다른 클라이언트의 코드가 노출되지 않도록 CLUe 팀에서 별도로 관리합니다. 클라이언트와의 연동이 완료되면 필요한 정보를 제공해 드립니다.

:::


### <Cmd sid='membership' product='clue' className='normal' />

<Cmd sid='membership' product='clue' className='normal' /> 설정은 CLUe 시스템이 다른 클라이언트나 제3자의 개입 없이 독립적으로 출입 제어를 결정합니다.

<Image src='/img/clue/clue-basic-setup-membership-setup.png' alt='Membership setup' />

웹 포털이나 모바일 애플리케이션만으로 CLUe를 사용하는 경우에도 적용됩니다.

<Image src='/img/clue/clue-basic-setup-membership-setup-web-mobile.png' alt='Membership setup web and mobile' />

<Cmd sid='membership' product='clue' className='normal' /> 설정을 통해 하위 공간에 적용할 크리덴셜을 선택할 수 있습니다.

<Image src='/img/clue/clue-basic-setup-access-service-membership.png' alt='Access Service Membership' />

:::note

**공간 그룹과 다르게 공간 설정을 변경할 수 있나요?**

네, 공간 설정 페이지에서 인증 방식을 구성할 수 있습니다. 다만 출입 방식을 <Cmd sid='membership' product='clue' />에서 <Cmd sid='vendor' product='clue' />로, 또는 그 반대로 변경할 수 없습니다. 따라서 같은 공간 그룹의 고객이 서로 다른 출입 방식을 필요로 한다면 각 용도에 맞게 별도의 공간 그룹을 만드는 것을 권장합니다.

:::
