장치의 로그를 **BioStar X**의 서버로 가져와서 이벤트를 확인할 수 있습니다. 이 기능은 <Cmd sid='setting.server' product='2' /> 설정에서 <Cmd sid='setting.server.logUploadType' product='2' /> 옵션을 수동으로 설정한 경우에 사용할 수 있습니다.

:::info

<Cmd sid='setting.server.logUploadType' product='2' /> 옵션에 대한 자세한 내용은 [다음 문서](settings-server-server#general)를 참고하세요.

:::


## 수동으로 가져오기

1. <Cmd sid='shell.header.launcher' /> 페이지에서 <Cmd sid='shell.header.setting' />을 클릭하세요.

1. 화면 왼쪽 사이드바에서 <Cmd>Event</Cmd> → <Cmd>Event Log Import</Cmd>를 클릭하세요.

1. <Cmd>Update Log from Devices</Cmd> 항목에서 로그를 가져올 기간을 선택하세요.

    * <Cmd sid='monitoring.updateLog.latest' product='2' />: 마지막으로 **BioStar X** 서버로 저장된 로그의 날짜 이후의 로그를 가져옵니다. 

    * <Cmd sid='monitoring.updateLog.all' product='2' />: 장치의 모든 로그를 가져옵니다.

    * 기간 설정(<Cmd sid='monitoring.updateLog.from' product='2' /> / <Cmd sid='monitoring.updateLog.to' product='2' />): 로그를 가져올 시작일과 종료일을 선택할 수 있습니다.

1. 로그를 가져오려면 <Cmd sid='button.updateLog' product='2' /> 버튼을 클릭하세요.


## 외부 저장매체로 가져오기

외부 저장매체(USB)에 저장된 이벤트 로그를 **BioStar X**에 불러올 수 있습니다.

1. <Cmd sid='shell.header.launcher' /> 페이지에서 <Cmd sid='shell.header.setting' />을 클릭하세요.

1. 화면 왼쪽 사이드바에서 <Cmd>Event</Cmd> → <Cmd>Event Log Import</Cmd>를 클릭하세요.

1. <Cmd sid='menu.dataImport' product='2' /> 항목의 버튼을 클릭하세요.

1. 로컬에 저장된 데이터 파일(*.tgz*)을 선택하세요.

데이터 파일을 성공적으로 가져오면 화면에 메시지가 나타납니다.

:::info

* 오래된 펌웨어 버전을 사용하는 장치에서 내보내기한 데이터는 **BioStar X**로 가져올 수 없습니다. 항상 최신 버전의 펌웨어를 사용하세요.

* FaceStation F2, FaceStation 2, FaceLite, BioStation A2, BioStation 2, X-Station 2, BioStation 3 모델에서 내보낸 데이터 파일만 가져올 수 있습니다.

* **BioStar X** 시스템에서 설정한 출입문, 엘리베이터, 구역 이벤트 로그가 아닌 경우 일부 정보가 빈 칸으로 나타날 수 있습니다.

:::
