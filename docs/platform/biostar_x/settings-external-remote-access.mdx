<Cmd sid='setting.menu.remoteAccessExternal.root' className='normal' />는 **사용자 소유의 ngrok 계정**으로 외부 네트워크에서 내부 네트워크로 접속할 수 있는 기능입니다. 이 기능을 사용하면 방화벽 설정이나 포트 포워딩 없이도 원격지에서 **BioStar X** 서버에 접속할 수 있습니다.

:::info

* <Cmd sid='setting.menu.remoteAccessExternal.root' className='normal' /> 기능은 추가 옵션을 통해 사용할 수 있습니다. 라이선스 정책에 대한 자세한 내용은 [다음 문서](licensing#features-add-ons)를 참고하세요.

* <Cmd sid='setting.menu.remoteAccessExternal.root' className='normal' /> 기능을 사용하려면 ngrok의 **Pay-as-you-go** 플랜을 구독해야 합니다. ngrok 플랜에 대한 자세한 내용은 [다음 링크](https://ngrok.com/pricing)를 참고하세요. 

  ngrok의 **Free** 또는 **Hobbyist** 플랜에서는 밴드 폭 제한, 동시 터널 제한 등으로 인해 안정적인 리모트 액세스 연결이 어려울 수 있습니다.

:::

:::note

<Cmd sid='setting.menu.remoteAccessExternal.root' className='bold' /> VS <Cmd sid='setting.menu.remoteAccess.root' className='bold' />

* <Cmd sid='setting.menu.remoteAccess.root' className='bold' /> 기능은 라이선스 구매를 통해서 사용자에게 ngrok 슈프리마 사이트의 bot 계정, 엔드포인트를 제공하여 외부 네트워크에서의 연결을 지원하는 기능입니다. <Cmd sid='setting.menu.remoteAccess.root' className='bold' /> 기능에 대한 자세한 내용은 [다음 문서](settings-remote-access)를 참고하세요.

* 반면, <Cmd sid='setting.menu.remoteAccessExternal.root' className='bold' /> 기능은 사용자가 소유한 ngrok 계정을 통해 외부 네트워크에서 내부 네트워크로의 연결을 지원하는 기능입니다. 사용자는 직접 ngrok 계정을 생성하고, ngrok 설정 정보를 입력해야 합니다.

:::

:::caution

* 리모트 액세스(Remote Access) 기능은 제3자 서비스인 ngrok의 터널링 서비스를 통해 제공됩니다. 기능 사용 전 반드시 [Remote Access 기능 사용 동의](../../legal/eula#remote-access-agreement) 약관을 확인하세요.

* BioStar X는 단순히 ngrok API를 호출하여 연결을 지원할 뿐, 인터넷 연결의 보안성, 안전성, 연속성은 ngrok 서비스에 전적으로 의존합니다. 따라서 본 기능 사용 중 발생할 수 있는 보안 사고, 데이터 손실, 시스템 침해 등에 대하여 슈프리마는 어떠한 책임도 지지 않습니다.

:::

## 언제 사용하나요? {#whenShoulduseit}

다음과 같은 상황에서 리모트 액세스 기능을 사용하세요.

- 현재 소유한 ngrok 계정이 있으면서, 외부 네트워크에서 사내 **BioStar X** 서버에 접속해야 할 때

- 방화벽이나 NAT 환경으로 인해 직접 접속이 어려운 경우

- 복잡한 네트워크 설정 없이 간편하게 원격 접속을 구성하고 싶을 때

- 임시로 외부 접속 권한을 제공해야 하는 경우


## 시작하기 전에 {#beforeStart}

### 네트워크 설정 확인 {#checkNetworksettings}

<Cmd sid='setting.menu.remoteAccessExternal.root' className='normal' /> 기능을 사용하려면 **BioStar X** 서버와 ngrok 간의 통신이 필요합니다. 서버의 방화벽 설정에서 다음을 확인하세요.

* **인바운드 규칙**: **BioStar X**가 사용 중인 포트(기본값: **443**)에 대한 인바운드 트래픽 허용

* **아웃바운드 규칙**: ngrok 서비스와의 통신을 위한 아웃바운드 트래픽 허용 (포트: **443**)

방화벽 설정에서 이 규칙들이 활성화되어 있지 않으면 ngrok 연결에 실패할 수 있습니다.

### ngrok 계정 및 플랜 준비 {#prepareNgrokAccount}

* ngrok 계정을 준비하세요. ngrok 계정이 없다면 [다음 링크](https://dashboard.ngrok.com/signup)에서 계정을 생성하세요.

* ngrok 플랜을 구독하세요. <Cmd sid='setting.menu.remoteAccessExternal.root' className='normal' /> 기능은 ngrok의 **Pay-as-you-go** 플랜 이상에서 안정적인 연결이 가능합니다. 플랜에 대한 자세한 내용은  [다음 링크](https://ngrok.com/pricing)를 참고하세요.

### Authtoken 확인 {#verifyAuthtoken}

ngrok 대시보드에서 **Authtoken**을 확인하세요. **Authtoken**은 **BioStar X**가 ngrok 서비스에 인증할 때 필요한 고유한 인증키입니다.

1. ngrok 대시보드에 로그인하세요.

1. 왼쪽 사이드바에서 **Getting Started** → **Your Authtoken** 메뉴를 클릭하세요.

1. **Authtoken** 값을 복사하세요.

:::caution

**Authtoken**은 민감한 인증 정보입니다. 타인과 공유하지 않도록 주의하세요.

:::


### 엔드 포인트 URL 준비 {#prepareTheEndpointUrl}

ngrok 대시보드에서 엔드포인트 URL을 설정하세요. URL은 `your-name.ngrok.app` 형식입니다.

1. ngrok 대시보드에 로그인하세요.

1. 왼쪽 사이드바에서 **Universal Gateway** → **Endpoints & Traffic Policy** 메뉴를 클릭하세요.

1. 화면 오른쪽 상단의 **New Endpoint** 버튼을 클릭하세요.

1. **Cloud Endpoint**를 선택하세요.

    <Image src='/img/common/biostarx-settings-external-remote-access-ngrok-select.png' alone />

1. **Bindings**에서 **Public**을 선택하고 **Create Cloud Endpoint** 버튼을 클릭하세요.

    <Image src='/img/common/biostarx-settings-external-remote-access-ngrok-cloud-endpoint.png' alone />

1. **Traffic Policy** 탭을 클릭하고 **Code** 탭으로 전환하세요.

1. `on_http_request.actions.config.url` 값을 확인하세요. 엔드포인트 URL 끝에 `.internal`을 추가하여 내부 트래픽 라우팅을 설정하세요.

    <Image src='/img/common/biostarx-settings-external-remote-access-ngrok-cloud-endpoint-setting.png' />

    예: `your-name.bsx.ngrok.app` → `your-name.bsx.ngrok.app.internal`

1. 설정을 완료한 후 화면 우측 상단의 **Save** 버튼을 클릭하세요.

:::info

`.internal` 도메인은 ngrok의 내부 네트워크 라우팅을 위한 특수 도메인입니다. 이를 통해 외부 요청이 내부 **BioStar X** 서버로 안전하게 전달됩니다.

:::

## 엔드포인트 URL 접속 {#endpointUrlAccess}

설정을 완료하고 <Cmd sid='setting.remoteAccess.field.hostURL' /> 항목에 입력한 엔드포인트 URL을 통해 외부 네트워크에서 **BioStar X** 서버에 접속할 수 있는지 확인하세요.
