사용자의 출입 권한 설정 방법을 안내합니다. 출입 권한을 통해 사용자가 어떤 공간을 출입할 수 있는지, 어떤 시간대에 출입할 수 있는지 등을 설정할 수 있습니다. 출입 권한은 사용자를 새로 추가하거나 수정할 때 설정할 수 있습니다.

<Cmd>Launcher</Cmd> > <Cmd>User</Cmd> 페이지로 이동해, 아래 두 가지 방식으로 출입 권한을 설정할 수 있습니다.

* **새로운 사용자**: 화면 오른쪽 상단의 <Cmd>New User</Cmd> 버튼을 클릭하세요. New User 창의 Permission 섹션에서 출입 권한을 설정하세요.

* **기존 사용자**: 사용자 목록에서 사용자를 선택하세요. 화면 오른쪽에 표시되는 프로필 화면에서 See More 버튼을 클릭하세요. 사용자 정보를 수정할 수 있는 창이 나타나면 Permission 섹션에서 출입 권한을 설정하세요.


## 권한 등급 {#permission-level}

권한 등급에 따라 BioStar X에서 접근할 수 있는 메뉴가 제한됩니다. Permission 섹션의 Operator Level 항목에서 사용자의 권한 등급을 선택하세요. 사용자에게 할당할 수 있는 권한 등급은 아래 목록을 참고하세요.

* Administrator: 모든 메뉴를 사용할 수 있는 관리자 등급입니다.

* User Operator: 사용자와 설정 메뉴만 사용할 수 있는 등급입니다.

* Monitoring Operator: 모니터링과 설정 메뉴를 사용할 수 있으며, 대시보드, 사용자, 장치, 출입문, 구역, 출입 통제 메뉴는 **보기** 권한만 가집니다.

* Video Operator: 모니터링과 설정 메뉴를 사용할 수 있는 등급입니다.

* TA Operator: 근태 관리 메뉴를 사용할 수 있으며 사용자 메뉴는 **보기** 권한만 가집니다.

* User: 자신의 사용자 정보와 근태 정보만을 확인할 수 있습니다.

* Visitor Operator: 방문자 메뉴만 사용할 수 있는 등급입니다.

:::note

사전 정의된 사용자 권한 외 새로운 사용자 권한을 추가하려면 [다음 문서](#)를 참고하세요.

:::


## 사용자 IP {#user-ip}

사용자 IP 주소를 등록하면, 계정에 등록한 IP 정보와 일치할 경우에만 PC에 로그인할 수 있습니다. 이를 통해 보안을 강화할 수 있습니다. Permission 섹션의 User IP 항목에서 사용자의 IP 주소를 입력하세요.

:::note

* IP 주소는 xxx.xxx.xxx.xxx 형식으로 입력할 수 있으며 각 옥텟은 0~255 범위의 숫자로만 입력할 수 있습니다.

* IP 주소를 등록하지 않으면 IP 정보와 관계없이 PC에 로그인할 수 있습니다.

* 클라우드를 사용할 때는 사용자 계정에 등록된 IP 주소와 PC의 IP 주소가 일치하지 않아도 접속할 수 있습니다.

- 클라우드를 사용할 때는 사용자 계정에 등록된 IP 주소와 PC의 IP 주소가 일치하지 않아도 접속할 수 있습니다.

- 클라우드를 사용할 때는 사용자 계정에 등록된 IP 주소와 PC의 IP 주소가 일치하지 않아도 접속할 수 있습니다.

:::


## 멀티팩터 인증

**멀티팩터 인증**을 활성화하면 사용자는 BioStar X에 로그인 시 지문 인증을 추가해 로그인할 수 있습니다. Permission 섹션의 Multi-Factor Auth 항목에서 사용자의 이중 인증을 설정하세요.

:::info

* **멀티팩터 인증**(Multi-factor authentication, MFA)이란, 2회 이상의 인증 절차를 통해 사용자를 인증하는 방식입니다. 멀티팩터 인증을 사용해 계정에 대한 보안을 강화할 수 있습니다.

* **멀티팩터 인증** 설정에 대한 자세한 내용은 [다음 문서](login-with-mfa)를 참고하세요.

:::


## 출입 그룹 {#access-group}

사용자의 출입 그룹을 설정해 조직별, 부서별로 출입을 제한할 수 있습니다. Permission 섹션의 Access Group 항목에서 출입 그룹을 선택하세요.

:::note

출입 그룹 추가 및 설정에 대한 자세한 내용은 [다음 문서](#)를 참고하세요.

:::


## 개인 인증 모드 {#auth-mode}

사용자에 따라 기본 인증 방식을 설정할 수 있습니다. Permission 섹션의 Private Auth Mode 항목에서 원하는 인증 방식을 선택하세요. 원하는 인증 방식를 추가로 선택할 수 있습니다.

:::note

* 확장 인증 모드를 활성화하면 얼굴과 지문을 포함하여 인증 모드를 조합할 수 있습니다. <Cmd>Extend AuthMode Usage</Cmd> 옵션은 FaceStation F2 및 BioStation 3, BioEntry W3 모델에서만 지원합니다.

* 추가한 인증 방식을 삭제하려면 <IcClose/> 버튼을 클릭하세요.

:::


## 장치 기본 인증 모드 제외 {#exclude-device-auth}

앞서 Private Auth Mode 항목에서 설정한 개인 인증만 사용하도록 설정할 수 있습니다. Permission 섹션의 Exclude Device Auth Mode 항목을 클릭해 기능을 활성화하세요. 

:::note

Exclude Device Auth Mode 항목을 비활성화하면 장치에 설정된 인증 방식과 개인 인증 방식 모두를 사용할 수 있습니다.

:::


## 출입 예외 {#access-exception}

사용자가 모든 출입문을 자유롭게 드나들도록 허용하거나 특정 출입문에 대해서만 출입을 제한하도록 예외를 설정할 수 있습니다. Permission 섹션의 Access Exception 항목에서 사용자가 출입할 수 있는 출입문(<Cmd>Allow</Cmd>)과 출입을 제한할 출입문(<Cmd>Deny</Cmd>)을 선택하세요.

:::note

추가한 출입문을 삭제하려면 <IcClose/> 버튼을 클릭하세요.

:::


## Extended Door Open Time

연장된 출입 시간 동안에도 출입을 허용할지 설정할 수 있습니다. 연장 출입 시간에 출입을 허용하려면 Permission 섹션의 Extended Door Open Time 항목을 클릭해 기능을 활성화하세요.
