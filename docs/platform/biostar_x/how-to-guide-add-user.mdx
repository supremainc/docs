새로운 사용자를 추가하는 방법을 안내합니다. 사용자의 사진 및 이름, 이메일, 전화번호 등과 같은 정보를 입력하고 사용자의 출입 권한 및 생체 정보를 설정할 수 있습니다.

## 사용자 추가 {#add-user}

1. <Cmd>Launcher</Cmd> 페이지에서 <Cmd>User</Cmd>를 클릭하세요.

1. <Cmd>User</Cmd> 페이지의 오른쪽 상단의 <Cmd>New User</Cmd> 버튼을 클릭하세요.

1. <Cmd>New User</Cmd> 창이 나타나면 사용자 정보를 입력하세요.

1. <Cmd>Information</Cmd> 섹션에서 사용자의 기본 정보를 입력하세요.

1. <Cmd>Permission</Cmd> 섹션에서 사용자의 출입 권한을 설정하세요.

1. <Cmd>Credential</Cmd> 섹션에서 <Cmd>1:1 Security Level</Cmd>을 설정하세요.

1. 사용자의 크리덴셜을 등록하세요.

1. 모든 정보를 입력하고 <Cmd>Save</Cmd> 버튼을 클릭하세요.

사용자 목록에서 추가한 사용자를 확인할 수 있습니다.

:::info

* 출입 권한 및 생체 인식 정보는 선택 사항입니다. 기본 정보를 입력하고 저장한 후에 수정하거나 추가할 수 있습니다.

* <Cmd>Information</Cmd> 섹션의 각 항목에 대한 자세한 내용은 [다음 문서](#information)를 참고하세요.

* <Cmd>Permission</Cmd> 섹션의 각 항목에 대한 자세한 내용은 [다음 문서](#permission)를 참고하세요.

* <Cmd>Credential</Cmd> 섹션의 각 항목에 대한 자세한 내용은 [다음 문서](#credential)를 참고하세요.

* 크리덴셜을 등록하고 <Cmd>1:1 Security Level</Cmd> 항목에서 적절한 보안 등급을 설정하세요. 보안 등급이 높을수록 인증률이 낮거나 본인 거부율(FRR)이 높을 수 있습니다.

* 크리덴셜 등록에 대한 자세한 내용은 [다음 문서](how-to-guide-enroll-credential)를 참고하세요.

:::

### 기본 정보 {#information}

<Cmd>Information</Cmd> 섹션에서 사용자의 기본 정보를 입력하세요.

* **사진**: 사용자의 사진을 등록할 수 있습니다. 웹캡으로 사진을 촬영하거나 PC에서 사진을 업로드할 수 있습니다. 프로필에 마우스를 오버하면 사용할 수 있는 도구가 표시됩니다.

  * <IcCamera/>: PC에 웹캡이 연결되어 있다면 버튼을 클릭해 사진을 촬영하고 등록할 수 있습니다.

  * <IcUpload/>: PC에 저장된 이미지 파일을 업로드할 수 있습니다.

  * <IcTrash/>: 현재 적용된 사진을 삭제할 수 있습니다.

  :::info

  모바일 출입 카드에 사용자의 사진을 표시하려면 반드시 사진을 추가해야 합니다.

  :::

* <Cmd>ID</Cmd>: 사용자에게 부여할 고유 ID를 입력하세요.

* First Name

* Last Name

* Email

* Telephone

* Department

* Title

* Group

* Period

* Status

* Custom Field

:::info

Custom Field를 추가하는 방법에 대한 자세한 내용은 다음 문서를 참고하세요.

:::

### 출입 권한 {#permission}

<Cmd>Permission</Cmd> 섹션에서 사용자의 출입 권한을 설정하세요.

* Operation Level

* User IP

* Multi-Factor Auth

* Access Group

* Private Auth Mode

* Exclude Device Auth Mode

* Access Exception

* Extended Door Open Time

### 생체 인식 정보 {#credential}

<Cmd>Credential</Cmd> 섹션에서 사용자의 생체 인식 정보를 등록하세요. 하나 이상의 인증 방법을 등록할 수 있습니다.

* <Cmd>1:1 Security Level</Cmd>

  장치에서 설정한 바이오메트릭 1:N 보안 등급과 상관없이 사용자에게 별도의 보안 등급을 부여할 수 있습니다. 장치의 1:N 보안 등급을 높게 설정해 자주 인증에 실패하는 사용자가 있다면, <Cmd>1:1 Security Level</Cmd>을 낮게 설정하여 인증 실패를 완화할 수 있습니다.

  :::note

  * Private Auth Mode를 Biometric으로 설정했다면, 입력된 사용자를 알 수 없기 때문에 1:1 보안 등급을 적용할 수 없으며, 사용자 특정이 가능한 카드 또는 ID로 인증 모드를 설정했을 때에만 1:1 보안 등급을 적용할 수 있습니다.
  
  * 적절한 보안 등급을 설정하세요. 보안 등급이 높을수록 지문 인증률이 낮거나 본인 거부율(FRR)이 높을 수 있습니다.

  :::

* <Cmd>Fingerprint</Cmd>: 사용자의 지문을 등록하고 지문 인식을 통해 사용자의 출입을 허용합니다. 지문 등록에 대한 자세한 내용은 [다음 문서](how-to-guide-enroll-credential#enroll-fingerprint)를 참고하세요.

* <Cmd>Face</Cmd>: 적외선 카메라로 촬영한 사용자의 얼굴을 등록하고 얼굴 인식을 통해 사용자의 출입을 허용합니다. 얼굴 등록에 대한 자세한 내용은 [다음 문서](how-to-guide-enroll-credential#enroll-face)를 참고하세요.

* <Cmd>Visual Face</Cmd>: 비주얼 페이스는 사용자의 얼굴을 비주얼 카메라로 촬영한 인증 수단입니다. 사용자의 비주얼 페이스를 등록하고 얼굴 인식을 통해 사용자의 출입을 허용합니다. 얼굴 등록에 대한 자세한 내용은 [다음 문서](how-to-guide-enroll-credential#enroll-face)를 참고하세요.

* <Cmd>Card</Cmd>: 사용자의 카드를 등록하고 카드 인식을 통해 사용자의 출입을 허용합니다. 카드 등록에 대한 자세한 내용은 [다음 문서](how-to-guide-enroll-credential#enroll-card)를 참고하세요.

* <Cmd>Mobile</Cmd>: 슈프리마의 에어팝 포탈(Airfob Portal)과 연동하여 모바일 출입 카드를 사용하면, 사용자에게 모바일 출입 카드를 발급할 수 있습니다. 모바일 출입 카드 등록에 대한 자세한 내용은 [다음 문서](how-to-guide-enroll-credential#enroll-mobile)를 참고하세요.

* <Cmd>QR/Barcode</Cmd>: 사용자의 모바일 기기를 통해 QR 코드 또는 바코드를 발급하여 사용자의 출입을 허용합니다. QR 코드 또는 바코드 등록에 대한 자세한 내용은 [다음 문서](how-to-guide-enroll-credential#enroll-qrbarcode)를 참고하세요.

* <Cmd>Passcode</Cmd>: 사용자에게 비밀번호를 할당하고 출입 장치에 비밀번호를 입력하여 출입을 허용합니다. Passcode 등록에 대한 자세한 내용은 [다음 문서](how-to-guide-enroll-credential#enroll-passcode)를 참고하세요.

* <Cmd>Access PIN</Cmd>: 사용자에게 PIN 번호를 할당하고 출입 장치에 PIN 번호를 입력하여 출입을 허용합니다. Access PIN 등록에 대한 자세한 내용은 [다음 문서](how-to-guide-enroll-credential#enroll-accesspin)를 참고하세요.
