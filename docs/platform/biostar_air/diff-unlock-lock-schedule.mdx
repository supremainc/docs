## 스케줄이 필요한 경우

**출입문 스케줄은 없어도 괜찮습니다**. 대부분의 시설에서는 기본 출입 제어만으로 충분합니다. 인증된 사용자가 크리덴셜을 제시하면 자동으로 출입문이 열리므로, 추가 스케줄 설정 없이도 정상 작동합니다.

아래의 상황에서 스케줄을 설정하세요.

* 특정 시간에 누구든지 자유롭게 출입할 수 있도록 해야 할 때

* 특정 시간에 모든 사용자의 출입을 차단해야 할 때

* 시간대별로 출입 정책을 다르게 적용해야 할 때

<Cmd sid='unlock_schedule' product='air' className='bold' />과 <Cmd sid='lock_schedule' product='air' className='bold' />의 혼동을 피하려면 각각의 작동 방식과 사용 시점을 명확히 이해해야 합니다.

:::info

출입문 스케줄 설정에 대한 자세한 내용은 <DocLink docId='platform/biostar_air/managing-door-schedules' />를 참고하세요.

:::


## <Cmd sid='unlock_schedule' product='air' className='bold' /> vs <Cmd sid='lock_schedule' product='air' className='bold' />

<div className='table-fixed'>

| 구분 | 열림 스케줄 | 잠김 스케줄 |
| -- | -- | -- |
| **스케줄 시간 중 상태** | 항상 열림 | 항상 잠김 |
| **인증 필요 여부** | 불필요, 누구든지 자유롭게 출입 | 크리덴셜 인증으로 열 수 없음 |
| **수동 개방** | 가능 | 사이트 관리자만 가능 |
| **릴레이 작동** | 계속 작동함 | 작동하지 않음 |

</div>

:::info

사이트 관리자는 BioStar Air 관리자 포털 또는 모바일 앱에서 지정된 관리자를 의미합니다.

:::


## 열림 스케줄

### 작동 방식

스케줄 시간 동안 출입문이 항상 열려있으며, 릴레이가 계속 작동합니다. 인증이 필요하지 않으므로 누구든지 자유롭게 출입할 수 있습니다.

### 사용 사례

* 공용 공간의 주 출입문

* 주거 단지 내 공용 편의 시설

* 업무 시간 중 사무실 로비 출입문


### 최초 인증 후 열림 (선택 사항)

#### 작동 방식

* 스케줄이 시작되면 출입문이 잠긴 상태로 시작합니다.

* 첫 인증 후 첫 번째 인증된 사용자가 크리덴셜을 제시한 순간 출입문이 열립니다.

* 스케줄이 끝날 때까지 모든 사용자가 자유롭게 출입할 수 있습니다.

:::info

최초 인증 후 열림 상태 유지 옵션(<Cmd sid='unlock_after_first_auth' product='air' />)에 대한 자세한 내용은 <DocLink docId='platform/biostar_air/managing-door-schedules#configuringUnlockSchedules' />를 참고하세요.

:::

#### 사용 사례

* 직원이 도착한 후에 고객 입장을 허용하는 소매점

* 첫 직원이 건물을 열면 이후 다른 직원들은 자유롭게 출입할 수 있는 사무실

* 시설 담당자가 출입을 개시하면 회원은 자유롭게 이용할 수 있는 멤버십 시설


## 잠김 스케줄

### 작동 방식

스케줄 시간 동안 출입문이 항상 잠겨있으며, 릴레이가 작동하지 않습니다. 유효한 크리덴셜을 제시해도 인증으로 열 수 없으며, 사이트 관리자만 수동으로 개방할 수 있습니다.

### 사용 사례

* 특정 시간 이후에 엄격히 폐쇄해야 하는 구역

* 수영장 또는 체육관, 기타 출입 제한 구역과 같은 시설
