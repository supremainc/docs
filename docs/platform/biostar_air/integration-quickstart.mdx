BioStar Air 데모 사이트를 설정하고 호환 장치를 연결·등록하여 클라우드 API 통합을 테스트하는 방법을 안내합니다. BioStar 2에서 작업한 경험이 있다면 많은 개념이 친숙하게 느껴질 것입니다. BioStar Air는 완전한 클라우드 기반 시스템입니다.


## 사전 요구 사항 {#pre-requisites}

시작하기 전에 다음 사항을 확인하세요.

* 공장 초기화된 펌웨어를 설치한 **BioStar Air 호환 장치**를 준비하세요.

   * XP2-AIR, XS2-AIR, BEW3-AIR, BS3-AIR, BLN2-AIR

   * 장치는 공인 슈프리마 대리점을 통해 구매할 수 있습니다.

* 장치는 이더넷을 통해 인터넷에 연결되어 있어야 합니다.

* 네트워크에서 아웃바운드 액세스를 허용하세요.

   * Port 443 (HTTPS)

   * Port 5671 (MQTT over TLS)


## 데모 환경과 API 키 설정 {#demo-setup}

<Steps>
<Step>

### 회원 가입 및 로그인

* [BioStar Air Developer](https://developers.biostarair.com/login)로 이동하세요.

* **회원 가입**을 클릭하고 양식을 작성한 후 승인을 기다리세요.

* 승인이 완료되면 크리덴셜을 사용해 로그인하세요.

</Step>
<Step>

### 데모 애플리케이션 생성

1. **API** → **Management**로 이동하세요.

1. **\+ Register** 버튼을 클릭하세요.

1. 애플리케이션 유형을 **Demo**로 선택하세요.

1. 애플리케이션 이름을 입력하고 **Register** 버튼을 클릭하세요.

1. 데모 사이트 로그인 크리덴셜(이메일 및 비밀번호)이 포함된 엑셀 파일을 다운로드하려면 **Download** 버튼을 클릭하세요.

</Step>
<Step>

### API 키 발급

1. **Application Management**에서 등록한 애플리케이션을 클릭하세요.

1. **API Management**로 스크롤하세요.

1. **Add** 버튼을 클릭하세요.

1. 이름을 입력하고 확인하세요. 생성된 키를 저장하세요.

</Step>
<Step>

### 데모 포털 로그인

1. [Demo Portal](https://demo.airfobspace.com/login)로 이동하세요.

1. 엑셀 파일의 이메일 및 비밀번호로 로그인하세요.

1. 방금 생성한 API 키를 입력하세요.

</Step>
</Steps>


## 장치 등록 {#registerDevices}

모바일 기기에 BioStar Air Demo App(안드로이드 전용)을 사용하세요.

1. 다음 링크를 클릭해 모바일 앱을 다운로드하세요. [DeviceRegistration.zip](https://moca-public-file-share.s3.ap-northeast-2.amazonaws.com/DeviceRegistration.zip)

1. 엑셀 파일의 크리덴셜로 로그인하세요.

1. 안드로이드 환경의 모바일 기기를 리더기 근처(BLE 범위)로 가져가세요.

   * **All Menu** → **Devices**에서 오른쪽 상단의 <IcAdd /> 버튼을 탭하세요.

   * 장치를 검색하고 **Register** 버튼을 탭하세요.

1. 장치에서 소리가 나고 재부팅되며 등록된 장치 목록에 나타납니다.

:::info

장치는 모바일 앱을 통해서만 등록할 수 있습니다. 웹 포털에서는 등록할 수 없습니다. API 기반 등록에는 파트너에게 제공되지 않는 암호화 인증서가 필요합니다.

:::


## 장치 관리

데모 앱 또는 데모 웹 포털에서 장치를 관리할 수 있습니다.

* **BioStar Air Demo App**

* **Demo Web Portal**: [https://demo.airfobspace.com/login](https://demo.airfobspace.com/login)



## Demo vs Production {#demo-vs-production}

* 테스트를 위한 **Demo** 사이트는 [개발자 포털](https://developers.biostarair.com/login)을 통해 생성됩니다.

* **Production** 사이트는 [파트너 포털](https://partner.biostarair.com/)을 통해 생성되며 사이트 ID, 사용자 이메일 및 비밀번호가 필요합니다. Production 사이트는 승인된 판매자 또는 슈프리마 지사만 생성할 수 있습니다.

:::info

* 사용자 ID와 계정 ID는 다릅니다. 혼동하지 마세요.

* 여러 사이트를 관리한다면 비밀번호를 안전하게 저장하세요.

* 항상 최신 Bearer 토큰을 사용하세요.

* 브라우저에서 복사한 Bearer 토큰은 Postman에서 재사용할 수 있습니다.

:::
