BioStar Air의 유연한 장치 관리 설정을 통해 모든 사이트(Site)의 요구 사항을 정확하게 구성할 수 있습니다. 이 문서에서 장치 관리는 등록, 구성, 펌웨어 관리, 모니터링, 지속적인 운영을 포함한 장치의 전체 라이프사이클을 의미합니다.

BioStar Air는 두 가지 장치 아키텍처를 지원합니다. 두 아키텍처 모두 동일한 사이트에서 함께 사용할 수 있으므로, 각 출입문나 구역에 가장 적합한 방식을 선택할 수 있습니다.


## 분산형 아키텍처

분산형 아키텍처에서는 각 엣지 장치가 자체 컨트롤러가 내장되어 있으며 이더넷을 통해 클라우드에 직접 연결됩니다. 네트워크 연결이 끊어지더라도 장치는 독립적으로 작동하며, 연결이 복구되면 자동으로 동기화됩니다.

엣지 장치는 RS-485를 통해 연결된 보조 장치의 컨트롤러 역할도 할 수 있습니다.

이 아키텍처는 다음과 같은 경우에 주로 사용됩니다.

* 신규 설치

* 바이오메트릭 장치

* 인프라 구축을 최소화해야 하는 경우


## 중앙 집중식 아키텍처

중앙 집중식 아키텍처에서는 장치가 RS-485 또는 Wiegand를 통해 컨트롤 패널에 연결됩니다. 컨트롤 패널은 이더넷을 통해 클라우드에 연결되고 연결된 장치들을 관리하는 컨트롤러로 작동합니다.

컨트롤 패널은 네트워크 연결이 끊기면, 연결된 장치들의 중복성 관리와 출입 제어 결정을 담당합니다.

이 아키텍처는 다음과 같은 경우에 주로 사용됩니다.

* 기존 저전압 배선이 있는 재구축 프로젝트

* 기존 출입 제어 시스템을 교체하는 사이트

* 중앙 집중식 보안실 또는 제어 패널을 사용하는 구축 환경
