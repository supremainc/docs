#### 4.2.1. 데이터베이스의 개인 데이터 암호화

**권고 사항:**  
모든 민감한 사용자 및 자격 증명 정보가 암호화된 상태로 저장되도록 **데이터베이스의 개인 데이터 암호화**를 활성화합니다.

**경로:** `설정 → 시스템 → 보안 → 데이터베이스의 개인 데이터 암호화`

**암호화되는 데이터 범위:**
- 프로필 이미지
- 사용자 ID, 이름, 전화번호, 사용자 IP, 이메일 주소
- 로그인 ID 및 로그인 암호
- 얼굴 템플릿, 지문 템플릿
- 카드 ID, 스마트 카드 레이아웃 키
- 사용자 및 방문자를 위한 맞춤형 정보
- 이미지 로그 파일

**개인 데이터 암호화 키:**
- 암호화를 **사용**으로 설정하면 구성 가능
- 키는 **32자**여야 하며 문자, 숫자, 기호 포함
- 키를 변경하면 모든 기존 데이터의 재암호화가 발생

**단계별 조치:**
1. 관리자로 로그인
2. `설정 → 시스템 → 보안`
3. **데이터베이스의 개인 데이터 암호화 = 사용** 설정
4. 선택적으로 **개인 데이터 암호화 키** 아래 **변경** 클릭 → 새 32자 키 입력

> ⚠️ 암호화 키 변경 시 모든 기존 개인 데이터가 재암호화됩니다(데이터베이스 크기에 따라 시간이 걸릴 수 있습니다). 유지 관리 기간에 예약하십시오. 키를 분실하면 암호화된 데이터를 복구할 수 없을 수 있습니다.

#### 4.2.2. 시각적 얼굴 이미지 저장

**권고 사항:**  
엄격한 운영 요구 사항이 없는 한 **시각적 얼굴 이미지 저장**을 비활성화합니다. 기본적으로 BioStar X는 **시각적 이미지**와 **얼굴 템플릿**(특징 추출된 생체 데이터)을 모두 저장합니다. 이 옵션을 비활성화하면 **템플릿**만 보관되어 개인 정보 위험이 줄어듭니다.

**중요한 이유:**
- **템플릿 대 이미지**: 템플릿은 생체 데이터의 수학적 표현으로 실제 얼굴로 역공학이 어렵습니다. 반면 이미지는 직접 식별 가능합니다.
- 많은 개인 정보 보호 규정(GDPR, PDPA 등)이 생체 이미지를 매우 민감한 데이터로 취급합니다.

**경로:**
1. `설정 → 서버`
2. **전문가 설정 표시 = 사용** 활성화
3. **전문가 설정**에서 **시각적 얼굴 이미지 저장** 찾기
4. 기본값: **사용** → 권장: **사용 안 함**으로 설정

**모범 사례:**
- 생체 인증에는 이미지가 아닌 **템플릿**만 보관
- 조사 또는 감사 목적으로 이미지가 필요한 경우: 별도의 보안 저장소에 보관하고 짧은 보존 기간(30–60일) 적용
- 업그레이드 후 설정이 기본값으로 복원되지 않았는지 주기적으로 검토
