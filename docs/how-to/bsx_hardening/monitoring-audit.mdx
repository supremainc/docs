#### 3.6.1. 로깅 정책 정의

최소한 다음 항목을 로깅합니다:
- 인증 시도(성공/실패)
- 권한 변경
- 구성 변경
- 데이터베이스 접근
- 방화벽 활동

보존 기간: 일반적으로 로컬에서 90–180일, 중앙화된 시스템에서는 더 길게 설정

---

#### 3.6.2. Windows 이벤트 로깅 및 모니터링 활성화

- 이벤트 뷰어에서 보안, 애플리케이션, 시스템 로깅 활성화 확인
- 로그 크기 구성(보안 로그는 최소 1GB)
- 감사 정책 설정: 로그온 이벤트, 권한 사용, 정책 변경, 계정 로그온, 개체 접근

---

#### 3.6.3. 중앙화된 로그 전달 및 SIEM 통합

- 중앙 SIEM 또는 로그 관리 시스템으로 로그 전달
- 지원 옵션: Splunk, Elastic Stack, Microsoft Sentinel, Windows Event Forwarding(WEF)

💡 **모범 사례**: 로그 전송 시 TLS로 암호화하여 가로채기 또는 변조 방지.

---

#### 3.6.4. 애플리케이션 수준 감사 활성화

- BioStar X → 설정 → 서버에서 **시스템 로그 수준 설정** 확인
- 로그 파일 위치: `[BioStar X 설치 경로]\logs\acs.log`
- BioStar X 서비스 계정과 관리자만 접근할 수 있도록 권한 설정
- SIEM 또는 중앙 로깅 솔루션으로 로그 전달

---

#### 3.6.5. 데이터베이스 감사 활성화

3.5.4 섹션에서 다룬 내용을 참고합니다. MariaDB는 `server_audit` 플러그인, SQL Server는 SQL Server Audit를 사용합니다.

---

#### 3.6.6. 방화벽 및 네트워크 활동 모니터링

- 방화벽 이벤트 로깅 활성화(`wf.msc → 속성 → 로깅`)
- 로그 저장 위치: `%systemroot%\system32\LogFiles\Firewall\pfirewall.log`
- IDS/IPS(Snort, Suricata, Zeek) 배포하여 이상 감지

> ⚠️ 방화벽 로그는 빠르게 증가합니다. 로그 순환 및 안전한 저장 구현.

---

#### 3.6.7. 검토 및 알림

자동화된 알림 설정 대상:
- 반복된 로그인 실패
- 권한 상승
- 예상치 못한 서비스 시작/종료
- 데이터베이스 스키마 수정
