### 6.1. BioStar X 기본 네트워크 포트

| 서비스 | 용도 | 기본 포트 | 참고 / 권장 사항 |
|--------|------|----------|-----------------|
| Unified Gateway Service | HTTPS (시스템 접근) | 443 | 관리자/운영자 로그인 기본값. 서비스 관리자에서 변경 가능. |
| Core Web Service | HTTPS (내부 UI) | 5002 | 443으로 프록시 처리 권장; 직접 접근 제한. |
|  | CloudNgrok | 52000 | 필요하지 않은 경우 비활성화. |
|  | Thrift RPC | 9310 | 신뢰할 수 있는 서비스로 제한. |
| Core Service | WebSocket | 9002 | 내부 전용. |
|  | FastCGI | 9000 | 로컬에서 바인딩해야 함. |
|  | API | 9010 | TLS 및 인증으로 보안. |
| Coordinator Service | 클라이언트 통신 | 21810 | 내부 전용. |
| Main Server | TCP 서버 | 51212 | **일반 텍스트** 장치 통신 — 가능하면 차단. |
|  | SSL 서버 | 51213 | **권장** 장치 통신(TLS). |
|  | gRPC | 51219 | 내부 서비스 통신만. |
| Cache Service | 클라이언트 | 10800 | Redis에 해당 — 외부에 노출 금지. |
|  | 통신 | 47500 | 내부 전용. |
|  | 검색 | 47100 | 내부 전용. |

### 6.2. BioStar X의 기본 보안 기능

| 기능 | UI 위치 | 기본값 | 권장 사항 |
|------|---------|--------|----------|
| 기본 Admin 로그인 ID & 암호 변경 | 사용자 모듈 → 기본 Admin 편집(UID:1) | admin / 설치 PW | 설치 직후 변경. |
| 접근 분리(역할) | 설정 → 사용자 → 계정 수준 | 사전 정의: Admin, Operator, Monitoring | 필요에 따라 맞춤 역할 생성. |
| 암호 강도 정책 | 설정 → 시스템 → 보안 | 중간 | 강력으로 설정하고 MFA 시행. |
| 세션 보안(동시 연결) | 설정 → 시스템 → 보안 | 허용 | 공유 로그인 방지를 위해 **비활성** 설정. |
| 다단계 인증 | 설정 → 시스템 → 보안 → MFA | 사용 안 함 | Admin/Operator/Auditor 계정에 활성화. |
| 장치와의 보안 통신 | 설정 → 시스템 → 보안 → 고급 | 사용 안 함 | 활성화 + SSL 포트(51213) 사용. |
| 장치 해시키 관리 | 설정 → 시스템 → 보안 → 고급 | 사용 안 함 | 고보안 배포에서 탬퍼 보호를 위해 활성화. |
| 시스템 백업 | 설정 → 시스템 → 시스템 백업 | 수동 | 자동 일별 백업 활성화. |
| 디렉터리 통합 | 설정 → 디렉터리 통합 | 구성되지 않음 | 가능한 경우 Entra ID 또는 AD와 통합. |
| 로그 설정(보존 및 수준) | 설정 → 서버 → 서버 | 60일, Info | 컴플라이언스에 맞게 조정. SIEM으로 전달. |
| 데이터베이스의 개인 데이터 암호화 | 설정 → 시스템 → 보안 | 사용 안 함 | 프로덕션에서 활성화. |
| 시각적 얼굴 이미지 저장 | 설정 → 서버 → 전문가 설정 | 사용 | 정당한 이유가 없는 한 비활성화. |

### 6.3. 컴플라이언스 매핑 (간략 보기)

| 보안 제어 | 지원하는 컴플라이언스 영역 |
|----------|--------------------------|
| 데이터베이스 암호화(PII, 생체 정보) | GDPR, CCPA, ISO 27001 |
| 시스템 로그 보존 + 전달 | ISO 27001, PCI-DSS |
| 관리자/운영자 계정 MFA | NIST 800-63, ISO 27001 |
| 장치와의 보안 통신(TLS) | GDPR, ISO 27001 |
| 백업 및 복원 기능 | ISO 27001, SOC 2 |
| 디렉터리 통합(AD/Entra ID) | SOX, ISO 27001, SOC 2 |
