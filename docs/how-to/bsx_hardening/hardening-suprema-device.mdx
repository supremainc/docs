본 섹션은 BioStar X에 연결된 Suprema 제조 접근 제어 장치(생체 인식 단말기, RFID 리더, 컨트롤러 등)에 대한 강화 권고 사항을 제공합니다.

**범위:**
- BioStar X에 연결된 Suprema 접근 제어 장치에 적용
- 펌웨어 유지 관리, 장치 이벤트 로그 처리, 관리자 구성, 장치 측 운영 제어 포함
- 장치 기능은 모델 및 펌웨어 버전에 따라 다를 수 있으며, 구성 변경 적용 전 항상 제품별 장치 매뉴얼 및 릴리스 노트를 참조하십시오.

### 5.1. 장치 펌웨어 최신 상태 유지

**권고 사항:**  
고객 환경에 대해 승인된 최신 지원 버전으로 Suprema 장치 펌웨어를 최신 상태로 유지합니다.

**일반 모범 사례:**
- 모든 연결된 장치의 인벤토리 유지(모델명, 장치 ID, 펌웨어 버전, 설치 위치, 역할)
- 업그레이드 전 Suprema 펌웨어 릴리스 노트 검토
- 광범위한 배포 전에 비프로덕션 장치에서 펌웨어 업그레이드 테스트
- 유지 관리 기간에 펌웨어 업그레이드 예약
- 업그레이드 중 안정적인 전원 공급 보장
- 업그레이드 후 장치가 BioStar X에 다시 연결되고 사용자, 도어, 이벤트, 인증 동작이 정상인지 확인

#### 5.1.1. 소프트웨어에서 펌웨어 업그레이드

**권장 접근 방식:**

1. 대상 장치 모델에 맞는 펌웨어 파일 확보
2. 릴리스 노트 및 호환성 참고 사항 검토
3. 장치가 온라인이며 BioStar X와 정상 통신 중인지 확인
4. 사용자, 자격 증명, 로그, 장치 구성 동기화 확인
5. 대상 장치 선택 후 펌웨어 업그레이드 실행
6. 업그레이드 완료 및 장치 재시작 후 펌웨어 버전 확인
7. 검증: BioStar X 연결, 인증, 도어 제어, 이벤트 로그 업로드 정상 여부 확인

> ⚠️ 다른 장치 모델용 펌웨어 적용 금지. 업그레이드 중 중단 금지. 여러 중요 출입구 장치를 동시에 업그레이드하지 마십시오.

#### 5.1.2. 장치/USB에서 직접 펌웨어 업그레이드

1. USB 메모리에 펌웨어 파일 저장 (하나의 펌웨어 파일만 유지)
2. USB를 장치에 연결
3. 관리자 수준 자격 증명으로 인증
4. 펌웨어 파일이 장치로 전송될 때까지 대기
5. 장치가 지시할 때 USB 제거
6. 펌웨어 업그레이드 완료 및 자동 재시작 대기
7. 펌웨어 버전 확인 및 장치 동작 검증

> ⚠️ USB를 통한 펌웨어 업그레이드를 진행하기 전에 장치 관리자가 구성되어 있어야 합니다. 업그레이드 중 전원 공급을 끊지 마십시오.

### 5.2. 장치 이벤트 로그 보존 및 보호

**권고 사항:**  
장치 이벤트 로그를 보존하고 BioStar X에 정기적으로 업로드되도록 합니다.

**장치 로그 정책:**
- 관리자가 삭제하거나 용량 한계로 덮어쓰이기 전까지 장치에 로컬로 보존
- 장기 감사 보존을 위해 장치 로컬 저장소에만 의존하지 마십시오
- 가능한 경우 BioStar X로의 자동 로그 업로드 활성화
- 수동 로그 업로드 사용 시 정기적인 로그 수집 일정 정의

**보안 탬퍼 고려 사항:**  
일부 Suprema 장치는 **보안 탬퍼** 동작을 지원합니다. 탬퍼 이벤트 발생 시 장치는 사용자, 로그, 암호화 키, SSL 인증서를 포함한 민감한 데이터를 삭제할 수 있습니다. 따라서 자동 로그 업로드를 활성화하여 탬퍼 전 로그를 BioStar X에 안전하게 저장하는 것이 중요합니다.

### 5.3. 장치 관리자 구성

**권고 사항:**  
지원되는 Suprema 장치에 장치 관리자를 구성하고 장치 관리자 메뉴에 대한 접근을 제한합니다. 장치는 관리자 보호 없이 프로덕션에 배포해서는 안 됩니다.

> 장치 관리자 권한은 BioStar X 사용자 권한과 별개입니다. 사용자를 장치 관리자로 지정하면 해당 사용자가 장치에서 로컬로 수행할 수 있는 작업을 제어하지만, 사용자의 BioStar X 애플리케이션 권한을 부여하거나 수정하지 않습니다.

**일반 모범 사례:**
- 프로덕션 배포 전 최소 한 명의 장치 관리자 등록
- 승인된 직원에게만 할당된 고유한 관리자 자격 증명 사용
- 강력한 인증 방법 사용(카드 + PIN, 생체 + PIN 등)
- 직원 역할이 변경되면 즉시 관리자 접근 제거

#### 5.3.1. 소프트웨어에서 장치 관리자 구성 (권장)

1. BioStar X에 관리자로 로그인
2. 장치 관리자 역할을 할 사용자 생성 또는 선택
3. 장치 기능에 따라 필요한 자격 증명(카드, PIN, 지문, 얼굴) 등록
4. 관리자 수준 권한 할당
5. 대상 장치에 사용자 전송 또는 동기화
6. 장치에서 관리자 자격 증명이 관리자 메뉴에 접근할 수 있는지 확인

#### 5.3.2. 장치에서 직접 장치 관리자 구성

1. 장치 메뉴 접근
2. 사용자 등록 또는 관리자 구성 메뉴 열기
3. 관리자 사용자 및 자격 증명 등록
4. 관리자 수준 권한 할당
5. 설정 저장
6. 관리자 메뉴에 다시 접근하여 관리자 인증 테스트

---

### 5.4. 장치 네트워크 및 로컬 서비스 강화

**권고 사항:**  
최소 기능 원칙에 따라 장치 네트워크 및 로컬 서비스 설정을 구성합니다.

**권장 조치:**
- 안정적이고 문서화된 장치 IP 주소 지정 사용 (DHCP 사용 시 IP 예약, 고정 IP 사용 시 IP 주소, 서브넷 마스크, 게이트웨이, DNS 서버, 장치 포트, 장치 ID, 설치 위치 문서화)
- 필요하지 않은 장치 측 기능 비활성화(RTSP, IP 인터폰 등)
- RTSP, 인터폰 등이 필요한 경우 강력한 자격 증명 구성 및 신뢰할 수 있는 시스템으로만 접근 제한
- 지원되는 경우 시간 동기화 활성화
- 기본값 복원 또는 공장 초기화 후 네트워크 설정, 관리자/운영자 수준, 루트 인증서, 보안 통신, BioStar X 연결 재확인

---

### 5.5. 장치 측 운영 강화 체크리스트

- ✅ 최신 지원 버전으로 펌웨어 업데이트 유지
- ✅ 장치 모델, 장치 ID, 펌웨어 버전, 위치, 역할의 인벤토리 유지
- ✅ 프로덕션 사용 전 장치 관리자 접근 구성
- ✅ 승인된 직원만으로 관리자 권한 제한
- ✅ 지원되는 경우 BioStar X와의 보안 통신 활성화
- ✅ 배포 위험 수준에 맞게 장치 해시키 관리/보안 탬퍼 활성화
- ✅ 가능한 경우 자동 로그 업로드 활성화
- ✅ 장치 이벤트 로그가 BioStar X에 정기적으로 업로드되는지 확인
- ✅ 장치 설치 및 케이블링을 물리적으로 보안
- ✅ 탬퍼 이벤트, 예상치 못한 재시작, 펌웨어 변경, 로그 공백을 보안 관련 이벤트로 취급
- ✅ 펌웨어 업그레이드, 장치 교체, 복원 작업 후 장치 구성 검토
- ✅ 기본값 복원 및 공장 초기화 작업을 승인된 관리자만으로 제한
