지원하는 인증 방식과 인증 절차를 설명합니다.

인증 모드​

장치의 인증 모드를 설정할 수 있습니다. Face, Fingerprint, PIN, Card / QR Code, ID를 조합하여 인증 모드로 사용할 수 있으며, 각 인증 모드마다 사용할 스케줄을 설정할 수도 있습니다.

Información

지문 인증은 BS3M-ODB, BS3M-OAPB 제품에서만 사용할 수 있습니다.

인증 모드 등록​

  1. 을 누른 뒤 관리자 권한 인증 수단으로 인증하세요.

  2. AUTHENTICATION → Auth Mode를 누르세요.

  3. 을 선택하세요.

  4. 을 누르면 추가할 수 있는 크리덴셜이 화면에 나타납니다. 원하는 크리덴셜을 추가하세요. 크리덴셜이 추가된 상태에서 을 누르면 추가한 크리덴셜을 삭제할 수 있습니다.

    • : 얼굴, : 지문, : 카드 / QR코드, : ID, : PIN

    • 선택한 인증 방법을 순서대로 모두 인증하도록 조합하거나, 선택한 인증 방법 중 일부만 선택해 인증하도록 조합할 수 있습니다. 예를 들어, + + 으로 조합할 경우 카드 / QR코드, 얼굴, PIN을 모두 인증해야 하며, + / 으로 조합할 경우 카드 / QR코드로 인증한 뒤 얼굴 또는 PIN을 입력해야 합니다.

  5. 원하는 인증 모드를 설정한 뒤 Schedule을 설정하세요.

    Información

    • Schedule은 BioStar X에서 설정할 수 있습니다. 설정한 스케줄이 없다면 Always만 선택할 수 있습니다.

    • Schedule 설정에 대한 자세한 내용은 BioStar X 관리자 설명서를 참고하세요.

    • QR Code 인증을 사용하려면 AUTHENTICATION → Operation에서 QR Authentication이 활성화 되어있어야 하며, 별도의 장치 라이선스가 필요합니다. 자세한 내용은 라이선스를 참고하세요.

  6. 설정을 저장하려면 OK를 누르세요.

인증 모드 편집​

  1. 을 누른 뒤 관리자 권한 인증 수단으로 인증하세요.

  2. AUTHENTICATION → Auth Mode를 누르세요.

  3. 수정할 인증 모드를 선택하세요.

  4. 인증 모드 등록을 참고하여 인증 모드를 수정한 뒤 OK를 누르세요.

인증 모드 삭제​

  1. 을 누른 뒤 관리자 권한 인증 수단으로 인증하세요.

  2. AUTHENTICATION → Auth Mode를 누르세요.

  3. 을 누른 뒤 삭제할 항목을 선택하세요. 여러 개를 선택할 수도 있습니다.

  4. 선택한 항목을 삭제하려면 OK를 누르세요.

인증 옵션​

서버 매칭​

서버 매칭을 설정할 수 있습니다.

  1. 을 누른 뒤 관리자 권한 인증 수단으로 인증하세요.

  2. AUTHENTICATION을 누른 뒤 Operation에서 Server Matching (Card, ID)를 활성화하세요.

    • Server Matching (Card, ID): 서버 매칭을 설정하면 장치에서 사용자 인증을 수행하지 않고, BioStar X에서 사용자 인증을 수행합니다. 서버 매칭은 장치에 사용자 정보가 너무 많을 때나 사용자 크리덴셜 정보가 저장된 장치를 외부에 노출하고 싶지 않을 때 유용하게 사용할 수 있습니다.
    Información

    • Server Matching은 카드, ID만 사용할 수 있습니다.

    • Server Matching을 사용할 경우 장치와 BioStar X에서 서버 매칭을 활성화해야 합니다. BioStar X 서버 매칭 설정에 대한 자세한 내용은 BioStar X 관리자 설명서를 참고하세요.

QR 인증​

QR코드 인증을 설정할 수 있습니다.

  1. 을 누른 뒤 관리자 권한 인증 수단으로 인증하세요.

  2. AUTHENTICATION을 누른 뒤 Operation에서 QR Authentication을 활성화하세요.

    • QR Authentication: 장치의 카메라를 통한 QR 코드 인증 사용 여부를 설정할 수 있습니다.

    • Use QR as Card: 사용자에게 발급한 CSN 카드 또는 Wiegand 카드와 동일한 데이터의 QR코드로 인증할 수 있도록 설정할 수 있습니다. QR Authentication이 활성화되어있는 경우에만 설정할 수 있습니다.

    Información

    QR Authentication을 사용하려면 별도의 장치 라이선스가 필요합니다. 자세한 내용은 라이선스를 참고하세요.

PIN​

비밀번호 입력 시 스크램블 키패드 사용 여부를 설정할 수 있습니다.

  1. 을 누른 뒤 관리자 권한 인증 수단으로 인증하세요.

  2. AUTHENTICATION을 누른 뒤 PIN에서 Scramble Keypad 사용 여부를 설정하세요.

    • Scramble Keypad: 비밀번호를 입력할 때 숫자 키의 위치가 무작위로 섞여 비밀번호 유출을 방지할 수 있습니다.

인증 결과 표시 옵션​

인증 결과 표시 옵션을 설정할 수 있습니다.

  1. 을 누른 뒤 관리자 권한 인증 수단으로 인증하세요.

  2. AUTHENTICATION을 누른 뒤 Auth Result Display Option에서 원하는 항목을 변경하세요.

    • User ID Display: 장치의 인증 결과 화면에서 사용자 ID가 표시되는 방법을 선택할 수 있습니다.

      • Display All: 사용자 ID를 모두 표시합니다.

      • Mask All but First Letter: 사용자 ID의 첫 문자만 표시합니다.

      • Hide All: 사용자 ID를 표시하지 않습니다.

    • User Name Display: 장치의 인증 결과 화면에서 사용자 이름이 표시되는 방법을 선택할 수 있습니다.

      • Display All: 사용자 이름을 모두 표시합니다.

      • Mask All but First Letter: 사용자 이름의 첫 문자만 표시합니다.

      • Hide All: 사용자 이름을 표시하지 않습니다.

얼굴​

얼굴 인증에 대한 설정을 변경할 수 있습니다.

  1. 을 누른 뒤 관리자 권한 인증 수단으로 인증하세요.

  2. AUTHENTICATION → Face을 선택하세요.

  3. 원하는 항목을 변경하세요.

    • Motion Sensor: 장치 주변에 움직임을 감지하는 감도를 설정합니다. Motion Sensor를 Off으로 설정한 경우, 얼굴을 인증하려면 화면을 터치한 뒤 인증을 진행하세요.

    • Enroll Timeout: 설정한 시간 동안 얼굴을 등록하지 못하면 얼굴 등록이 취소됩니다.

    • Auth Timeout: 설정한 시간 동안 인증을 완료하지 못하면 인증에 실패합니다.

    • Duplicate Check: 얼굴 등록 시 중복 얼굴 여부를 검사할 수 있습니다.

    • Face Detect Setting: 얼굴 인증 시 장치가 얼굴을 검출할 영역의 위치와 크기를 설정할 수 있습니다.

      Información

      최적의 성능을 위해 Face Detect Setting의 하위 옵션은 기본 설정 값 사용을 권장합니다. 각 옵션의 기본 값은 다음과 같습니다.

      • 최대 얼굴 회전 각도: 15

      • 얼굴 인식 거리: 최소 30 cm, 최대 100 cm

      • 얼굴 인식 영역 넓히기: 비활성화

    • ToM Enrollment: ToM Enrollment을 활성화하면 BioStar X에서 발급받은 Template on Mobile을 등록할 수 있습니다. Template on Mobile은 사용자의 바이오 메트릭 템플릿을 모바일 기기에 저장하는 모바일 액세스 카드로 BioStar X 서버, Airfob Portal과 장치에 사용자 데이터를 저장하지 않고도 바이오 메트릭으로 인증할 수 있습니다.

      Información

      Template on Mobile에 대한 자세한 내용은 BioStar X 관리자 설명서를 참고하세요.

    • Store Visual Face Image: 비주얼 페이스에 실제 이미지와 템플릿을 모두 저장하도록 하거나 실제 이미지는 저장하지 않고 템플릿만 저장하도록 설정할 수 있습니다. 비활성화할 경우 저장된 비주얼 페이스의 이미지가 모두 삭제되며, 앞으로 등록되는 비주얼 페이스도 이미지를 제외한 템플릿만을 저장합니다.

      Información

      이 설정을 비활성화한 후에는 BioStar 2 v2.9.7 이상에 연결하여 사용하세요.

    • Security Level: 1:N 인증을 위한 보안 등급을 설정할 수 있습니다.

    • Operation Mode: 얼굴 인증 시 장치의 작동 모드를 설정할 수 있습니다.

      • Fusion Matching: 비주얼 및 적외선 매칭이 모두 동작하여 얼굴 인증의 정확도를 높일 수 있습니다.

      • Fast Matching: 사용자는 장치의 인증 범위 안에서 걸어가면서도 얼굴을 인증하는 등 빠르게 인증을 수행할 수 있습니다.

    • Fake Detection: 사진 등의 위조 얼굴을 이용한 사용자 인증을 방지할 수 있습니다. Operation Mode를 Fusion Matching으로 설정할 때 활성화됩니다.

      Información

      • 위조 얼굴 검출 성능을 개선하여 iBeta Level 1 ISO 30107-3 제시형 공격 탐지 시험에 통과하였습니다.

      • 햇빛이 강한 환경에서는 Fake Detection 기능을 Normal으로 사용하는 것을 권장합니다.

    • Advanced Settings: 마스크 검출 등 얼굴 인증에 대한 고급 설정을 변경할 수 있습니다.

      • Mask Detection: 마스크 검출 사용 여부를 설정할 수 있습니다.

      • Detection Mode: 마스크 착용 여부를 확인하는 방식을 설정할 수 있습니다. Mask Detection을 Enabled으로 설정할 경우 활성화됩니다.

        Información

        • Require Mask (Hard): 마스크를 착용하지 않은 사용자는 인증이 거부되며, 마스크 미착용 이벤트가 로그로 기록됩니다.

        • Require Mask (Soft): 마스크를 착용하지 않은 사용자도 인증할 수 있으나 마스크 미착용 이벤트가 로그로 기록됩니다.

        • Require No Mask: 마스크를 착용하지 않은 사용자만 인증할 수 있으며, 이 모드를 사용할 경우 마스크 확인 모드가 Check Before Authentication으로 고정됩니다.

      • Mask Detection Level: 마스크 착용을 감지하는 감도를 설정할 수 있습니다. Mask Detection을 Enabled으로 설정할 경우 활성화됩니다.

      • Check Mode: 장치의 용도에 따라 확인 모드를 설정할 수 있습니다.

        Información

        • Check After Authentication: 인증을 수행한 뒤 사용자의 마스크 착용 여부를 확인합니다.

        • Check Before Authentication: 사용자의 마스크 착용 여부를 확인한 뒤 인증을 수행합니다.

        • Check Only: 장치를 마스크 착용 여부를 확인하는 용도로만 사용할 수 있습니다. 이 모드를 사용할 경우 인증과 관계없이 마스크를 착용한 사용자는 모두 출입할 수 있습니다.

지문 BS3M-ODB, BS3M-OAPB​

지문 인증에 대한 설정을 변경할 수 있습니다.

  1. 을 누른 뒤 관리자 권한 인증 수단으로 인증하세요.

  2. AUTHENTICATION → Fingerprint을 선택하세요.

  3. 원하는 항목을 변경하세요.

    • Security Level: 1:N 인증을 위한 보안 등급을 설정할 수 있습니다.

    • Matching Timeout: 지문 매칭 시 대기 시간을 설정할 수 있습니다. 설정한 시간 동안 인증을 완료하지 못하면 인증에 실패합니다.

    • View Image: 지문을 스캔했을 때 원본 이미지를 볼 수 있습니다.

    • Sensor Sensitivity: 지문 인식 센서의 민감도를 설정할 수 있습니다. 센서 감도를 높여 정밀한 지문 정보를 얻으려면 센서 감도를 높게 설정하세요.

    • 1:N Fast Mode: 지문 인증 속도를 설정할 수 있습니다. Auto을 선택하면 장치에 등록된 총 지문 템플릿의 개수에 따라 인증 속도가 설정됩니다.

    • Template Format: 지문 템플릿 형식을 설정할 수 있습니다. SUPREMA가 기본으로 설정되어 있으며, 템플릿 형식을 변경할 경우 이전에 저장된 모든 지문을 사용할 수 없으므로 변경 시 유의하세요.

    • Sensor Mode: Auto On으로 설정하면 지문 센서가 사용자의 손가락을 인식하여 켜집니다. Always On으로 설정하면 지문을 스캔하지 않을 때에도 항상 센서가 켜져 있습니다.

    • Live Finger Detection: 위조 지문 검사 등급을 설정할 수 있습니다. 위조 지문 검사 등급이 높을수록 실제 사람 지문에 대한 거부율도 높아질 수 있습니다.

    • Advanced Enrollment: 높은 품질의 지문 정보를 저장하기 위해 스캔한 지문의 품질을 검사할 수 있습니다. 활성화하면 지문 품질이 낮을 경우 사용자에게 알려주어 지문을 올바르게 스캔하도록 도와줍니다.

    • Duplicate Check: 지문 등록 시 중복 지문 여부를 검사할 수 있습니다.

근태​

근태 모드 등록 방법을 설정할 수 있습니다.

  1. 을 누른 뒤 관리자 권한 인증 수단으로 인증하세요.

  2. AUTHENTICATION → T&A를 선택하세요.

  3. 원하는 항목을 설정하세요.

    • T&A Mode: 근태 모드를 사용할 방법을 설정할 수 있습니다.

    • T&A Code: 근태 이벤트를 등록할 수 있습니다. 근태 이벤트를 등록하려면 T&A Code → 을 선택한 뒤 화면에 표시할 아이콘, T&A Event Name, Schedule을 설정하세요.

    • T&A Required: 사용자가 인증할 때 근태 이벤트를 반드시 선택하도록 설정할 수 있습니다. T&A Mode를 By User으로 설정했을 때 사용할 수 있습니다.

    • Fixed T&A Key: 관리자가 선택한 근태 이벤트만 사용하도록 설정할 수 있습니다. T&A Mode를 Fixed으로 설정했을 때 사용할 수 있습니다.

    • Job Code: 작업 코드 사용 여부를 선택할 수 있습니다.
