Administración del sistema

Configuración del sistema​

Requisitos previos​

Licencia​

  1. Licencia de software

    1. Genetec

      • Security Center: 1 ea.

      • Conexión SDK: GSC-1SDK-SUPREMA-READ (una licencia admite 2 conexiones).

        1. Inicio de sesión de Config Tool: 2 ea

        2. Inicio de sesión de Security Desk: 1 ea

        3. Complemento extractor Suprema: 1 ea por Genetec Security Center Server

        4. Complemento de dispositivo Suprema: 1 ea por complemento de dispositivo Suprema

          Información

          Un complemento de dispositivo admite hasta 4 enlaces en la nube (256 dispositivos Suprema), y puede agregar tantos complementos de dispositivo Suprema como necesite.

          • Se necesitan en total 3 GSC-1SDK-SUPREMA-READ por servidor.

    2. Suprema

      • Licencia del complemento: INT_GEN_GSC

  2. Licencia de dispositivo

    1. Genetec

      • GSC-Sy-X-1R(E): 1 ea por dispositivo Suprema asignado a la puerta como lector.

    2. Suprema

      • Solo los dispositivos cuyo ID esté registrado en la licencia del servidor pueden registrarse en el complemento.

      • SKU: modelo de dispositivo - GEN.

Base de datos​

  1. Permisos para la base de datos SQL

    Para garantizar el funcionamiento correcto del complemento (Device Plugin y Extractor DB), otorgue a la cuenta SQL los siguientes permisos:

    1. Roles del servidor:

      • db_creator

      • processadmin

      • public

    2. Roles de la base de datos:

      • db_backupoperator

      • db_datareader

      • db_datawriter

  2. Requisitos de tamaño de la base de datos

    La capacidad de almacenamiento requerida de la base de datos varía según el número de usuarios enrolados con datos biométricos. Cuando se enrolan plantillas faciales, el complemento Suprema Extractor genera y almacena las plantillas correspondientes en su base de datos.

    • Cada plantilla facial requiere aproximadamente 104 KB de espacio en la base de datos.

    • Asegure suficiente capacidad de almacenamiento según el número esperado de usuarios biométricos.

Protocolo RIO​

Para garantizar el funcionamiento correcto del complemento, habilite el protocolo RIO para Synergis Cloud Link o Softwire.

Proceso​

  1. Ingrese la siguiente URL en su navegador.

    https://[Unit IP]/Features/DuiRIO/Enabled/Set?value=true

  2. Inicie sesión en Synergis Cloud Link o Softwire.

  3. Haga clic en la URL http://[IPaddress]/Features/DuiRIO/Enabled/Set?value=true.

    Cuando el protocolo RIO está habilitado, la pestaña RIO aparece en la página Hardware dentro de Config Tool → Access Control → Cloud Link.

Ajuste de calidad de imagen del perfil​

Si desea extraer plantillas faciales de las imágenes de perfil de los titulares de tarjetas en GSC, ajuste el tamaño de la imagen para garantizar su calidad.

  • Ruta: Config Tool → Access Control → General settings

Ajuste del dispositivo​

  1. Eliminación de todos los usuarios y registros de los dispositivos.

  2. Inicialice los dispositivos a valores de fábrica.

  3. Configure el tipo de ID de usuario como alfanumérico.

    Información

    IP y PW predeterminados para el administrador del dispositivo.

    • ID: su

    • PW: 1234

    Puede cambiar la PW desde la página Config Tool → Suprema Plugin Configuration → Superadmin Credential.

  4. Ajuste de red.

    • Admite DHCP.

    • Ajuste de red requerido según la política del sitio.

  5. Ajuste del tipo de ID de usuario.

    El tipo de ID de usuario debe configurarse como alfanumérico antes de conectar el dispositivo al complemento Suprema.

Registro Dispositivo​

  1. En Config Tool, haga clic en Plugins debajo de Tasks.

  2. Haga clic en Suprema Device Plugin.

  3. Vaya a la pestaña Device Management.

  4. Haga clic en el botón Search Device. Aparecen todos los dispositivos disponibles.

    Si el modo de comunicación del dispositivo está configurado como Device To Server, haga clic en el botón Refresh List en el panel inferior.

  5. Seleccione el dispositivo que desea agregar marcando la casilla Select y haga clic en el botón Add Devices. Para enrolar todos los dispositivos, haga clic en el botón Add All Devices.

Editar ajustes e información del dispositivo​

Se pueden editar los ajustes y la información de los dispositivos enrolados.

  1. En Config Tool, haga clic en Plugins debajo de Tasks.

  2. Abra la flecha izquierda de Suprema Device Plugin y seleccione un dispositivo enrolado. Aparece el área Device Settings.

  3. Configure las opciones deseadas y luego haga clic en el botón Apply en la parte inferior para aplicar los ajustes editados.

Red​

Puede configurar los ajustes de conexión de red del dispositivo.

N.ºElementoDescripción
TCP/IP

Puede configurar los ajustes de conexión TCP/IP del dispositivo.

  • Use DHCP: Seleccione esta opción para permitir que el dispositivo use una dirección IP dinámica. Si selecciona esta opción, no se pueden ingresar ajustes de red.

  • IP Address, Subnet Mask, Gateway: Ingrese los ajustes de red del dispositivo.

  • Device Port: Ingrese un puerto que usará el dispositivo.

  • DNS Server: Ingrese la dirección del servidor DNS.

Server

Puede ingresar los ajustes de conexión para usar en el modo de servidor.

  • Device → Server Connection: Seleccione esta opción para configurar los ajustes de conexión a los dispositivos. Si selecciona esta opción, se pueden ingresar los ajustes de red del servidor.

  • Server Address: Ingrese la dirección IP o el nombre de dominio del servidor.

  • Server Port: Ingrese el número de puerto del servidor.

Serial

Puede configurar el modo de conexión y la velocidad en baudios de los dispositivos conectados por RS-485.

  • RS-485: Configure un modo RS-485.

  • Baud Rate: Configure la velocidad en baudios de la conexión RS-485.

Autenticación​

Puede configurar los ajustes de autenticación del usuario del dispositivo.

Dispositivo de autenticación facial​

N.ºElementoDescripción
Auth Mode

Puede configurar los modos de autenticación del dispositivo. Use cualquier combinación de huella digital, ID, tarjeta, PIN y rostro como modos de autenticación.

Haga clic en +Add y cree un modo de autenticación arrastrando y soltando las opciones disponibles.

Auth Timeout

Cuando se usa una combinación de varias credenciales en Auth Mode, el sistema espera este tiempo para autenticar la segunda credencial.

Configure un tiempo de espera para autenticar la segunda credencial después de autenticar la primera. Si la segunda credencial no se autentica dentro de este tiempo, la autenticación falla.

Full Access

Puede otorgar acceso total a los usuarios enrolados en el dispositivo sin configurar grupos de acceso.

Face

Puede configurar los ajustes detallados de la autenticación facial.

  • 1:N Security Level: Puede configurar un nivel de seguridad para usar en la autenticación facial. Cuanto mayor sea el nivel de seguridad, mayor será la tasa de rechazo falso (FRR) y menor la tasa de aceptación falsa (FAR).

  • Enrollment Time: Si no se enrola un rostro durante el tiempo configurado al enrolar el rostro de un usuario, el enrolamiento del rostro se cancela.

  • Motion Sensor: Configure la sensibilidad para detectar movimiento cerca del dispositivo.

  • Face Detect Setting: Configure el entorno para reconocer el rostro del usuario durante la autenticación facial.

    • Maximum Head Pose Angle: Configure el ángulo máximo de rotación de la cabeza.

    • Detection Distance: Configure la distancia mínima y máxima de detección.

    • Wide Search: Si se configura en ON, busca un rostro en toda la imagen de la cámara.

Use Suprema Handler

Esta opción habilita o deshabilita la comunicación RIO entre el dispositivo Suprema y Cloud Link.

  • Enable: Los eventos generados por el dispositivo Suprema se envían a Cloud Link a través del complemento.

  • Disable: Los eventos generados por el dispositivo Suprema no se envían al complemento.

Desactive esta opción cuando el dispositivo Suprema se use como lector conectado a un controlador Mercury.

N.ºElementoDescripción

Configure un tiempo de espera para autenticar la segunda credencial después de autenticar la primera. Si la segunda credencial no se autentica dentro de este tiempo, la autenticación falla.

Full Access

Puede otorgar acceso total a los usuarios enrolados en el dispositivo sin configurar grupos de acceso.

Fingerprint

Puede configurar los ajustes detallados de la autenticación por huella digital.

  • 1:N Security Level: Puede configurar un nivel de seguridad para usar en la autenticación por huella digital. Cuanto mayor sea el nivel de seguridad, mayor será la tasa de rechazo falso (FRR) y menor la tasa de aceptación falsa (FAR).

  • Scan Timeout: Si no se enrola un dedo durante el tiempo configurado al enrolar el dedo de un usuario, el enrolamiento del dedo se cancela.

  • Sensor Sensitivity: Configure la sensibilidad del sensor de reconocimiento de huellas dactilares. Establece la sensibilidad más alta para obtener información de huella dactilar precisa.

  • 1:N Fast Mode: Puede configurar la velocidad de la autenticación por huella digital. Al seleccionar Auto, la velocidad de autenticación se ajusta según la cantidad total de plantillas de huellas digitales inscritas en el dispositivo.

  • Template Format: Puede revisar el formato configurado de la plantilla de huella digital.

  • Matching Timeout: Configure el tiempo de espera para la comparación de huellas dactilares. Si la autenticación no se completa dentro del tiempo especificado, fallará.

  • Fingerprint LFD: Configure el nivel de detección de huella dactilar falsa. Niveles más altos de detección de huellas falsas pueden aumentar las tasas de rechazo de las huellas reales.

  • Sensor Mode: Configure el modo de funcionamiento del sensor de huellas dactilares.

    • Auto On: El sensor de huellas dactilares se enciende al reconocer el dedo del usuario.

    • Always On: El sensor de huellas dactilares permanece siempre encendido.

Mascarilla​

Puede configurar los ajustes detallados de la detección de mascarillas.

Los dispositivos de autenticación facial pueden detectar mascarillas y restringir el acceso a usuarios sin mascarilla.

N.ºElementoDescripción
Mask Detection

Puede configurar si se usa o no la detección de mascarillas. Si selecciona Use (Deny access when failed to detect mask), rechaza la autenticación de usuarios que no llevan mascarilla y guarda los registros de eventos. Si selecciona Use (Allow access after leaving log when failed to detect mask), los usuarios que no llevan mascarilla pueden autenticarse, pero los registros de eventos siguen guardándose.

Mask Detect LevelPuede configurar la sensibilidad para la detección de mascarillas.
Check Mode

  • Check after authentication: Autentique y luego verifique si el usuario lleva mascarilla.

  • Check before authentication: Verifique si el usuario lleva mascarilla y luego autentique. Este modo no intenta autenticar la identidad de un usuario si no lleva mascarilla.

  • Check without authentication: Detecte si un usuario lleva mascarilla sin autenticarlo. Cuando este modo está activado, cualquier usuario que lleve mascarilla puede acceder, sin importar la autenticación.

Asignar dispositivo a Cloud Link​

  1. En Config Tool, haga clic en Access Control debajo de Tasks.

  2. Vaya a la pestaña Manage Suprema Devices en la parte superior de la pantalla.

  3. Ingrese el Cloud Link Host (dirección IP), el Cloud Link Username (ID) y la Cloud Link Password, y luego haga clic en el botón Validate Login.

  4. Haga clic en el botón View Devices To Add.

  5. Seleccione el dispositivo que desea asignar marcando la casilla Select y haga clic en el botón Assign o Assign Selected. Para asignar todos los dispositivos, haga clic en el botón Assign All.

Configuración del formato de tarjeta​

  1. En Config Tool, haga clic en Suprema Plugin Configuration en Tasks.

  2. Seleccione el tipo y formato de tarjeta.

    • Orden de bytes: admite MSB o LSB

    • Tipo de CSN Card_Format: después de habilitar el tipo de CSN Card_Format y seleccionar un formato Wiegand, el plugin convierte los datos de la tarjeta recibidos del dispositivo al formato Wiegand seleccionado y los entrega a Cloud Link.

    • Configuración de tarjeta Wiegand: los dispositivos pueden leer tarjetas Wiegand según el formato de tarjeta Wiegand seleccionado.

    • Credencial personalizada: al usar un formato de credencial personalizada, verifique antes la compatibilidad con los dispositivos Suprema.

  3. Haga clic en botón de guardar para aplicar el tipo y formato de tarjeta seleccionados a todos los dispositivos.

Configuración de puertas​

  1. En Config Tool, haga clic en Area view debajo de Tasks.

  2. Seleccione la puerta y vaya a la pestaña Hardware.

  3. Asigne dispositivos Suprema a Reader, Request to exit y Entry sensor.

Limitación de la configuración de entradas

En la arquitectura Suprema, los tipos de entrada (por ejemplo, REX, Door Contact) solo pueden configurarse después de asignar el dispositivo a una puerta.

Causa raíz:

  • El SDK de Suprema requiere que las entradas se asignen a nivel de puerta, no a nivel de dispositivo.

  • Por lo tanto, en Genetec no hay forma de preconfigurar los tipos de entrada antes de asignar el dispositivo a una puerta.

Precaución

Cualquier cambio en el tipo de entrada realizado desde:

Config Tool → Access Control → Cloud Link → Peripherals Tab no se aplicará al dispositivo Suprema directamente.
