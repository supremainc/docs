Configurar Formato De Tarjeta Inteligente

Configura el diseño de tarjetas inteligentes como MIFARE, iCLASS, DESFire e iCLASS Seos.

Agregar tarjeta inteligente​

  1. Haz clic en Settings en la página Launcher.

  2. Haz clic en Credential → Card Format en la barra lateral izquierda.

  3. Haz clic en Smart Card en la lista de formatos de tarjeta de la izquierda.

  4. Haz clic en Add Smart Card en la parte superior izquierda de la pantalla.

  5. Cuando aparezca la pantalla Add New Smart Card, configura cada elemento. Para obtener más información, consulta lo siguiente.

  6. Una vez que completes todos los ajustes, haz clic en Apply.

Tipos de tarjeta inteligente​

Define el tipo de tarjeta inteligente deseado en la opción Smart Card Type.

  • Suprema Smart Card

  • Custom Smart Card

Información

  • Las tarjetas inteligentes personalizadas solo admiten MIFARE, DESFire y FeliCa.

  • FeliCa solo se admite en el firmware BioStation 3 versión 1.3.0 o superior.

Guía de opciones de configuración​

Configura el diseño de la tarjeta en la sección Info.

Información

  • Los campos Primary Key y Secondary Key admiten solo hexadecimal. Ingresa los valores de la clave en el campo derecho y haz clic en Convert to HEX. Usa el valor convertido como clave del sitio.

  • Convert to HEX está disponible solo cuando la opción Smart Card Type es Suprema Smart Card.

Configurar información básica​

  • Name: Ingresa el nombre de la tarjeta inteligente.

  • Secondary Key: Configura si se usará una clave secundaria del sitio. Establécelo en Active para habilitar la configuración de la clave secundaria del sitio. Cuando se configura una clave secundaria del sitio, se utilizará para la autenticación cuando la clave primaria de la tarjeta no coincida.

Configurar tarjeta inteligente​

Configura la estructura de la tarjeta inteligente.

Tarjeta inteligente Suprema​

Los campos Primary Key y Secondary Key admiten solo hexadecimal. Ingresa el valor de la clave en cada campo y haz clic en Convert to HEX. Usa el valor convertido como clave del sitio.

MIFARE

  • Security Level: Configure el tipo de cifrado en SL1 o SL3.

  • Primary Key: Clave que cifra las comunicaciones entre la tarjeta inteligente y el lector de tarjetas.

  • Secondary Key: Se usa para la autenticación cuando la clave de sitio primaria de la tarjeta no coincide. La clave de sitio secundaria solo se puede ingresar después de activar la opción Secondary Key en la parte superior de la sección.

  • Start Block Index: Seleccione el bloque inicial donde se guardará cada plantilla. Este bloque es el índice del bloque donde se almacena la información del usuario; configúrelo en un bloque almacenable si el usuario ya usa la tarjeta inteligente.
iCLASS

  • Primary Key: Clave que cifra las comunicaciones entre la tarjeta inteligente y el lector de tarjetas.

  • Secondary Key: Se usa para la autenticación cuando la clave de sitio primaria de la tarjeta no coincide. La clave de sitio secundaria solo se puede ingresar después de activar la opción Secondary Key en la parte superior de la sección.

  • Start Block Index: Seleccione el bloque inicial donde se guardará cada plantilla. Este bloque es el índice del bloque donde se almacena la información del usuario; configúrelo en un bloque almacenable si el usuario ya usa la tarjeta inteligente.
DESFire

  • DESFire Advanced: Utilice tarjetas DESFire emitidas por terceros. Solo se puede configurar DESCire.

  • Primary Key: Clave que cifra las comunicaciones entre la tarjeta inteligente y el lector de tarjetas.

  • Secondary Key: Se usa para la autenticación cuando la clave de sitio primaria de la tarjeta no coincide. La clave de sitio secundaria solo se puede ingresar después de activar la opción Secondary Key en la parte superior de la sección.

  • ID Field: Configure para leer un área específica de la tarjeta. Funciona como un directorio que contiene File ID.

  • File ID: Configure el ID de archivo.

  • Security Level: Configure el tipo de cifrado en DES/3DES o AES.

iClass Seos

  • Primary Key: Clave que cifra las comunicaciones entre la tarjeta inteligente y el lector de tarjetas.

  • Secondary Key: Se usa para la autenticación cuando la clave de sitio primaria de la tarjeta no coincide. La clave de sitio secundaria solo se puede ingresar después de activar la opción Secondary Key en la parte superior de la sección.

  • ADF Address Value: Dirección ADF donde se almacenan las credenciales digitales.
Diseño​

Modifica el diseño que registra la información del usuario, facial y de huellas digitales.

  • Template Count: Establece el número de plantillas de huellas digitales que se incluirán en el diseño.

  • Template Size: Define la cantidad de bytes que usa cada plantilla de huellas digitales.

  • Use Face Template: Elige si se usará la plantilla facial.

  • Face Template Size: Establece la cantidad de bytes que usa la plantilla facial.

Información

Use Face Template está disponible en los modelos FaceStation F2, BioStation 3 y BioEntry W3.

Tarjeta inteligente personalizada​

Las tarjetas inteligentes personalizadas solo admiten MIFARE, DESFire y FeliCa.

Información

FeliCa solo se admite en el firmware BioStation 3 versión 1.3.0 o superior.

MIFARE

  • Security Level: Configure el tipo de cifrado en SL1 o SL3.

  • Primary Key: Clave que cifra las comunicaciones entre la tarjeta inteligente y el lector de tarjetas.

  • Secondary Key: Se usa para la autenticación cuando la clave de sitio primaria de la tarjeta no coincide. La clave de sitio secundaria solo se puede ingresar después de activar la opción Secondary Key en la parte superior de la sección.

  • Block Index: Seleccione el bloque inicial donde se guardará cada plantilla. Este bloque es el índice del bloque donde se almacena la información del usuario; configúrelo en un bloque almacenable si el usuario ya usa la tarjeta inteligente.

  • Skip Bytes: Configure el punto de inicio para leer el número de tarjeta.

  • Data Size: (Cuando los valores configurados de Primary Key y Secondary Key coinciden con la configuración de la tarjeta) Configure el tamaño de los datos de la tarjeta que se leerán.

DesFire

  • DESFire Advanced: Utilice tarjetas DESFire emitidas por terceros. Solo se puede configurar DESCire.

  • Primary Key: Clave que cifra las comunicaciones entre la tarjeta inteligente y el lector de tarjetas.

  • Secondary Key: Se usa para la autenticación cuando la clave de sitio primaria de la tarjeta no coincide. La clave de sitio secundaria solo se puede ingresar después de activar la opción Secondary Key en la parte superior de la sección.

  • ID Field: Configure para leer un área específica de la tarjeta. Funciona como un directorio que contiene File ID.

  • File ID: Configure el ID de archivo.

  • Security Level: Configure el tipo de cifrado en DES/3DES o AES.

  • Skip Bytes: Configure el punto de inicio para leer el número de tarjeta.

  • Data Size: (Cuando los valores configurados de Primary Key y Secondary Key coinciden con la configuración de la tarjeta) Configure el tamaño de los datos de la tarjeta que se leerán.

Felica

  • System Code: Ingrese el código de sistema para leer tarjetas FeliCa en hexadecimal, con un máximo de 4 caracteres.

  • Service Code: Ingrese el código de servicio para leer tarjetas FeliCa en hexadecimal, con un máximo de 4 caracteres.

  • ID Field: Configure para leer un área determinada de la tarjeta. Configure hasta 8 bloques para lectura haciendo clic en + Add.

    Block Number: Configure el bloque que se leerá de la tarjeta. (0-150)
