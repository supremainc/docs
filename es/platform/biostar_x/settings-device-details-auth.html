Configuración De Autenticación

Se proporcionan instrucciones para varios ajustes relacionados con la autenticación de usuarios en el dispositivo. La sección Authentication ofrece información paso a paso sobre los métodos de autenticación y las principales opciones de autenticación compatibles con el dispositivo. Tenga en cuenta que los cambios en los ajustes se aplicarán al dispositivo real y configure la política de autenticación según su entorno, consultando las características y precauciones de cada opción.

Información

  • Las opciones de red disponibles pueden variar según el tipo de dispositivo.

  • Si el usuario ha realizado cambios, haga clic en Apply. Los cambios no se aplicarán al dispositivo si no se guardan.

Configuración del método de autenticación​

Configure el modo de autenticación del dispositivo, los permisos, la coincidencia con el servidor y el tiempo de espera de autenticación.

  • Auth Mode: Configure el modo de autenticación del dispositivo combinando una o más credenciales. Para obtener más información, consulte lo siguiente.

    • Haga clic en para modificar el modo de autenticación agregado.

    • Haga clic en para eliminar el modo de autenticación agregado.

  • Full Access: A los usuarios registrados en el dispositivo se les pueden otorgar todos los permisos de acceso sin definir ajustes de grupo de acceso.

    Información

    Cuando esta opción está configurada en Use, el dispositivo no se puede registrar en Access Level ni en Floor Level.

  • Auth Timeout: Es el tiempo de espera para autenticar la segunda credencial cuando se utilizan varias credenciales en la opción Auth Mode. Si la credencial no se autentica dentro del tiempo especificado, la autenticación fallará.

  • Server Matching

    • Active: Activa la coincidencia con el servidor. Configure la función de coincidencia con el servidor para autenticar con la información de usuario almacenada en el servidor donde está instalado BioStar X.

    • Inactive: Desactiva la coincidencia con el servidor. En este caso, la autenticación se realiza utilizando la información de usuario registrada en el dispositivo.

    Ver los dispositivos y versiones de firmware que admiten la función Server Matching

    • Versión de firmware de CoreStation 1.0.0 o superior

    • Firmware de BioEntry P2 1.0.0 o posterior

    • Firmware de BioEntry W2 1.0.0 o posterior

    • Firmware de BioStation L2 1.0.0 o posterior

    • Firmware de BioStation A2 1.0.0 o posterior

    • Firmware de BioStation 2 1.2.0 o posterior

    • Firmware de BioLite Net 2.2.0 o posterior

    • Firmware de BioEntry Plus 2.2.0 o posterior

    • Firmware de BioEntry W 2.2.0 o posterior

    • Firmware de XPass 2.2.0 o posterior

    • Firmware de XPass S2 2.2.0 o posterior

    • Firmware de BioLite N2 1.0.0 o posterior

    • Versión de firmware de XPass D2 1.0.0 o superior

    • Firmware de XPass 2 1.0.0 o posterior

    • FaceStation 2 versión de firmware 1.4.0 o superior

    • FaceStation F2 versión de firmware 1.0.0 o superior

    • X-Station 2 versión de firmware 1.0.0 o superior

    • BioStation 3 firmware 1.0.0 o superior

    • BioEntry W3 versión de firmware 1.0.0 o superior

    • FaceLite no admite la función de coincidencia con el servidor.

    • Los dispositivos FaceStation F2, BioStation 3 y BioEntry W3 no pueden usar la coincidencia con el servidor para la autenticación facial.

  • Face Detection Level: Establezca los pasos del algoritmo para reconocer rostros con la cámara integrada cuando el usuario se autentique.

    • Normal: Detecta rostros a una distancia equivalente a la longitud del brazo de una persona.

    • High: Requiere que el usuario esté más cerca que la longitud del brazo de una persona para detectar un rostro.

    • Not Use: No utiliza la funcionalidad de reconocimiento facial.

    Información

    Esta opción es compatible con BioStation A2.

  • User ID Display: Controle si se muestra u oculta la ID de usuario en el dispositivo cuando la autenticación es exitosa.

    • Display All / Mask All but First Letter / Hide All

  • User Name Display: Controle si se muestra u oculta el nombre de usuario en el dispositivo cuando la autenticación es exitosa.

    • Display All / Mask All but First Letter / Hide All

Agregar modo de autenticación​

Configure el modo de autenticación del dispositivo en la opción Auth Mode. Configure el modo de autenticación combinando credenciales como huella digital, ID, tarjeta, PIN y rostro.

  1. En la opción Auth Mode, haga clic en Add.

  2. Cuando aparezca la ventana Add New Auth Mode, arrastre el modo de autenticación deseado al área central.

    Agregue hasta cinco modos de autenticación.

  3. Seleccione el elemento deseado en la opción Schedule.

  4. Cuando finalices la configuración, haz clic en Apply.

Información

Si el horario deseado no está disponible en la opción Schedule, haga clic en Add New Schedule para agregar un nuevo horario. Agregue un nuevo horario. Para obtener más información sobre la configuración de horarios, consulta lo siguiente.

Configuración de autenticación de huellas dactilares​

En el grupo Fingerprint, configure los detalles relacionados con la autenticación de huellas dactilares del dispositivo.

  • 1:N Security Level: Establezca el nivel de seguridad utilizado para autenticar huellas dactilares. Los niveles de seguridad más altos incrementan la tasa de rechazo falso (FRR) pero disminuyen la tasa de aceptación falsa (FAR).

  • Scan Timeout: Establezca el tiempo de espera para el escaneo de huellas dactilares. Si no se escanea dentro del tiempo establecido, la autenticación fallará.

  • Sensor Sensitivity: Establezca la sensibilidad del sensor de reconocimiento de huellas dactilares. Establece la sensibilidad más alta para obtener información de huella dactilar precisa.

  • 1:N Fast Mode: Establezca la velocidad de autenticación de huellas dactilares. Cuando se selecciona Automatic, se ajusta el rendimiento de la autenticación según el total de plantillas de huellas dactilares registradas en el dispositivo.

  • Template Format: Vea el formato de plantilla de huella dactilar configurado.

  • Matching Timeout: Establezca el tiempo de espera para la comparación de huellas dactilares. Si la autenticación no se completa dentro del tiempo especificado, fallará.

  • View Image: Muestre las imágenes de huellas dactilares en la pantalla durante la autenticación.

  • Sensor Mode: Establezca el modo de funcionamiento del sensor de huellas dactilares.

    • Auto On: El sensor de huellas dactilares se enciende al reconocer el dedo del usuario.

    • Always On: El sensor de huellas dactilares permanece siempre encendido.

  • Advanced Enrollment: Evalúe la calidad de las huellas dactilares escaneadas para guardar información de alta calidad. Establézcalo en Use para notificar al usuario si la calidad de la huella dactilar es baja y ayudar a escanearla correctamente.

  • Fingerprint LFD: Establezca el nivel para la detección de huellas dactilares falsas. Niveles más altos de detección de huellas falsas pueden aumentar las tasas de rechazo de las huellas reales.

  • Duplicate Check: Verifique registros duplicados al enrolar huellas dactilares.

Información

  • Las opciones disponibles para la configuración pueden variar según el tipo de dispositivo.

  • Cambiar la opción Template Format en los ajustes de autenticación por huella digital hará que todas las huellas almacenadas previamente queden inutilizables. Asegúrate de seleccionar una plantilla en la opción Template Format antes de enrolar las huellas de los usuarios.

  • La opción View Image es compatible con los modelos BioStation 2, BioStation A2, BioStation L2, BioLite N2, FaceStation F2 (FSF2-ODB) y X-Station 2 (XS2-ODPB, XS2-OAPB).

  • La opción Fingerprint LFD es compatible con los modelos BioStation A2, BioStation L2, BioEntry W2, BioLite N2, FaceStation F2 (FSF2-ODB) y X-Station 2 (XS2-ODPB, XS2-OAPB).

Ajustes de autenticación facial​

En el grupo IR Face / Face, puedes configurar los detalles relacionados con la autenticación facial del dispositivo.

  • 1:N Security Level: Define el nivel de seguridad utilizado para autenticar rostros. Los niveles de seguridad más altos incrementan la tasa de rechazo falso (FRR) pero disminuyen la tasa de aceptación falsa (FAR).

  • Enrollment Time: Si el rostro del usuario no se enrola dentro del tiempo establecido al registrarlo, el enrolamiento se cancelará.

  • Motion Sensor: Define la sensibilidad para detectar movimiento alrededor del dispositivo.

  • Ambient Brightness: El dispositivo puede detectar la luminosidad ambiental y ajustar la intensidad del LED IR.

  • Enhanced fake face enrollment block: Ajusta el nivel de detección de rostros falsos. Incrementar el nivel de detección de rostros falsos puede aumentar la tasa de rechazo de rostros reales.

  • Light Brightness: Ajusta manualmente el brillo de la luz del dispositivo. Define el brillo seleccionando Normal o High, o selecciona Not Use para apagar la luz.

  • Quick Enrollment: Define si se habilita el enrolamiento rápido de rostros.

    • Enable: Configura el procedimiento de enrolamiento de rostros en un solo paso.

    • Disabled: Configura el procedimiento de enrolamiento de rostros en 3 pasos. Establece Disabled para enrolar plantillas de rostros de alta calidad.

  • Duplicate Check: Verifica registros duplicados al enrolar rostros.

  • Face Detect Setting: Configura los ajustes ambientales para reconocer el rostro del usuario durante la autenticación.

    • Maximum Head Pose Angle: Define el ángulo máximo de rotación permitido para el rostro.

    • Detection Distance: Define las distancias mínima y máxima para reconocer el rostro.

    • Wide Search: Establece ON para permitir el reconocimiento facial en toda la vista de la cámara.

  • Operation Mode: Define el modo de operación del dispositivo durante la autenticación facial.

    • Fusion Matching Mode: Realiza emparejamiento visual e infrarrojo para mejorar la precisión de la autenticación facial.

    • Fast Matching Mode: Permite la autenticación rápida de usuarios que caminan dentro del rango de autenticación del dispositivo.

  • Fake Detection: Evita la autenticación de usuarios mediante rostros falsos, como fotografías. Se activa cuando la opción Operation Mode está configurada en Fusion Matching Mode.

Información

  • Las opciones disponibles para la configuración pueden variar según el tipo de dispositivo.

  • Las opciones Ambient Brightness, Enhanced fake face enrollment block y Quick Enrollment son compatibles con los modelos FaceStation 2 y FaceLite.

  • La opción Light Brightness es compatible con firmware FaceStation F2 versión 1.1.0 o superior.

  • Las opciones Face Detect Setting y Operation Mode son compatibles con los modelos FaceStation F2 y BioStation 3.

Ajustes de QR/código de barras​

En el grupo QR/Barcode, puedes configurar los detalles para la autenticación mediante QR/código de barras en el dispositivo.

  • Use QR/Barcode through Camera: Define si se utiliza la autenticación QR/código de barras a través de la cámara del dispositivo.

  • Camera Timeout: Define el tiempo de espera de escaneo de la cámara. Si un QR/código de barras no se escanea dentro del tiempo especificado, la autenticación fallará.

  • Use QR as Card: Configura la autenticación de códigos QR con los mismos datos que las tarjetas CSN o Wiegand emitidas a los usuarios.

  • Motion Sensor: Define la sensibilidad para detectar movimiento e iniciar el escaneo de la cámara.

  • Use QR/Barcode through Scanner: Define si se utiliza el escáner del dispositivo para la autenticación QR/código de barras.

  • Scanner Timeout: Define el tiempo de espera de escaneo. Si un QR/código de barras no se escanea dentro del tiempo especificado, la autenticación fallará.

Información

  • La opción Use QR/Barcode through Camera requiere una licencia de dispositivo independiente. Para más información, consulta el siguiente.

  • Los dispositivos que admiten las opciones Use QR/Barcode through Camera, Camera Timeout, Use QR as Card y Motion Sensor son los siguientes.

    • X-Station 2 (XS2-ODPB, XS2-OAPB, XS2-DPB, XS2-APB) versión de firmware 1.2.0 o superior

    • BioStation 3 (BS3-DB, BS3-APWB) versión de firmware 1.1.0 o superior

  • Los dispositivos que admiten las opciones Use QR/Barcode through Scanner y Scanner Timeout son los siguientes.

    • X-Station 2 (XS2-QDPB, XS2-QAPB)

Ajustes de autenticación con tarjeta​

En el grupo Card Type, puedes definir los tipos de tarjetas que se usarán en el dispositivo.

Información

Solo se mostrarán en el grupo Card Type los tipos de tarjeta compatibles con el dispositivo.

CSN Card

Puedes seleccionar los tipos y formatos de tarjetas CSN y definir el orden de bytes.

  • Byte Order

    • MSB: Procesa los datos de la tarjeta en orden de unidades de byte grandes a pequeñas, almacenando secuencialmente el número de serie de la tarjeta (CSN) de izquierda a derecha.

    • LSB: Procesa los datos de la tarjeta en orden de unidades de byte pequeñas a grandes, almacenando el número de serie de la tarjeta (CSN).

  • Format Type

    • General: Lee y muestra el número de serie (CSN) de la tarjeta sin conversión adicional.

    • Wiegand: Transforma y muestra el número de serie (CSN) de la tarjeta según el formato Wiegand definido por el usuario. Para información sobre cómo establecer un nuevo formato Wiegand, consulta el siguiente.

Wiegand Card​

Selecciona los tipos de tarjetas Wiegand y define los formatos Wiegand. Elija el formato Wiegand que se utilizará en la opción Wiegand Format.

Información

Para información sobre cómo establecer un nuevo formato Wiegand, consulta el siguiente.

Suprema Smart Card​

Seleccione los tipos de tarjetas inteligentes Suprema y configure la estructura y el orden de bytes de la tarjeta.

  • Suprema Smart Card Layout: Elija la estructura de tarjeta inteligente Suprema. Para obtener información sobre cómo configurar un nuevo diseño de tarjeta inteligente, consulte lo siguiente.

  • Suprema Smart Card Output Byte Order

    • MSB: Procesa los datos de la tarjeta en orden de unidades de bytes grandes a pequeñas.

    • LSB: Procesa los datos de la tarjeta en orden de unidades de bytes pequeñas a grandes.

Custom Smart Card​

Seleccione los tipos de tarjetas inteligentes personalizadas emitidas por terceros y configure la estructura y el orden de bytes de la tarjeta.

  • Custom Smart Card Layout: Elija la estructura de tarjeta inteligente personalizada. Para obtener información sobre cómo configurar un nuevo diseño de tarjeta inteligente, consulte lo siguiente.

  • Custom Smart Card Byte Order

    • MSB: Procesa los datos de la tarjeta en orden de unidades de bytes grandes a pequeñas.

    • LSB: Procesa los datos de la tarjeta en orden de unidades de bytes pequeñas a grandes.

Ver dispositivos y versiones de firmware que admiten tarjetas inteligentes personalizadas

  • XPass D2 versión de firmware 1.7.1 o superior

  • BioEntry P2 versión de firmware 1.5.1 o superior

  • BioEntry W2 versión de firmware 1.8.0 o superior

  • BioStation 2a versión de firmware 1.1.0 o superior

  • X-Station 2 versión de firmware 1.3.0 o superior

  • BioStation 3 versión de firmware 1.3.0 o superior

  • BioEntry W3 versión de firmware 1.0.0 o superior

  • BioLite N2 versión de firmware 1.6.2 o superior

CSN Mobile​

Configure el método para reconocer tarjetas móviles.

  • NFC: Reconoce tarjetas móviles mediante comunicación NFC desde dispositivos móviles.

  • BLE: Reconoce tarjetas móviles mediante comunicación Bluetooth Low Energy (BLE).

Template on Mobile​

Especifique cómo funciona el reconocimiento en Template on Mobile y configure el orden de bytes para los biométricos enrolados directamente por el usuario en el dispositivo.

  • ToM Output Byte Order

    • MSB: Procesa los datos de la tarjeta en orden de unidades de bytes grandes a pequeñas.

    • LSB: Procesa los datos de la tarjeta en orden de unidades de bytes pequeñas a grandes.

Ver dispositivos y versiones de firmware que admiten plantillas en el móvil

  • BioStation 3 versión de firmware 1.2.0 o superior

  • BioEntry W3 versión de firmware 1.0.0 o superior

Configuración de autenticación por PIN​

En el grupo PIN, active o desactive la opción Scramble Keypad para la autenticación por PIN del dispositivo.

Información

La opción Scramble Keypad solo está disponible en los productos Suprema que admiten interfaces de usuario a través de la pantalla LCD.
