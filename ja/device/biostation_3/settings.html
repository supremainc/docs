設定

システムおよびデバイスの設定方法について案内します。

画面とサウンド​

デバイスの画面とサウンドの設定を変更できます。

  1. を選択し、管理者権限の認証手段で認証します。

  2. SETTINGS → DISPLAY & SOUND を押してください。

  3. 必要な項目を変更してください。

    • Home Screen：ホーム画面の背景に表示する項目を選択できます。

    • Language：画面に表示される言語を選択できます。

    • Device Private Msg.：ユーザーが認証する際に画面に表示する個人メッセージの使用有無を設定できます。 Device Private Msg. は BioStar Device SDK を使用して設定できます。 設定していない場合、デバイスで Device Private Msg. を有効にしても、認証が成功したときにメッセージは表示されません。

    • Server Private Msg.：ユーザーが認証する際に画面に表示される個人メッセージの使用有無を設定できます。 Server Private Msg.はBioStar X APIを使用して設定できます。 設定していない場合、デバイスでServer Private Msg.を有効にしても、認証成功時にメッセージは表示されません。

    • Screen Saver：スクリーンセーバーの使用有無を設定できます。 スクリーンセーバーを使用すると、デバイスを使用していないときに画面の明るさを下げ、不要な電力消費を抑えることができます。

    • Menu Timeout：メニュー画面が自動的に消えるまでの時間（秒）を設定できます。 設定した時間内にボタン入力がない場合は、ホーム画面に戻ります。

    • Message Timeout：設定完了メッセージや通知メッセージなどが自動的に消えるまでの時間（秒）を設定できます。

    • Backlight Timeout：画面のバックライトが消灯するまでの時間（秒）を設定できます。

    • Voice Instruction：通知音の代わりに音声ガイダンスを使用できます。

    • Volume：音量を設定できます。

    • Intercom Speaker Volume：IPインターホン使用時にスピーカーから出力される音量を調整できます。

    • Intercom Microphone Volume：IPインターホン使用時にマイクに入力される音量を調整できます。

情報

BioStar Device SDK の詳細については、Suprema ナレッジベースの BioStar Device SDK ページ を参照してください。

通信​

イーサネット、サーバー、OSDP 通信など、デバイスの通信設定を変更できます。

イーサネット​

  1. を選択し、管理者権限の認証手段で認証します。

  2. SETTINGS → NETWORK → Ethernetを選択します。

  3. 必要な項目を設定してください。

    • DHCP：DHCPの使用有無を設定します。 DHCP設定をオフにすると、IP Address, Gateway, Subnet Maskをユーザーが編集できます。

    • Device Port：デバイスのポートを設定できます。

    • IP Address：デバイスのIPアドレスを確認できます。 編集するにはDHCP設定をオフにします。

    • Gateway：デバイスのゲートウェイを確認できます。 編集するにはDHCP設定をオフにします。

    • Subnet Mask：デバイスのサブネットマスクを確認できます。 編集するにはDHCP設定をオフにします。

    • DNS：ドメインネームサーバーのアドレスを設定できます。

  4. 設定を保存するにはOKを押します。

情報

EthernetはWirelessと同時には使用できません。

無線 LAN​

  1. を選択し、管理者権限の認証手段で認証します。

  2. SETTINGS → NETWORK → Wirelessを選択します。

  3. 接続可能なネットワークの一覧が表示されます。

    • Hidden Network：ネットワーク情報を直接入力して無線ネットワークを追加し、接続できます。

      • SSID：SSIDを入力します。 SSID には英字（大文字と小文字を区別）、数字、特殊記号を含めて最大 32 文字まで入力できます。

      • Password：パスワードを入力します。 パスワードには英字、数字、特殊記号のみ入力できます。

  4. 接続したいネットワークを選択し、パスワードを入力します。 OKを押すと無線ネットワークに接続されます。

  5. 使用する無線ネットワークのを押してからDHCP設定をオフにすると、IP Address, Gateway, Subnet Maskをユーザーが編集できます。

情報

  • WirelessはBS3-APWBモデルでのみ使用できます。

  • WirelessはEthernetと同時には使用できません。

  • Wirelessで接続するには無線ルーターが必要です。 無線ルーターの設置および設定については、無線ルーターのユーザーガイドを参照してください。

サーバー​

  1. を選択し、管理者権限の認証手段で認証します。

  2. SETTINGS → NETWORK → Serverを選択します。

  3. 必要な項目を設定してください。

    • Device → Server：デバイスから入力した情報を持つサーバーに直接接続信号を送信できます。 Device → Server設定をオンにすると、Server IP, Server URL, Server Portをユーザーが入力できます。

    • Server IP：BioStar XがインストールされたPCのIPを入力します。

    • Server URL：Server IPの代わりにServer URLを入力できます。

    • Server Port：BioStar XがインストールされたPCのポートを入力します。

  4. 設定を保存するにはOKを押します。

OSDP 通信設定​

  1. を選択し、管理者権限の認証手段で認証します。

  2. SETTINGS → NETWORKを選択します。

  3. OSDP Settingで必要な項目を設定します。

    • RS-485：RS-485の動作モードを選択できます。

    • Baud Rate：必要な通信速度を選択できます。

    • Authentication Result：Intelligent Slaveを使用する場合、他社コントローラーまたはデバイスの認証結果を画面に表示するかどうかを選択できます。 RS-485モードをDefaultまたはSlaveに設定したときに使用できます。

デバイス​

IPインターホン​

IP インターホンのアカウントと内線を設定できます。

  1. を選択し、管理者権限の認証手段で認証します。

  2. SETTINGS → DEVICE → Intercomを押します。

    • Intercom：IPインターホンの有効化を設定できます。 IPインターホン設定をオンにすると、SIP AccountとExtensionsを設定できます。

    • SIP Account：SIPサーバーにデバイスを登録できるよう、各項目を正しく入力してください。

    • Intercom Video Resolution：デバイスから送出されるIPインターホン映像の解像度を選択できます。

    • Extensions：内線番号を追加および削除できます。

    • Display Extension Number：内線番号の表示可否を設定できます。

    情報

    • 内線番号にDisplay Nameを設定していない場合、発信画面で受信者を識別できなくなります。

    • 内線番号には英字（大文字・小文字）、数字、特殊記号 + - @ . のみ入力できます。

  3. 設定を保存するにはOKを押します。

ライセンス​

デバイスライセンスを有効化したり、Licenses in Slave DevicesとActivated Licenses in Deviceを確認できます。

  1. を選択し、管理者権限の認証手段で認証します。

  2. SETTINGS → DEVICE → Licenseをタップします。

  3. デバイスライセンスを有効化するには、ライセンスが保存された USB をデバイスに接続し、 を選択してください。

  4. 有効化できるライセンスが画面に表示されます。 必要なライセンスを有効化してください。

情報

  • デバイスライセンスを発行するには、購入先にお問い合わせください。

  • デバイスライセンス 1 つにつき、1 つの機能を使用できます。

  • 1 つのデバイスライセンスファイルに複数のデバイスライセンスを含めることができます。 （最大 100 個まで対応）

  • デバイスライセンスファイルは暗号化されているため、任意に変更できません。

  • デバイスライセンスはデバイスIDを基準に発行されます。 不正な方法でデバイスIDを変更した場合、ライセンスの保証サービスは提供されません。

リレー​

デバイスでOpen TimeとExit Buttonの入力ポートを設定できます。

  1. を選択し、管理者権限の認証手段で認証します。

  2. SETTINGS → DEVICE → Relayをタップします。

    • Relay：リレーの有効化を設定できます。 リレー設定をオンにすると、Open Time、Exit Buttonを設定できます。

    • Open Time：正しくユーザー認証が行われたときにドアを開けておく時間を設定できます。

    • Exit Button：退室ボタンが接続されている入力ポートを選択してください。

    • Switch：退室ボタン入力ポートのスイッチ（N/O または N/C）の種類を選択できます。

日付と時刻​

日付と時刻を設定できます。 正確なログデータを収集するために適切に設定してください。

  1. を選択し、管理者権限の認証手段で認証します。

  2. SETTINGS → DEVICE → Date & Timeをタップします。

  3. 必要な項目を変更してください。

    • Date & Time：現在の日付と時刻を確認できます。 直接修正するには、Time Syncをオフにしてください。

    • Time Zone：現在の地域の時間基準を設定できます。

    • Time Sync：サーバーと時刻を同期できます。 サーバーと時刻を同期するには、Time Syncをオンにしてください。

    • Date Format：日付形式を設定できます。 YYYY/MM/DD、MM/DD/YYYY、DD/MM/YYYYを選択できます。

    • Time Format：時刻形式を設定できます。 AM/PM、24-Hourを選択できます。

サマータイム​

デバイスで夏時間（DST）を適用できます。 開始日時と終了日時を適切に設定してください。

  1. を選択し、管理者権限の認証手段で認証します。

  2. SETTINGS → DEVICE → Daylight Saving Timeを選択してから、をタップしてサマータイムを追加します。

  3. 必要な項目を変更してください。

    • Name：サマータイムの名前を入力してください。

    • Start Month / Start Week / Start Day / Start Time：サマータイム適用を開始する月、週、曜日、時刻を設定してください。

    • End Month / End Week / End Day / End Time：サマータイム適用を終了する月、週、曜日、時刻を設定してください。

  4. 設定を保存するにはOKを押します。

デバイス情報​

モデル名、デバイス ID、シリアル番号、デバイスおよびファームウェアのバージョン、カーネル、MAC アドレスを確認できます。

  1. を選択し、管理者権限の認証手段で認証します。

  2. SETTINGS → DEVICE → Device Infoを選択します。 Model Name、Device ID、Serial No.、HW、FW、Kernel、MAC、WiFi MACの情報を確認できます。

情報

  • MAC項目を長押しすると、偽顔検出のアルゴリズムバージョンを確認できるFAS_IR_SP情報が表示されます。 FAS_IR_SP項目をタップすると、全体ハッシュ値をポップアップメッセージで確認できます。

  • WiFi MAC項目はBS3-APWBモデルでのみ確認できます。

メモリ情報​

メモリ使用情報を確認できます。

  1. を選択し、管理者権限の認証手段で認証します。

  2. SETTINGS → DEVICE → Memory Infoを選択します。 デバイスのメモリ使用状況を確認できます。

USB メモリ​

デバイスに USB メモリを接続してユーザー情報をインポートしたり、ログとユーザー情報をエクスポートしたりできます。 ファームウェアをアップグレードすることもできます。

  1. を選択し、管理者権限の認証手段で認証します。

  2. SETTINGS → DEVICE → USB Memoryを選択します。

  3. 必要な項目を選択して設定を変更してください。

    • Import：接続されたUSBメモリからユーザー情報をインポートできます。

    • Export：接続されたUSBメモリにエクスポートする情報を選択できます。

    • FW Upgrade：接続されたUSBメモリにファームウェアファイルが保存されている場合、ファームウェアをアップグレードできます。

注意

ファームウェアのアップグレード中に電源が切れないよう注意してください。 故障の原因になります。

情報

  • USB メモリはファイルシステムが exFAT 形式の製品を使用することを推奨します。 FAT32 形式の USB メモリを使用すると、保存されている顔データが 50,000 件を超える場合にデバイスが誤動作する可能性があります。

  • USB メモリの空き容量を確認してください。 顔データ 100,000 件の場合、15 GB 以上の空き容量が必要です。

セキュリティタンパー​

セキュリティタンパーの使用可否を設定できます。 セキュリティタンパーが発生した場合、すべてのユーザー、すべてのログ、セキュリティキーが削除されます。

  1. を選択し、管理者権限の認証手段で認証します。

  2. SETTINGS → DEVICEを選択します。

  3. タンパー発生時にすべてのユーザー、すべてのログ、セキュリティキーが削除されるように設定するには、Secure Tamper設定をオンにしてください。

RTSP​

サードパーティ製ビデオ管理システム (VMS) と接続して、製品のビジュアルカメラで撮影した映像をモニタリングします。

  1. を選択し、管理者権限の認証手段で認証します。

  2. SETTINGS → DEVICE → RTSP を選択してください。

  3. 必要な項目を選択して設定を変更してください。

    • RTSP：RTSP の有効化を設定できます。 RTSP 設定を有効にすると、ID、Password、RTSP Address、RTSP Port、RTSP Video Resolution を設定できます。

    • ID：ID を入力してください。 ID には英字（大文字と小文字を区別）、数字、一部の記号 - _ を組み合わせて最大 32 文字まで入力できます。

    • Password：パスワードを入力します。 パスワードには英字（大文字または小文字）と数字を組み合わせて 8～32 文字を入力できます。

    • RTSP Address：RTSP アドレスを入力してください。

    • RTSP Port：RTSP ポート番号を入力してください。

    • RTSP Video Resolution：デバイスから出力される RTSP 映像の解像度を選択できます。

情報

デバイスをスレーブデバイスとして構成した場合、RTSP 機能は使用できません。

マスター管理者​

マスター管理者の認証手段を変更できます。

  1. を選択し、管理者権限の認証手段で認証します。

  2. SETTINGS → DEVICE → Master Admin を選択してください。

  3. 希望する認証手段を設定してください。

情報

  • 少なくとも 2 種類以上のクレデンシャルを登録する必要があります。

  • この機能はファームウェア v1.4.0 以上の新しいデバイスでのみ使用できます。

管理者 2 段階認証​

ファームウェアをアップグレードしたデバイスは Master Admin 設定を提供しませんが、Admin 2-step Authentication オプションを使用して全管理者の権限を強化し、デバイスのセキュリティを向上させることができます。

  1. を選択し、管理者権限の認証手段で認証します。

  2. SETTINGS → DEVICE → Admin 2-step Authentication を選択してください。

情報

  • Admin 2-step Authentication はファームウェアを v1.4.0 以上にアップグレードした後に使用でき、アップグレード後に下位バージョンへダウングレードすることはできません。

  • Admin 2-step Authentication オプションを有効にしていない場合、1 段階認証がデフォルトです。

  • すべての管理者に2種類以上のクレデンシャルが登録されていない状態で有効化すると、設定は失敗しエラーメッセージが表示されます。 すべての管理者に2種類以上のクレデンシャルを登録してから、再試行してください。

注意

Admin 2-step Authentication を有効にした後にすべての管理者のクレデンシャルを 2 種未満に削除すると、BioStar X に接続できない状況では管理者メニューにアクセスできなくなります。 そのため、管理者クレデンシャルを削除する際は十分に注意してください。

デバイスの再起動​

デバイスを再起動できます。

  1. を選択し、管理者権限の認証手段で認証します。

  2. SETTINGS → DEVICE → Restart Device を選択してください。

  3. デバイスを再起動するには OK を押してください。 前の画面に戻るには Cancel を押してください。

デフォルト値に戻す​

デバイス設定、ネットワーク設定、ユーザー権限などが初期化されます。

  1. を選択し、管理者権限の認証手段で認証します。

  2. SETTINGS → DEVICE → Restore Default を選択してください。

    • All settings：デバイスに保存されているすべての設定を初期化します。 デバイス設定をすべて初期化するには OK を押してください。 前の画面に戻るには Cancel を押してください。

    • Keep Network Settings：ネットワーク設定を除くすべての設定を初期化します。 ネットワーク設定を除くすべての設定を初期化するには OK を押してください。 前の画面に戻るには Cancel を押してください。

    • Factory Default：デバイスに保存されているすべての情報とルート証明書を削除し、設定を初期化します。

    • Delete the Root Certificate：デバイスに保存されているルート証明書を削除します。

  3. Restore Default を実行すると、デバイスが再起動します。

情報

  • 初期化を実行すると、ユーザー権限も同時に初期化されます。 初期化を実行した後、必ずユーザー権限を再設定してください。

  • 初期化を実行しても言語設定は変更されません。

  • Factory Default メニューは、デバイスにルート証明書が保存されている場合に使用できます。

  • Delete the Root Certificate メニューは、デバイスにルート証明書が保存され、全管理者が設定されている場合に使用できます。

ユーザー ID の種類​

デバイスに登録するユーザー ID の形式を選択できます。

  1. を選択し、管理者権限の認証手段で認証します。

  2. SETTINGS → DEVICE → User ID Type を選択してください。

    • Number：ユーザー ID は数字のみ入力できます。

    • Alphanumeric：ユーザー ID は英字、数字、一部記号 _ - を組み合わせて入力できます。

  3. 必要な項目を選択してください。
