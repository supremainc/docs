XPass D2 を設定する

XPass D2 に適用する設定をあらかじめテンプレートとして作成し、個別デバイスに直接適用できます。 PC で管理プログラムに接続して設定を変更したりコマンドカードを使用する方法と比べて、はるかに速く設定を適用できます。

情報

デバイスのファームウェアバージョンが低い場合、いくつかの設定が適用されないことがあります。 すべての設定を適用するには、デバイスファームウェアをアップグレードするを参照してファームウェアをアップグレードしてください。

テンプレートを追加する​

RS-485 通信、カード形式、キーパッド設定、PIN、LED、ブザーなどを設定した後、テンプレートとして追加できます。 テンプレートを追加すると、複数の個別デバイスを毎回設定せずに、迅速かつ簡単に設定を適用できます。

  1. モバイル端末で Suprema Device Manager を起動します。

  2. メイン画面で XPass D2 を選択します。

  3. Template Management → を押します。

  4. テンプレートを作成するデバイスモデルを選択します。

  5. 各項目を設定した後、テンプレートを保存するには Save を押します。

情報

テンプレートは最大 100 個まで追加できます。

テンプレート名および管理者パスワードの設定​

  • Template Name：希望するテンプレート名を入力します。

  • Admin Password：管理者パスワードを入力します。

情報

  • 管理者パスワードは 6～32 桁の数字で設定できます。

  • 管理者パスワードを紛失しないよう注意してください。 管理者パスワードを紛失した場合、テンプレートを適用したデバイスに接続するにはそのデバイスを初期化する必要があります。

  • 管理者パスワードの変更について詳しくは、管理者パスワードを変更するを参照してください。

インターフェース​

  • OSDP：デバイスとマスターデバイスの通信で使用する OSDP アドレスを設定します。 0～126 の数値で設定できます。

  • Baudrate：RS-485 の転送速度を設定します。

認証​

  • Card Type：デバイスで使用するカードタイプを設定できます。

    • CSN Card：CSN カードタイプを選択し、バイト順序を設定できます。

    • Suprema Smart Card Layout：Suprema が発行したスマートカードタイプを選択できます。

    • Custom Smart Card Layout：他社が発行したスマートカードタイプを選択できます。

    • Mobile：モバイルカードタイプを設定できます。

情報

バイト順序を MSB に設定すると、カードデータを処理する際に上位バイトから下位バイトの順に処理し、カードのシリアル番号（CSN）を保存するときに左から右へ順に処理します。 LSB は下位バイトから上位バイトの順に処理し、カードのシリアル番号（CSN）を保存します。

ウィーガンドカードフォーマット​

  • Format：デバイスに指定する Wiegand カード形式を設定できます。 設定した形式でカードデータが処理されます。

    • New Wiegand Format：希望するフォーマットがない場合、 を押して新しい Wiegand フォーマットを追加できます。

      • Name：希望する Wiegand フォーマットの名前を入力します。

      • Total Bits：合計ビット数を入力します。

      • ID field：使用する ID の開始ビットと終了ビットを入力します。 Add を押すと、ID フィールドを追加できます。

      • Parity field：使用するパリティフィールドの位置と開始ビット、終了ビットを入力します。 Add を押すと、パリティフィールドを追加できます。

        情報

        パリティフィールドを追加するには、合計ビット数を入力します。

  • Pulse width (μs)：Wiegand 信号のパルス幅を設定できます。

  • Pulse interval (μs)：Wiegand 信号のパルス間隔を設定できます。

Suprema スマートカードレイアウト​

  • Secondary Key：補助サイトキーの使用可否を設定できます。 補助サイトキーを設定すると、カードの基本キーが一致しない場合に補助サイトキーを使用して認証を実行できます。

  • MIFARE：MIFARE カードを設定できます。

    • Encryption Type：暗号化方式を Crypto-1 または AES-128 に設定できます。

    • Primary Key：スマートカードとデバイス間の通信を暗号化するキーです。

    • Secondary Key：カードの基本サイトキーが一致しない場合に使用して認証を実行できます。 補助サイトキーの項目は、前レベルの Secondary Key を有効にしたときのみ表示されます。

    • Start Block Index：各テンプレートを保存する開始ブロックを選択します。 このブロックはユーザー情報を保存するブロックのインデックスです。既にスマートカードを使用している場合は、保存可能なブロックを設定します。 MIFAREのみ設定できます。

  • DESFire：DESFireカードを設定できます。

    • DESFire Advanced：他社が発行したDESFireカードを使用できます。 DESFireのみ設定します。

    • Primary Key：スマートカードとデバイス間の通信を暗号化するキーです。

    • Secondary Key：カードの基本サイトキーが一致しない場合に使用して認証を実行できます。 補助サイトキーの項目は、前レベルの Secondary Key を有効にしたときのみ表示されます。

    • App ID：アプリケーションIDを設定してください。 ファイルIDを含む一種のディレクトリとして機能します。 DESFireのみ設定します。

    • File ID：ファイルIDを設定してください。 DESFireのみ設定します。

    • Encryption Type：暗号化方式をDES/3DESまたはAESに設定できます。

  • Output Byte Order：スマートカード出力のバイト順序を設定できます。

情報

  • DESFire Advancedを使用するには、App Master Key、App Master Key Index、File Read Access Key、File Read Access Key Index、App ID、File ID、Encryption Typeの情報を正確に入力してください。

  • App Master Key、File Read Access Keyは最大32バイトの16進数のみ入力できます。

  • バイト順序を MSB に設定すると、カードデータを処理する際に上位バイトから下位バイトの順に処理し、カードのシリアル番号（CSN）を保存するときに左から右へ順に処理します。 LSB は下位バイトから上位バイトの順に処理し、カードのシリアル番号（CSN）を保存します。

カスタムスマートカードレイアウト​

  • Secondary Key：補助サイトキーの使用可否を設定できます。 補助サイトキーを設定すると、カードの基本キーが一致しない場合に補助サイトキーを使用して認証を実行できます。

  • MIFARE：MIFARE カードを設定できます。

    • Encryption Type：暗号化方式を Crypto-1 または AES-128 に設定できます。

    • Primary Key：スマートカードとデバイス間の通信を暗号化するキーです。

    • Secondary Key：カードの基本サイトキーが一致しない場合に使用して認証を実行できます。 補助サイトキーの項目は、前レベルの Secondary Key を有効にしたときのみ表示されます。

    • Block Index：各テンプレートが保存される開始ブロックを選択してください。 このブロックはユーザー情報を保存するブロックのインデックスです。既にスマートカードを使用している場合は、保存可能なブロックを設定します。 MIFAREのみ設定できます。

    • Skip Bytes：カード番号を読み取る開始位置を設定できます。

    • Data Size：（設定したPrimary KeyとSecondary Keyがカードの設定値と同じ場合）読み取るカードのデータサイズを設定できます。

  • DESFire：DESFireカードを設定できます。

    • DESFire Advanced：他社が発行したDESFireカードを使用できます。 DESFireのみ設定します。

    • Primary Key：スマートカードとデバイス間の通信を暗号化するキーです。

    • Secondary Key：カードの基本サイトキーが一致しない場合に使用して認証を実行できます。 補助サイトキーの項目は、前レベルの Secondary Key を有効にしたときのみ表示されます。

    • App ID：アプリケーションIDを設定してください。 ファイルIDを含む一種のディレクトリとして機能します。 DESFireのみ設定します。

    • File ID：ファイルIDを設定してください。 DESFireのみ設定します。

    • Encryption Type：暗号化方式をDES/3DESまたはAESに設定できます。

    • Skip Bytes：カード番号を読み取る開始位置を設定できます。

    • Data Size：（設定したPrimary KeyとSecondary Keyがカードの設定値と同じ場合）読み取るカードのデータサイズを設定できます。

  • Byte Order：スマートカード出力のバイト順序を設定できます。

情報

  • DESFire Advancedを使用するには、App Master Key、App Master Key Index、File Read Access Key、File Read Access Key Index、App ID、File ID、Encryption Typeの情報を正確に入力してください。

  • App Master Key、File Read Access Keyは最大32バイトの16進数のみ入力できます。

  • バイト順序を MSB に設定すると、カードデータを処理する際に上位バイトから下位バイトの順に処理し、カードのシリアル番号（CSN）を保存するときに左から右へ順に処理します。 LSB は下位バイトから上位バイトの順に処理し、カードのシリアル番号（CSN）を保存します。

キーパッド設定​

  • Keypad：キーパッドの使用可否を設定できます。 キーパッドを有効にすると、キーパッドでカードIDを入力して認証できます。

  • Single key input：キーパッドでカードIDを入力する際、キーを押すたびに値を送信します。

  • Buffered key input：キーパッドでカードIDを入力する際、すべてのキーを押した後に「#」キーを押すと、全値を一括で送信します。

  • Blink LED for PIN after tagging card：デバイスの認証モードをカード＋PINに設定した場合、ユーザーがカードを提示するとLEDを動作させてPIN入力を促すことができます。

  • Backlight：キーパッドのバックライトをオンまたはオフにできます。

情報

  • Keypad SettingはXPD2-GKDBデバイスのテンプレート設定画面にのみ表示されます。

  • BacklightはXPD2-GKDBハードウェアV02M、ファームウェアv1.7.1以上でのみサポートされます。

  • デバイスでキーパッドを通じて入力された値は、基本的に4ビット単位で送信されます。 キーパッドで入力したカードIDは、Wiegandカード形式に従ってカードIDと同じ形式で送信されます。

その他​

  • Secure Tamper：デバイスでタンパーイベントが発生した場合、デバイスに保存されたセキュリティキーを削除するよう設定できます。 セキュリティタンパー機能を使用するには有効にしてください。

LED / ブザー​

  • Normal：通常時にデバイスLEDに表示される色を設定できます。

  • Scan Card：デバイスにカードをスキャンする際にデバイスLEDに表示される色とブザー音の再生回数を設定できます。

  • Auth Sucess：認証成功時にデバイスLEDに表示される色とブザー音の再生回数を設定できます。

  • Auth Fail：認証失敗時にデバイスLEDに表示される色とブザー音の再生回数を設定できます。

情報

  • LEDは最大3色を交互に繰り返し表示するよう設定できます。 スロットを押して色を選択してください。

  • を選択すると、該当スロットの色表示順序をスキップし、次の順序のスロットに設定された色が表示されます。

テンプレートを適用する​

追加したテンプレートはBLE通信を利用して複数台のデバイスに同一に適用できます。

情報

デバイスのファームウェアバージョンが低い場合、いくつかの設定が適用されないことがあります。 すべての設定を適用するには、デバイスファームウェアをアップグレードするを参照してファームウェアをアップグレードしてください。

  1. モバイル機器でBluetooth機能を有効にした後、Suprema Device Managerを起動してください。

  2. メイン画面で XPass D2 を選択します。

  3. Choose a templateを押してください。 選択可能なテンプレートの一覧が表示されます。

  4. テンプレート一覧から目的のテンプレートを選択してください。

  5. Startを押してください。

  6. テンプレートを適用するデバイスにモバイル機器の背面を完全に接触させてください。

  7. テンプレート適用が完了したら、OKを押してください。

情報

  • Suprema Device Managerで変更した設定はデバイスにのみ適用され、サーバーには同期されません。

  • デバイスがマスターデバイスと接続されているか、Wiegand出力設定が変更されている場合、Suprema Device Managerにデフォルトキーで接続できません。 Suprema Device Managerに接続するにはデバイスを初期化してください。

テンプレートを管理する​

テンプレートを編集する​

  1. モバイル端末で Suprema Device Manager を起動します。

  2. メイン画面で XPass D2 を選択します。

  3. Template Managementを押してください。

  4. テンプレート一覧から編集するテンプレートを選択します。

  5. 「テンプレートを追加」を参照してテンプレートを編集します。

  6. 変更した設定を保存するには Save を、新しいテンプレートとして保存するには Save as new template を押します。

テンプレートを削除する​

  1. モバイル端末で Suprema Device Manager を起動します。

  2. メイン画面で XPass D2 を選択します。

  3. Template Managementを押してください。

  4. テンプレート一覧で削除するテンプレートを選択します。

  5. テンプレートを削除するには → OK を押します。

テンプレートをエクスポートする​

XPass D2 設定のテンプレートをアプリからエクスポートできます。

  1. モバイル端末で Suprema Device Manager を起動します。

  2. メイン画面で XPass D2 を選択します。

  3. Template Managementを押してください。

  4. テンプレート一覧画面の上部で を押してから Template Export を選択します。

  5. エクスポートするテンプレートを選択した後、 を押します。

情報

テンプレートは Suprema Device Manager アプリケーションのインストールファイルと同じ場所に作成される DeviceManager_TemplateFiles フォルダーに保存されます。

テンプレートをインポートする​

XPass D2 設定のテンプレートをアプリにインポートできます。

  1. モバイル端末で Suprema Device Manager を起動します。

  2. メイン画面で XPass D2 を選択します。

  3. Template Managementを押してください。

  4. テンプレート一覧画面の上部で を押してから Template Import を選択します。

  5. テンプレートが保存されているフォルダーでインポートするテンプレートを選択します。

情報

テンプレートは Suprema Device Manager アプリケーションのインストールファイルと同じ場所に作成される DeviceManager_TemplateFiles フォルダーに保存されます。

デバイスの検索と接続​

周辺に設置された XPass D2 を検索して接続できます。 Suprema Device Manager と接続すると、デバイスのファームウェアのアップグレードやデバイスの再起動、テンプレートのパスワード変更など、さまざまな機能を利用できます。

  1. モバイル機器でBluetooth機能を有効にした後、Suprema Device Managerを起動してください。

  2. メイン画面で XPass D2 を選択します。

  3. Search Device を押します。 接続可能なデバイスの一覧が表示されます。

  4. デバイス一覧でデバイス ID を確認してデバイスを選択するか、接続するデバイスにモバイル機器を近づけます。

  5. 接続パスワードを入力します。 入力したパスワードを画面に表示するには を押します。

  6. デバイスの接続を完了するには OK を押します。

デバイスのファームウェアをアップグレードする​

デバイスのファームウェアを簡単にアップグレードできます。

情報

  • ファームウェアをアップグレードするには、モバイル機器にファームウェアファイルをダウンロードする必要があります。 最新ファームウェアはSuprema ダウンロードセンターからダウンロードできます。

  • ファームウェアのアップグレード中は、デバイスとモバイル機器の距離を 1 m 以内に保ってください。

  1. 「デバイスの検索と接続」を参照して、ファームウェアをアップグレードするデバイスと接続します。

  2. → FW upgrade を押します。

  3. ファームウェアファイルが保存されているパスでファームウェアを選択します。 ファームウェアのアップグレードが実行されます。

  4. ファームウェアのアップグレードを完了するには OK を押します。

デバイスを再起動する​

デバイスを再起動できます。

  1. 「デバイスの検索と接続」を参照して、再起動するデバイスと接続します。

  2. → Device Restart を押します。

  3. デバイスを再起動すると、モバイル機器との BLE 通信が切断されます。 自動で再接続するには、Auto Connect 機能を有効にします。 詳細な設定方法は「アプリを設定する」を参照してください。

管理者パスワードを変更する​

テンプレートの管理者パスワードを変更できます。

  1. 「デバイスの検索と接続」を参照し、パスワードを変更するテンプレートが適用されたデバイスと接続します。

  2. → Change Password を押します。

  3. 現在のパスワードと新しいパスワードを入力します。

  4. パスワードの変更を完了するには OK を押します。

情報

  • 管理者パスワードは 6～32 桁の数字で設定できます。

  • 管理者パスワードを紛失しないよう注意してください。 管理者パスワードを紛失した場合、テンプレートを適用したデバイスに接続するにはそのデバイスを初期化する必要があります。
