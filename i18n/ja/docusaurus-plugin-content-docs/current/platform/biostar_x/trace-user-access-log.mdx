ユーザーの入退室履歴を追跡して確認することは、セキュリティを強化し運用効率を高めるうえで非常に重要です。 入退室履歴を活用することで、セキュリティの脅威を防ぎ、運用を最適化できます。 入退室権限のない場所や許可されていない時間帯に入退室を試みたユーザーを特定できます。 これにより、セキュリティの脅威に迅速に対応できます。 繰り返される認証失敗や異常な時間帯の入退室などの異常兆候を把握し、セキュリティの脅威を早期に発見できます。

次の手順に従ってユーザーの入退室履歴を確認できます。

1. <Cmd sid='shell.header.launcher' /> ページで <Cmd sid='shell.header.user' /> をクリックします。

2. ユーザーリストで入退室履歴を確認するユーザーをクリックします。

3. 画面の右側に選択したユーザーのプロフィール情報が表示されます。

<Image src='/img/biostarx-users-list-view-profile.png' />

プレビュー画面の下部にある <Cmd sid='team.event.trace' /> セクションで、そのユーザーの入退室履歴を確認できます。 

:::info

* リアルタイムの入退室イベントは <Cmd sid='shell.header.arena' /> ページで確認できます。 詳細は [以下](monitoring-event) を参照してください。

* 過去の入退室イベントは <Cmd sid='shell.header.data' /> ページで確認できます。 詳細は[次のドキュメント](how-to-guide-data)を参照してください。

:::
