複数の顧客やパートナーから、ユーザーリストを一括で追加する際に問題が発生するという報告がありました。 ほとんどの問題は、形式が不正なCSVファイルが原因です。 一括追加機能が期待通りに動作するよう、以下の指示に正確に従ってください。

:::info

v2.10から、BioStar Airは不正な項目を含むCSVファイルをアップロードした場合、検証レポートを返します。 エラーを修正した後、ファイルを再アップロードしてください。

<Image src='/img/air/bsair-add-bulk-import-user-csv-error.png' />

:::

## CSVファイルの準備

### CSVテンプレートのダウンロード

1. 画面左のサイドバーで <Cmd sid='user_menu' product='air' /> → <Cmd sid='user_submenu' product='air' /> に移動します。

2. 画面右上の <Cmd sid='add_user_btn' product='air' /> → <Cmd sid='user_actions_import_title' product='air' /> をクリックします。

3. <Cmd sid='user_actions_import_title' product='air' /> パネルが表示されたら、<Cmd sid='user_actions_import_download_btn_text' product='air' /> ボタンをクリックします。

4. CSVテンプレートファイル(*biostar-import-user-csv-example.csv*)をローカルに保存します。

### CSVテンプレートの正確な形式を使用する

* **列(column)を手動で追加または削除しないでください**。 予期しない列(column)があるとエラーが発生する可能性があります。

* 追加フィールド（例：従業員IDまたは部署）を含める必要がある場合は、ユーザー一覧の上部にある<Cmd sid='user_setting_templ_action' product='air' />(<IcAirUtemplate />)機能を使って該当するフィールドを追加してください。 フィールドを追加すると、ダウンロードしたCSVファイルに該当する列が自動的に含まれます。

各列の要件を参照してデータを正しく入力してください。

#### <Cmd sid='user_csv_example_header1' product='air' className='normal' />

* 各ユーザーの一意の数値IDです。

* <Cmd sid='user_csv_example_header1' product='air' className='bold' /> 列は**空にすることができ**、システムが自動的に値を割り当てます。

* ユーザーIDを手動で入力する場合

  * **数字のみ** を使用してください。 

  * **一意の値** を使用してください。

  * 文字、空白、ハイフン、または特殊文字を含めないでください。

:::caution

文字を含む会社固有の従業員IDを使用する場合は、<Cmd sid='user_csv_example_header1' product='air' className='bold' /> フィールドを使用しないでください。 代わりに、<Cmd sid='user_setting_templ_action' product='air' /> 機能を使用して別のカスタムユーザーフィールドを作成してください。

:::

#### <Cmd sid='user_csv_example_header7' product='air' className='normal' />

* 有効なメール形式を使用してください。 [検証式](#excelFormulas)を参照してメール形式を検証できます。

  * 空白を含めることはできません。

  * `@`（アットサイン）を含める必要があります。

  * ドメインにはピリオド(`.`)およびトップレベルドメインを含める必要があります。

    > 例：example.com、gmail.com、suprema.co.kr

  * アドレスの末尾に追加のピリオドがあってはなりません。

  * 特殊文字または不可視文字があってはなりません。

  * メールアドレスである必要があります。 '提供されていません' または '提供しない' のようなエラーが発生する可能性があります。

* メールアドレスは**一意である必要があります**。 ファイル内や既存ユーザー間での重複は許可されません。

* 以下のクレデンシャルを発行する場合、メールアドレスは必須です。

  * <Cmd sid='card_type_mobile' product='air' />

  * <Cmd sid='card_type_option3' product='air' />

  * <Cmd sid='card_type_option4' product='air' />

* RFIDカードのみを所持しているユーザーはメールアドレスの入力は不要です。

#### <Cmd sid='user_csv_example_header8' product='air' className='normal' />

* モバイルSMSでクレデンシャルを送信しない場合は任意です。

* メールアドレスと同様に**一意である必要があり**、ファイル内や既存ユーザー間での重複は許可されません。

* クレデンシャルを配布する際は、メールを推奨します。 SMSメッセージは追加料金が発生します。

#### 日付 (<Cmd sid='user_csv_example_header5' product='air' className='normal' /> / <Cmd sid='user_csv_example_header6' product='air' className='normal' />)

すべての日付フィールドは以下の正確な形式に従う必要があります。

```plaintext
YYYY-MM-DDTHH:MM:SS.sssZ

Example:
1995-05-09T00:00:00.000Z
```

'Z'を欠いていたり、ダッシュ(`—`)の代わりにスラッシュ(`/`)を使用したり、時間を省略するなどの些細な違いもアップロード失敗の原因になります。

#### クレデンシャルの要件

| クレデンシャル形式          | メールまたは電話番号 |
| --- | --- |
| <Cmd sid='card_type_rf' product='air' /> Only | いいえ。                       |
| <Cmd sid='card_type_mobile' product='air' /> / <Cmd sid='card_type_option3' product='air' /> / <Cmd sid='card_type_option4' product='air' /> | はい、1つ必要です。    |

* クレデンシャルはメールアドレスまたは電話番号を通じて送信される必要があります。

* 両方を提供することはできますが、**モバイル形式のクレデンシャル**には**最低1つが必要**です。

* メールは無料です。 SMSは料金が発生します。

## 事前検証のためのExcel数式 {#excelFormulas}

データをアップロードする前に、**ヘルパー列**で次の数式を使ってデータを確認することを推奨します。

### メールまたは電話番号の重複検証

```excel-formula title="Detect Duplicate Emails or Phone Numbers"
=IF(E2="", "", IF(COUNTIF(E$2:E2, E2)>1, "Duplicate", "First Occurrence"))
```

### 日付形式の検証

```excel-formula title="Validate ISO 8601 Date Format (Column Q)"
=IF(Q2="", "", IF(AND(LEN(Q2)=24, 
  ISNUMBER(VALUE(MID(Q2,1,4))), MID(Q2,5,1)="-", 
  ISNUMBER(VALUE(MID(Q2,6,2))), MID(Q2,8,1)="-", 
  ISNUMBER(VALUE(MID(Q2,9,2))), MID(Q2,11,1)="T", 
  ISNUMBER(VALUE(MID(Q2,12,2))), MID(Q2,14,1)=":", 
  ISNUMBER(VALUE(MID(Q2,15,2))), MID(Q2,17,1)=":", 
  ISNUMBER(VALUE(MID(Q2,18,2))), MID(Q2,20,1)=".", 
  ISNUMBER(VALUE(MID(Q2,21,3))), RIGHT(Q2,1)="Z"), 
  "Valid", "Invalid"))
```

### メール形式の検証

```excel-formula title="Validate Email Format (Column E)"
=IF(TRIM(E2)="", "", IF(AND(
  ISNUMBER(FIND("@", E2)),
  ISNUMBER(FIND(".", E2)),
  FIND("@", E2) > 1,
  FIND(".", E2, FIND("@", E2)) > FIND("@", E2) + 1,
  LEN(E2) - LEN(SUBSTITUTE(E2, "@", "")) = 1,
  ISERROR(FIND(" ", E2))
), "Valid", "Invalid"))
```

## エラーが発生しますか？ {#common-errors}

* 正しいテンプレートをダウンロードし、列を追加していませんか？

* モバイルクレデンシャルを受け取るユーザーのすべての必須フィールドが記入されていますか？

* メールアドレスと電話番号はファイル内および既存ユーザー間で一意ですか？

* 日付とメール形式は上記の通り正確ですか？

:::info

* サービスチームはファイルの修正サービスを提供しません。 ファイルがアップロードされない場合は、上記の検証ツールを使用して問題を特定し、解決してください。

* この文書は、サービスチームによる手動のトラブルシューティングなしでアップロードを正常に完了できるように支援するために作成されています。

:::
