:::caution

<Cmd sid="ui_128" product='dev' /> 機能は、LCD画面によるユーザーインターフェースに対応するSuprema製品のうち、CE製品のみで使用できます。

:::

<Cmd sid="ui_128" product='dev' /> 機能は、デバイスの**完全な管理者権限を強化**し、**デバイスのセキュリティを向上**させ、**不正アクセスおよび設定変更を防止**するための機能です。<br/>この機能はEU地域向けの**CE製品**のみに適用され、**CE製品**では**必ずマスター管理者を登録**する必要があります。 登録済みのマスター管理者のみが管理者メニューに入って、デバイス設定を変更できます。

:::info

* この機能はEU地域向けの**CE製品**のセキュリティを強化するために導入された必須ポリシーであり、**グローバル製品（Non-CE）**では対応していません。

* マスター管理者は直接削除できず、**工場出荷時リセット時のみ削除されます**。

:::

## ファームウェアアップグレード時の動作ロジック {#firmware-upgrade-logic}

既存で使用中の製品のファームウェアをアップグレードする場合、**アップグレード前のバージョン**と**アップグレード対象バージョン**の組み合わせによって <Cmd sid="ui_128" product='dev' /> および <Cmd sid="ui_073" product='dev' /> の対応状況が異なる場合があります。

:::note

詳細なバージョン別の対応状況は、Suprema技術サポートチーム（CS@suprema.co.kr）にお問い合わせください。

:::

## 新しいデバイス <Badge only>CE製品</Badge>

CE製品の新しいデバイスでは、最初から <Cmd sid="ui_128" product='dev' /> の登録が必須であり、登録しない場合はデバイスの使用が制限されます。

:::info

* 初回起動時に <Cmd sid="ui_128" product='dev' /> の登録画面が表示され、登録を完了しないと次の画面に進めません。

* BioStar X（または BioStar 2）と接続してデバイス全体の管理者を割り当てても、<Cmd sid="ui_128" product='dev' /> をデバイスで直接登録しなければ、すべての機能を使用できません。

* 新しいデバイスはファームウェアをダウングレードできません。

:::

### マスター管理者を登録する

1. 新しいデバイスを初回起動すると、<Cmd sid="ui_128" product='dev' /> の登録画面が表示されます。

   <Image src='/img/how-to/setting-master-admin-1.png' caption width='300px' />

2. EU地域のセキュリティポリシーにより、マスター管理者は**最低2種類以上**のクレデンシャルを登録する必要があります。 必要な2種類のクレデンシャルを選択して登録してください。

   <Image src='/img/how-to/setting-master-admin-2.png' caption width='300px' />

   <Cmd sid="ui_128" product='dev' /> に登録できるクレデンシャルは次のとおりです。

   <Table>
   <Tbody>
   <Row>
   <Th>クレデンシャルの種類</Th>
   <Th>登録可能数</Th>
   <Th>詳細条件</Th>
   </Row>
   <Row>
   <Td>カード</Td>
   <Td>最大4個</Td>
   <Td>CSN、Wiegandタイプのみ対応<br/>同一タイプ内での重複登録は不可</Td>
   </Row>
   <Row>
   <Td>顔</Td>
   <Td>最大2個</Td>
   <Td>同じアルゴリズムを搭載したデバイスでのみ登録可能</Td>
   </Row>
   <Row>
   <Td>指紋</Td>
   <Td>最大2個</Td>
   <Td>-</Td>
   </Row>
   <Row>
   <Td>PIN</Td>
   <Td>1個</Td>
   <Td>最低8桁以上（CE規制適用）</Td>
   </Row>
   </Tbody>
   </Table>

   :::note

   **登録条件**

   * 少なくとも 2 種類以上のクレデンシャルを登録する必要があります。

   * デバイスがサポートするすべてのクレデンシャルを認証手段として使用できます。

   :::

3. 2種類以上のクレデンシャルを登録した後、<Cmd>完了</Cmd>ボタンを押すと <Cmd sid="ui_128" product='dev' /> の登録が完了します。

   <Image src='/img/how-to/setting-master-admin-3.png' caption width='300px' />

## ファームウェアアップグレードデバイス <Badge only>CE製品</Badge>

ファームウェアをアップグレードしたCE製品は <Cmd sid="ui_128" product='dev' /> 設定を提供しませんが、<Cmd sid="ui_073" product='dev' /> オプションで管理者の権限を強化し、デバイスのセキュリティを向上できます。

### 管理者2段階認証の設定 {#admin-two-step-authentication}

:::info

* デバイスに登録された管理者のクレデンシャルが2種類以上の場合のみ、<Cmd sid="ui_073" product='dev' /> を有効化できます。

  * PINは単独クレデンシャルとして使用できないため、ID + PINの組み合わせは対応していません。

* アップグレードされたデバイスではファームウェアをダウングレードできません。

:::

* **設定** → **デバイス** → <Cmd sid="ui_073" product='dev' /> メニューで管理者2段階認証を有効化できます。

* <Cmd sid="ui_073" product='dev' /> オプションは、既存で使用していたデバイスのファームウェアを最新バージョンにアップグレードした場合にのみ表示されます。

<Image src='/img/how-to/setting-two-step-authentication.png' caption width='300px' />

:::info

* すべての管理者に2種類以上のクレデンシャルが登録されていない場合、有効化に失敗し、エラーメッセージが表示されます。<br/>すべての管理者に2種類以上のクレデンシャルを登録してから、再試行してください。

  * PINは単独クレデンシャルとして使用できないため、ID + PINの組み合わせは対応していません。

<Image src='/img/how-to/error-two-step-authentication.png' caption width='300px' />

:::

:::warning

* <Cmd sid="ui_073" product='dev' /> を有効化した後、すべての管理者のクレデンシャルを2種類未満に削除すると、BioStar X（またはBioStar 2）に接続できない状況では管理者メニューにアクセスできなくなります。 そのため、管理者クレデンシャルを削除する際は十分に注意してください。

:::
