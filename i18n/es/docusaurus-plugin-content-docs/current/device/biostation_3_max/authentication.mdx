Describe los métodos de autenticación compatibles y los procedimientos de autenticación.

## Modo de autenticación

Configura el modo de autenticación. Combine <Cmd sid="ui_228" product="dev" />PIN, <Cmd sid="ui_040" product="dev" />Tarjeta / código QR<Cmd sid="ui_312" product="dev" />, y <Cmd sid="ui_020" product="dev" />ID<Cmd sid="ui_323" product="dev" /> para crear modos de autenticación y establecer un horario para cada modo.

:::info

La autenticación por huella está disponible solo en BS3M-ODB y BS3M-OAPB.

:::

### Registrar modo de autenticación {#registering-auth-mode}

1. Presiona <Image src="/img/common/ico-menu.svg" ico alone /> y autentícate con la credencial de nivel de administrador.

2. Presione <Cmd sid="ui_256" product="dev" /> → <Cmd sid="ui_261" product="dev" />.

3. Seleccione <Image src="/img/common/ico-add.svg" ico alone />.

4. Al presionar <Image src="/img/common/ico-add1.svg" ico alone />, se mostrará en la pantalla la credencial que se puede añadir. Selecciona las credenciales que deseas añadir. Presione <Image src="/img/common/ico-cancel.svg" ico alone /> mientras la credencial esté añadida para eliminarla.

   <Image src="/img/device/bs3m-add-auth-mode.png" />

   * <Image src="/img/common/ico-authmode-face-bs3m.png" ico alone />: Rostro, <Image src="/img/common/ico-authmode-fingerprint-bs3m.png" ico alone />: Huella, <Image src="/img/common/ico-authmode-card-qr-bs3m.png" ico alone />: Tarjeta / código QR, <Image src="/img/common/ico-authmode-id-bs3m.png" ico alone />: ID, <Image src="/img/common/ico-authmode-pin-bs3m.png" ico alone />: PIN

   * Combine los métodos de autenticación seleccionados para autenticar todos en orden o elija autenticar solo algunos de ellos. Por ejemplo, con la combinación <Image src="/img/common/ico-authmode-card-qr-bs3m.png" ico alone /> + <Image src="/img/common/ico-authmode-face-bs3m.png" ico alone /> + <Image src="/img/common/ico-authmode-pin-bs3m.png" ico alone />, autentíquese con tarjeta / código QR, rostro y PIN. Con la combinación <Image src="/img/common/ico-authmode-card-qr-bs3m.png" ico alone /> + <Image src="/img/common/ico-authmode-face-bs3m.png" ico alone /> / <Image src="/img/common/ico-authmode-pin-bs3m.png" ico alone />, primero autentíquese con tarjeta / código QR y luego presente el rostro o ingrese el PIN.

5. Después de configurar el modo de autenticación deseado, configure <Cmd sid="ui_203" product="dev" />.

   :::info

   * <Cmd sid="ui_203" product="dev" /> se puede configurar en BioStar X. Si no se establece un horario, solo se puede seleccionar <Cmd sid="ui_361" product="dev" />.

   * Para obtener más información sobre la configuración de <Cmd sid="ui_203" product="dev" />, consulte la [Guía del administrador de BioStar X](../../platform/biostar_x/settings-schedule).

   * Para usar la autenticación <Cmd sid="ui_042" product="dev" />, debe hacer clic en <Cmd sid="ui_256" product="dev" /> y <Cmd sid="ui_265" product="dev" /> debe tener activado <Cmd sid="ui_041" product="dev" />, además se requiere una licencia de dispositivo adicional. Para obtener más información, consulta [Licencia](settings#license).

   :::

6. Para guardar la configuración, presione <Cmd sid="ui_037" product="dev" />.

### Modificar modo de autenticación

1. Presiona <Image src="/img/common/ico-menu.svg" ico alone /> y autentícate con la credencial de nivel de administrador.

2. Presione <Cmd sid="ui_256" product="dev" /> → <Cmd sid="ui_261" product="dev" />.

3. Selecciona el modo de autenticación que deseas modificar.

4. Modifique el modo de autenticación consultando [Registrar modo de autenticación](#registering-auth-mode) y presione <Cmd sid="ui_037" product="dev" />.

### Eliminar modo de autenticación

1. Presiona <Image src="/img/common/ico-menu.svg" ico alone /> y autentícate con la credencial de nivel de administrador.

2. Presione <Cmd sid="ui_256" product="dev" /> → <Cmd sid="ui_261" product="dev" />.

3. Presione <Image src="/img/common/ico-delete.svg" ico alone /> y seleccione el modo de autenticación que desea eliminar. Es posible seleccionar varias opciones.

4. Para eliminar el elemento seleccionado, presione <Cmd sid="ui_037" product="dev" />.

## Operación

### Coincidencia en el servidor

Configura la coincidencia con el servidor.

1. Presiona <Image src="/img/common/ico-menu.svg" ico alone /> y autentícate con la credencial de nivel de administrador.

2. Después de presionar <Cmd sid="ui_256" product="dev" />, active <Cmd sid="ui_265" product="dev" /> en <Cmd sid="ui_187" product="dev" />.

   <Image src="/img/device/bs3m-operation.png" />

   * <Cmd sid="ui_187" product="dev" />: Cuando se configura la coincidencia con el servidor, la autenticación del usuario la realiza BioStar X sin que el dispositivo realice la autenticación. Se puede usar cuando el dispositivo tiene demasiada información de usuarios o cuando el dispositivo está instalado en exteriores.

   :::info

   * <Cmd sid="ui_185" product="dev" /> solo puede usar tarjetas e IDs.

   * Si usa <Cmd sid="ui_185" product="dev" />, debe habilitar la coincidencia con el servidor tanto en el dispositivo como en BioStar X. Para obtener más información sobre la configuración de coincidencia con el servidor en BioStar X, consulte la [Guía del administrador de BioStar X](../../platform/biostar_x/settings-server-server#servermatching).

   :::

### Autenticación QR

Active la autenticación mediante código QR.

1. Presiona <Image src="/img/common/ico-menu.svg" ico alone /> y autentícate con la credencial de nivel de administrador.

2. Después de presionar <Cmd sid="ui_256" product="dev" />, active <Cmd sid="ui_265" product="dev" /> para <Cmd sid="ui_041" product="dev" />.

   <Image src="/img/device/bs3m-operation.png" />

   * <Cmd sid="ui_041" product="dev" />: Configure si desea usar la autenticación por código QR mediante la cámara del dispositivo.

   * <Cmd sid="ui_329" product="dev" />: Puede configurar la autenticación con un código QR que contenga los mismos datos que la tarjeta CSN o Wiegand emitida al usuario. Esto solo se puede configurar cuando <Cmd sid="ui_041" product="dev" /> está activado.

   :::info

   Se requiere una licencia de dispositivo adicional para usar <Cmd sid="ui_041" product="dev" />. Para obtener más información, consulta [Licencia](settings#license).

   :::

## PIN

Configura si se utiliza el teclado aleatorio al ingresar contraseñas.

1. Presiona <Image src="/img/common/ico-menu.svg" ico alone /> y autentícate con la credencial de nivel de administrador.

2. Después de presionar <Cmd sid="ui_256" product="dev" />, configure si desea usar <Cmd sid="ui_040" product="dev" /> en <Cmd sid="ui_204" product="dev" />.

   <Image src="/img/device/bs3m-pin.png" />

   * <Cmd sid="ui_204" product="dev" />: Las posiciones de las teclas numéricas se pueden mezclar aleatoriamente al ingresar una contraseña para evitar filtraciones.

## Opciones de visualización del resultado de autenticación

Configura la opción de visualización del resultado de autenticación.

1. Presiona <Image src="/img/common/ico-menu.svg" ico alone /> y autentícate con la credencial de nivel de administrador.

2. Después de presionar <Cmd sid="ui_256" product="dev" />, cambie los elementos deseados en <Cmd sid="ui_259" product="dev" />.

   <Image src="/img/device/bs3m-auth-result-display-option.png" />

   * <Cmd sid="ui_169" product="dev" />: Seleccione cómo aparece el ID de usuario en la pantalla de resultados de autenticación del dispositivo.

     * <Cmd sid="ui_307" product="dev" />: Mostrar el ID de usuario completo.

     * <Cmd sid="ui_316" product="dev" />: Mostrar solo el primer carácter del ID de usuario.

     * <Cmd sid="ui_357" product="dev" />: No mostrar el ID de usuario.

   * <Cmd sid="ui_177" product="dev" />: Seleccione cómo aparece el nombre de usuario en la pantalla de resultados de autenticación del dispositivo.

     * <Cmd sid="ui_307" product="dev" />: Mostrar el nombre de usuario completo.

     * <Cmd sid="ui_316" product="dev" />: Mostrar solo el primer carácter del nombre de usuario.

     * <Cmd sid="ui_357" product="dev" />: No mostrar el nombre de usuario.

## Rostro

Cambie la configuración de la autenticación facial.

1. Presiona <Image src="/img/common/ico-menu.svg" ico alone /> y autentícate con la credencial de nivel de administrador.

2. Seleccione <Cmd sid="ui_256" product="dev" /> → <Cmd sid="ui_228" product="dev" />.

3. Cambie el elemento deseado.

   <Image src="/img/device/bs3m-face.png" />

   * <Cmd sid="ui_140" product="dev" />: Configure la sensibilidad para detectar movimiento alrededor del dispositivo. Cuando <Cmd sid="ui_140" product="dev" /> se establece en <Cmd sid="ui_091" product="dev" />, toque la pantalla para autenticar su rostro.

   * <Cmd sid="ui_115" product="dev" />: Si el rostro no se registra dentro del tiempo establecido, el registro se cancelará.

   * <Cmd sid="ui_260" product="dev" />: Si la autenticación no se completa dentro del tiempo establecido, esta fallará.

   * <Cmd sid="ui_117" product="dev" />: Puede verificar rostros duplicados durante el registro de rostro.

   * <Cmd sid="ui_229" product="dev" />: Configure la posición y el tamaño del área donde el dispositivo detecta rostros durante la autenticación.

     :::info

     Para un rendimiento óptimo, se recomienda usar la configuración predeterminada de las subopciones de <Cmd sid="ui_229" product="dev" />. Los valores predeterminados de cada opción son los siguientes:

     * Ángulo máximo de orientación de la cabeza: 15

     * Distancia de detección: mín. 30 cm, máx. 100 cm

     * Búsqueda amplia: desactivada

     :::

   * <Cmd sid="ui_060" product="dev" />: Al activar <Cmd sid="ui_060" product="dev" />, puede enrolar la <Cmd sid="ui_340" product="dev" /> emitida por BioStar X. <Cmd sid="ui_340" product="dev" /> permite la autenticación biométrica sin almacenar datos del usuario en el servidor de BioStar X, en Airfob Portal ni en el dispositivo, ya que guarda la plantilla biométrica del usuario en una tarjeta de acceso móvil.

     :::info

     Para obtener más información sobre <Cmd sid="ui_340" product="dev" />, consulte la [Guía del administrador de BioStar X](../../platform/biostar_x/enroll-credential-mobile#template-on-mobile).

     :::

   * <Cmd sid="ui_157" product="dev" />: Puede configurar para almacenar tanto las imágenes reales como las plantillas en Visual Face, o para guardar solo las plantillas sin almacenar las imágenes reales. Si se desactiva, se eliminarán todas las imágenes almacenadas de rostros visuales y los rostros visuales enrolados posteriormente almacenarán solo las plantillas, sin imágenes.

     :::info

     Para administrar esta configuración, utilice Biostar 2 v2.9.7 o superior.

     :::

   * <Cmd sid="ui_149" product="dev" />: Puede configurar el nivel de seguridad para la autenticación 1:N.

   * <Cmd sid="ui_278" product="dev" />: Puede configurar el modo de funcionamiento del dispositivo para la autenticación facial.

     * <Cmd sid="ui_359" product="dev" />: La coincidencia visual e infrarroja funciona conjuntamente para aumentar la precisión de la autenticación facial.

     * <Cmd sid="ui_159" product="dev" />: Los usuarios pueden autenticarse rápidamente mientras caminan dentro del rango de autenticación del dispositivo.

   * <Cmd sid="ui_247" product="dev" />: Puede evitar la autenticación con rostros suplantados, como fotografías. Se activa cuando <Cmd sid="ui_278" product="dev" /> se establece en <Cmd sid="ui_359" product="dev" />.

     :::info

     * El rendimiento de la función anti-spoofing facial se ha mejorado para aprobar la prueba iBeta Level 1 ISO 30107-3 de detección de ataques de presentación.

     * Con luz solar intensa, se recomienda usar la función <Cmd sid="ui_247" product="dev" /> como <Cmd sid="ui_152" product="dev" />.

     :::

   * <Cmd sid="ui_070" product="dev" />: Puede cambiar la configuración avanzada de la autenticación facial, como la detección de mascarilla.

     * <Cmd sid="ui_123" product="dev" />: Puede configurar si desea usar la detección de mascarilla.

     * <Cmd sid="ui_067" product="dev" />: Puede configurar el método para verificar si se lleva mascarilla. Se activa cuando <Cmd sid="ui_123" product="dev" /> se establece en <Cmd sid="ui_161" product="dev" />.

       :::info

       * <Cmd sid="ui_127" product="dev" />: A los usuarios que no lleven mascarilla se les niega la autenticación y se registra un evento de incumplimiento de mascarilla.

       * <Cmd sid="ui_126" product="dev" />: Los usuarios que no lleven mascarilla aún pueden autenticarse, pero se registra un evento de incumplimiento de mascarilla.

       * <Cmd sid="ui_125" product="dev" />: Solo los usuarios que no lleven mascarilla pueden autenticarse y, cuando se utiliza este modo, el modo de verificación de mascarilla se fija en <Cmd sid="ui_369" product="dev" />.

       :::

     * <Cmd sid="ui_124" product="dev" />: Configure la sensibilidad para detectar el uso de mascarilla. Se activa cuando <Cmd sid="ui_123" product="dev" /> se establece en <Cmd sid="ui_161" product="dev" />.

     * <Cmd sid="ui_368" product="dev" />: Configure el modo de confirmación según el propósito del dispositivo.

       :::info

       * <Cmd sid="ui_267" product="dev" />: Después de la autenticación, verifica si el usuario lleva mascarilla.

       * <Cmd sid="ui_369" product="dev" />: Autentica después de comprobar si el usuario lleva mascarilla.

       * <Cmd sid="ui_370" product="dev" />: El dispositivo solo se puede usar para verificar si se lleva mascarilla. Cuando este modo está activado, cualquier usuario que lleve mascarilla puede acceder, sin importar la autenticación.

       :::

## Huella <Badge only>BS3M-ODB, BS3M-OAPB</Badge> {#fingerprint}

Cambia los ajustes de autenticación por huella digital.

1. Presiona <Image src="/img/common/ico-menu.svg" ico alone /> y autentícate con la credencial de nivel de administrador.

2. Seleccione <Cmd sid="ui_256" product="dev" /> → <Cmd sid="ui_312" product="dev" />.

3. Cambie el elemento deseado.

   <Image src="/img/device/auth-fingerprint.png" width={300} />

   * <Cmd sid="ui_149" product="dev" />: Puede configurar el nivel de seguridad para la autenticación 1:N.

   * <Cmd sid="ui_130" product="dev" />: Configure el tiempo de espera para la comparación de huellas. Si la autenticación no se completa dentro del tiempo establecido, falla.

   * <Cmd sid="ui_254" product="dev" />: Ver la imagen sin procesar al escanear una huella.

   * <Cmd sid="ui_197" product="dev" />: Configure la sensibilidad del sensor de huellas. Establece la sensibilidad más alta para obtener información de huella dactilar precisa.

   * <Cmd sid="ui_003" product="dev" />: Configure el rendimiento de la autenticación por huella. Cuando selecciona <Cmd sid="ui_275" product="dev" />, la velocidad de autenticación se ajusta según el número total de plantillas de huellas almacenadas en el dispositivo.

   * <Cmd sid="ui_342" product="dev" />: Configure el formato de la plantilla de huella. El valor predeterminado es <Cmd sid="ui_057" product="dev" />. Cambiar el formato de la plantilla bloquea el uso de todas las huellas dactilares almacenadas previamente, así que procede con precaución.

   * <Cmd sid="ui_198" product="dev" />: Defínalo en <Cmd sid="ui_277" product="dev" /> para encender el sensor de huella cuando detecte el dedo del usuario. Defínalo en <Cmd sid="ui_362" product="dev" /> para mantener el sensor siempre encendido, incluso cuando no se estén escaneando huellas.

   * <Cmd sid="ui_249" product="dev" />: Configure el nivel de detección de huellas falsas. Niveles más altos de detección de huellas falsas pueden aumentar las tasas de rechazo de las huellas reales.

   * <Cmd sid="ui_116" product="dev" />: Inspeccione la calidad de la huella escaneada para guardar datos de alta calidad. Cuando está activado, el dispositivo notifica al usuario si la calidad de la huella es baja y le ayuda a escanear correctamente.

   * <Cmd sid="ui_118" product="dev" />: Verifique huellas duplicadas durante el enrolamiento.

## T&A (Tiempo y Asistencia)

Configura cómo registrar el modo T&A.

1. Presiona <Image src="/img/common/ico-menu.svg" ico alone /> y autentícate con la credencial de nivel de administrador.

2. Seleccione <Cmd sid="ui_256" product="dev" /> → <Cmd sid="ui_079" product="dev" />.

3. Cambie el elemento deseado.

   <Image src="/img/device/bs3m-ta.png" />

   * <Cmd sid="ui_083_list" product="dev" />: Configure cómo usar el modo de asistencia.

   * <Cmd sid="ui_086" product="dev" />: Agregue eventos de tiempo y asistencia. Para registrar eventos de asistencia, seleccione <Cmd sid="ui_086" product="dev" /> → <Image src="/img/common/ico-add.svg" ico alone /> y luego configure el icono que se mostrará en la pantalla, <Cmd sid="ui_085" product="dev" /> y <Cmd sid="ui_203" product="dev" />.

   * <Cmd sid="ui_082" product="dev" />: Exija a los usuarios seleccionar un evento de tiempo y asistencia durante la autenticación. Disponible cuando <Cmd sid="ui_083_list" product="dev" /> está configurado en <Cmd sid="ui_199" product="dev" />.

   * <Cmd sid="ui_081" product="dev" />: Permita a los usuarios usar solo los eventos de tiempo y asistencia seleccionados por el administrador. Disponible cuando <Cmd sid="ui_083_list" product="dev" /> está configurado en <Cmd sid="ui_071" product="dev" />.

   * <Cmd sid="ui_279" product="dev" />: Seleccione si desea usar códigos de trabajo.
