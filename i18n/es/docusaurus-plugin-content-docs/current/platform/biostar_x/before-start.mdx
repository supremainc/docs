:::tip

Esta guía proporciona información importante que debe conocer antes de iniciar **BioStar X**. **BioStar X** es compatible con sistemas operativos de 64 bits. Verifique los requisitos del sistema de la PC donde desea instalar **BioStar X** y luego instálelo.

:::

## Notas previas a la instalación {#before}

Antes de instalar **BioStar X**, verifique lo siguiente y continúe con la instalación.

* **BioStar X** solo se puede instalar en un sistema operativo de 64 bits.

* Si está utilizando MS SQL 2012 Express, consulte el siguiente enlace para instalar el [Service Pack 3](https://www.microsoft.com/download/details.aspx?id=49996).

* Si está utilizando MS SQL 2014 Express, consulte el siguiente enlace para instalar el [Service Pack 2](https://www.microsoft.com/ko-kr/download/details.aspx?id=53168).

* Si está utilizando MS SQL Server, configure la opción `Collation` del servidor y de cada tabla en CI (sin distinción de mayúsculas).

* Al hacer una copia de seguridad de una base de datos de una versión anterior de **BioStar X**, desactive todos los servicios antes de continuar. Además, si no respalda y restaura juntas las bases de datos AC y TA, no podrá utilizar la base de datos TA.

* Si desea respaldar la base de datos de **BioStar X**, asegúrese de respaldar también el *enckey* en la carpeta *\Program Files\BioStar X\util* y los archivos *system.conf* y *setting.conf* en la carpeta *\Program Files\BioStar X*. De lo contrario, la base de datos no estará disponible.

* Consulte [lo siguiente](service-settings) para conocer el puerto predeterminado que utiliza **BioStar X**. Si otro programa ocupa el mismo puerto, **BioStar X** podría no funcionar correctamente.

## Verificar la base de datos {#checkdb}

Si está utilizando una base de datos configurada por el usuario, verifique los siguientes elementos antes de instalar **BioStar X**.

### MariaDB {#mariadb}

1. Cambie las siguientes opciones en la sección `[mysql]` del archivo *my.cnf*.

   ```ini title='my.cnf'
   character-set-server=utf8
   collation-server=utf8_unicode_ci
   max_connections = 600
   ```

2. Agregue las siguientes opciones en la sección `[mysql]` del archivo *my.cnf*.

   ```ini title='my.cnf'
   log_bin_trust_function_creators = 1
   group_concat_max_len = 102400
   ```

3. Conéctese a MariaDB con privilegios de root y ejecute el siguiente comando.

   ```sql title='SQL'
   GRANT SUPER ON . TO user_id@'localhost' IDENTIFIED BY "password";
   GRANT SUPER ON . TO user_id@'%' IDENTIFIED BY "password";
   ```

### MS SQL Server {#ms-sql-server}

:::info

* Configure las opciones de `Collation` para todas las bases de datos y tablas como insensibles a mayúsculas (CI).

* Los nombres de base de datos solo pueden contener números, letras en inglés (distinguiendo mayúsculas y minúsculas) y los símbolos especiales `-` `_`.

:::

#### Configurar el puerto {#port-setting}

* Ejecute **SQL Server Configuration Manager** y establezca **TCP/IP Protocol** para **Protocols for SQLEXPRESS** en el número de puerto deseado.

* Reinicie los **SQL Server Services** para aplicar los ajustes.

#### Crear el usuario y la base de datos {#create-a-user-and-database}

1. Inicie sesión con la cuenta **sa** usando **SQL Server Authentication** en **SQL Server Management Studio**.

2. Haga clic derecho en **Security** y haga clic en **New Login**.

3. Ingrese el nombre deseado en el campo **Login Name** y seleccione **SQL Server Authentication**.

4. Ingrese la contraseña deseada en los campos **Password** y **Confirm password**, luego desmarque **Enforce password policy**.

5. Haga clic en **OK** para guardar los ajustes.

6. Haga clic derecho en **Database** y seleccione **New Database**.

7. Ingrese el nombre deseado en el campo Database Name.

8. Ingrese el nombre de inicio de sesión configurado en el paso **3** en el campo **Owner**.

:::tip

Establezca los siguientes valores recomendados en la sección **Database Files**. Configure **Autogrowth** usando tamaños fijos (MB/GB) en lugar de porcentajes y evite incrementos muy pequeños que puedan causar eventos de crecimiento frecuentes.

<Image src="/img/biostarx-before-start-new-database.png" caption />

#### Pequeña (menos de 10 GB)

* Datos: **Initial Size** 1 - 5 GB, **Autogrowth** 100 - 512 MB

* Registro: **Initial Size** 512 MB - 2 GB, **Autogrowth** 100 - 512 MB

* TempDB: **Initial Size** 512 MB - 1 GB

> p. ej. Datos 2 GB (**Autogrowth** 256 MB) / Registro 1 GB (**Autogrowth** 128 MB) / TempDB 512 MB

#### Mediano (10 - 500 GB)

* Datos: **Initial Size** 10 - 50 GB, **Autogrowth** 512 MB - 1 GB

* Registro: **Initial Size** 5 - 10 GB, **Autogrowth** 512 MB - 1 GB

* TempDB: **Initial Size** 2 - 8 GB

> p. ej. Datos 20 GB (**Autogrowth** 1 GB) / Registro 5 GB (**Autogrowth** 512 MB) / TempDB 4 GB

#### Empresa (500 GB o más)

* Datos: **Initial Size** 100 GB o más, **Autogrowth** 1 - 4 GB

* Registro: **Initial Size** 20 ~ 100 GB, **Autogrowth** 1 ~ 4 GB

* TempDB: **Initial Size** 8 - 32 GB

> p. ej. Datos 100 GB (**Autogrowth** 2 GB) / Registro 50 GB (**Autogrowth** 2 GB) / TempDB 16 GB

:::

:::caution

* No configure **Autogrowth** con incrementos muy pequeños ni deje **Maxsize** en **Unlimited**. En su lugar, establezca límites apropiados según el crecimiento esperado y supervise el uso.

* En entornos con muchas transacciones, respalde los registros periódicamente para evitar que el tamaño de los archivos de registro aumente.

:::

#### Configurar la base de datos de autenticación de Windows {#windows-authentication-database-settings}

<Steps>
<Step>

##### Configuración previa {#preliminary-setting}

Si utiliza **Microsoft Windows Active Directory**, complete las configuraciones previas que se indican a continuación antes de configurar la base de datos de autenticación de Windows.

1. Inicie sesión con una cuenta de administrador en **SQL Server Management Studio**.

2. Haga clic derecho en **Security** y haga clic en **New Login**.

3. Seleccione **Windows Authentication** y luego haga clic en **Search**.

4. En la ventana **Select a user or group**, haga clic en Ubicación, seleccione la ruta de Active Directory y haga clic en **OK**.

5. Ingrese el nombre de usuario en el campo **Enter object name to select**, luego haga clic en **Check Names** > **OK**.

6. Haga clic en **Server Roles** en **Select a page**.

7. Seleccione **sysadmin** y luego haga clic en **OK**.

8. Haz clic en **User Mapping** en la página **Select**.

9. Selecciona **ac**, **master**, **ta**, **ve** y establece **Default Schema** en **dbo**.

10. Haga clic en **OK** para guardar los ajustes.

</Step>

<Step>

##### Configurar la autenticación de Windows de MS SQL con una cuenta de Active Directory {#settingsforusingmssql}

1. Ejecuta *services.msc*.

2. En la ventana **Properties** > **Logon** de la base de datos de SQL Server, selecciona **Specify account** e inicia sesión como usuario de dominio.

3. Agrega el puerto 1433 como excepción en el Firewall de Windows.

4. Crea las bases de datos vacías **ac**, **ta**, **ve** en SQL Server con autenticación **sysadmin**.

5. Configura al usuario de dominio para que use la autenticación de Windows en SQL Server y asigna todos los permisos excepto **sysadmin** a las bases de datos **ac**, **ta**, **ve**.

6. Conecta todos los servicios, excepto el equipo local, en *services.msc*.

7. Establece al usuario de dominio como administrador en el equipo de servicio local.

8. Conéctate a MS SQL ODBC.

</Step>

<Step>

##### Configuración de la base de datos {#database-setting}

1. Después de ejecutar **SQL Server Configuration Manager**, haz clic en **Client Protocol** en **SQL Native Client**.

2. Selecciona **TCP/IP** y verifica el puerto predeterminado.

3. Haz clic en **Protocols for SQL EXPRESS** en **SQL Server Network Configuration**.

4. Comprueba que el puerto ODBC en **TCP/IP** esté configurado con el mismo valor que el puerto predeterminado.

5. Inicie sesión con una cuenta de administrador en **SQL Server Management Studio**.

6. Haz clic en **Security** > **Log In**, luego haz doble clic en **NT AUTHORITY\SYSTEM**.

7. Haga clic en **Server Roles** en **Select a page**.

8. Selecciona **public** y **sysadmin**, luego haz clic en **OK**.

9. Haz clic en **User Mapping** en la página **Select**.

10. Selecciona las bases de datos **ac**, **ta**, **ve** y haz clic en **OK** para guardar.

</Step>
</Steps>
