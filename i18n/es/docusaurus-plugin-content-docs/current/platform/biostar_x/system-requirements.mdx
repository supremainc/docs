**BioStar X** ofrece una solución confiable y escalable de gestión de seguridad integrada para entornos empresariales grandes. El sistema puede gestionar eficientemente más de mil dispositivos mediante una arquitectura distribuida compuesta por un servidor principal y un servidor de comunicación.

El servidor principal gestiona las funciones centrales del sistema, mientras que el servidor de comunicación se encarga de la comunicación con los dispositivos, distribuyendo la carga y optimizando el rendimiento. Además, los administradores pueden acceder al sistema a través de un navegador web en los equipos cliente y realizar todas las tareas de gestión. Consulta los siguientes requisitos del sistema para configurar el entorno óptimo según el tamaño de tu organización.

:::info

Para obtener más información sobre la configuración y optimización de licencias, comunícate con el [Servicio de soporte técnico de Suprema](https://support.supremainc.com).

:::

## Servidor principal {#main-server}

El servidor principal es el servidor donde se instala el sistema de gestión central de **BioStar X**. El usuario accede al sistema mediante un navegador web para gestionarlo y realizar todas las tareas, como configurar la información de usuarios o los permisos de acceso. También se encarga de supervisar el estado general del sistema y de gestionar los registros de eventos o las alarmas.

:::info

* **BioStar X** solo se puede instalar en sistemas operativos de 64 bits.

* La seguridad de comunicación de MS SQL Server admite TLS 1.2.

* Si MS SQL Server y **BioStar X** se instalan en diferentes equipos, instala el [Controlador Microsoft OLE DB para SQL Server](https://learn.microsoft.com/sql/connect/oledb/download-oledb-driver-for-sql-server) en el equipo donde esté instalado **BioStar X**.

* Para entornos empresariales, se recomienda instalar una base de datos MS SQL Server.

* Si utilizas una base de datos MS SQL Server y **BioStar X T&A**, deberás instalar el [Controlador Microsoft ODBC 17 para SQL Server](https://learn.microsoft.com/en-us/sql/connect/odbc/download-odbc-driver-for-sql-server?view=sql-server-ver16#version-17) correspondiente a tu entorno.

* El entorno virtual de Windows proporcionado por Boot Camp en macOS no es compatible.

:::

### Requisitos de licencias de SQL Server {#sqlserver-requriements}

Si utilizas SQL Server como base de datos, se requiere una licencia adecuada de SQL Server según la cantidad de dispositivos conectados para el funcionamiento estable de **BioStar X**. Consulta las configuraciones de licencias recomendadas para cada entorno y selecciona la licencia adecuada al tamaño y las necesidades de tu organización.

:::info

Para obtener más información sobre las licencias de SQL Server, consulta [Microsoft SQL Server Licensing](https://www.microsoft.com/sql-server/sql-server-2022-pricing).

:::

## Servidor de comunicación {#communication-server}

El servidor de comunicación es un servidor dedicado a la comunicación con dispositivos de control de acceso. Al instalarlo por separado del servidor principal, ayuda a gestionar muchos dispositivos de forma confiable. Un solo servidor de comunicación puede conectar hasta 1,000 dispositivos, y se pueden agregar varios servidores de comunicación según sea necesario.

## Cliente {#client}

El cliente es el equipo desde el cual los administradores acceden a **BioStar X** a través de un navegador web para operar el sistema.
