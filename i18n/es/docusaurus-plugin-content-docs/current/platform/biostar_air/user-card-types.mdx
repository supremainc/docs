A partir de la versión 2.6 de BioStar Air se introdujeron dos tipos de usuario para mejorar la gestión.

* <Cmd sid="user_regular_card_type" product="air" />: Un usuario que es miembro regular de una organización o instalación y que requiere acceso a largo plazo. Se pueden emitir todos los tipos de credenciales.

* <Cmd sid="user_temporary_card_type" product="air" />: Un visitante o miembro no regular que necesita acceso limitado y de corta duración. Esto puede incluir huéspedes, visitantes externos, contratistas temporales, mensajeros de entrega, etc. Solo se pueden emitir las credenciales <Cmd sid="user_form_input20" product="air" />, incluyendo el código QR y LinkPass.

:::info

Para obtener más información sobre la emisión de credenciales al agregar o editar usuarios, consulte <DocLink docId='platform/biostar_air/adding-individual-users#issuingCredentials' />.

:::

## Tipos de tarjetas para usuarios regulares {#regular-user-card-typs}

Las credenciales que puede emitir a los usuarios regulares son las siguientes.

:::info

Para obtener más información sobre cómo usar credenciales móviles con la aplicación Suprema Pass, consulte <DocLink docId='platform/biostar_air/opening-doors-with-mobile-credentials' />.

:::

### <Cmd sid='card_type_mobile' product='air' className='normal' /> {#mobileCard}

* Se asigna automáticamente un **ID de tarjeta** único, aunque también puede establecerse manualmente.

* Al usar un **Airfob Tag** o **Airfob Patch**, integre el sistema de control de acceso existente utilizando el número de la tarjeta móvil como número de serie de la tarjeta.

* Para compatibilidad con un sistema anterior, el ID de la tarjeta móvil puede coincidir con el número de tarjeta RF del sitio anterior.

### <Cmd sid='card_type_rf' product='air' className='normal' /> {#rfCard}

* Agregue hasta ocho tarjetas RFID por usuario.

* Configure <Cmd sid="rf_card_type" product="air" /> (CSN o Wiegand), <Cmd sid="wiegand_card_id" product="air" />, y <Cmd sid="rf_card_name" product="air" /> para simplificar la gestión cuando se asignan varias tarjetas.

* Seleccione un lector en <Cmd sid="newest_rf_card_text_device" product="air" /> para escanear una tarjeta RFID existente con un dispositivo Suprema conectado.

* Si una tarjeta RF existente tiene un **CSN** (número de serie de la tarjeta), ingrese el CSN manualmente.

### <Cmd sid='card_type_option3' product='air' className='normal' /> {#qrCode}

* Los usuarios regulares pueden usar el **código QR** de una tarjeta móvil.

* Acceda al código QR tocando el botón en la interfaz de la tarjeta móvil.

### <Cmd sid='card_type_option4' product='air' className='normal' /> {#linkPass}

* Envíe un enlace web a la dirección de correo electrónico o al número de teléfono del usuario.

* El usuario abre el enlace recibido en un navegador web y hace clic en <Cmd sid="web_card_preview_button1" product="air" /> para obtener acceso.

:::info

Para obtener más información sobre las credenciales LinkPass, consulte <DocLink docId='platform/biostar_air/linkpass-credentials' />.

:::

### Autenticación facial {#face}

* Agregue hasta dos imágenes de perfil facial por usuario.

* Regístrate mediante un lector conectado o cargando una imagen.

* Envía al usuario un enlace por correo electrónico y, a continuación, captura una foto con la cámara del dispositivo móvil para registrarlo.

### PIN {#pin}

Establece un PIN de 4 a 16 dígitos para usar como parte de la autenticación de dos factores (2FA).

## Tipos de tarjetas para usuarios temporales {#temporary-user-card-typs}

Los usuarios temporales no reciben tarjetas móviles ni tarjetas RFID. En su lugar, emite **credenciales web** que no requieren la descarga de una aplicación. Las opciones disponibles son las siguientes:

### <Cmd sid='card_type_option3' product='air' className='normal' /> {#TempQrCode}

* Envía por correo electrónico o por mensaje de texto.

* Accesible a través de un navegador web.

* Usa un **código QR dinámico**, que se actualiza cada cinco minutos, para mejorar la seguridad.

### <Cmd sid='card_type_option4' product='air' className='normal' /> {#TempLinkPass}

Envía al usuario un enlace web por correo electrónico o por mensaje de texto. El usuario obtiene acceso al hacer clic en <Cmd sid="web_card_preview_button1" product="air" /> en un navegador web.

:::info

Para obtener más información sobre las credenciales LinkPass, consulte <DocLink docId='platform/biostar_air/linkpass-credentials' />.

:::

## Administrar niveles de acceso {#managing-access-levels}

Configura los **niveles de acceso** para controlar dónde se pueden usar las credenciales, independientemente de si el usuario es regular o temporal.

:::caution

Si el usuario es temporal, asigna con cuidado niveles de acceso restrictivos y evita emitir credenciales con acceso ilimitado.

:::
