Los ajustes flexibles de gestión de dispositivos de BioStar Air permiten configurar cada sitio exactamente según sus requisitos. En este documento, la gestión de dispositivos se refiere al ciclo de vida completo de un dispositivo, que incluye la inscripción, los ajustes, la gestión de firmware, la supervisión y la operación continua.

BioStar Air admite dos arquitecturas de dispositivos diferentes. Ambas pueden utilizarse en el mismo sitio, lo que permite a las organizaciones elegir el mejor enfoque para cada puerta o área.

## Arquitectura distribuida

En una arquitectura distribuida, cada dispositivo edge contiene su propio controlador y se conecta directamente a la nube mediante Ethernet. Los dispositivos siguen operando de forma independiente durante interrupciones de red y se sincronizan automáticamente cuando se restablece la conectividad.

Los dispositivos edge también pueden actuar como controladores para lectores secundarios conectados vía RS-485.

Esta arquitectura se utiliza comúnmente para:

* Nuevas instalaciones

* Dispositivos biométricos

* Despliegues en los que es importante minimizar la infraestructura.

## Arquitectura centralizada

En una arquitectura centralizada, los lectores se conectan a un panel de control mediante RS-485 o Wiegand. El panel de control se conecta a la nube vía Ethernet y actúa como controlador para los dispositivos aguas abajo.

El panel de control gestiona la redundancia y las decisiones de acceso para los lectores conectados durante interrupciones de red.

Esta arquitectura se utiliza comúnmente para:

* Proyectos de modernización con cableado de bajo voltaje existente

* Sitios que reemplazan sistemas heredados de control de acceso

* Despliegues con salas de seguridad o paneles centralizados
