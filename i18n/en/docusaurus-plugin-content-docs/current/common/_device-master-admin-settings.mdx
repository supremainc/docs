## Set Up Master Administrator <Badge only>CE model only</Badge>

The <Cmd sid='ui_128' product='dev' /> feature enhances the overall admin privileges of the device to improve device security and prevent unauthorized access and configuration changes. This feature applies only to **CE products** for the EU region, and **Master Administrator enrollment is required** on **CE products**. Only the enrolled master administrator can access the administrator menu and change device settings.

:::info

* This feature is a required policy introduced to enhance security for **CE products** in the EU region, and it is not supported on **Global (Non-CE) products**.

* The master admin cannot delete directly and will be deleted only during <Cmd sid='ui_072' product='dev' />.

:::

### Register master admin on a new device

</Xclude>

On a new device for CE products, you must enroll <Cmd sid="ui_128" product='dev' /> from the start. If you do not enroll it, device use is restricted.

:::info

* At first boot, the <Cmd sid="ui_128" product='dev' /> enrollment screen appears. If you do not complete enrollment, you cannot move to the next screen.

* Even if you connect to BioStar X (or BioStar 2) and assign a device-wide administrator, you must enroll <Cmd sid="ui_128" product='dev' /> directly on the device to use all features.

* New devices cannot perform firmware downgrades.

:::
