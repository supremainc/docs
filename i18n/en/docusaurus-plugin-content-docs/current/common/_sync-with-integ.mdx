This feature allows you to synchronize user information changed in <strong>{props.name}</strong>.

* <Cmd sid='settings.activeDirectory.syncMode.label' product='2' />: Select the desired synchronization mode.

  * <Cmd sid='settings.activeDirectory.syncMode.addEdit' product='2' />: When synchronizing users from the linked directory service, add users who don't exist in **BioStar X** and update users whose information has changed, without deleting any.

    <Image src='/img/biostarx-setting-directory-sync-add-update-mode.png' />

  * <Cmd sid='settings.activeDirectory.syncMode.addEditDelete' product='2' />: Synchronize **BioStar X** user information to match the linked directory service. Unlike <Cmd sid='settings.activeDirectory.syncMode.addEdit' product='2' /> mode, users deleted in the directory service are also removed.

    <Image src='/img/biostarx-setting-directory-sync-add-update-delete-mode.png' />

* <Cmd sid='settings.activeDirectory.autoSync.label' product='2' />: Set the automatic synchronization interval. Synchronization runs automatically at the interval set in <Cmd sid='settings.activeDirectory.syncInterval.label' product='2' />. Enter the interval in minutes. The minimum value is **30** minutes, and the maximum value is **10,080** minutes (7 days).

  <Image src='/img/biostarx-setting-directory-sync-auto-sync-interval.png' />

* <Cmd sid='settings.activeDirectory.lastSynchronized' product='2' />: You can check the date and time of the most recent synchronization.

:::caution

* When using <Cmd sid='settings.activeDirectory.syncMode.addEdit' product='2' /> mode, if a group with the same name exists, it is retained rather than deleted.

* When using <Cmd sid='settings.activeDirectory.syncMode.addEditDelete' product='2' /> mode

  * **If a group with the same name already exists**: the group is not created, and the existing group's information is retained. However, all users in that group are removed and then re-added to reflect the directory service's latest information. However, users in that group who share the same name are retained.

  * **Users created in BioStar X**: Users created directly in **BioStar X** are not linked to the directory service. When synchronizing, update these users so their information is linked to the directory service.

  * **Limitations**: After synchronization, if a group with the same name already exists, that group's information and members may not synchronize properly.

* If you upgrade to **BioStar X v1.0.2** with directory integration enabled, the synchronization mode defaults to <Cmd sid='settings.activeDirectory.syncMode.addEditDelete' product='2' />.

:::

:::info

* Each time you click <Cmd sid='settings.activeDirectory.SyncNow' product='2' />, user information is synchronized with the directory service using the selected synchronization mode.

* When you select <Cmd sid='settings.activeDirectory.syncMode.addEditDelete' product='2' /> mode, a warning message will appear. To continue, click <Cmd sid='button.continue' product='2' />. To cancel, click <Cmd sid='button.cancel' product='2' />.

  <Image src='/img/biostarx-setting-directory-sync-warn-message.png' />

* To exclude specific users from synchronization when integrating, see <DocLink inner='#excludeintegration' />.

:::
