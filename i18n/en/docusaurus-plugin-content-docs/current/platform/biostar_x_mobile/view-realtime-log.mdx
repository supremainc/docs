View events that occur on the BioStar X server in real time. On the app home screen, tap <Cmd>REAL-TIME LOG</Cmd>.

<Image src='/img/mobile/bsx-mobile-app-realtime-log.png' width='500px' className='none' alone />

Use the <Cmd>REAL-TIME LOG</Cmd> feature in the following situations.

* Monitor access attempts during commute hours

* Check currently received events in real time

## View real-time logs

<Columns>
<Column className='width--23'>

<Image src='/img/mobile/bsx-mobile-app-realtime-log-list.png' alone />

</Column>
<Column className='padding--l40'>

On the app home screen, tap <Cmd>REAL-TIME LOG</Cmd>. When you first open the Real-time Log (<Cmd>Real-time Log</Cmd>) screen, the list is empty.

* When a new event occurs, it is automatically added to the list in real time. There is no limit to the number of rows in the list, and new events are added at the top.

* To pause real-time event reception, tap <IcAppPause title='Pause' width='20px' height='20px' /> in the upper right corner of the screen. When paused, new events are not added to the list, but they are queued in the background. To resume real-time event reception, tap <IcAppPlay title='Play' width='20px' height='20px' />. Events added to the queue appear before newer events.

* To clear the real-time log list, tap <IcAppClean title='Clean' width='20px' height='20px' /> in the upper right corner of the screen. The log list is cleared, and new events are added in real time as they occur.

* Filter the real-time log list to show only related logs such as certain events, users, doors, or devices. For more information, refer to <DocLink inner='#viewSpecificLog' />.

:::info

* If you move to another menu and return to <Cmd>Real-time Log</Cmd>, the log list is cleared, and new logs are added in real time as they occur.

* The date and time shown in the logs follow the BioStar X server locale settings. For more information on date and time display format settings in BioStar X, refer to <DocLink docId='platform/biostar_x/settings-preferences' />.

* You can also open the Real-time Log screen by tapping Side menu (<IcHamberg title='Hamburger icon' width={20} height={20} />) → <Cmd>Monitoring</Cmd> → <Cmd>Real-time Log</Cmd>.

:::

</Column>
</Columns>

## View only certain logs {/* #viewSpecificLog */}

<Columns>
<Column className='width--23'>

<Image src='/img/mobile/bsx-mobile-app-realtime-log-filter.png' alone />

</Column>
<Column className='padding--l40'>

Filter the Real-time Log list to show only related logs such as certain events, users, doors, or devices. 

1. Tap <IcAppFilter title='Filter' width='20px' height='20px' /> in the upper right corner of the screen. 

2. Set the following filter conditions to show only the logs you want.

   * <Cmd>Door</Cmd>: A certain door (for example, the first-floor main entrance)

   * <Cmd>Device</Cmd>: A certain device (for example, an event generated by a certain access control device)

   * <Cmd>User</Cmd>: A certain user (for example, access records for a VIP user)

   * <Cmd>Event</Cmd>: A certain event type (for example, facial authentication failure, intrusion alarm, and more)

   * <Cmd>Elevator</Cmd>: A certain elevator (for example, an elevator on a certain floor)

   * <Cmd>Advanced AC</Cmd>: Certain advanced access control events (for example, anti-passback, fire alarm, scheduled open/lock, and more)

3. To filter the log list with the set conditions, tap <Cmd>Apply</Cmd>.

:::info

* To reset the filter conditions, tap <IcAppFilter title='Filter' width='20px' height='20px' /> → <IcAppReset title='Reset' width='20px' height='20px' /> in the upper right corner of the screen.

* Elevators and advanced access control are available when the **Advanced** license or higher is active. For more information on the licensing policy, see <DocLink docId='platform/biostar_x/licensing' />.

:::

</Column>
</Columns>

## View log details

<Columns>
<Column className='width--23'>

<Image src='/img/mobile/bsx-mobile-app-realtime-log-details.png' width='252px' alone />

</Column>
<Column className='padding--l40'>

Tap a log entry to view detailed information about that log.

* <Cmd>Event Log</Cmd>: The event type. Shows which type of event occurred.

* <Cmd>Datetime</Cmd>: The date and time the event occurred.

* <Cmd>Door</Cmd>: The door where the event occurred.

* <Cmd>Device</Cmd>: The device where the event occurred.

* <Cmd>User</Cmd>: The user who triggered the event. For example, for an access authentication event, the user who attempted access appears.

Tap the buttons at the bottom of the details popup to perform the following actions.

* To copy the event information to the clipboard, tap <Cmd>EVENT INFORMATION COPIED</Cmd>.

* To close the details popup, tap <Cmd>OK</Cmd>.

</Column>
</Columns>
