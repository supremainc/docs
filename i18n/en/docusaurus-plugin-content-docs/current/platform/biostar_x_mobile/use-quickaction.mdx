**Quick Action** is a feature that lets you easily control the operation of multiple doors during an emergency. Create Quick Action in BioStar X to use it in the mobile app.

:::info

* For more information on the **Quick Action** feature, see <DocLink docId='platform/biostar_x/settings-custom-interface' />.

* In the mobile app, you can run the Quick Action feature or view detailed information about Quick Action entries. To create or edit a Quick Action item, log in to the BioStar X web interface and configure it.

:::

## Run a Quick Action

<Columns>
<Column className='width--23'>

<Image src='/img/mobile/bsx-mobile-quickaction-list.png' />

</Column>
<Column className='padding--l40'>

You must create Quick Actions in the BioStar X web interface before using them in the mobile app.

1. Tap <Cmd>QUICK ACTION</Cmd> on the app home screen.

2. In the Quick Action list, tap the <Cmd>Activate</Cmd> button for the Quick Action entry you want to run.

When Quick Action runs successfully, a success message appears.

:::info

* Depending on the Quick Action setting, a pop-up may appear asking whether to run it.

* Each Quick Action entry shows the most recent activation date and time.

:::

</Column>
</Columns>

## Quick Action details

<Columns>
<Column className='width--23'>

<Image src='/img/mobile/bsx-mobile-quickaction-details.png' />

</Column>
<Column className='padding--l40'>

To view detailed information configured for a Quick Action entry, tap the entry in the Quick Action list.

* <Cmd>Last Activation</Cmd>: The date and time when the Quick Action was last run.

* <Cmd>Confirm before run</Cmd>: Indicates whether a pop-up appears asking whether to run the Quick Action.

* <Cmd>Activate</Cmd>: The button that runs the Quick Action.

* <Cmd>Description and Operator Level</Cmd>: The Quick Action description and the administrator level required to run it.

* <Cmd>Trigger and Actions</Cmd>: The doors controlled when the Quick Action runs and the actions applied to those doors.

:::info

For more information on trigger conditions and actions, see <DocLink docId='platform/biostar_x/settings-triger-and-action' />.

:::

</Column>
</Columns>
