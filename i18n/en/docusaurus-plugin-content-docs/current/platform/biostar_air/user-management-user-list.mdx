The user list is an essential management tool for administrators responsible for access control in an organization. You can perform all user management tasks, including organizational management through user grouping, immediate response to new or departing employees with real-time activation/suspension, integrated management of various credentials, enhanced security through validity period monitoring, and preparation for reporting and auditing through filtering and report generation.

## View user list

The user list shows all users registered in BioStar Air at a glance. View each user's details and use the filter and sort features to quickly find specific users. Edit, delete, and assign groups to users directly from the user list.

### User information

The table in the user list provides the following information.

* <Cmd sid="user_table_column8" product="air" />: Indicates whether the user is a regular or temporary user. For more information on regular and temporary users, refer to <DocLink docId='platform/biostar_air/user-card-types' />.

* <Cmd sid="user_table_column1" product="air" />: The user's name.

* <Cmd sid="user_info_id" product="air" />: The user's unique ID.

* <Cmd sid="user_table_column2" product="air" />: Shows the credentials issued to the user. For more information on the types of credentials, refer to <DocLink docId='platform/biostar_air/user-card-types' />.

* <Cmd sid="user_table_column3" product="air" />: The validity period of the credentials issued to the user.

* <Cmd sid="user_table_column4" product="air" />: Indicates whether the user is active or inactive.

* <Cmd sid="user_table_column5" product="air" />: The group to which the user belongs. For more information on user groups, refer to <DocLink docId='platform/biostar_air/managing-user-groups' />.

* <Cmd sid="user_table_column6" product="air" />: The access level assigned to the user. For more information on access levels, refer to <DocLink docId='platform/biostar_air/manage-access-level' />.

* <Cmd sid="user_table_column7" product="air" />: The user's permissions for floors they can access. For more information on floor levels, refer to <DocLink docId='platform/biostar_air/manage-floor-level' />.

### Additional features {#additionalFunctions}

The features available at the top of the user list are as follows.

* <IcAirGroup width='24px' height='24px' title='Manage Groups' />: Assign, remove, or move the selected users to a group. For more information, refer to <DocLink inner='#manageUserGroups' />.

* <IcAirLock width='24px' height='24px' title='Set Access' />: Change the access level of the selected users. For more information about access levels, see <DocLink docId='platform/biostar_air/manage-access-level' />.

  <Image src='/img/air/bsair-user-list-tool-aclevel.png' />

* <IcAirMtag width='24px' height='24px' title='Mobile Consecutive Tag' />: Allow the selected users to tag mobile cards consecutively without waiting time. The Suprema Pass app supports this starting from version 2.9.0.
   <Image src='/img/air/bsair-user-list-tool-mobile-tag.png' />

* <IcAirUtemplate width='24px' height='24px' title='User Template' />: Apply a user template to the selected users. Add any other information needed in addition to the default user information.

  <Image src='/img/air/bsair-user-list-user-template.png' width='43%' />

* <IcAirDown width='24px' height='24px' title='Download' />: Export the selected user information to a PDF or CSV file.

* <IcAirUpface width='24px' height='24px' title='Image Upload' />: Upload user images in bulk. The name of each image file to upload must match the user's ID.

   For example, the file 601.jpg corresponds to user ID 601.

   :::note

   **File format requirements**

  * Supported formats: *.jpg*, *.jpeg*, *.png*

  * File size limit: **1 MB** per image

   :::

* <IcAirFaceLink width='26px' height='26px' title='Send Face Registration Link' />: Send a face enrollment link to the selected users by email or text message. When a user clicks the link and enrolls their face on the webpage, they are granted access via facial authentication.

   :::note

   The link cannot be sent to users without a registered email address or phone number.

   :::

* <IcAirFilterReset width='26px' height='26px' title='Clear Filters' />: Reset all filters applied to the user list. This feature is available only when filters are applied. For more information on applying filtering, refer to <DocLink inner='#userFiltering' />.

* <IcAirReport width='26px' height='26px' title='Report' />: Configure user reports. After a report is generated, it is sent to the administrator's email. For more information on report configuring, refer to <DocLink inner='#userReport' />.

* <IcAirMask width='24px' height='24px' title='Apply Masking' />: Mask(`***`) the information such as name and ID in the user list to protect privacy. This feature is useful in environments requiring privacy. Select the toggle button to enable this feature, and click <Cmd sid='save_btn' product='air' />.

  <Image src='/img/air/bsair-user-list-user-masking.png' />

## Manage user groups {#manageUserGroups}

This guide describes how to assign or move, exclude user from user group. For more information on creating and managing user group, refer to <DocLink docId='platform/biostar_air/managing-user-groups' />.

<Tabs groupId='platform'>
<TabItem value='web' label='Web'>

### Assign users to a group {#assigning-users-to-a-group}

After creating a user group, you can add users to the group from the user list.

1. Click <Cmd sid="user_menu" product="air" /> → <Cmd sid="user_submenu" product="air" /> in the left sidebar.

2. Click the checkbox next to the users to assign to the group in the user list.

3. Click <IcAirGroup title='Manage Groups' /> at the top left of the user list.

   <Image src='/img/air/bsair-user-group-assign.png' />

4. When the <Cmd sid='manage_user_groups_header' product='air' /> window appears, select the parent group or subgroup to assign the users.

   <Image src='/img/air/bsair-user-group-assign-popup.png' width='50%' />

5. Complete the selecting, and save group settings, click <Cmd sid='manage_grops_ok_btn' product='air' />.

Confirm that the users are assigned to the new group in the <Cmd sid='user_table_column5' product='air' /> column of the user list.

<Image src='/img/air/bsair-user-group-assign-result.png' />

Return to <Cmd sid='user_groups_submenu' product='air' /> to confirm that the group's <Cmd sid='user_groups_table_col2' product='air' /> is updated.

### Remove or move group {#disableGroup}

You can exclude users from a user group or move them to another group.

1. Click <Cmd sid="user_menu" product="air" /> → <Cmd sid="user_submenu" product="air" /> in the left sidebar.

2. Click the checkbox next to the users to remove from or move between groups in the user list.

   :::note

   When moving users, first select users from the same group, then proceed.

   :::

3. Click <IcAirGroup /> at the top left of the user list.

4. When the <Cmd sid='manage_user_groups_header' product='air' /> window appears, perform one of the following:

   * To unassign the group, clear the selected group's checkbox.

   * To move the group, select the checkbox for the current group and then choose the target group.

5. Click <Cmd sid='manage_grops_ok_btn' product='air' /> to save the group settings.

Confirm the group changes in the <Cmd sid='user_table_column5' product='air' /> column of the user list.

</TabItem>

<TabItem value='mobile' label='Mobile'>

<Image src='/img/air/bsair-app-mng-user-group.png' className='none' width='60%' />

* To view users in a group, tap <IcBw /> to the right of the desired group.

* To add a new group, tap <IcAdd title='Add' /> in the top right.

* To edit a group name, long-press the desired group and tap <IcAirEdit title='Edit' /> at the bottom.

* To delete a group, long-press the desired group and tap <IcAirRemove title='Remove' /> at the bottom. Select one or more groups to delete them.

## Manage user status {#manageUserStatus}

Activate or suspend a user's access permissions. Reissue credentials as well.

### Individual user

<TabItem value='web' label='Web'>

## Delete users {#deleteUser}

:::caution

Deleted users cannot be recovered. Be sure to verify before deleting.

:::

### Individual user {#deleteSingleUser}

1. Click <Cmd sid="user_menu" product="air" /> → <Cmd sid="user_submenu" product="air" /> in the left sidebar.

2. Click the user you want to delete in the list.

   <Image src='/img/air/bsair-user-list.png' />

3. Click <span className="air-remove"><IcAirRemove /></span> in the user details.

   <Image src='/img/air/bsair-user-details-remove.png' />

4. When a warning message appears, click <Cmd sid='delete_user_yes_btn' product='air' />.

### Delete multiple users {#deleteMultipleUsers}

1. Click <Cmd sid="user_menu" product="air" /> → <Cmd sid="user_submenu" product="air" /> in the left sidebar.

2. Click the checkbox of each user you want to delete in the list. You can select more than one user.

   <Image src='/img/air/bsair-user-list-multi-select.png' />

3. Click <IcAirMore /> → <Cmd sid="delete_action" product="air" /> at the top left of the screen.

   <Image src='/img/air/bsair-user-list-multi-select-remove-opt.png' />

4. When a warning message appears, click <Cmd sid='delete_user_yes_btn' product='air' />.

## Create a user report {#userReport}

Create a user report and send it to the administrator's email. 

1. Click <IcAirReport /> at the top of the user list.

2. When the <Cmd sid="report_popup_title" product="air" /> window appears, enter a report name.

   <Image src='/img/air/bsair-user-report-setting.png' />

3. Select the items to include in the report from each filter option.

4. Click <Cmd sid="report_form_text_add" product="air" /> to save the report.

5. A <Cmd sid="report_form_text_title" product="air" /> is created and a download link is sent to the administrator's email.

<Image src='/img/air/bsair-user-report-setting-list.png' />

:::info

In <Cmd sid="report_form_text_title" product="air" />, search for, edit, download, or delete generated reports.

:::
