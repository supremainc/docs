This document provides step-by-step instructions on adding and managing individual users in BioStar Air. It covers the entire process of selecting the user type (regular or temporary), issuing the appropriate credentials, and setting access levels and validity periods. It also includes how to edit or delete the added user’s information.

:::tip

Refer to the following video for step-by-step guidance on the entire process of adding a new user.

<YouTube className='fit--content' videoId="FTgpsYPlP8k" />

:::

## Add users on the web

### Add users {#addingUsers}

1. Click <Cmd sid="user_menu" product="air" /> → <Cmd sid="user_submenu" product="air" /> in the left sidebar.

2. A list of users appears.

3. Click <Cmd sid="add_user_btn" product="air" /> → <Cmd sid="add_user_btn" product="air" /> at the top right of the screen.

<Image src='/img/air/bsair-gettingstarted-user-group-mng-add-user.png' />

When the <Cmd sid='add_a_user' product='air' /> window appears, enter and configure the required information in each section. Set the required fields in all sections and click <Cmd sid="add_user_card_activate_btn" product="air" /> at the bottom right.

### Enter user information {#userInformation}

Enter the new user's basic information in the <Cmd sid="user_detail" product="air" /> section.

<Image src='/img/air/bsair-gettingstarted-user-group-mng-add-user-details.png' />

* <Cmd sid="user_form_input1" product="air" />: Enter a specific ID manually or allow the system to generate it sequentially.

* <Cmd sid="user_form_input3" product="air" />: Select the user's default language.

* <Cmd sid="user_form_input5" product="air" />: Select the group to which the user belongs.

* <Cmd sid="user_form_input2" product="air" />: Enter the user's full name.

* <Cmd sid="user_form_input18" product="air" />: Enter the user's title.

* <Cmd sid="user_form_input4" product="air" />: Enter the name of the user's department.

* <Cmd sid="user_form_input11" product="air" />: Upload the user's profile image. For more information, see <DocLink inner='#uploadProfileImage' />.

:::info

* <span className="require">\*</span> indicates a required field.

* Enter the user's email and phone number in <Cmd sid="user_form_input8" product="air" /> in the <Cmd sid="credential_detail" product="air" /> section.

:::

### Upload profile image {#uploadProfileImage}

1. Hover over the profile image area and click the plus icon.

   <Image src='/img/air/bsair-upload-profile-image.png' />

2. Select the image file saved locally.

3. When the <Cmd sid="crop_image_header" product="air" /> window appears, adjust the photo to the desired size. Click <Cmd sid="fit_this_image" product="air" /> to resize the image automatically.

   <Image src='/img/air/bsair-upload-profile-image-adjust.png' />

4. Click <Cmd sid="save_btn" product="air" /> to save the uploaded image as the profile image.

### Select user type {#selecting-user-type}

Select whether the user you are adding in the <Cmd sid="credential_detail" product="air" /> section is a regular user or a visitor. The credential types that can be issued vary by user type.

* <Cmd sid="user_regular_card_type" product="air" />: A user who is a regular member of an organization or facility and requires long-term, ongoing access. All credential types can be issued.

  <Image src='/img/air/bsair-add-user-credential-user-type-regular.png' />

* <Cmd sid="user_temporary_card_type" product="air" />: A visitor or non-regular member who needs short-term, limited access. This can include guests, external visitors, short-term contractors, delivery couriers, and so on. Only <Cmd sid="user_form_input20" product="air" /> credentials, including QR code and LinkPass, can be issued.

  <Image src='/img/air/bsair-add-user-credential-user-type-temporary.png' />

:::info

* <Cmd sid="user_regular_card_type" product="air" /> users are automatically assigned the full access level, whereas <Cmd sid="user_temporary_card_type" product="air" /> users must manually select an access level.

* <Cmd sid="user_table_column8" product="air" /> is supported in BioStar Air version 2.6 or higher.

:::

### Issue credentials {#issuingCredentials}

Enter user information, and then set the credentials to use for access authentication in the <Cmd sid="credential_detail" product="air" /> section. Each credential has a toggle switch to enable or disable it.

* <Cmd sid="card_type_mobile" product="air" />: Use the Suprema Pass mobile app as an access authentication method. The administrator can edit the card ID number.

  <Image src='/img/air/bsair-add-user-credential-credential-type-mobile.png' />

* <Cmd sid="card_type_rf" product="air" />: Add up to eight RFID cards per user for access authentication. Click <Cmd sid="add_btn" product="air" />.

  <Image src='/img/air/bsair-add-user-credential-credential-type-rfcard.png' />

   Configure the card type (CSN or Wiegand), card ID, and card name. Select a reader in <Cmd sid="newest_rf_card_text_device" product="air" />. Alternatively, scan a physical RFID card with the connected reader.

  <Image src='/img/air/bsair-add-user-credential-credential-type-rfcard-2type.png' className='none' />

* <Cmd sid="user_form_input20" product="air" />: Use <Cmd sid="card_type_option3" product="air" /> or <Cmd sid="card_type_option4" product="air" /> as access authentication methods.

  <Image src='/img/air/bsair-add-user-credential-credential-type-web.png' />

   :::note

  <Cmd sid="card_type_option3" product="air" /> appears only after enabling <Cmd sid="site_setting_qr_label" product="air" /> at <Cmd sid="settings_menu" product="air" /> → <Cmd sid="site_submenu" product="air" /> → <Cmd sid="site_setting_qr_reader_tab" product="air" />.

   :::

* <Cmd sid="card_type_option5" product="air" />: Add up to two face profile images for access authentication. You can upload an existing image or register it using a connected reader. Alternatively, email the user a link so they can enroll directly using their mobile device's camera.

  <Image src='/img/air/bsair-add-user-credential-credential-type-face.png' />

  * <Cmd sid="select_file" product="air" />: Select an image file saved locally to upload.

  * <Cmd sid='register_on_device' product='air' />: Use the connected reader to enroll a face image.

  * <Cmd sid="register_on_mobile" product="air" />: Scan a QR code with a mobile device’s camera to open a link for face image enrollment.

    <Image src='/img/air/bsair-add-user-credential-credential-type-face-qr.png' width='50%' />

* <Cmd sid="card_type_option6" product="air" />: Set a 4–16-digit PIN to use as part of two-factor authentication (2FA).

  <Image src='/img/air/bsair-add-user-credential-credential-type-pin.png' />

### Send invitation link {#sendingInvitationLink}

Email the Suprema Pass app download link to users who use <Cmd sid="card_type_mobile" product="air" /> or <Cmd sid="user_form_input20" product="air" />. Enter the email address (required) and phone number (optional).

<Image src='/img/air/bsair-add-user-credential-credential-type-invite-link.png' />

:::info

Request SMS credit recharge in <Cmd sid="settings_menu" product="air" /> → <Cmd sid="credit_center_submenu" product="air" /> to use the <Cmd sid="user_form_input10" product="air" /> option.

:::

### Set access levels and validity {#setting-access-levels-and-validity}

After selecting credentials, configure the credential validity period, access levels, and floor levels in the <Cmd sid="access_detail" product="air" /> section.

<Image src='/img/air/bsair-add-user-access-info.png' />

* <Cmd sid="card_valid_period" product="air" />: Set the validity period of the credentials issued to the user. Click <IcInfinity title='No Expiration Date' /> to leave the expiration date open-ended.

* <Cmd sid="user_form_input12" product="air" />: Select the access levels for the user. Grant full access or choose a predefined access level.

* <Cmd sid="user_form_input19" product="air" />: Select the floor levels the user can access.

* <Cmd sid='user_device_manager' product='air' />: Allow a user to manage access permissions on a specific reader.

* <Cmd sid="mobile_consecutive_tag" product="air" />: Allows consecutive mobile card tags without wait time. This feature is supported in Suprema Pass app version 2.9.0 or higher.

:::info

* <Cmd sid="user_form_input12" product="air" /> can be assigned permanently (expires with the credential) or temporarily (expires before the credential). For more information on access-level settings, see <DocLink docId='platform/biostar_air/manage-access-level' />.

* For more information on floor-level settings, see <DocLink docId='platform/biostar_air/manage-floor-level' />.

* <Cmd sid='user_device_manager' product='air' /> option is available only to X-Station 2 users who have access to the X-Station 2 device's admin menu.

:::

### Select card template {#selectingCardTemplate}

Click <IcAirL /> or <IcAirR /> under Card Template in the right panel to change the card template design.

<Image src='/img/air/bsair-add-user-select-card-template.png' width='70%' />

:::info

For more information about creating new card templates, see <DocLink docId='platform/biostar_air/managing-card-templates' />.

:::

### Save or activate the user {#saving-or-activating-the-user}

After configuring all user settings, choose one of the following options.

* Click <Cmd sid="save_btn" product="air" /> to save the user for later activation.

* Click <Cmd sid="add_user_card_activate_btn" product="air" /> to issue and activate the credentials immediately.

## Add users with the mobile app {#addingUsersWithMobileApp}

<Columns>
<Column className='width--40'>

<Image src='/img/air/bsair-app-mng-user-add.png' className='none' />

</Column>
<Column className='padding--l40'>

The mobile app also allows adding new users.

1. Tap <IcAdd /> at the top of the user list.

2. Enter the required details in <Cmd sid='add_user_form_tab' product='air' />.

3. Select whether the user is regular or temporary in <Cmd sid='select_user_card_type_label' product='air' />.

4. Issue credentials for access in <Cmd sid='credential_detail' product='air' />.

5. Enter the user's email address or mobile phone number in <Cmd sid='user_form_input8' product='air' />.

6. Set the validity period and access level.

7. To issue the card later, tap <Cmd sid='save_btn' product='air' />, or tap <Cmd sid='add_user_card_activate_btn' product='air' /> to issue the credentials immediately.

</Column>
