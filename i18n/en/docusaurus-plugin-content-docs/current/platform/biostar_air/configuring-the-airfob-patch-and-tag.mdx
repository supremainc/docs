## Register the Airfob Patch in the BioStar Air app {/* #registerApp */}

### Before you start

* Install the BioStar Air app on your mobile device. Install the mobile app through the following link.

  * [iOS](https://apps.apple.com/us/app/airfob-pro/id1659959609)

  * [Android](https://play.google.com/store/apps/details?id=com.mocainc.afs2pro)

* Log in using the same ID and password you use for the BioStar Air admin portal.

* Make sure BLE is enabled on your mobile device.

* Attach the Airfob Patch to the reader permanently or temporarily. For more information, see <DocLink docId='platform/biostar_air/install-airfob-patch' />.

:::info

For more information on using the BioStar Air mobile app, see <DocLink docId='platform/biostar_air/managing-sites-with-app' />.

:::

### Register Airfob Patch

1. Open the BioStar Air app on your mobile device and log in.

2. On the BioStar Air home screen, tap <Cmd sid='site_setting_tab_management' product='air' /> at the bottom.

3. Bring your mobile device within 10 cm of the Airfob Patch, then tap the <IcAdd /> button in the upper-right corner of the screen.

4. In the device search screen, tap <Cmd sid="button.register" product="2" /> for Airfob Patch to register.

5. When the device registration window appears, tap <Cmd sid="device_btn_ok" product="air" />.

Return to the device list and confirm the added Airfob Patch.

<Step>

### Choose the output method {/* #choose-the-output-method */}

* **None**: Select this option when you do not want to send a card ID. Useful for time and attendance to extend battery life.

* **Fixed Device**: Sends the pre-programmed fixed card ID. Useful for door locks or intercoms that require each RFID card to be programmed individually. Only this fixed card ID appears in the device logs.

* **User Card ID**: Sends a unique card ID for each user that matches the existing RFID card number in the access control system. This enables seamless integration without issuing new physical cards.

* **Direct Input**: Manually specify the card ID to be sent. Useful for custom configurations where one ID must be used on multiple units.

</Step>
<Step>

### Save your settings {/* #save-your-settings */}

* Keep the mobile device near the unit, tap <Cmd sid="save_btn" product="air" />, and confirm the selections.

* The update complete message appears.

</Step>
