Set layout of smart cards such as MIFARE, iCLASS, DESFire, iCLASS Seos.

## Add smart card {#addSmartformat}

1. Click <Cmd sid='shell.header.setting' /> on the <Cmd sid='shell.header.launcher' /> page.

2. Click <Cmd>Credential</Cmd> â†’ <Cmd>Card Format</Cmd> in the left sidebar.

3. Click <Cmd sid='common.smartCard' product='2' />.

4. Click <Cmd sid='button.func.addSmartCard' product='2' /> in the top left of the screen.

5. When the <Cmd sid='setting.cardFormat.addNewSmartCard' product='2' /> screen appears, configure each item. Refer to the [following](#smartCardType) for more information.

   <Image src='/img/biostarx-settings-credentail-smart-format-add.png' />

6. Once all settings are completed, click <Cmd sid='button.apply' product='2' />.

### Smart card types {#smartCardType}

Enable the <Cmd sid='setting.cardFormat.smartCardType' product='2' /> option to set a custom smart card layout.

<Image src='/img/biostarx-settings-credentail-smart-format-type.png' />

:::info

* Custom smart cards only support MIFARE, DESFire, and FeliCa. 

* FeliCa is only supported on BioStation 3 firmware version 1.3.0 or higher.

:::

### Setting options guide {#setting}

Set the card layout in the <Cmd sid='common.info' product='2' /> section.

<Image src='/img/biostarx-settings-credentail-smart-format-info.png' />

:::info

* The <Cmd sid='setting.cardFormat.primaryKey' product='2' /> and <Cmd sid='setting.cardFormat.secondaryKey' product='2' /> support only hexadecimal. Enter the key values to be used in the right field and click <Cmd sid='setting.cardFormat.convertToHex' product='2' />. Use the converted value as the site key.

* <Cmd sid='setting.cardFormat.convertToHex' product='2' /> is only available when the <Cmd sid='setting.cardFormat.smartCardType' product='2' /> option is <Cmd sid='common.supremaSmartCard' product='2' />.

:::

#### Set basic information

* <Cmd sid='common.name' product='2' />: Enter the smart card name.

* <Cmd sid='setting.cardFormat.secondaryKey' product='2' />: Configure whether to use a secondary site key. Set to <Cmd sid='common.active' product='2' /> to enable secondary site key configuration. When a secondary site key is set, it will be used for authentication when the card's primary site key does not match.

#### Set smart card

* <Cmd sid='setting.card.format.desfireAvd' product='2' />: You can use DESFire cards issued by third parties. Only DESCire can be configured.

  :::note

  Enter the below information accurately to use this feature.

  <Cmd sid='setting.cardFormat.appMasterKey' product='2' />, <Cmd sid='setting.cardFormat.appMasterKey' product='2' /> <Cmd sid='device.info.inputIndex' product='2' />, <Cmd sid='setting.cardFormat.fileReadAccessKey' product='2' />, <Cmd sid='setting.cardFormat.fileReadAccessKey' product='2' /> <Cmd sid='device.info.inputIndex' product='2' />, <Cmd sid='setting.cardFormat.fileWriteAccessKey' product='2' /> <Cmd sid='device.info.inputIndex' product='2' />, <Cmd sid='setting.cardFormat.appId' product='2' />, <Cmd sid='setting.cardFormat.fileId' product='2' />, <Cmd sid='setting.cardFormat.encryptionType' product='2' />

  :::

* <Cmd sid='setting.cardFormat.primaryKey' product='2' />: The key that encrypts communications between the smart card and the card reader.

* <Cmd sid='setting.cardFormat.secondaryKey' product='2' />: Used for authentication when the card's primary site key does not match. The secondary site key can only be entered when the option <Cmd sid='setting.cardFormat.secondaryKey' product='2' /> is enabled.

* <Cmd sid='setting.cardFormat.startBlockIndex' product='2' /> / <Cmd sid='setting.cardFormat.blockIndex' product='2' />: Select the starting block where each template will be stored. This block is the index of the block where user information is stored; set it to a storable block if the user is already using the smart card.

  :::note

  Can only be set for MIFARE and iCLASS.

  :::

* <Cmd sid='setting.cardFormat.idField' product='2' />: Configure to read a specific area on the card. You can set up to 8 blocks to read by clicking <Cmd sid='button.addplus' product='2' />.

  :::note

  Only DESCire can be configured.

  :::

* <Cmd sid='setting.cardFormat.fileId' product='2' />: Set the file ID.

  :::note

  Only DESCire can be configured.

  :::

* <Cmd sid='setting.cardFormat.encryptionType' product='2' />: Set the encryption type to <Cmd>DES/3DES</Cmd> or <Cmd>AES</Cmd>.

  :::note

  Only DESCire can be configured.

  :::

* <Cmd sid='setting.cardFormat.ADF' product='2' />: The ADF address where digital credentials are stored.

  :::note

  Only accessible on iCLASS Seos cards.

  :::

* <Cmd sid='setting.cardFormat.skipBytes' product='2' />: Set the starting point for reading the card number.

  :::note

  Can only be set for custom smart cards MIFARE, DESFire, and FeliCa.

  :::

* <Cmd sid='setting.cardFormat.dataSize' product='2' />: (When the set <Cmd sid='setting.cardFormat.primaryKey' product='2' /> and <Cmd sid='setting.cardFormat.secondaryKey' product='2' /> values are the same as the card's configuration) Configure the data size of the card to be read.

  :::note

  Can only be set for custom smart cards MIFARE, DESFire, and FeliCa.

  :::

* <Cmd sid='setting.cardFormat.systemCode' product='2' />: Enter the system code to read FeliCa cards in hexadecimal, with a maximum of 4 characters.

  :::note

  Can only be set for FeliCa in custom smart cards.

  :::

* <Cmd sid='setting.cardFormat.serviceCode' product='2' />: Enter the service code to read FeliCa cards in hexadecimal, with a maximum of 4 characters.

  :::note

  Can only be set for FeliCa in custom smart cards.

  :::

* <Cmd sid='setting.cardFormat.appId' product='2' />: Set the application ID, serving as a type of directory that includes <Cmd sid='setting.cardFormat.fileId' product='2' />. You can set up to 8 blocks to read by clicking <Cmd sid='button.addplus' product='2' />.

  * <Cmd sid='setting.cardFormat.blockNumber' product='2' />: Set the block to read from the card. (0\-150)

  :::note

  Can only be set for FeliCa in custom smart cards.

  :::

#### Layout {#layout}

You can modify the layout that records user information, facial, and fingerprint information.

* <Cmd sid='setting.cardFormat.numberOfTemplate' product='2' />: Set the number of fingerprint templates to include in the layout.

* <Cmd sid='setting.cardFormat.templateSize' product='2' />: Set the number of bytes used by each fingerprint template.

* <Cmd sid='setting.cardFormat.useFace' product='2' />: Choose whether to use the face template.

* <Cmd sid='setting.cardFormat.faceTemplateSize' product='2' />: Set the number of bytes used by the face template.

:::info

<Cmd sid='setting.cardFormat.useFace' product='2' /> is available on FaceStation F2, BioStation 3, and BioEntry W3 models.

:::
