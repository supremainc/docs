The <Cmd sid='setting.menu.system.auditTrail' /> feature records and tracks all activities occurring in the system. All activities such as user login/logout, setting changes, and system administration tasks are recorded and can be utilized for security audits and system monitoring.

* **Security audit**: Detect unauthorized access attempts or privilege abuse.

* **Troubleshooting**: Track the occurrence of system errors and setting change history.

* **Compliance**: Retain access logs and generate audit reports.

## Use audit trail {#audit-trail-guide}

1. Click <Cmd sid='shell.header.setting' /> on the <Cmd sid='shell.header.launcher' /> page.

2. Click <Cmd sid='setting.menu.system.root' /> → <Cmd sid='setting.menu.system.auditTrail' /> on the left sidebar.

3. The audit trail list appears on the screen.

<Image src='/img/biostarx-settings-system-audit-trail.png' />

For more information on major tracking items, refer to [the following](#audit-trail-items).

### Types of tracked actions {#method}

Click <Cmd sid='auditTrail.method' product='2' /> option in the <Cmd sid='auditTrail.filter.title' product='2' /> section to categorize audit trails by the following action types.

<Image src='/img/biostarx-settings-system-audit-trail-action-filter.png' />

* <Cmd sid='audit.method.1' product='2' />: Create new item

* <Cmd sid='audit.method.3' product='2' />: Modify existing item

* <Cmd sid='audit.method.4' product='2' />: Remove item

* <Cmd sid='audit.method.2' product='2' />: Execute system functions (login, device control, etc.)

### Classification by permission level {#permision}

Audit trails can be categorized according to admin level. Click <Cmd sid='user.bioStarOperator' product='2' /> option in the <Cmd sid='auditTrail.filter.title' product='2' /> section to filter by the following permission levels.

* <Cmd sid='audit.permission.1' product='2' />: Access all system functions

* <Cmd sid='audit.permission.2' product='2' />: User management functions

* <Cmd sid='audit.permission.3' product='2' />: Monitoring functions

* <Cmd sid='audit.permission.253' product='2' />: Video-related functions

* <Cmd sid='audit.permission.254' product='2' />: Attendance management functions

:::info

<Cmd sid='audit.permission.253' product='2' /> and <Cmd sid='audit.permission.254' product='2' /> permission levels are available through additional options for **Advanced** license or higher. For more information on licensing policy, refer to [the following](licensing#features-add-ons).

:::

## Period setting query

Set the period to check the audit trail history for a specific duration.

### Query with predefined periods

Select <Cmd sid='auditTrail.filter.one_month' product='2' /> or <Cmd sid='auditTrail.filter.three_month' product='2' /> from the filter list on the left screen.

<Image src='/img/biostarx-settings-system-audit-trail-defined-period.png' />

### Query with custom periods

For detailed period settings, set the start date and end date in the <Cmd sid='auditTrail.filter.title' product='2' /> section under the <Cmd sid='common.datetime' product='2' /> option.

<Image src='/img/biostarx-settings-system-audit-trail-set-period.png' />

## Filter settings

Filter the audit trail list efficiently using various criteria.

<Image src='/img/biostarx-settings-system-audit-trail-list.png' />

In the <Cmd sid='auditTrail.filter.title' product='2' /> section, you can set the following options. Filter options match the column items of the audit trail list. However, the <Cmd sid='auditTrail.content' product='2' /> column cannot be filtered.

* <Cmd sid='common.datetime' product='2' />: Date and time when the event occurred

* <Cmd sid='common.gnb.user' product='2' />: User ID of the user who performed the event

* <Cmd sid='user.bioStarOperator' product='2' />: User's [permission level](#permision)

* <Cmd sid='auditTrail.ip' product='2' />: IP address from which the user accessed

* <Cmd sid='auditTrail.menu' product='2' />: Menu category of the modified item

* <Cmd sid='auditTrail.target' product='2' />: Modified target

* <Cmd sid='auditTrail.method' product='2' />: Performed [action](#method)

* <Cmd sid='auditTrail.content' product='2' />: Details during the event occurrence

## Save filter

Set the filter and click the <Cmd sid='auditTrail.filter.title' product='2' /> section's <Cmd sid="ta.report.condition.saveFilter.button" product='2' /> button to save the filter you set to the filter list on the left screen.

<Image src='/img/biostarx-settings-system-audit-trail-add-filter.png' />

* Click <IcEditUL /> to edit the filter name. Enter the filter name and press <Kbd>Enter</Kbd> to change the filter name.

* Click <IcTrash2 /> to delete the filter. The filter will be removed from the filter list.

## Export to CSV

Export the audit trail list to a CSV file for use with external analysis tools.

1. In the <Cmd sid='auditTrail.filter.title' product='2' /> section, set filters if needed.

2. Click <IcMoreW /> → <Cmd sid='audit.user.csv_export' product='2' /> on the right side of the section.

3. When the CSV export window appears, enter <Cmd sid='auditTrail.export.reason' product='2' />(optional) and click the <Cmd sid='button.download' product='2' /> button.

   <Image src='/img/biostarx-settings-system-audit-trail-export-csv.png' />

:::tip

If you have set filtering, only the filtered results will be exported to the CSV file.

:::
