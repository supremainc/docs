<Cmd sid='common.imageLog' product='2' /> allows for image verification of events that occur on devices with cameras. It can record face images of users passing through doors or situations during event occurrences using the device's camera. The recorded images can be viewed through real-time events on the <Cmd sid='shell.header.arena' /> page.

<Image src='/img/biostarx-setting-device-imagelog-example.png' />

:::info

* This feature is supported by the BioStation A2, FaceStation 2, FaceStation F2, X-Station 2, and BioStation 3 models.

* For detailed information on viewing event logs on the <Cmd sid='shell.header.arena' /> page, refer to [the following document](monitoring-event).

* The settings configured in the <Cmd sid='shell.header.setting' /> → <Cmd sid='common.imageLog' product='2' /> menu do not reflect on the device. For detailed information on the device's <Cmd sid='common.imageLog' product='2' /> settings, refer to [the following document](settings-device-details-advanced#image-log-settings).

:::

## Add Basic Settings {#add-image-log-settings}

You can set events and schedules to generate image logs on the device. An image log will be generated if the conditions of events and schedules are met.

1. Click <Cmd sid='shell.header.setting' /> on the <Cmd sid='shell.header.launcher' /> page.

2. In the left sidebar, click <Cmd>Device</Cmd> → <Cmd>Image Log</Cmd>.

3. Click the <Cmd sid='button.plusAdd' product='2' /> button to the right of the <Cmd sid='common.preset' product='2' /> section.

   <Image src='/img/biostarx-setting-device-imagelog-add.png' />

4. When an event is added at the bottom of the list, click the <IcDown /> button for the desired event and select your options.

   <Image src='/img/biostarx-setting-device-imagelog-select-event.png' />

5. Click the <IcDown /> button in the schedule for the selected event and choose the preferred option.

   <Image src='/img/biostarx-setting-device-imagelog-select-schedule.png' />

6. Click <Cmd sid='button.apply' product='2' /> at the bottom of the screen to save the settings.

:::info

If the desired schedule is not available, click <Cmd sid='button.plusAddSchedule' product='2' /> to add one. For more details on schedule settings, refer to [the following document](settings-schedule).

:::

## Modify Basic Settings {#edit-image-log-settings}

You can change the events and schedules of existing image logs.

1. Click <Cmd sid='shell.header.setting' /> on the <Cmd sid='shell.header.launcher' /> page.

2. In the left sidebar, click <Cmd>Device</Cmd> → <Cmd>Image Log</Cmd>.

3. In the list of the <Cmd sid='common.preset' product='2' /> section, click the <IcDown /> button for the event you want to modify and select your options.

   <Image src='/img/biostarx-setting-device-imagelog-select-event.png' />

4. Click the <IcDown /> button in the schedule for the selected event and choose the preferred option.

   <Image src='/img/biostarx-setting-device-imagelog-select-schedule.png' />

5. Click <Cmd sid='button.apply' product='2' /> at the bottom of the screen to save the settings.

:::info

If the desired schedule is not available, click <Cmd sid='button.plusAddSchedule' product='2' /> to add one. For more details on schedule settings, refer to [the following document](settings-schedule).

:::

## Delete Basic Settings {#delete-image-log-settings}

You can delete events and schedules of existing image logs.

1. Click <Cmd sid='shell.header.setting' /> on the <Cmd sid='shell.header.launcher' /> page.

2. In the left sidebar, click <Cmd>Device</Cmd> → <Cmd>Image Log</Cmd>.

3. In the list of the <Cmd sid='common.preset' product='2' /> section, click the <IcTrash2 /> button for the event you want to delete.

   <Image src='/img/biostarx-setting-device-imagelog-remove.png' />

4. Click <Cmd sid='button.apply' product='2' /> at the bottom of the screen to save the settings.

## <Cmd sid='setting.imageLog.deleteOption' product='2' className='bold' /> {#image-log-delete-options}

You can set delete options so that image log files do not take up space on the **BioStar X** server. Image logs that exceed the set file size or duration will be automatically deleted. 

1. Click <Cmd sid='shell.header.setting' /> on the <Cmd sid='shell.header.launcher' /> page.

2. In the left sidebar, click <Cmd>Device</Cmd> → <Cmd>Image Log</Cmd>.

3. Set the conditions for deleting image logs in the <Cmd sid='setting.imageLog.deleteOption' product='2' /> section.

   * <Cmd sid='setting.imageLog.deleteOption' product='2' />: Choose the file size unit (<Cmd>MB</Cmd>/<Cmd>GB</Cmd>) or duration unit (<Cmd sid='setting.imageLog.deleteOption.day' product='2' />/<Cmd sid='setting.imageLog.deleteOption.week' product='2' />/<Cmd sid='setting.imageLog.deleteOption.month' product='2' />).

   * <Cmd sid='setting.imageLog.deleteOption.log.amount' product='2' />: Set the unit of the condition defined in <Cmd sid='setting.imageLog.deleteOption' product='2' />.

   * <Cmd sid='setting.imageLog.deleteCheckDuration' product='2' />: Choose the cycle for deleting image logs.

4. Click <Cmd sid='button.apply' product='2' /> at the bottom of the screen to save the settings.

## <Cmd sid='setting.imageLog.storagePath' product='2' className='bold' /> {#image-log-save-path}

You can set the path where image logs are stored. 

1. Click <Cmd sid='shell.header.setting' /> on the <Cmd sid='shell.header.launcher' /> page.

2. In the left sidebar, click <Cmd>Device</Cmd> → <Cmd>Image Log</Cmd>.

3. Enter the path for storing image logs in the <Cmd sid='setting.imageLog.imageLogFilePath' product='2' /> input field of the <Cmd sid='setting.imageLog.storagePath' product='2' /> section.

4. Click <Cmd sid='button.apply' product='2' /> at the bottom of the screen to save the settings.

:::info

* The storage path is not created automatically, so it must be created in advance for storing image logs.

* For example, entering `.\imagelog\` will store image logs under the path where BioStar X is installed. 

  > *C:\\Program Files\\BioStar 2(x64)\\imagelog\\*

:::

## <Cmd sid='setting.image_log.title' product='2' className='bold' /> {#image-log-user-profile}

You can configure the device to display the user's profile picture when monitoring event logs and real-time logs if the device cannot shoot image logs.

1. Click <Cmd sid='shell.header.setting' /> on the <Cmd sid='shell.header.launcher' /> page.

2. In the left sidebar, click <Cmd>Device</Cmd> → <Cmd>Image Log</Cmd>.

3. Check the checkbox for the <Cmd sid='setting.image_log.display' product='2' /> item in the <Cmd sid='setting.image_log.title' product='2' /> section.

4. Click <Cmd sid='button.apply' product='2' /> at the bottom of the screen to save the settings.

:::info

* When this option is enabled, if an image log occurs, the user profile picture will not be shown, and the recorded image log will be displayed instead.

* For detailed information on viewing event logs on the <Cmd sid='shell.header.arena' /> page, refer to [the following document](monitoring-event).

:::
