This guide describes how to set up access groups using access levels and user group information. Access groups are crucial elements that define permission for accessing doors. Setting up access groups enables management of user permissions to access doors. 

## Before start {#beforeStart}

Confirm the following before registering access groups.

* Set up access levels. For more details on registering access levels, refer to the [following](settings-access-control-manage-access-level).

* Set up floor levels. For more details on registering floor levels, refer to the [following](#).

* Set up user groups. For more details on registering user groups, refer to the [following](user-group-manage).

## Register access groups {#addAccessGroup}

1. Click <Cmd sid='shell.header.setting' /> on the <Cmd sid='shell.header.launcher' /> page.

1. Click <Cmd>Access Control</Cmd> in the left sidebar.

1. Click <Cmd sid='button.func.addAccessGroup' product='2' />.

1. On the <Cmd sid='accessGroup.addNewAccessGroup' product='2' /> screen, enter <Cmd sid='common.name' product='2' /> and <Cmd sid='common.description' product='2' />.

    <Image src='/img/biostarx-settings-add-access-group.png' />

1. In the <Cmd sid='common.accessLevel' product='2' /> field, click <Cmd sid='button.addplus' product='2' /> to select the access levels to apply to the access group.

1. In the <Cmd sid='ag.floorLevel.title' product='2' /> field, click <Cmd sid='button.addplus' product='2' /> to select the floor levels to apply to the access group.

1. In the <Cmd sid='accessGroup.userGroup' product='2' /> field, click <Cmd sid='button.addplus' product='2' /> to select the user groups to apply to the access group.

1. In the <Cmd sid='common.user' product='2' /> field, click <Cmd sid='button.addplus' product='2' /> to select users for the access group.

1. To register the access group, click <Cmd sid='button.apply' product='2' /> at the bottom of the screen.

:::info

* If the desired access level is not available, you will need to register a new access level. For more details on registering access levels, refer to the [following](settings-access-control-manage-access-level).

* If the desired floor level is not available, you will need to register a new floor level. For more details on registering floor levels, refer to the [following](#).

* To delete added items, click <IcTrash2 />.

* To search for a desired item, click <IcSearch />.

:::

## Edit access groups {#editAccessGroup}

1. Click <Cmd sid='shell.header.setting' /> on the <Cmd sid='shell.header.launcher' /> page.

1. Click <Cmd>Access Control</Cmd> in the left sidebar.

1. Click the <Cmd sid='access_control.tab.access_group' product='2' /> tab.

1. Click the access group you want to edit from the list.

1. Modify the desired items.

1. Once modifications are complete, click <Cmd sid='button.apply' product='2' /> at the bottom of the screen.

## Delete access groups {#deleteAccessGroup}

1. Click <Cmd sid='shell.header.setting' /> on the <Cmd sid='shell.header.launcher' /> page.

1. Click <Cmd>Access Control</Cmd> in the left sidebar.

1. Click the <Cmd sid='access_control.tab.access_group' product='2' /> tab.

1. Check the checkbox for the access group you wish to delete from the list.

1. Click <Cmd sid='button.deleteAccessGroup' product='2' /> in the upper right corner of the screen.
