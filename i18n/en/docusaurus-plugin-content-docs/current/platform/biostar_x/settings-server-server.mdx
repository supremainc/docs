You can change the basic information settings of the BioStar X server, user and device management, server matching, system log levels, and settings for saving facial images.

1. Click <Cmd sid='shell.header.setting' /> on the <Cmd sid='shell.header.launcher' /> page.

2. Click <Cmd sid='setting.menu.server.root' /> → <Cmd sid="common.server" product='2' /> in the left sidebar.

3. Set the desired items in each section.

   * <Cmd sid='common.general' product='2' />: You can set the IP address, port number, session expiration time, event log upload method, and log retention period.

   * <Cmd sid='setting.server.userDeviceManagement' product='2' />: You can change user and device management settings.

   * <Cmd sid='setting.server.logLevelManagement' product='2' />: You can change system log level settings.

   * <Cmd sid='settings.expertSettings.title' product='2' />: You can set it to save both the actual image and the template of the face or save only the template without storing the actual image.

4. Click <Cmd sid='button.apply' product='2' /> at the bottom right of the screen to save the settings.

## Basic information settings {#basic-setting}

You can check or set basic information for **BioStar X** in the <Cmd sid='common.general' product='2' /> section.

<Image src='/img/biostarx-settings-server-general.png' className='none' />

* <Cmd sid="setting.server.biostarIPAdress" product='2' />: You can check the server IP address.

* <Cmd sid="setting.server.logUploadType" product='2' />: Select an event log upload method. If real-time communication with the server is difficult, set this to <Cmd sid="common.manual" product='2' />.

* <Cmd sid="setting.server.biostarPort" product='2' />: You can change the server port. Enter the desired port number.

* <Cmd sid="setting.server.logRetainDays" product='2' />: The log retention period can be set from 1 to 120 days.

* <Cmd sid="common.sessionTimeout" product='2' />: Set a session timeout period. If there is no operation after logging into BioStar X for the configured time, you will be automatically logged out.

## User and device management {#user-and-device-management}

In the <Cmd sid='setting.server.userDeviceManagement' product='2' /> section, you can set the synchronization and authentication methods between users and devices.

<Image src='/img/biostarx-settings-server-management.png' className='none' />

### <Cmd sid="setting.server.automaticUserSync" product='2' className='normal' /> {#user-sync}

In the <Cmd sid="setting.server.automaticUserSync" product='2' /> option, you can choose how to synchronize user information between the server and devices.

* <Cmd sid="setting.server.syncOption.allDevices" product='2' />: User information is synchronized automatically between the server and all registered devices.

* <Cmd sid="setting.server.syncOption.deviceSync" product='2' />: User information from the server is synchronized with all devices registered on the server. However, user information modified on the device is not synchronized to the server, and only user information added on the device is synchronized to the server.

* <Cmd sid="setting.server.syncOption.smartSync" product='2' />: Only devices belonging to an access group with changes are automatically synchronized with the server.

:::info

* If you select the <Cmd sid="setting.server.syncOption.smartSync" product='2' /> option, users saved on devices that do not belong to the access group cannot be managed by the server. To use this option, go to <Cmd sid='shell.header.setting' /> → <Cmd sid='setting.menu.device.root' /> to select each device, and then click <Cmd sid="device.sync_all" product='2' /> to proceed with synchronization. For more information about the <Cmd sid="device.sync_all" product='2' /> feature, refer to [the following](settings-device-using-functions).

* Even if you select the <Cmd sid="setting.server.syncOption.smartSync" product='2' /> option, special access groups designated for specific purposes will synchronize regardless of the device access group.

  * Dual authentication access group set up in the Devices and Elevators.

  * Always pass group for anti-passback setting

  * Always pass group for schedule lock setting

  * Open authentication access group for schedule open setting

  * Access/Deactivate group for guard setting 

* Even if you select the <Cmd sid="setting.server.syncOption.smartSync" product='2' /> option, users designated as device administrators will synchronize regardless of the access group.

:::

### <Cmd sid="setting.server.fingerprintTemplateFormat" product='2' className='normal' /> {#fingerprint-template-format}

In the <Cmd sid="setting.server.fingerprintTemplateFormat" product='2' /> option, you can choose the fingerprint template format. Select from the following items.

* <Cmd>SUPREMA</Cmd>: Suprema fingerprint template format. Used when enrolling fingerprints on Suprema devices.

* <Cmd>ISO</Cmd>: ISO international fingerprint template format.

* <Cmd>ANSI378</Cmd>: Fingerprint template format defined by the American National Standards Institute (ANSI).

:::info

If there are existing user fingerprint templates on the device, they cannot be changed to another format.

:::

### <Cmd sid="setting.server.custom_field.title" product='2' className='normal' /> {#custom-field}

In the <Cmd sid="setting.server.custom_field.title" product='2' /> option, you can add fields to input additional information about the user. The added fields will be displayed on the user information screen.

1. To add a custom user field, click the <Cmd sid="button.plusAdd" product='2' /> button.

2. Enter the name of the custom user field and select the desired item type.

   <Image src='/img/biostarx-settings-server-management-custome-field.png' />

   * <Cmd sid="setting.server.custom_field.type.numericbox" product='2' />: Allows entry of values from 0 to 4294962795; characters cannot be entered.

   * <Cmd sid="setting.server.custom_field.type.textbox" product='2' />: Allows entry of up to 32 characters of numbers and letters.

   * <Cmd sid="setting.server.custom_field.type.combobox" product='2' />: Allows addition of up to 20 items of 32 characters; each item is separated by a semicolon (;). (e.g., Select1;Select2;Select3)

     <Image src='/img/biostarx-settings-server-management-custome-field-select.png' />

   * <Cmd sid='setting.server.custom_field.type.fileupload' product='2' />: You can add a field for file uploads.

     :::note

     * The maximum file size is 1MB.

     * The file name can be a minimum of 1 character and a maximum of 120 characters.

     * File name requirements: The file name must use alphanumeric characters, underscores (`_`), and hyphens (`-`), and use a period (`.`) to separate the extension.

     :::

3. Click the <Cmd sid='button.apply' product='2' /> button at the bottom right of the screen to save the added custom user field.

:::info

* To change the order of the custom user fields, change the numbers in the <Cmd sid='setting.server.custom_field.order' product='2' /> column. The position of that field will change.

* After modifying the custom user fields, click the <Cmd sid='button.apply' product='2' /> button at the bottom right of the screen.

* Click the <IcTrash2 /> button to delete a custom user field. Deleted fields cannot be recovered.

* For more information on user information registration and modification, refer to [the following](how-to-guide-user).

* User information with added custom user fields can be printed on a card. For more information, refer to [the following link](settings-card-printer#adding-custom-user-fields).

:::

## Server matching {#servermatching}

In the <Cmd sid='common.serverMatching' product='2' /> section, you can set up the [server matching](../../reference/glossary#server_matching) feature to authenticate user information on **BioStar X** without authenticating on the device.

<Image src='/img/biostarx-settings-server-server-matching.png' className='none' />

* <Cmd sid="setting.server.serverMatching.use" product='2' />: Activates/Deactivates server matching.

* <Cmd sid="setting.server.serverMatching.matchingConcurrencyCount" product='2' />: You can configure how many matchings can be done simultaneously.

  :::note

  This option may vary depending on the CPU performance of the server PC where **BioStar X** is installed.

  :::

* <Cmd sid="setting.server.serverMatching.fastMode" product='2' />: You can configure the fingerprint matching speed.

* <Cmd sid="setting.server.serverMatching.securityLevel" product='2' />: You can set the security level for server matching for fingerprints. The higher the security level is set, the more the false rejection rate (FRR) can occur.

:::info

<Cmd sid='common.serverMatching' product='2' /> is available when feature add-ons are enabled with an **Advanced** or higher license. For more information about the license policy, refer to [the following](licensing).

:::

## System log level settings {#systemloglevel}

In the <Cmd sid='setting.server.logLevelManagement' product='2' /> section, you can set the level of system logs stored in the database.

<Image src='/img/biostarx-settings-server-system-log-level.png' className='none' />

System logs are managed according to predefined categories, and the log levels are classified into <Cmd sid="setting.server.logLevel.trace" product='2' />, <Cmd sid="setting.server.logLevel.debug" product='2' />, <Cmd sid="setting.server.logLevel.info" product='2' />, <Cmd sid="setting.server.logLevel.warning" product='2' />, and <Cmd sid="setting.server.logLevel.error" product='2' />. The high level contains all lower level logs. For example, if you select <Cmd sid="setting.server.logLevel.trace" product='2' />, it will include and save all lower levels of <Cmd sid="setting.server.logLevel.debug" product='2' />, <Cmd sid="setting.server.logLevel.info" product='2' />, <Cmd sid="setting.server.logLevel.warning" product='2' />, and <Cmd sid="setting.server.logLevel.error" product='2' /> logs.

## Saving facial images {#storevisualfaceimage}

When enrolling a face as a credential, you can set it to save both the actual image and template, or to save only the template without the actual image.

:::note

**Before using**

Before activating the <Cmd sid="settings.expertSettings.enableOptionLabel" product='2' /> option, if any changes have been made in the <Cmd sid="common.server" product='2' /> menu, be sure to save those changes first. Click <Cmd sid="button.apply" product='2' /> at the bottom right of the screen.

:::

1. In the <Cmd sid="settings.expertSettings.title" product='2' /> section, set the <Cmd sid="settings.expertSettings.enableOptionLabel" product='2' /> option to <Cmd sid="common.use" product='2' />.

   <Image src='/img/biostarx-settings-server-expert.png' className='none' />

2. Click the <Cmd sid='settings.expertSettings.enableButton' product='2' /> button.

3. When you enter the <Cmd sid="settings.expertSettings.detail.title" product='2' /> screen, set the <Cmd sid="settings.expertSettings.detail.storeVisualFaceImageOptionLabel" product='2' /> option.

   <Image src='/img/biostarx-settings-server-expert2.png' />

   * <Cmd sid="common.use" product='2' />: This is the default setting. Saves both the actual image and the template of the face credential.

   * <Cmd sid="common.notUse" product='2' />: Does not save the actual image of the face credential and saves only the template. Read the warning popup that appears when you select this option carefully before setting it.

4. Click <Cmd sid="button.apply" product='2' /> to save the settings.

:::caution

**Precautions when disabling the <Cmd sid="settings.expertSettings.detail.storeVisualFaceImageOptionLabel" product='2' /> option**

* If you disable this feature, all stored facial images will be deleted, and future enrolled facial images will only save the templates excluding the images.

* This feature cannot be disabled if the enrolled facial images do not have both types of templates. For more information about the <Cmd sid='team.menu.faceMigration' /> feature, refer to [the following](migration-face).

* After disabling this feature, if you include invalid face image templates in the <Cmd sid="menu.dataImport" product='2' />, you will not be able to import the data file. To resolve this issue, temporarily enable this feature before importing the data file.

:::
