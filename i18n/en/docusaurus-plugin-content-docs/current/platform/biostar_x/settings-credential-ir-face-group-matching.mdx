<Cmd sid="setting.menu.credential.irFaceGroupMatching" className="normal" /> authenticates users by setting matching groups based on user groups. Use this feature to allow access via IR face authentication only to users in certain user groups.

1. Click <Cmd sid="shell.header.setting" /> on the <Cmd sid="shell.header.launcher" /> page.

2. Click <Cmd sid="setting.menu.credential.root" /> â†’ <Cmd sid="setting.menu.credential.irFaceGroupMatching" /> in the left sidebar.

3. Change the <Cmd sid="globalSetting.facegroup.detail.useOption" product="2" /> option to <Cmd sid="common.use" />.

   <Image src="/img/biostarx-settings-credential-irface-group-match.png" />

4. Click <Cmd sid="button.addplus" product="2" /> in the <Cmd sid="globalSetting.facegroup.detail.deviceTap.title" product="2" />.

5. When the <Cmd sid="globalSetting.facegroup.detail.deviceTap.title" product="2" /> window appears, select the device and click <Cmd sid="button.add" product="2" />.

   :::note

   Only the FaceStation 2 model can be selected.

   :::

6. Click <Cmd sid="button.addplus" product="2" /> in the <Cmd sid="globalSetting.facegroup.detail.groupSettings" product="2" />.

7. Set the <Cmd sid="globalSetting.facegroup.detail.groupName" product="2" /> and <Cmd sid="common.userGroup" product="2" /> fields.

8. Click <Cmd sid="button.apply" product="2" /> at the bottom right of the screen to save the settings.

:::info

* Up to 10 matching groups can be created.

* Each group can contain up to 3,000 IR face templates.

* Multiple user groups can be assigned to a single matching group.

* The total number of IR face templates in a matching group cannot exceed 5,000.

* If a user group contains more than 3,000 IR face templates, it cannot be set as a matching group.

* This feature is supported only by the FaceStation 2 model.

* Delete all configured devices and group settings to disable the group matching feature.

:::
