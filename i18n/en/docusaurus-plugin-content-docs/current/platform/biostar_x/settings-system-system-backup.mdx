You can back up key information such as the database of **BioStar X**, various settings, and keys. Use the <Cmd sid="common.backup" product='2' /> feature to prevent data loss on the server or when migrating to a new PC.

The supported backup methods are as follows.

* **Manual Backup**: The user manually creates the backup file.

* **Automatic Backup**: The system automatically creates backup files at predefined intervals.

:::info

* When restoring **BioStar X** from a backup file created on a previous PC to a new PC, you need to reissue the license.

* If the SQL Server database and **BioStar X** are installed on different servers, you cannot use **BioStar X** system backup and restore.

* For more information on system restore, refer to [the following](settings-system-system-restore).

:::

## Manual backup

Users can create backup files manually.

1. Click <Cmd sid='shell.header.setting' /> on the <Cmd sid='shell.header.launcher' /> page.

2. Click <Cmd sid='setting.menu.system.root' /> → <Cmd sid="common.backup" product='2' /> on the left sidebar of the screen.

   <Image src='/img/biostarx-settings-system-system-backup-general.png' className='none' />

3. In the <Cmd sid='common.backup.general' product='2' /> section, set the path to save the backup file and the maximum number of backup files.

   * <Cmd sid='backup.filePath' product='2' />: Enter the path where the backup file will be saved. The save path should be an existing directory path.

   * <Cmd sid='backup.numberFileToKeep' product='2' />: Set the maximum number of backup files.

4. To create a backup file, click the <Cmd sid="backup.backupNow" product='2' /> button.

Backup files are saved in *zip* format at the path specified in the <Cmd sid='backup.filePath' product='2' /> option. The file name is generated in the format `BioStar_X_Backup_YYYYMMDD_HHMMSS.zip`. Here, `YYYYMMDD` represents the backup date, and `HHMMSS` represents the backup time.

:::info

* If the maximum number of backups are exceeded, the oldest backup files are automatically deleted.

* <Cmd sid="backup.numberFileToKeep" product='2' />, only numbers between 1 and 100 are entered.

:::

## Automatic backup settings

When you set automatic backup, the system creates backup files at predefined intervals.

1. Click <Cmd sid='shell.header.setting' /> on the <Cmd sid='shell.header.launcher' /> page.

2. Click <Cmd sid='setting.menu.system.root' /> → <Cmd sid="common.backup" product='2' /> on the left sidebar of the screen.

   <Image src='/img/biostarx-settings-system-system-backup-automatic.png' className='none' />

3. In the <Cmd sid='common.backup.autoBackup' product='2' /> section, select the <Cmd sid='common.backup.frequency' product='2' /> option.

   * <Cmd sid="backup.daily" product='2' />: Automatically back up daily. Set the backup time in the <Cmd sid='backup.time' product='2' /> option.

   * <Cmd sid="backup.weekly" product='2' />: Automatically back up weekly. Set the day for automatic backup in the <Cmd sid='backup.day' product='2' /> option, and set the backup time in the <Cmd sid='backup.time' product='2' /> option.

   * <Cmd sid="backup.monthly" product='2' />: Automatically back up monthly. Set the date for automatic backup in the <Cmd sid='backup.date' product='2' /> option, and set the backup time in the <Cmd sid='backup.time' product='2' /> option.

4. Click <Cmd sid="button.apply" product='2' /> to save the settings.

Backup files are saved in *zip* format at the path specified in the <Cmd sid='common.backup.general' product='2' /> section's <Cmd sid='backup.filePath' product='2' /> option. The file name is generated in the format `BioStar_X_Backup_YYYYMMDD_HHMMSS.zip`. Here, `YYYYMMDD` represents the backup date, and `HHMMSS` represents the backup time.

:::info

* If you select the automatic backup frequency as <Cmd sid="backup.monthly" product='2' /> and set the date to 29, 30, or 31, automatic backups will not occur in months without that date.

* The time is based on the standard time zone set in <Cmd sid='setting.menu.preferences.root' />. For more information on setting the time zone, refer to [the following](settings-preferences#language-timezone).

:::
