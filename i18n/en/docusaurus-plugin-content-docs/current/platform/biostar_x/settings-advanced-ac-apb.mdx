Anti-passback provides enhanced features compared to anti-passback generated based on doors or devices. Set up anti-passback to force users to use designated doors and devices upon entry and exit. This prevents unauthorized re-entry and enhances security.

:::info

<Glossary termid='apb' />

:::

## Register anti-passback {#addAPBZone}

1. Click <Cmd sid='shell.header.setting' /> on the <Cmd sid='shell.header.launcher' /> page.

2. Click <Cmd sid='setting.menu.advancedAc.root' /> in the left sidebar.

3. Click <Cmd sid='button.func.addZone' product='2' />.

4. Select <Cmd sid='zone.antiPassback' product='2' /> and click <Cmd sid='button.apply' product='2' />.

5. Enter the name of the additional anti-passback feature in the <Cmd sid='common.info' product='2' /> section.

   <Image src='/img/biostarx-settings-adv-ac-add-apb.png' />

6. Set the details of the anti-passback feature in the <Cmd sid='common.setting' product='2' /> section. For more information, refer to [the following](#setting).

7. Set the actions to perform when an anti-passback violation occurs in the <Cmd sid='common.alarm' product='2' /> section.

8. Select the entry groups that can bypass anti-passback in the <Cmd sid='zone.apb.bypass' product='2' /> section. Users belonging to this group can bypass anti-passback.

9. After completing all settings, click <Cmd sid='button.apply' product='2' /> at the bottom of the screen.

:::info

The <Cmd sid='common.alarm' product='2' /> section can only be used after configuring both <Cmd sid='zone.entryDevices' product='2' /> and <Cmd sid='zone.exitDevices' product='2' />.

:::

## Setting options guide {#setting}

The details for anti-passback settings are as follows.

* <Cmd sid='zone.globalZone' product='2' />: Set the application range.

  * <Cmd sid='zone.globalZone.local' product='2' />: Can be set with entry devices connected via RS-485.

  * <Cmd sid='zone.globalZone.global' product='2' />: Can be set to all devices registered in **BioStar X**.

* <Cmd sid='zone.pause.activatedAndDeactivated' product='2' />: You can activate or temporarily deactivate the settings.

* <Cmd sid='zone.apbType' product='2' />: Select the type of anti-passback.

  * <Cmd sid='door.apbType.hard' product='2' />: Creates anti-passback events and cannot enter.

  * <Cmd sid='door.apbType.soft' product='2' />: Creates anti-passback events and can enter.

* <Cmd sid='door.resetTime' product='2' />: Set the waiting time until re-entry is allowed after a user violates anti-passback rules. After the specified time has passed following an alarm, the user's access is automatically allowed.

  * Can be set within the range of **1** minute to **10080** minutes (7 days).

  * Setting to **0** minutes will prevent the alarm from clearing automatically until the administrator clears it manually.

* <Cmd sid='zone.apbUseDoorSensor' product='2' />: Set the scope in which anti-passback rules apply.

  * <Cmd sid='common.on' product='2' />: Anti-passback rules apply according to the actual operation of the entry and exit devices.

  * <Cmd sid='common.off' product='2' />: Rules apply based on user authentication regardless of the door's operation.

  * <Cmd sid='zone.apbUseEachDoorSensorSetting' product='2' />: When using APB based on entry for the corresponding door, the anti-passback rules apply according to the sensor usage option setting.

* <Cmd sid='zone.entryDevices' product='2' />: Select the device to be used when entering. You can choose from the list of registered devices.

* <Cmd sid='zone.exitDevices' product='2' />: Select the device to be used when exiting. You can choose from the list of registered devices.

* <Cmd sid='zone.apbFailAction' product='2' />: Set the door action when communication with devices configured with anti-passback in **BioStar X** is interrupted. This feature is available when the <Cmd sid='zone.globalZone' product='2' /> option is set to <Cmd sid='zone.globalZone.global' product='2' />.

  * <Cmd sid='zone.apbFailAction.noAction' product='2' />: The door opens when the user is authenticated normally.

  * <Cmd sid='zone.apbFailAction.recordLog' product='2' />: An anti-passback violation alarm occurs and the door opens.

  * <Cmd sid='zone.apbFailAction.doorLocked' product='2' />: An anti-passback violation alarm occurs and the door does not open.
