Guidelines for registering new users. You can enter information such as the user's photo, name, email, and phone number, and set the user's access permissions and biometric information.

Enter user information and set access permissions, then enroll credentials.

1. Click <Cmd sid='shell.header.user' /> on the <Cmd sid='shell.header.launcher' /> page.

2. Click the <Cmd sid='team.user.newUser' /> button in the upper right corner of the <Cmd sid='shell.header.user' /> page.

   <Image src='/img/biostarx-user-add-user-01.png' />

3. Enter user information on the <Cmd sid='user.addNewUser' product='2' /> screen.

   <Image src='/img/biostarx-user-add-user-02.png' />

4. Enter the user's basic information in the <Cmd sid='team.user.information' /> section.

5. Enroll the user's biometric information in the <Cmd sid='team.user.credential.title' /> section.

6. Set **BioStar X** operation and access permissions for the user in the <Cmd sid='team.user.permission.title' /> section.

7. In the <Cmd sid='common.advanced' product='2' /> section, you can enter a message to display on the device when the user accesses it.

8. Enter all information and click <Cmd sid='user.save' product='2' /> in the upper right corner of the screen.

The registered user appears in the user list. For detailed information on input and settings by section, refer to the below.

<NextStep notitle>
<NextItem to='add-user-basic-info'>

Enter the user's basic information in the <Cmd sid='team.user.information' /> section. You can check the detailed information for each field.

</NextItem>
<NextItem to='enroll-credential'>

Enroll the user credentials and set access authentication method in the <Cmd sid='team.user.credential.title' /> section.

</NextItem>
<NextItem to='set-permission'>

Check how to set the user's access permissions in the <Cmd sid='team.user.permission.title' /> section.

</NextItem>
<NextItem to='set-user-advanced'>

In the <Cmd sid='common.advanced' product='2' /> section, you can set the message to display on the device when the user accesses it or exclude that user from synchronization when linking with <Cmd sid='settings.activeDirectory.title' product='2' className='normal' />.

</NextItem>
</NextStep>

:::info

* Access permissions and biometric information are optional. You can modify or add after saving the basic information.

* To print user information as a card using the card template, click <Cmd sid="button.printCard" product='2' /> at the top right of the screen. For more information, refer to [the following](settings-card-printer).

* Click <Cmd sid="button.cancel" product='2' /> at the top right of the screen to cancel user registration.

:::
