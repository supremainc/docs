You can grant separate security levels to users regardless of the biometric 1:N security level set on the device. If users frequently fail authentication when the device's 1:N security level is set high, consider lowering the <Cmd sid='user.fingerSecurityLevel' product='2' /> to mitigate failed authentications. 

<Image src='/img/biostarx-new-user-credential-securitylevle.png' />

Set the desired level in the <Cmd sid='user.fingerSecurityLevel' product='2' /> field. The available values are as follows:

* <Cmd sid='securityLevel.deviceDefault' product='2' />

* <Cmd sid='securityLevel.lowest' product='2' />

* <Cmd sid='securityLevel.low' product='2' />

* <Cmd sid='securityLevel.normal' product='2' />

* <Cmd sid='securityLevel.high' product='2' />

* <Cmd sid='securityLevel.highest' product='2' />

Click <Cmd sid='user.save' product='2' /> in the upper right corner of the screen to save the settings.

:::info

* If you set <Cmd sid="user.authMode.privateMode" product='2' /> in the <Cmd sid="common.credential" product='2' /> section to <Cmd sid="user.authMode.biometric" product='2' />, you cannot apply the 1:1 security level as the input user cannot be identified, and you can only apply the 1:1 security level when a card or ID that allows user identification is set in the authentication mode. For more information about <Cmd sid='user.authMode.privateMode' product='2' />, refer to [the following](enroll-credential-authmode).

* Set an appropriate security level. A higher security level may result in lower fingerprint authentication rates or higher [False Rejection Rates (FRR)](../../reference/glossary#frr).

:::
