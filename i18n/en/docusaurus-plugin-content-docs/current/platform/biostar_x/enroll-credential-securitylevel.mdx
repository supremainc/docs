You can assign a separate security level to users regardless of the biometric 1:N security level set on the device. If users frequently fail authentication when the device's 1:N security level is set high, consider lowering the <Cmd sid='user.fingerSecurityLevel' product='2' /> to mitigate failed authentications. 

<Image src='/img/biostarx-new-user-credential-securitylevle.png' />

Set the desired level in the <Cmd sid='user.fingerSecurityLevel' product='2' /> field. The available values are as follows:

* <Cmd sid='securityLevel.deviceDefault' product='2' />

* <Cmd sid='securityLevel.lowest' product='2' />

* <Cmd sid='securityLevel.low' product='2' />

* <Cmd sid='securityLevel.normal' product='2' />

* <Cmd sid='securityLevel.high' product='2' />

* <Cmd sid='securityLevel.highest' product='2' />

To save the settings, click the <Cmd sid='button.apply' product='2' /> button in the upper-right corner of the screen.

:::info

* If the <Cmd sid='user.authMode.privateMode' product='2' /> option in the <Cmd sid='common.permission' product='2' /> section is set to <Cmd>Biometric</Cmd>, the 1:1 security level cannot be applied because the user is unknown. The 1:1 security level can only be applied when authentication modes using user-specific cards or IDs are set. For more information about <Cmd sid='user.authMode.privateMode' product='2' />, refer to [the following document](set-permission#privateAuthMode).

* Set an appropriate security level. A higher security level may result in lower fingerprint authentication rates or higher [False Rejection Rates (FRR)](../../reference/glossary#frr).

:::
