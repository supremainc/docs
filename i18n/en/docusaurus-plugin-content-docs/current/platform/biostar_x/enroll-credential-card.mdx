Guide users on how to enroll access cards. Explain the enrollment methods for CSN cards, Wiegand cards, and smart cards, as well as the card reading/formatting features. Refer to the manual of the device for the types of cards supported by the device.

:::note

**Card enrollment using a USB agent**

| MIFARE     |  ✅  |    ❌    |      ✅     |

## CSN card {#csncard}

1. In the <Cmd sid='common.credential' product='2' /> section, click the <Cmd sid='button.addCard' product='2' /> button.

2. When the <Cmd sid='user.header.card' product='2' /> window appears, select <Cmd>CSN</Cmd> from the <Cmd sid='user.cardType' product='2' /> options.

   <Image src='/img/biostarx-new-user-credential-card-csn.png' />

3. Select your desired card enrollment method from the <Cmd sid='common.registrationOption' product='2' /> field.

   * <Cmd sid='common.registerByCardReader' product='2' />: Select the device from the <Cmd sid='common.device' product='2' /> list to scan the card and click the <Cmd sid='common.readCard' product='2' /> button.

   * <Cmd sid='common.assignCard' product='2' />: Select the card you want to assign to the user from the list. You can search through the input field.

   * <Cmd sid='common.enterManually' product='2' />: Enter the card number manually or click the <Cmd sid='common.useUserId' product='2' /> button.

4. Click the <Cmd sid='button.enroll' product='2' /> button.

## Wiegand card {#wiegandcard}

1. In the <Cmd sid='common.credential' product='2' /> section, click the <Cmd sid='button.addCard' product='2' /> button.

2. When the <Cmd sid='user.header.card' product='2' /> window appears, select <Cmd>Wiegand</Cmd> from the <Cmd sid='user.cardType' product='2' /> options.

   <Image src='/img/biostarx-new-user-credential-card-wiegand.png' />

3. Select the desired format from the <Cmd sid='user.cardEnrol.cardFormat' product='2' /> field.

4. Select your desired card enrollment method from the <Cmd sid='common.registrationOption' product='2' /> field.

   * <Cmd sid='common.registerByCardReader' product='2' />: Select the device from the <Cmd sid='common.device' product='2' /> list to scan the card and click the <Cmd sid='common.readCard' product='2' /> button.

   * <Cmd sid='common.assignCard' product='2' />: Select the card you want to assign to the user from the list. You can search through the input field.

   * <Cmd sid='common.enterManually' product='2' />: Enter the facility code and card ID manually.

5. Click the <Cmd sid='button.enroll' product='2' /> button.

:::info

If the desired card data format is not available in the <Cmd sid='user.cardEnrol.cardFormat' product='2' /> options, refer to [the following](settings-credential-wiegand-format).

:::

## Smart card {#smardcard}

Enroll a security credential card, access-on card, or custom smart card.

1. In the <Cmd sid='common.credential' product='2' /> section, click the <Cmd sid='button.addCard' product='2' /> button.

2. When the <Cmd sid='user.header.card' product='2' /> window appears, select <Cmd sid='common.smartCard' product='2' /> from the <Cmd sid='user.cardType' product='2' /> options.

   <Image src='/img/biostarx-new-user-credential-card-smartcard.png' />

3. Select the device for card enrollment from the <Cmd sid='common.device' product='2' /> field.

4. Once selected, the card layout format set in the <Cmd sid='user.cardLayoutFormatName' product='2' /> field will be displayed.

5. Select the desired card type from <Cmd sid='user.cardMode' product='2' />.

   * <Cmd sid='user.secureCredentialCard' product='2' />: User information (card ID, PIN, access group, duration, fingerprint template, face template, private authentication mode) can be stored on the card.

   * <Cmd sid='user.accessOnCard' product='2' />: User information (card ID, PIN, fingerprint template, face template) can be stored on the card.

   * <Cmd sid='common.customSmartCard' product='2' />: Enroll a smart card issued by a third party. Click the <Cmd sid='common.readCard' product='2' /> button to enroll the card ID.

6. Select the desired fingerprint template.

7. Click the <Cmd sid='button.writeCard' product='2' /> button.

:::info

* To issue a smart card, the correct card type must be set. For more information about card types, refer to [the following](settings-credential-smart-format).

* The user information stored on the smart card uses information stored in **BioStar X**. Failure to store updated user information may result in incorrect information being stored on the smart card. Additionally, if the updated user information is not synchronized with the device, the device may fail to perform authentication.

* To use a face template, select <Cmd sid='setting.cardFormat.useFace' product='2' /> in the <Cmd sid='setting.cardFormat.layout' product='2' /> section when adding a new smart card in the <Cmd sid='shell.header.setting' /> → <Cmd sid='setting.menu.credential.root' /> → <Cmd sid='setting.menu.credential.cardFormat' /> menu. For more information, refer to [the following](settings-credential-smart-format#layout).

* <Cmd sid='user.accessOnCard' product='2' className='normal' /> cannot set the card ID manually.

* For devices and firmware versions supporting Custom Smart Card, refer to the list below.

  <details>
  <summary>View devices and firmware versions that support custom smart cards</summary>

  * XPass D2 firmware version 1.7.1 or higher

  * BioEntry P2 firmware version 1.5.1 or higher

  * BioEntry W2 firmware version 1.8.0 or higher

  * BioStation 2a firmware version 1.1.0 or higher

  * X-Station 2 firmware version 1.3.0 or higher

  * BioStation 3 firmware version 1.3.0 or higher

  * BioEntry W3 firmware version 1.0.0 or higher

  * BioLite N2 firmware version 1.6.2 or higher

  </details>

* To set up the smart card layout, refer to [the following](settings-device-details-auth).

* To format the smart card and rewrite information, refer to [the following](#readformatcard).

:::

## Reading card/format {#readformatcard}

You can format smart cards and rewrite information.

1. In the <Cmd sid='common.credential' product='2' /> section, click the <Cmd sid='button.addCard' product='2' /> button.

2. When the <Cmd sid='user.header.card' product='2' /> window appears, select <Cmd sid='common.readCard' product='2' /> from the <Cmd sid='user.cardType' product='2' /> options.

   <Image src='/img/biostarx-new-user-credential-card-readcard.png' />

3. Select the device capable of reading smart cards from the <Cmd sid='common.device' product='2' /> field.

4. Select the <Cmd sid='user.cardModeLong' product='2' />.

5. Click the <Cmd sid='common.readCard' product='2' /> button.

6. Verify the card information and click the <Cmd sid='button.formatCard' product='2' /> button.

:::info

* The list of devices in the <Cmd sid='common.device' product='2' /> field will only display devices with the smart card layout set. For more information about setting this up, refer to [the following](settings-device-details-auth).

* Custom smart cards cannot be formatted.

:::
