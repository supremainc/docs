You can generate T&A reports from user attendance events collected through the system, and edit time records or export them as CSV or PDF files.

Easily use the pre-set 7 report filters, and administrators can set filters themselves.

## Before start

### Before using the multilingual report

**BioStar X** supports both Korean and English by default. To use multilingual reports, set your desired language according to the following instructions.

1. Navigate to the following path. *C:\Program Files\BioStar X\plugin\ta\dist\setup\report_fonts*

2. Create a folder with the name of the language you want to use. Refer to the ISO 639-1 standards for language names. For example, to use Spanish, create a folder named "es."

3. Copy and paste the fonts into the created folder. Only one TrueType font is supported.

### Before updating the report,

**BioStar X** uses MariaDB as the default database. If you are using MS SQL Server database, please check the following items first.

When using **BioStar X** with MS SQL Server, if there are many registered users, the memory usage on the PC may accumulate each time the report is updated. Reset the maximum server memory of the MS SQL Server database.

1. Run **Microsoft SQL Server Management Studio**.

2. Right-click on the **BioStar X** database in the object explorer.

3. Click on **Properties** in the popup menu.

4. Click **Memory** and reduce the value of **Maximum server memory** item.

:::info

For more information on MariaDB and MS SQL Server settings during **BioStar X** installation, refer to [the following](custom-install).

:::

## Check the report

### Generate the report

This provides instructions on generating attendance records reports for registered users.

1. Click <Cmd sid='shell.header.tna' /> on the <Cmd sid='shell.header.launcher' /> page.

2. Click the <Cmd sid='ta.report.title' product='2' /> tab on the left sidebar of the screen.

   <Image src='/img/biostarx-tna-report.png' />

3. Click the desired report filter item from the left sidebar of the screen.

4. In <Cmd sid='common.userGroup' product='2' /> or <Cmd sid='common.user' product='2' />, click the <IcSearch /> button, then specify the group or user.

5. To generate the report, click the <Cmd sid='button.updateReport' product='2' /> button.

6. When the list of reports appears at the bottom of the screen, you can click the desired item to view details.

<Image src='/img/biostarx-tna-report-update-list.png' />

### Set the filter conditions

You can set filter conditions to generate a new T&A record report.

<Image src='/img/biostarx-tna-report-filter-setting.png' />

* <Cmd sid='common.name' product='2' />: Enter the report name.

* <Cmd sid='ta.report.condition.reportType' product='2' />: Select the desired report type. The available report types are as follows.

  * <Cmd sid='report.daily' product='2' />, <Cmd sid='report.dailySummary' product='2' />, <Cmd sid='report.individual' product='2' />, <Cmd sid='report.individualSummary' product='2' />, <Cmd sid='report.leave' product='2' />, <Cmd sid='report.exception' product='2' />, <Cmd sid='report.modifiedPunchLogHistory' product='2' />, <Cmd sid='report.workweek' product='2' />

  :::info

  By selecting <Cmd sid='ta.report.condition.reportType' product='2' /> as <Cmd sid='report.individual' product='2' />, you can set whether to output user entry records.

  <Image src='/img/biostarx-tna-report-individual-types.png' />

  * <Cmd sid='report.individual.optionSummary' product='2' />: This outputs only the user's check-in and check-out records in the report.

  * <Cmd sid='report.individual.optionDetail' product='2' />: This outputs all user entry records in the report.

  :::

* <Cmd sid='grid.columnSetting' product='2' />: You can change the order of the report table columns or hide them.

  <Image src='/img/biostarx-tna-report-filter-setting-column.png' />

  :::info

  After changing the column order, click the <Cmd sid='columnSetting.defaultColumn' product='2' /> button to restore to defaults.

  :::

* <Cmd sid='report.filter' product='2' />: This will only activate when <Cmd sid='ta.report.condition.reportType' product='2' /> is set to <Cmd sid='report.leave' product='2' /> or <Cmd sid='report.exception' product='2' />, and you can choose specific conditions for absence or exception records.

* <Cmd sid='common.userGroup' product='2' /> / <Cmd sid='common.user' product='2' />: Select the user group or user for report generation.

* <Cmd sid='ta.report.condition.saveFilter.button' product='2' />: You can save the configured attendance report as a filter.

  <Image src='/img/biostarx-tna-report-filter-saving.png' />

### Set report period

This provides instructions on setting the report period.

<Image src='/img/biostarx-tna-report-period-setting.png' />

You can set the report period to <Cmd sid='timeCard.daily' product='2' />, <Cmd sid='timeCard.weekly' product='2' />, or <Cmd sid='timeCard.monthly' product='2' />. To generate a report for a specific period, select <Cmd sid='timeCard.custom' product='2' />.

### Export report

* <Cmd sid='button.downloadToCsv' product='2' />: You can save the generated report as a CSV file.

* <Cmd sid='button.exportPdf' product='2' />: You can save the generated report as a PDF file.

## Generate working alarm time report

You can generate T&A reports for users who have reached the designated working hours or notify the administrator via email. The alert working hours report is generated weekly.

1. Click <Cmd sid='shell.header.tna' /> on the <Cmd sid='shell.header.launcher' /> page.

2. Click the <Cmd sid='ta.report.title' product='2' /> tab on the left sidebar of the screen.

3. Click <Cmd>Working Alarm Time Report</Cmd> in the filter list on the left sidebar.

   <Image src='/img/biostarx-tna-report-working-alarm.png' />

4. Set each item in <Cmd sid='ta.report.condition.filterConditions' product='2' /> and <Cmd sid='ta.report.condition.reportPeriod' product='2' />.

5. To generate the report, click the <Cmd sid='button.updateReport' product='2' /> button.

:::info

To send notifications to the administrator via email for users who have reached the designated working hours, set <Cmd sid='ta.report.condition.autoEmail' product='2' />. For more information, refer to [the following](#autoEmail).

:::

### Automated email setting

You can automatically send alarm emails to the administrator for users who have reached the designated working hours.

<Image src='/img/biostarx-tna-report-working-alarm-auto-email.png' />

* <Cmd sid='common.email' product='2' />: Click the checkbox to automatically send emails to the administrator.

* <Cmd sid='setting.dst.dayOfWeek' product='2' />: Select the day of the week to send emails to the administrator.

* <Cmd sid='shift.time' product='2' />: Set the time to send emails to the administrator.

* <Cmd sid='setting.triggerAction.recipient' product='2' />: You can specify the administrator who will receive the email. Click the <Cmd sid='button.edit' product='2' /> button to open the <Cmd sid='setting.triggerAction.recipient' product='2' /> popup window. Enter the email address. You can specify more than one administrator.

  <Image src='/img/biostarx-tna-report-working-alarm-auto-email-receipt.png' />

:::info

* To set the <Cmd sid='ta.report.condition.autoEmail' product='2' /> feature, you must set filter conditions and then save the filter.

* The sender information of automatically sent emails can be set in <Cmd sid='setting.setting' product='2' />(<IcSet />) â†’ <Cmd sid='setting.triggerAction.senderInfor' product='2' /> in the left sidebar of the screen. For more information, refer to [the following](tna-settings#senderInfo).

:::
