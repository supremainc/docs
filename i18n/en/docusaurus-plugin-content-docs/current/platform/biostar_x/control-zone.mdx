This guide describes how to control zones. With the features provided, you can respond quickly to issues and manage the zone efficiently.

1. Click <Cmd sid='shell.header.arena' /> on the <Cmd sid='shell.header.launcher' /> page or select <Cmd sid='shell.header.arena' /> from the shortcut list at the top left of the screen.

2. In the <Cmd sid='arena.tree.door.title' /> tab on the left sidebar, go to <Cmd sid='arena.tree.door.allZones' />. 

3. Select the desired zone and right-click. You can select the desired feature from the popup menu.

:::info

The features provided differ depending on the type of zone. The types of zones are as follows.

* **Anti-Passback** zone

* Fire alarm zone

* Schedule lock/unlock zone

* Intrusion Alarm Zone

* Interlock zone

* Muster Zone

* Occupancy limit zone

For more information about the zone settings, refer to the [following](#).

:::

## Clear Anti Pass Back {#apb-release}

You can clear the APB alarm in the **Anti-Passback** zone. Click <Cmd sid='arena.tree.contextMenu.clearApb' /> in the popup menu. A completion message appears at the top right of the screen.

:::info

* The <Cmd sid='arena.tree.contextMenu.clearApb' /> feature is only available in **Anti-Passback** zones.

* For more information on how to set up **Anti-Passback** in the zone, refer to the [following](#).

* For more information about **Anti-Passback**, refer to the [following](../../reference/glossary#apb).

:::

## Clear Alarm {#alarm}

You can deactivate the alarm that occurred in this zone. Click <Cmd sid='arena.tree.contextMenu.clearAlarm' /> in the popup menu. A completion message appears at the top right of the screen.

:::info

The <Cmd sid='arena.tree.contextMenu.clearAlarm' /> feature cannot be used in occupancy limit zone.

:::

## Activate/Deactivate Zone {#zone-revitalizationdisabled}

You can activate a zone or deactivate an unused zone. The deactivated area is indicated by the <IcDisable width='20px' height='20px' /> icon. Click <Cmd sid='arena.tree.contextMenu.enable' /> or <Cmd sid='arena.tree.contextMenu.disable' /> in the popup menu.

## Control Security Zone {#security-zone-control}

You can control the security status of the security zone. Click <Cmd sid='arena.tree.contextMenu.arm' /> or <Cmd sid='arena.tree.contextMenu.disarm' /> in the popup menu.

:::info

* The <Cmd sid='arena.tree.contextMenu.arm' /> and <Cmd sid='arena.tree.contextMenu.disarm' /> features are only available in security zones.

* For more information about the security zone settings, refer to the [following](#).

:::

## View Muster Zone Report {#confirmation-of-muster-reports}

To view a report for a muster zone, click <Cmd sid='arena.tree.contextMenu.musterReport' />. When the new window opens, you will be taken to a page where you can check the report for the muster zone.

:::info

* The <Cmd sid='arena.tree.contextMenu.musterReport' /> feature is only available in the muster zone.

* For more information about the muster zone settings, refer to the [following](#).

:::

## Monitor Occupancy Limit Zone {#monitoring-of-restrictions-on-the-number-of-people}

You can check the status of an occupancy zone. Click <Cmd sid='arena.tree.contextMenu.occupancyMonitoring' /> in the popup menu. When a new window opens, you will be taken to a page that monitors occupancy limit zone.

:::info

* The <Cmd sid='arena.tree.contextMenu.occupancyMonitoring' /> feature is only available in occupancy limit zones.

* For more information about the occupancy limit zone settings, refer to the [following](#).

:::

## View Details {#detail-view}

You can check detailed information about the zone and the event history. Click <Cmd sid='arena.tree.contextMenu.viewDetail' /> in the popup menu. Detailed information and event history for the zone appear on the right side of the screen.

* **Zone Details**: You can check the name, location, status information, and more for the zone.

* <Cmd sid='arena.recentEvent' />: You can check the recent event list that occurred in the selected zone.

:::info

For detailed information on each item in the zone's details section, refer to the [following](#).

:::
