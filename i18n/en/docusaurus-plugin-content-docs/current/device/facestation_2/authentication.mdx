## Authentication mode

Set the authentication mode. You can use <Cmd sid='ui_228' product='dev' />, <Cmd sid='ui_040' product='dev' />, <Cmd sid='ui_322' product='dev' />, and <Cmd sid='ui_020' product='dev' /> in combination as authentication modes, and you can set schedules for each authentication mode.

### Registering auth mode {#registering-auth-mode}

1. Click <Image src='/img/common/ico-menu2.svg' ico alone /> and authenticate with administrator privileges.

2. Press <Cmd sid='ui_256' product='dev' /> → <Cmd sid='ui_261' product='dev' />.

3. Select <Image src='/img/common/ico-add2.svg' ico alone />.

4. Press <Image src='/img/common/ico-add3.svg' ico alone /> to see the credentials that can be added on the screen. Select the credentials to add. You can delete the added credentials by pressing <Image src='/img/common/ico-cancel1.svg' ico alone /> when credentials are added.

5. After setting the desired authentication mode, set <Cmd sid='ui_203' product='dev' />.

### Modifying auth mode

1. Click <Image src='/img/common/ico-menu2.svg' ico alone /> and authenticate with administrator privileges.

2. Press <Cmd sid='ui_256' product='dev' /> → <Cmd sid='ui_261' product='dev' />.

3. Select the authentication mode to modify.

4. Modify the authentication mode by referring to [Registering auth mode](#registering-auth-mode) and press <Cmd sid='ui_037' product='dev' />.

### Delete auth mode

1. Click <Image src='/img/common/ico-menu2.svg' ico alone /> and authenticate with administrator privileges.

2. Press <Cmd sid='ui_256' product='dev' /> → <Cmd sid='ui_261' product='dev' />.

3. Press <Image src='/img/common/ico-delete.svg' ico alone /> and select the authentication mode to delete. You can select multiple options.

4. To delete the selected item, press <Cmd sid='ui_037' product='dev' />.

## Operation

1. Click <Image src='/img/common/ico-menu2.svg' ico alone /> and authenticate with administrator privileges.

2. After pressing <Cmd sid='ui_256' product='dev' />, change the desired item in <Cmd sid='ui_265' product='dev' />.

   <Image src='/img/device/fs2-auth-option.png' />

   * <Cmd sid='ui_310' product='dev' />: Adjusts the intensity of the IR LED by detecting ambient brightness. Change settings according to the installation environment.

   * <Cmd sid='ui_140' product='dev' />: Set the sensitivity to detect motion around the device. When <Cmd sid='ui_140' product='dev' /> is set to <Cmd sid='ui_091' product='dev' />, touch the screen to authenticate your face.

   * <Cmd sid='ui_260' product='dev' />: If authentication is not completed within the set time, authentication fails.

   * <Cmd sid='ui_185' product='dev' />: When server matching is set, the device does not perform user authentication; BioStar X performs user authentication. It can be used when the device has too much user information or when the device is installed outside.

     :::info

     * <Cmd sid='ui_185' product='dev' /> can only use cards and IDs.

     * If you use <Cmd sid='ui_185' product='dev' />, you must enable server matching on both the device and BioStar X. For more information about BioStar X server matching settings, refer to the [BioStar X Administrator Guide](../../platform/biostar_x/settings-server-server#servermatching).

     :::

   * <Cmd sid='ui_232' product='dev' />: Set the sensitivity for face position, angle, and distance when registering a face. Set high for precise face templates.

   * <Cmd sid='ui_115' product='dev' />: If the face is not enrolled within the set time, the face registration will be canceled.

   * <Cmd sid='ui_117' product='dev' />: You can check for duplicate faces during face registration.

## Facial recognition

You can change the configuration for facial authentication.

1. Click <Image src='/img/common/ico-menu2.svg' ico alone /> and authenticate with administrator privileges.

2. After pressing <Cmd sid='ui_256' product='dev' />, change the desired item in <Cmd sid='ui_231' product='dev' />.

   <Image src='/img/device/fs2-auth.png' />

   * <Cmd sid='ui_149' product='dev' />: You can set the security level for 1:N authentication.

   * <Cmd sid='ui_078' product='dev' />: You can check whether group matching is used. Using group matching allows you to group multiple users' faces to increase authentication speed.

   * <Cmd sid='ui_360' product='dev' />: Enable this option if the firmware of the device that registers face templates is v1.0.1 or earlier.

   * <Cmd sid='ui_248' product='dev' />: This can prevent user registration using spoofed faces such as photos. You can set the intensity from <Cmd sid='ui_004' product='dev' /> to <Cmd sid='ui_006' product='dev' />, and as the level increases, the inspection intensity strengthens.

   * <Cmd sid='ui_158' product='dev' />: You can set whether to use fast face registration. When fast face registration is enabled, the face registration process is set to one step, and when disabled, it is set to three steps.

     :::info

     * <Cmd sid='ui_078' product='dev' /> can be set in BioStar X.

     * If <Cmd sid='ui_248' product='dev' /> is set to a high level, the registration of real faces may be rejected. If the registration fails, try again or lower the inspection intensity.

     * To register high-quality face templates, disable <Cmd sid='ui_158' product='dev' />.

     :::

## Attendance Mode

You can set how to register T&A mode.

1. Click <Image src='/img/common/ico-menu2.svg' ico alone /> and authenticate with administrator privileges.

2. Click <Cmd sid='ui_256' product='dev' />, then change the desired item in <Cmd sid='ui_083_list' product='dev' />.

   <Image src='/img/device/fs2-auth1.png' />

   * <Cmd sid='ui_083_list' product='dev' />: Set how to use attendance mode.

   * <Cmd sid='ui_086' product='dev' />: Add time and attendance events.

   * <Cmd sid='ui_082' product='dev' />: Require users to select a time and attendance event during authentication. Available when <Cmd sid='ui_083_list' product='dev' /> is set to <Cmd sid='ui_199' product='dev' />.

   * <Cmd sid='ui_081' product='dev' />: Allow users to use only time and attendance events selected by the administrator. Available when <Cmd sid='ui_083_list' product='dev' /> is set to <Cmd sid='ui_071' product='dev' />.

   * <Cmd sid='ui_279' product='dev' />: Select whether to use job codes.
