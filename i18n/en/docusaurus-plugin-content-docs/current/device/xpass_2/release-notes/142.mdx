# Firmware Version 1.4.2 (Build No. 1.4.2_241025)

Release<Separator/>2024-10-28

:::caution

Devices with the new BLE chip cannot downgrade to a lower version after upgrading the firmware to v1.3.1 or higher. Devices with existing BLE chips can be downgraded to a lower version.

For more information, check the serial number of the device and contact the [Suprema](https://supremainc.com).

:::

## New Features and Improvements

---

1. Supports for device firmware upgrade via the Device Manager application.

1. Added feature to set the baud rate of slave devices.

1. Improved to get user update succeeded event log using the GetLogWithFilter function in the SDK.

1. Improved RS-485 communication quality.

## Bug Fixes

---

<BugLists>

1. When the device log is full and a new event occurs, unintended logs are retrieved with the newly generated event. <Badge>1.3.1</Badge>

1. When running the Device Manager application v1.1.8 on certain iOS, the device does not connect. <Badge>1.0.0</Badge>

1. When the **Input card ID from keypad** option is enabled, there is an issue where the card ID value recorded in the event log differs from the entered value when an unassigned card ID is input using the keypad. <Badge>1.0.0</Badge>

1. When a user with a 32-character user ID authenticates to the device while the device is not connected to the server, the authentication event log is not updated in BioStar 2 even after the device is reconnected to the server. <Badge>1.0.0</Badge>

1. The user ID stored as a 32-character string is truncated to only 31 characters in a specific event log. <Badge>1.0.0</Badge>

1. After a firmware upgrade, authentication failure occurs with the previously used cards. <Badge>1.4.1</Badge>

1. When the **Time Synchronization with Server** option is disabled and the device time is changed, the device restarts abnormally. <Badge>1.4.1</Badge>

1. When the **RS-485** mode is set to **Slave**, the **Custom Smart Card Layout** and **Byte Order** of the device are not saved as set. <Badge>1.4.1</Badge>

1. When the device is restarted with the **Primary Key** set only in the **Suprema Smart Card Layout** and the **Secondary Key** set only in the **Custom Smart Card Layout**, the device recognizes the Custom Smart Card as a CSN card. <Badge>1.4.1</Badge>

1. Some smart cards fail to be recognized in environments using the new SE chip firmware. <Badge>1.4.1</Badge>

1. DESFire smart cards with the **Custom Smart Card Layout** set to **Secondary Key** are recognized as CSN cards. <Badge>1.4.1</Badge>

1. Custom smart cards with the **DESFire Advanced** option enabled fail to be recognized. <Badge>1.4.1</Badge>

</BugLists>
