# Firmware Version 1.1.0 (Build No. 1.1.0_210813)

Release<Separator/>2021-08-13

## New Features and Improvements

---

1. Added devices that support fingerprint

    - X-Station 2 Finger (XS2-ODPB, XS2-OAPB)

1. Intelligent Slave Support

    - Intelligent Slave: A function that enables 1:1 or 1:N matching directly from the Suprema device and transmits the authentication result as OSDP card data to the third-party controller.

1. Improved manually turning the secure tamper on or off even when the hash key is set.

## Bug Fixes

---

<BugLists>

1. When setting an elevator as a schedule unlock zone, related logs were not displayed properly. <Badge>1.0.0</Badge>

1. The device was rebooted abnormally when selecting all data, exporting them via a USB flash drive, and then importing user data on the device storing maximum user and text log data (500,000 users, 5,000,000 text logs). <Badge>1.0.0</Badge>

1. When scanning a card with XPass D2 (XPD2-GDB, XPD2-MDB) connected as a slave, it did not fail immediately and waited for input after setting the Private Mode to Card + PIN. <Badge>1.0.0</Badge>

1. The firmware upgrade failed in the device connected as a slave. <Badge>1.0.0</Badge>

1. It was not able to enroll fingerprint in some devices that were connected as a slave. <Badge>1.0.0</Badge>

1. When the authentication mode was set to Card + Fingerprint in the slave device, the authentication failed. <Badge>1.0.0</Badge>

1. The RS-485 communication did not work properly when connecting the device to a third-party controller after activating the Secure Communication mode. <Badge>1.0.0</Badge>

1. The firmware upgrade from v1.0.3 or earlier to v1.1.0 failed. <Badge>1.0.0</Badge>

    - When upgrading the firmware from 1.0.3 or earlier to 1.1.0 or later, upgrade to 1.1.0 firmware first, and then upgrade to the later version firmware again.

1. The connection to the server was lost when accessing the Ethernet menu on the device. <Badge>1.0.0</Badge>

1. When accessing the menu through ID authentication on a device with T&A mode set and moving back to the home screen, the T&A caption was not displayed. <Badge>1.0.0</Badge>

1. If some ports of the OM-120 connected as a slave are set as a door relay and the device is rebooted, the relay was operated abnormally. <Badge>1.0.0</Badge>

1. An abnormal IP address was set when using DHCP. <Badge>1.0.0</Badge>

1. When the connection to BioStar 2 was lost, an 'Upgrade Failed' message was displayed regardless of the situation. <Badge>1.0.0</Badge>

1. When searching event logs on the device, the date selection popup screen was still displayed after the Menu Timeout. <Badge>1.0.0</Badge>

</BugLists>
