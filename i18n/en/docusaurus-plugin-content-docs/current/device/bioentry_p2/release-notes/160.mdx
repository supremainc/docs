# Firmware Version 1.6.0 (Build No. 1.6.0_260212)

Release<Separator/>2026-03-17

:::caution

- After upgrading the firmware to v1.5.1 or later, it is not able to downgrade to an earlier version of the firmware.

- BEP2-OA type of BioEntry P2 model support HID iCLASS card type through SE processor.<br/>For devices with the new SE processor, after upgrading the firmware to v1.4.4 or later, you cannot downgrade to a lower version and downgrade the firmware customized in the lower version. Devices with existing SE processor can be downgraded to a lower version.

- For more information, check the serial number of the device and contact the [Suprema](https://supremainc.com).

:::

## New Features and Improvements

---

1. Reflects global cybersecurity standards and the latest security requirements.

1. Added identification code to support recognition of the latest cards.
   * DESFire EV3

1. Improved time synchronization behavior between server and devices.

1. Supports feature to set the baud rate of slave devices.

1. Improved to distinguish devices that support the **Secure Tamper** feature.

1. Supports the **Ignore Repeated Signals Duration** feature, which ignores repetitive authentication signals from the controller when the device is connected to a 3rd party controller via Wiegand.

1. Supports not logging events when the **Ignore Repeated Signals Duration** feature is activated.

1. Supports SL1, SL3, and SL1/SL3 Mix Mode for security level compatibility with MIFARE Plus EV1 cards.
   * SL1: Compatible mode with MIFARE Classic
   * SL3: Advanced security mode based on AES

1. Improved to not recognize SEOS configuration cards as CSN for 1 minute after device booting.

1. Improved so that when authentication is attempted on a door set to **Manual Lock**, an authentication failure event occurs and a locked message appears.

1. Supports real-time monitoring of the device's multiple statuses in BioStar X.

1. Improved compatibility to support accessing the detail page and changing settings for **Master Admin** supported devices connected as slaves.
