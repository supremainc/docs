Setting up a line-crossing detection event detects people who cross configured virtual lines and triggers real-time events. Specify the crossing direction (A→B, B→A, or both) to monitor a specific direction only. Optionally add a face area to prioritize face tracking; enrolled users are automatically identified.

## When to use this?

* Monitor and count doorway traffic

* Monitor entry in a specific direction only (detect one-way traffic)

* Detect entry into specific areas outside business hours

* Track left/right area movement relative to the line

## Create line-crossing detection event

Use the <Cmd sid='live' product='cam' /> menu to detect people crossing a configured line in a specific area and trigger events. Also, setting a face area prioritizes face tracking in that area; enrolled users are automatically identified.

<Steps>
<Step>

### Start event creation

1. Click <Cmd sid='settings' product='cam' /> in the upper-left of the screen.

2. Click <Cmd sid='ai_and_event' product='cam' /> → <Cmd sid='ai_analysis' product='cam' /> in the left sidebar.

   <ProdImg src='/img/cam-server/ai-analysis-events.png' />

3. Click <Cmd sid='add_button' product='cam' /> at the top right of the screen.

4. In the <Cmd sid='select_ai_analysis_type' product='cam' /> window, click <Cmd sid='ai_analysis_line_crossing' product='cam' />.

   <ProdImg src='/img/cam-server/ai-analysis-events-add-line-crossing.png' />

5. When the <Cmd sid='face_analytics_consent_title' product='cam' /> pop-up appears, review the details, select the consent checkbox, and click <Cmd sid='face_analytics_consent_agree' product='cam' />.

   <ProdImg src='/img/cam-server/ai-analysis-notice-face-auth.png' width='50%' />

   :::note

   * To use the Facial Authentication feature, comply with local regulations. Cannot proceed without consent.

   * The <Cmd sid='face_analytics_consent_title' product='cam' /> popup appears only once. Subsequent AI event creation assumes consent.

   :::

6. Complete the required settings in the right <Cmd sid='ai_analysis_line_crossing' product='cam' /> panel.

   <ProdImg src='/img/cam-server/ai-analysis-events-add-line-crossing-settings.png' />

</Step>
<Step>

### Set event name

Enter the AI event name in the <Cmd sid='name' product='cam' /> input field. The name can be up to 64 characters long.

</Step>
<Step>

### Virtual line settings

Place the virtual line at the appropriate location in Live View. The camera detects people crossing configured virtual lines and triggers events. Set at least one virtual line. Click <Cmd sid='add_button' product='cam' /> to add up to 10. An orange line appears in Live View; drag corners or points to adjust the shape. Drag the center to move the line.

<ProdImg src='/img/cam-server/ai-analysis-events-add-line-crossing-settings-position.png' />

* <Cmd sid='name' product='cam' />: Edit the virtual line's name. Assign a clear, descriptive name. The virtual line name appears on hover in Live View.

* <Cmd sid='condition' product='cam' />: Set the crossing condition for the virtual line. Select one of the three direction options below.

  * <Cmd>A -> B</Cmd>: Trigger an event when crossing from A to B.

  * <Cmd>B -> A</Cmd>: Trigger an event when crossing from B to A.

  * <Cmd>A \<-> B</Cmd>: Trigger an event when crossing in both directions.

:::info

* Click <IcCamReset title='reset' /> to reset the configured virtual line's shape and size.

* Click <IcCamTrash title='delete' /> to delete an added virtual line.

:::

</Step>
<Step>

### Face area settings (optional)

Define the area for face detection. When set, the system prioritizes face tracking in that area. Face areas can overlap with virtual lines. Click <Cmd sid='add_button' product='cam' /> to add up to 10 face areas. A blue box appears in Live View; drag corners or points to adjust the shape. Drag the center to move the line.

<ProdImg src='/img/cam-server/ai-analysis-events-add-line-crossing-settings-face-area.png' />

* <Cmd sid='name' product='cam' />: Edit the face area's name. Assign a clear, descriptive name. The face area name appears on hover in Live View.

:::info

* Enrolled users are automatically identified in the <Cmd sid='live' product='cam' /> menu and shown by name. Unenrolled users appear as <Cmd sid='unknown_face' product='cam' />.

* Click <IcCamReset title='reset' /> to reset the configured face area's shape and size.

* Click <IcCamTrash title='delete' /> to delete an added face area.

:::

</Step>
<Step>

### Set event trigger conditions

Set the event trigger conditions.

* <Cmd sid='every_delay' product='cam' />: Set the event trigger delay. Events trigger only after the configured time elapses; set up to 60 seconds.

</Step>
<Step>

### Save settings

After completing all settings, click <Cmd sid='create' product='cam' /> in the top right. The interface returns to the previous screen, and the new AI event is added to the list.

<ProdImg src='/img/cam-server/ai-analysis-events-add-line-crossing-settings-complete.png' />

</Step>
</Steps>

## Check in Live

Click the <Cmd sid='live' product='cam' /> menu at the top left of the screen. Confirm the <Image src='/img/cam-server/ico-content-ai-show.svg' alt='AI On' alone ico /> icon at the top right of the camera feed is on. Configured virtual lines and face areas are overlaid on the video. An event occurs each time a person crosses a virtual line. If face areas are set, the camera detects faces there; enrolled users are automatically identified and their names are displayed.

<ProdImg src='/img/cam-server/ai-analysis-events-line-crosing-result.png' />

:::info

* Unenrolled users appear as <Cmd sid='unknown_face' product='cam' />. Enrolled users appear with their name.

* Poor lighting or backlight may prevent the camera from detecting faces. Improve area lighting or adjust the camera position and angle so faces are clearly visible.

* Faces must face the camera to be detected. Profile or rear views may not be detected.

* If too many events occur, adjust the <Cmd sid='every_delay' product='cam' /> option. Increasing the event delay reduces event frequency.

:::
