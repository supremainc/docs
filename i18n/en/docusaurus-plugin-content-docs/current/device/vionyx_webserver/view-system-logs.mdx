You can view and manage logs for camera user access, event occurrences, and system operations. Filter logs by the desired criteria in the web interface, export them in CSV format, or download encrypted device logs when more detailed analysis is needed.

Check logs in the following situations.

* To check who accessed the camera and when

* To track when a specific event occurred

* To check major system operations such as boot and reboot

* To review access records for security monitoring

* To collect system logs for troubleshooting

:::info

* Logs are retained automatically according to the device's memory size.

* Each log type is retained for approximately one month.

* When memory is full, the oldest logs are deleted automatically.

* Only administrators and operators with read/write permission can view system logs.

:::

## View logs

1. Click the <Cmd sid='system' product='cam' /> menu in the upper-left corner of the screen.

2. Click <Cmd sid='system_logs' product='cam' /> in the left sidebar. View all logs.

   <ProdImg src='/img/cam-server/cam-webserver-system-log.png' />

3. To filter logs by log category, level, and type, select the desired criteria in <Cmd sid='all_logs' product='cam' />, <Cmd sid='all_levels' product='cam' />, and <Cmd sid='all_types' product='cam' /> at the upper left of the log list.

   <ProdImg src='/img/cam-server/cam-webserver-system-log-filter.png' className='none' />

4. Click <IcSearch title='Search' /> at the top of the log list to filter logs based on the selected criteria.

:::info

* To view logs for a specific period, click the date and time input fields. When the calendar and time picker appear, select the desired start date and time. Set the end date and time the same way.

  <ProdImg src='/img/cam-server/cam-webserver-system-log-date.png' width='40%' />

* Scroll down the list to refresh it and keep viewing older logs.

* For more information about log types, see <DocLink docId='device/vionyx_webserver/system-log-type' />.

:::

### Log types

Logs are categorized as follows.

* <Cmd sid='ai_analysis' product='cam' />: View event records related to AI analysis.

* <Cmd sid='access' product='cam' />: Records of users accessing the camera web interface.

* <Cmd sid='event' product='cam' />: Records of event occurrences, Webhook sends, and rule executions.

* <Cmd sid='system' product='cam' />: Records of system operations such as boot, reboot, and initialization. This log is **viewable by administrators only**.

:::info

Select <Cmd sid='log_category_download' product='cam' /> to download the device's encrypted log file. For more information, see <DocLink inner='#downloadDeviceLogs' />.

:::

### Log levels

Logs are categorized by the following levels.

* <Cmd sid='log_level_info' product='cam' />: View general informational message logs.

* <Cmd sid='log_level_warning' product='cam' />: View warning message logs.

* <Cmd sid='log_level_error' product='cam' />: View error message logs.

## Export logs

Download logs in CSV format for external analysis or archiving.

1. Click the <Cmd sid='system' product='cam' /> menu in the upper-left corner of the screen.

2. Click <Cmd sid='system_logs' product='cam' /> in the left sidebar.

3. Select the log type to export in <Cmd sid='all_logs' product='cam' /> at the upper left of the log list. Set a search period to export logs for a specific time range.

4. Click <Cmd sid='csv_export' product='cam' /> at the top right of the screen.

5. When the <Cmd sid='csv_export' product='cam' /> popup appears, select the language to export.

   <ProdImg src='/img/cam-server/cam-webserver-system-log-export.png' />

6. Click <Cmd sid='ok' product='cam' />.

Download the log list as a CSV file.

:::info

* CSV filename format: `system-logs-YYYYMMDD_HHMMSS.csv`. 

  > Example: *system-logs-20240326_153000.csv*

* The file is saved to the browser's default download folder or to a user-selected location.

:::

## Log retention and management

### Automatic log deletion

When the camera's memory is full, the oldest logs are deleted automatically.

<div className='table-fixed'>

| **Log type** | **Retention period** |
| --- | --- |
| **System Logs** | Approximately one month |
| **Access Logs** | Approximately one month |
| **Event Logs** | Approximately one month |

</div>

### How to retain important logs

To retain important logs long-term, follow these steps:

* Export logs to CSV regularly.

* Store exported files on external storage (NAS, cloud storage, etc.).

* Include the export date and log period in the file.

## Device log download {#downloadDeviceLogs}

Download log files stored on the device in encrypted form. Device logs are detailed log files that include all events and operational records generated by the camera system for troubleshooting. The file is encrypted to keep it secure, and provide it to <Support /> when needed for troubleshooting.

1. Click the <Cmd sid='system' product='cam' /> menu in the upper-left corner of the screen.

2. Click <Cmd sid='system_logs' product='cam' /> in the left sidebar.

3. Select <Cmd sid='log_category_download' product='cam' /> in <Cmd sid='all_logs' product='cam' /> at the upper left of the log list.

   <ProdImg src='/img/cam-server/cam-webserver-system-log-download-device-log.png' />

4. Click <Cmd sid='log_category_download' product='cam' /> at the upper right of the log list.

Download the device log file locally. The file name uses the format `device_logs_{device-serial-number}_YYYYMMDDTHHMMSS.zip.enc`.

> Example: *device_logs_1234567890_20240326T153000.zip.enc*
