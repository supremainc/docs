This document provides essential technical reference information for deploying and securing BioStar X. Find network port configurations, recommended security settings, and key compliance requirements at a glance. Refer to the external resources section for additional support and detailed guidelines.

## Default BioStar X network ports

| Service | Purpose | Default Port(s) | Notes / Recommendations |
| --- | --- | --- | --- |
| Unified Gateway Service | HTTPS (system access) | 443 | Default for admin/operator login. Changeable via <Cmd>Service Manager</Cmd> → <Cmd>Service Settings</Cmd>. |
| Core Web Service | HTTPS (internal UI) | 5002 | Prefer proxied to 443; restrict direct access. |
| ^ | CloudNgrok | 52000 | Disable if not required. |
| ^ | Thrift RPC | 9310 | Restrict to trusted services. |
| Core Service | WebSocket | 9002 | Internal only. |
| ^ | FastCGI | 9000 | Should be bound locally. |
| ^ | API | 9010 | Secure with TLS & authentication. |
| Coordinator Service | Client Communication | 21810 | Internal only. |
| Main Server | TCP Server | 51212 | **Plaintext** device comms — block if possible. |
| ^ | SSL Server | 51213 | **Preferred** device comms (TLS). |
| ^ | gRPC | 51219 | Internal service comms only. |
| Cache Service | Client | 10800 | Equivalent to Redis — do not expose externally. |
| ^ | Communication | 47500 | Internal only. |
| ^ | Discovery | 47100 | Internal only. |

## Default security features in BioStar X

| Feature | Location in UI | Default | Recommendation |
| --- | --- | --- | --- |
| Change default admin login ID & password | <Cmd sid='shell.header.user' /> → Edit Default Admin (`UID:1`) | admin / installer PW | Change immediately after install. |
| Access separation (roles) | <Cmd sid='shell.header.user' /> → Select a user → <Cmd sid='common.permission' product='2' /> → <Cmd sid='user.accountLevel' product='2' /> | Predefined: <Cmd sid='setting.account.administrator' product='2' />, <Cmd sid='setting.account.userOperator' product='2' />, <Cmd sid='setting.account.monitoringOperator' product='2' /> | Create custom roles as needed. |
| Password strength policy | <Cmd sid='shell.header.setting' /> → <Cmd sid='setting.menu.system.root' /> → <Cmd sid='setting.menu.system.security' /> → <Cmd sid='setting.systemSecurity.loginPassword' product='2' /> → <Cmd sid='setting.server.passwordLevel' product='2' /> | <Cmd sid='setting.server.passwordLevelMedium' product='2' /> | Set to <Cmd sid='setting.server.passwordLevelStrong' product='2' />, enforce MFA. |
| Session security (Simultaneous connection) | <Cmd sid='shell.header.setting' /> → <Cmd sid='setting.menu.system.root' /> → <Cmd sid='setting.menu.system.security' /> → <Cmd sid='setting.systemSecurity.sessionSecurity' product='2' /> → <Cmd sid='setting.systemSecurity.useSimultaneousConnection' product='2' /> | <Cmd sid='common.active' product='2' /> | Set to <Cmd sid='common.inactive' product='2' /> to prevent shared logins. |
| Multi-factor authentication | <Cmd sid='shell.header.user' /> → Select a user → <Cmd sid='common.permission' product='2' /> → <Cmd sid='common.fingerprintLogin' product='2' /> | <Cmd sid='common.notUse' product='2' /> | Enable for <Cmd sid='setting.account.administrator' product='2' />, <Cmd sid='setting.account.userOperator' product='2' />, <Cmd sid='setting.account.monitoringOperator' product='2' /> accounts. |
| Secure communication with device | <Cmd sid='shell.header.setting' /> → <Cmd sid='setting.menu.system.root' /> → <Cmd sid='setting.menu.system.security' /> → <Cmd sid='setting.server.secure.comm.title' product='2' /> → <Cmd sid='setting.sever.secure.comm.menu' product='2' /> | <Cmd sid='common.notUse' product='2' /> | Enable + use SSL port (51213). |
| Device hashkey management | <Cmd sid='shell.header.setting' /> → <Cmd sid='setting.menu.system.root' /> → <Cmd sid='setting.menu.system.security' /> → <Cmd sid='setting.server.secure.comm.title' product='2' /> → <Cmd sid='setting.server.secure.comm.data_encryption' product='2' /> | <Cmd sid='common.notUse' product='2' /> | Enable for tamper protection in high-security deployments. |
| System Backup | <Cmd sid='shell.header.setting' /> → <Cmd sid='setting.menu.system.root' /> → <Cmd sid='setting.menu.system.systemBackup' /> | <Cmd sid='common.manual' product='2' /> | Enable <Cmd sid='common.backup.autoBackup' product='2' />. |
| Directory Integration | <Cmd sid='shell.header.setting' /> → <Cmd sid='setting.menu.system.root' /> → <Cmd sid='setting.menu.directoryIntegration.root' /> | Not Configured | Integrate with Entra ID or Active Directory where available. |
| Log settings (Retention & Levels) | <Cmd sid='shell.header.setting' /> → <Cmd sid='setting.menu.system.root' /> → <Cmd sid='setting.menu.server.root' /> → <Cmd sid='setting.menu.server.server' /> | <Cmd sid='setting.server.logRetainDays' product='2' />: 60 days<br /><Cmd sid='setting.server.logLevel' product='2' />: <Cmd sid='setting.server.logLevel.info' product='2' /> | Adjust per compliance. Forward to SIEM. |
| Encrypt personal data on database | <Cmd sid='shell.header.setting' /> → <Cmd sid='setting.menu.system.root' /> → <Cmd sid='setting.menu.system.security' /> → <Cmd sid='setting.server.secure.comm.title' product='2' /> → <Cmd sid='setting.sever.secure.comm.encrypt.db' product='2' /> | <Cmd sid='common.notUse' product='2' /> | Enable in production. |
| Store visual face image | <Cmd sid='shell.header.setting' /> → <Cmd sid='setting.menu.system.root' /> → <Cmd sid='setting.menu.server.server' /> → <Cmd sid='settings.expertSettings.enableButton' product='2' /> → <Cmd sid='settings.expertSettings.detail.storeVisualFaceImageOptionLabel' product='2' /> | <Cmd sid='common.use' product='2' /> | Disable unless justified. |
