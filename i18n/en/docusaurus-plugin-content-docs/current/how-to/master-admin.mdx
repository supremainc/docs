:::caution

<Cmd sid="ui_128" product='dev' /> feature is available only on **CE products** that support a user interface through the LCD screen.

:::

The <Cmd sid="ui_128" product='dev' /> feature is designed to **enhance overall administrator privileges**, **improve device security**, and **prevent unauthorized access and setting changes**.<br/>This feature applies only to **CE products** for the EU region, and **Master Administrator enrollment is required** on **CE products**. Only the enrolled master administrator can access the administrator menu and change device settings.

:::info

* This feature is a required policy introduced to enhance security for **CE products** in the EU region, and it is not supported on **Global (Non-CE) products**.

* The Master Admin cannot be deleted directly, and they are **deleted only through factory reset**.

:::

## Behavior Logic During Firmware Upgrade {/* #firmware-upgrade-logic */}

When you upgrade the firmware of an existing device, support for <Cmd sid="ui_128" product='dev' /> and <Cmd sid="ui_073" product='dev' /> can vary based on the combination of the **version before upgrade** and the **target version**.

:::note

For version-specific support details, contact Suprema Technical Support at CS@suprema.co.kr.

:::

## New Device <Badge only>CE products</Badge>

On a new device for CE products, you must enroll <Cmd sid="ui_128" product='dev' /> from the start. If you do not enroll it, device use is restricted.

:::info

* At first boot, the <Cmd sid="ui_128" product='dev' /> enrollment screen appears. If you do not complete enrollment, you cannot move to the next screen.

* Even if you connect to BioStar X (or BioStar 2) and assign an overall administrator for the device, you must enroll <Cmd sid="ui_128" product='dev' /> directly on the device to use all features.

* New devices cannot perform firmware downgrades.

:::

## Firmware-Upgraded Device <Badge only>CE products</Badge>

A CE product that has been upgraded from firmware does not provide <Cmd sid="ui_128" product='dev' /> settings, but you can use the <Cmd sid="ui_073" product='dev' /> option to enhance overall administrator privileges and improve device security.

### Set Admin 2-step Authentication {/* #admin-two-step-authentication */}

:::info

* <Cmd sid="ui_073" product='dev' /> can be activated only if all enrolled administrators have at least two types of credentials.

  * PIN cannot be used as a standalone credential, so the ID + PIN combination is not supported.

* The firmware upgrade device cannot be downgraded to a lower version after upgrading the firmware.

:::

* You can activate Admin 2-step Authentication in the **Settings** → **Device** → <Cmd sid="ui_073" product='dev' /> menu.

* The <Cmd sid="ui_073" product='dev' /> option appears only when you upgrade firmware to the latest version on an existing device.

<Image src='/img/how-to/setting-two-step-authentication.png' caption width='300px' />

:::info

* If not all administrators have at least two types of credentials, activation will fail and an error message will appear.<br/>Enroll two types of credentials for all administrators and try again.

  * PIN cannot be used as a standalone credential, so the ID + PIN combination is not supported.

<Image src='/img/how-to/error-two-step-authentication.png' caption width='300px' />

:::

:::warning

* If you enable <Cmd sid="ui_073" product='dev' /> and delete all administrators' credentials to fewer than two types, you cannot access the administrator menu when you cannot connect to BioStar X (or BioStar 2). Therefore, exercise caution when deleting administrator credentials.

:::
